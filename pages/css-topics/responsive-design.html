01

Qu'est-ce que le responsive design ?

Le responsive design (conception adaptative) est une méthodologie de développement web qui permet à une interface de s'afficher de manière optimale sur tous les supports : mobiles, tablettes, ordinateurs de bureau et téléviseurs. Le contenu demeure invariant ; seule la mise en page se reconfigure en fonction de l'espace disponible. Cette approche a été formalisée par Ethan Marcotte en 2010 et repose sur trois piliers fondamentaux : une grille fluide, des images flexibles et des media queries.

💧
Analogie : l'eau et le récipient

Considérons de l'eau versée dans différents récipients. Dans un verre étroit, elle adopte une forme haute et fine. Dans un saladier, elle s'étale largement. Il s'agit pourtant de la même eau — autrement dit, du même contenu. Le responsive design incarne précisément cette capacité du contenu à épouser la morphologie de son conteneur, en l'occurrence l'écran.

Les trois piliers du responsive design

  • Grille fluide : les largeurs sont exprimées en % ou en fr, jamais en unités fixes px
  • Images flexibles : les images ne doivent jamais excéder la largeur de leur conteneur (max-width: 100%)
  • Media queries : des points de rupture conditionnant les changements de mise en page

À l'heure actuelle, plus de 60 % du trafic web provient d'appareils mobiles. Google utilise le mobile-first indexing : c'est la version mobile d'un site qui est prise en compte pour le référencement. Un site non responsive est donc un site invisible.

02

La balise meta viewport

La mise en œuvre du responsive design débute par une balise HTML qui spécifie au navigateur mobile la manière d'interpréter la largeur de la page. En l'absence de celle-ci, les terminaux mobiles affichent la version desktop du site avec un zoom arrière important, rendant le texte illisible.

HTML 
<meta name="viewport" content="width=device-width, initial-scale=1.0">





<meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=2">
⚠️
Sans viewport, pas de responsive

Si la balise viewport est omise, le site ne sera jamais véritablement responsive. Les iPhones et autres appareils mobiles restitueront la page comme si elle était conçue pour un écran de 980 px de large, avec un zoom arrière. L'utilisateur devra pincer-zoomer pour lire — une expérience désastreuse.

03

Les media queries — syntaxe @media

Les media queries sont des règles CSS conditionnelles : elles ne s'appliquent que lorsque certaines conditions relatives au type de média ou aux caractéristiques du dispositif sont satisfaites (largeur d'écran, orientation, préférence utilisateur). Leur syntaxe générale est @media type and (condition) { règles }.

printImpression et aperçu avant impressionMasquer les interactions, ajuster les marges et les polices

CSS — Live editor 
/* Mobile-first : base = mobile, puis couches successives */
.grille {
  display: grid;
  grid-template-columns: 1fr;
  gap: 1rem;
}

@media screen and (min-width: 768px) {
  .grille { grid-template-columns: 1fr 1fr; }
}

@media screen and (min-width: 1024px) {
  .grille { grid-template-columns: 1fr 1fr 1fr; }
}

/* Préférence utilisateur */
@media (prefers-color-scheme: dark) {
  :root { --bg: #0D1117; --text: #e6edf3; }
}

@media (prefers-reduced-motion: reduce) {
  * { animation-duration: 0.01ms !important; }
}

/* Orientation */
@media (orientation: landscape) {
  .sidebar { width: 250px; }
}
RENDU
Redimensionnez la fenêtre pour voir les media queries en action. Les préférences système (mode sombre, réduction de mouvement) sont détectées automatiquement.
📌
Où placer les media queries ?

Deux approches : ① Dans le même fichier CSS après les règles de base (recommandé). ② Dans un fichier CSS distinct grâce à l'attribut media sur <link>. L'approche centralisée facilite la maintenance et évite des requêtes HTTP superflues.

04

Breakpoints et approches : mobile-first vs desktop-first

Un breakpoint est une largeur d'écran à partir de laquelle la mise en page se modifie. Il n'existe pas de valeurs universelles — tout dépend du contenu. Les breakpoints les plus fréquents (empruntés à Bootstrap) sont : 576 px (petits écrans), 768 px (tablettes), 992 px (desktop), 1200 px (grands écrans) et 1400 px (très grands écrans). La règle d'or est de les baser sur le contenu, pas sur des appareils spécifiques.

Référence complète — Breakpoints courants

BreakpointMobile-first (min-width)Desktop-first (max-width)Cible
576px@media (min-width: 576px)@media (max-width: 575px)Grands mobiles en paysage
768px@media (min-width: 768px)@media (max-width: 767px)Tablettes
992px@media (min-width: 992px)@media (max-width: 991px)Desktop
1200px@media (min-width: 1200px)@media (max-width: 1199px)Grands écrans
1400px@media (min-width: 1400px)@media (max-width: 1399px)Très grands écrans
CSS — Mobile-first 
/* MOBILE-FIRST (recommandé) */
/* Base = mobile (0‑575px) */
.grille { display: grid; grid-template-columns: 1fr; gap: 1rem; }

@media (min-width: 768px) { .grille { grid-template-columns: 1fr 1fr; } }
@media (min-width: 1024px) { .grille { grid-template-columns: 1fr 1fr 1fr; } }

/* DESKTOP-FIRST (alternative) */
.grille { display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 1.5rem; }

@media (max-width: 1023px) { .grille { grid-template-columns: 1fr 1fr; } }
@media (max-width: 767px) { .grille { grid-template-columns: 1fr; } }
💡
Mobile-first (min-width) est recommandé

Cette approche est plus performante (les mobiles téléchargent moins de CSS), plus élégante (progressive enhancement) et mieux adaptée au web contemporain. Desktop-first (max-width) peut servir pour adapter un site desktop existant, mais pour tout nouveau projet, privilégiez mobile-first.

05

Unités responsives : %, vw, vh, clamp()

Les unités relatives constituent le fondement du responsive design. vw, vh, % et la fonction clamp() permettent aux éléments de s'adapter à la taille de l'écran sans recourir à des media queries. La fonction clamp(MIN, PRÉFÉRÉ, MAX) est particulièrement puissante : elle définit une valeur fluide entre un minimum et un maximum, éliminant le besoin de breakpoints pour les tailles de texte.

CSS 
/* Viewport units */
.hero    { height: 60vh; }           /* 60 % de la hauteur de l'écran */
.section { padding: 5vw; }          /* padding proportionnel à la largeur */
.title   { font-size: clamp(1.5rem, 4vw, 3rem); } /* taille fluide */

/* clamp() — la fonction reine du responsive */
.text { font-size: clamp(1rem, 2.5vw, 2rem); }
.card { padding: clamp(1rem, 3vw, 2.5rem); }

/* Pourcentage + max-width */
.container { width: 100%; max-width: 1100px; margin: 0 auto; }
📌
clamp() — une alternative moderne aux media queries

clamp(MIN, PRÉFÉRÉ, MAX) permet une valeur fluide entre un minimum et un maximum. Un titre en clamp(1.5rem, 4vw, 3rem) croît progressivement avec l'écran sans jamais excéder 3 rem ni descendre sous 1.5 rem, éliminant le besoin de breakpoints pour les tailles de texte.

06

Images responsives et l'élément <picture>

Une image qui excède la largeur de son conteneur compromet la mise en page. La règle fondamentale est max-width: 100%. Pour la performance, on utilise srcset (servir la bonne résolution) et l'élément <picture> (servir le bon format).

Référence complète — <picture> et <source>

Élément / AttributDescriptionContexte d'utilisation
<picture>Conteneur générique pour plusieurs sources d'imageFournir différents formats ou résolutions selon le contexte
<source srcset="..." type="...">Définit une source alternative avec son type MIMEServir du WebP ou AVIF avec fallback JPEG/PNG
<source srcset="..." media="...">Source conditionnelle basée sur une media queryCharger une image différente selon la taille d'écran (art direction)
<img src="..." alt="...">Image de fallback, toujours présente et obligatoireFallback pour les navigateurs qui ne supportent pas <picture>
srcset="photo-400w.jpg 400w"Liste d'images avec leurs largeurs en pixels (w)Permet au navigateur de choisir la meilleure résolution
srcset="photo.jpg 1x, photo@2x.jpg 2x"Liste d'images avec leur densité de pixels (x)Cibler les écrans Retina (préférer la syntaxe w + sizes)
sizes="(max-width: 480px) 100vw, 800px"Indique la taille d'affichage de l'image dans chaque breakpointNécessaire pour que le navigateur choisisse la bonne source parmi les descriptors w
CSS + HTML 
/* CSS — la règle d'or */
img, video, iframe { max-width: 100%; height: auto; }

&-img { width: 100%; height: 300px; object-fit: cover; }

/* srcset — servir la bonne taille */
<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w, photo-800.jpg 800w, photo-1200.jpg 1200w"
     sizes="(max-width: 480px) 100vw, (max-width: 1024px) 50vw, 800px"
     alt="Paysage">

/* picture — différents FORMATS */
<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Paysage">
</picture>
RENDU
srcset + max-width: 100% = images adaptées à tous les écrans et résolutions.
💡
La règle fondamentale pour toutes les images

Intégrez cette règle dans le reset CSS : img, video, iframe { max-width: 100%; height: auto; }. Sans elle, une image de 1920 px dépasserait de 300 px sur un mobile — rédhibitoire pour le responsive.

07

Container queries — le futur du responsive

Les container queries (@container) permettent à un composant de réagir à la taille de son propre conteneur plutôt qu'à celle de la fenêtre. Supportées par tous les navigateurs modernes depuis 2023, elles constituent une avancée majeure pour les composants réutilisables.

Référence complète — @container

Propriété / RègleDescriptionContexte d'utilisation
container-type: inline-sizeDéfinit le conteneur comme un contexte de requête basé sur la largeurNécessaire pour activer les container queries sur ce conteneur
container-type: sizeDéfinit le conteneur sur la largeur ET la hauteurPlus rare, utile pour les requêtes sur les deux axes
container-name: nomDonne un nom au conteneur pour le cibler dans @containerÉvite les conflits quand plusieurs conteneurs sont imbriqués
container: nom / inline-sizeShorthand pour container-name et container-typecontainer: card / inline-size
@container nom (min-width: Npx)Requête conditionnelle sur la taille du conteneur nomméAdapter un composant à l'espace disponible dans son parent
@container (min-width: Npx)Requête sur le conteneur le plus proche sans nom spécifiqueUtilisation simple quand un seul conteneur est présent
CSS 
/* 1. Définir un conteneur */
.card-wrapper { container: card / inline-size; }

/* 2. Utiliser @container */
@container card (min-width: 400px) {
  .card { display: flex; flex-direction: row; }
  .card-img { width: 150px; height: 150px; }
}

@container card (max-width: 399px) {
  .card { display: flex; flex-direction: column; }
  .card-img { width: 100%; height: auto; }
}
📌
Container queries vs media queries

Les media queries réagissent au viewport (la fenêtre entière). Les container queries réagissent à la taille du parent. Une carte placée à la fois dans une sidebar étroite et dans la zone principale large s'adapte automatiquement à chaque contexte. Supportées par Chrome 105+, Firefox 110+, Safari 16+.

08

Exercices pratiques

EX 01 📱 Page responsive complète
  1. Créez exo-responsive.html avec header, navigation, contenu principal, sidebar, footer.
  2. En mobile (< 576 px) : tout en 1 colonne, menu en hamburger.
  3. En tablette (≥ 768 px) : 2 colonnes pour la section principale.
  4. En desktop (≥ 1024 px) : 3 colonnes avec sidebar visible.
  5. Utilisez clamp() pour les tailles de texte.
  6. Images responsives avec max-width: 100%.
EX 02 🔄 Migration mobile-first
  1. Écrivez un fichier CSS en desktop-first, puis réécrivez-le en mobile-first.
  2. Ajoutez clamp() pour au moins 3 propriétés (font-size, padding, gap).
  3. Utilisez max-width: 100%; height: auto; sur toutes les images.
  4. Ajoutez un breakpoint à 576 px, 768 px et 1024 px.
🔍
Exercice bonus — Container queries

Créez un composant "carte-produit" avec container-type: inline-size. La carte passe en mode horizontal quand le conteneur dépasse 400 px. Placez la même carte dans une sidebar étroite et dans la zone principale — elle s'adapte automatiquement.

Bilan & mini-glossaire

Compétences acquises

  • Les principes du responsive design : grille fluide, images flexibles, media queries
  • La balise meta viewport — obligatoire pour tout site responsive
  • Syntaxe des media queries : @media, types, opérateurs logiques, min/max-width, orientation, prefers-*

  • Unités responsives : %, vw, vh, clamp(min, préféré, max)
  • Images responsives : max-width: 100 %, srcset, picture, object-fit
  • Container queries (@container) pour des composants réutilisables

Mini-glossaire

Responsive design
Approche de conception web qui adapte la mise en page à la taille de l'écran.
Viewport
Zone visible du navigateur. Balise meta obligatoire pour indiquer la largeur réelle au mobile.
Media query
Règle CSS conditionnelle basée sur les caractéristiques du dispositif.
Breakpoint
Largeur d'écran à partir de laquelle la mise en page change.
Mobile-first
Approche qui part du mobile (styles de base) et ajoute des couches pour les écrans plus larges.
clamp()
Fonction CSS : clamp(MIN, PRÉFÉRÉ, MAX). Crée une valeur fluide entre un min et un max.
srcset
Attribut HTML listant plusieurs versions d'une image pour différentes résolutions d'écran.
Container query
Media query basée sur la taille du conteneur parent plutôt que sur celle du viewport.
object-fit
Propriété CSS pour contrôler comment une image remplit son cadre (cover, contain, fill).
