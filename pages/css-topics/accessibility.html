01

Pourquoi l'accessibilité CSS ?

L'accessibilité web (abrégée en a11y) vise à rendre les sites utilisables par l'ensemble des personnes, indépendamment de leurs capacités ou de leurs appareils. Le CSS joue un rôle déterminant dans cette démarche : il peut aussi bien améliorer que compromettre l'expérience utilisateur pour les personnes malvoyantes, daltoniennes, épileptiques, ou présentant des troubles moteurs.

L'accessibilité ne constitue pas un « nice to have » — elle représente un droit fondamental dans l'écosystème du web contemporain. De la même manière que les rampes d'accès profitent à tous (personnes en fauteuil, parents avec poussette), les pratiques CSS accessibles servent l'ensemble des utilisateurs.

L'impact du CSS sur l'accessibilité

👁️ Visuel

Contrastes insuffisants, textes trop petits, polices illisibles — le CSS peut rendre un site inaccessible aux malvoyants.

🎯 Navigation

Styles de focus invisibles, ordre de tabulation perturbé par position: absolute, éléments cliquables trop petits.

🌀 Mouvement

Animations rapides, défilement parallaxe, clignotements — sources de nausée ou de crises d'épilepsie.

💡
L'accessibilité bénéficie à tous les utilisateurs

Les pratiques CSS accessibles (contrastes élevés, textes dimensionnés de manière adaptée, navigation au clavier) servent l'ensemble des utilisateurs : usage en plein soleil, écran de qualité médiocre, mobilité réduite momentanée, ou utilisateur âgé.

02

Contrastes et couleurs accessibles

Le contraste entre la couleur du texte et celle du fond constitue le facteur principal de lisibilité. Les WCAG (Web Content Accessibility Guidelines) définissent des ratios minimums à respecter pour garantir une perception adéquate du contenu. Le ratio de contraste est calculé à partir de la luminance relative de chaque couleur, produisant une valeur comprise entre 1:1 (aucun contraste) et 21:1 (contraste maximal, noir sur blanc).

Ratios de contraste WCAG

NiveauTexte normal (<18px)Texte large (≥18px gras ou ≥24px)Composants d'interface
AA (minimum)4.5:13:13:1
AAA (renforcé)7:14.5:1—

Le ratio de contraste se calcule avec la formule : (L1 + 0.05) / (L2 + 0.05), où L1 est la luminance relative de la couleur la plus claire et L2 celle de la couleur la plus foncée. Des outils comme WebAIM Contrast Checker ou le sélecteur de couleurs des DevTools permettent de vérifier ces valeurs.

CSS — Contrastes
/* ✅ BON — contraste suffisant (AA) */ .good-contrast { color: #1a1a2e; /* texte très foncé */ background: #f5f5f5; /* fond clair */ /* ratio ≈ 15:1 ✅ */ } /* ❌ MAUVAIS — contraste insuffisant */ .bad-contrast { color: #cccccc; /* gris clair */ background: #ffffff; /* fond blanc */ /* ratio ≈ 1.5:1 ❌ illisible */ } /* Sélection de couleurs accessibles */ :root { --text-primary: #1a1a2e; --text-secondary: #4a4a6a; /* assez foncé pour AA */ --text-muted: #8a8aaa; /* décoratif uniquement */ --bg-primary: #ffffff; --bg-secondary: #f0f0f5; }
RENDU
✅ Bon contraste (15:1)
❌ Mauvais contraste (1.5:1)
✅ Texte blanc sur fond foncé
⚠️
La couleur comme unique vecteur d'information

Environ 8 % des hommes présentent une forme de daltonisme (déficience rouge-vert principalement). Il ne faut jamais utiliser la couleur comme seul indicateur d'information. Un champ de formulaire ne doit pas virer au rouge uniquement — ajoutez une icône, un texte d'erreur ou un symbole. De même, les liens soulignés sont préférables aux liens distingués uniquement par leur couleur.

03

outline et outline-offset

La propriété outline dessine un contour autour d'un élément, sans affecter sa taille ni sa position dans le flux (contrairement à border). Elle est essentielle pour les styles de focus accessibles. La propriété outline-offset permet d'ajuster la distance entre le contour et le bord de l'élément.

Référence complète — outline

ValeurDescriptionContexte d'utilisation
<outline-width>Épaisseur du contour : thin (1px), medium (3px), thick (5px), ou une valeur <length> comme 2px.Définit la visibilité du contour
<outline-style>Style du contour : none, hidden, dotted, dashed, solid, double, groove, ridge, inset, outset.Apparence visuelle du contour
<outline-color>Couleur du contour. Accepte invert (inversion des couleurs — déprécié) et toute valeur de couleur CSS.Contraste du contour
nonePas de contour. Utilisé avec précaution : ne jamais retirer le contour sans fournir une alternative visible.Désactivation (avec alternative)

Référence complète — outline-offset

ValeurDescriptionContexte d'utilisation
<length>Distance entre le bord de l'élément et le contour. Valeur positive (extérieur) ou négative (intérieur). Ex: 2px, -2px.Espacement du contour de focus
initialValeur par défaut (0). Le contour est collé au bord de l'élément.Réinitialisation
CSS 
/* Style de contour de base */
.focus-style {
  outline: 2px solid #3498db;
  outline-offset: 3px;
}

/* Contour avec box-shadow (alternative élégante) */
.custom-focus {
  outline: none; /* supprime le contour par défaut */
  box-shadow: 0 0 0 3px rgba(52, 152, 219, .4);
  border-radius: 4px;
}

/* Contour négatif (à l'intérieur) */
.inner-outline {
  outline: 2px solid #e63946;
  outline-offset: -4px;
}
⚠️
Ne jamais supprimer outline sans alternative

outline: none sans fournir de style de focus alternatif rend la navigation au clavier impossible pour les utilisateurs qui ne peuvent pas utiliser de souris. Si vous remplacez l'outline par défaut par un box-shadow ou un autre indicateur visuel, assurez-vous que ce dernier est au moins aussi visible que l'outline original.

04

:focus-visible et :focus-within

Les pseudo-classes :focus-visible et :focus-within offrent un contrôle précis sur les styles de focus. :focus-visible ne s'applique qu'en navigation clavier (idéal), tandis que :focus-within s'applique à un parent quand un de ses enfants reçoit le focus.

Référence complète — :focus-visible

CaractéristiqueDescriptionContexte d'utilisation
:focus-visiblePseudo-classe qui applique le style uniquement lorsque le navigateur détermine que le focus doit être visible (navigation au clavier, pas au clic souris).Style de focus moderne — recommandé pour tous les éléments interactifs
:focus (classique)S'applique dès qu'un élément reçoit le focus, y compris lors d'un clic souris.Style de focus traditionnel — peut être inesthétique après un clic
Support navigateurTous les navigateurs modernes depuis 2022. Pour les anciens, combiner avec :focus en fallback.Dégradation élégante

Référence complète — :focus-within

CaractéristiqueDescriptionContexte d'utilisation
:focus-withinPseudo-classe qui cible un élément lorsqu'il contient un élément enfant ayant le focus.Mise en évidence de conteneurs (formulaires, cartes) dont un champ est actif
HéritageRemonte dans tous les ancêtres jusqu'à la racine du document.Navigation dans des arbres complexes
CSS
/* ❌ MAUVAIS — supprimer le focus */ *:focus { outline: none; } /* JAMAIS ! */ /* ✅ BON — focus visible au clavier uniquement */ a:focus-visible, button:focus-visible { outline: 2px solid #3498db; outline-offset: 3px; border-radius: 4px; } /* ✅ Focus avec halo box-shadow */ .custom-focus:focus-visible { outline: none; box-shadow: 0 0 0 3px rgba(52, 152, 219, .4); border-radius: 4px; } /* ✅ Mise en évidence du conteneur parent */ .card:focus-within { border-color: #3498db; box-shadow: 0 0 0 3px rgba(52, 152, 219, .2); }
RENDU
Lien focusé
✅ Carte avec :focus-within activé
📌
Distinction entre :focus et :focus-visible

La pseudo-classe :focus s'applique dès qu'un élément reçoit le focus, y compris lors d'un clic souris. :focus-visible ne s'applique que lorsque le navigateur détecte une navigation au clavier. Cette approche constitue la solution idéale : les utilisateurs souris ne perçoivent pas de halo inesthétique après un clic, tandis que les utilisateurs clavier disposent systématiquement d'un repère visible.

05

prefers-reduced-motion

Certaines personnes souffrent de troubles vestibulaires, de nausées ou de migraines déclenchées par les mouvements et animations. La media query prefers-reduced-motion permet de détecter si l'utilisateur a configuré son système d'exploitation pour réduire les animations.

Référence complète — prefers-reduced-motion

ValeurDescriptionContexte d'utilisation
no-preferenceL'utilisateur n'a pas exprimé de préférence (valeur par défaut implicite).Animations normales
reduceL'utilisateur a demandé la réduction des animations via les paramètres système.Désactiver ou ralentir les animations
CSS 
/* Approche globale : désactiver toutes les animations */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* Approche fine : désactiver seulement le décoratif */
@media (prefers-reduced-motion: reduce) {
  .parallax, .confetti, .floating-bg {
    animation: none !important;
    transition: none !important;
  }
  /* Conserver les animations fonctionnelles (loader) */
  .loading-spinner {
    animation-duration: 1.5s; /* ralentie, pas arrêtée */
  }
}

/* Défilement doux adapté */
html { scroll-behavior: smooth; }
@media (prefers-reduced-motion: reduce) {
  html { scroll-behavior: auto; }
}
⚠️
Préservation des animations fonctionnelles

Il convient d'éviter l'approche binaire consistant à appliquer animation: none à l'ensemble des éléments. Une barre de progression ou un spinner de chargement constituent des animations fonctionnelles qui fournissent un feedback nécessaire. Il est préférable de les ralentir plutôt que de les supprimer.

06

prefers-color-scheme (mode sombre)

La media query prefers-color-scheme détecte si l'utilisateur a opté pour un thème clair (light) ou sombre (dark) au niveau de son système d'exploitation. Elle constitue le fondement technique du « dark mode » automatique.

Référence complète — prefers-color-scheme

ValeurDescriptionContexte d'utilisation
no-preferenceL'utilisateur n'a pas exprimé de préférence de thème.Utiliser le thème clair par défaut
lightL'utilisateur préfère un thème clair (fond blanc, texte foncé).Thème clair standard
darkL'utilisateur préfère un thème sombre (fond foncé, texte clair).Thème sombre, réduction de la fatigue oculaire
CSS 
/* Thème clair (défaut) */
:root {
  --bg: #ffffff;
  --text: #1a1a2e;
  --text-secondary: #4a4a6a;
  --border: #ddd;
  --accent: #3498db;
}

/* Thème sombre automatique */
@media (prefers-color-scheme: dark) {
  :root {
    --bg: #121212;
    --text: #e0e0e0;
    --text-secondary: #a0a0b0;
    --border: #333;
    --accent: #5dade2;
  }
}

body {
  background: var(--bg);
  color: var(--text);
}

/* Ajustement des images en mode sombre */
@media (prefers-color-scheme: dark) {
  img { opacity: .85; filter: brightness(.9); }
}

/* Forcer le mode avec une classe (toggle manuel) */
.dark-theme { --bg: #121212; --text: #e0e0e0; }
.light-theme { --bg: #ffffff; --text: #1a1a2e; }
💡
Articulation entre mode automatique et basculement manuel

La meilleure expérience utilisateur s'obtient en trois étapes : ① Appliquer le thème automatique via prefers-color-scheme. ② Ajouter un bouton de basculement manuel qui modifie une classe sur <html>. ③ Stocker la préférence dans localStorage pour assurer sa persistance entre les pages.

07

prefers-contrast et forced-colors

Ces media queries permettent d'adapter l'interface aux besoins de contraste de l'utilisateur. prefers-contrast détecte une préférence logicielle, tandis que forced-colors détecte l'activation du mode contraste élevé au niveau du système (notamment sous Windows).

Référence complète — prefers-contrast

ValeurDescriptionContexte d'utilisation
no-preferenceL'utilisateur n'a pas exprimé de préférence (valeur par défaut).Styles normaux
moreL'utilisateur a demandé un contraste accru (textes plus foncés, bordures plus visibles).Accessibilité renforcée, malvoyance
lessL'utilisateur a demandé un contraste réduit (textes plus doux).Fatigue visuelle, migraines
customL'utilisateur utilise un jeu de couleurs personnalisé (expérimental).Palettes personnalisées

Référence complète — forced-colors

ValeurDescriptionContexte d'utilisation
nonePas de mode de couleurs forcées (valeur par défaut).Styles normaux
activeMode contraste élevé du système activé (Windows High Contrast Mode). Les couleurs système remplacent les couleurs CSS.Assure une visibilité minimale
CSS 
/* Contraste accru */
@media (prefers-contrast: more) {
  :root {
    --text: #000000;       /* texte 100% noir */
    --text-secondary: #222;
    --border: #000;
  }
  a { text-decoration: underline; }
}

/* Contraste réduit */
@media (prefers-contrast: less) {
  :root {
    --text: #555;
    --border: #e0e0e0;
  }
}

/* Mode contraste élevé Windows */
@media (forced-colors: active) {
  .btn {
    border: 2px solid ButtonText;
  }
  .icon {
    forced-color-adjust: none; /* préserve les couleurs d'origine */
  }
}
📌
Disponibilité variable des media queries

Les media queries prefers-reduced-motion et prefers-color-scheme sont les plus largement prises en charge. prefers-reduced-transparency et inverted-colors sont moins répandues — vérifiez leur compatibilité sur caniuse.com avant utilisation en production.

08

Contenu masqué pour lecteurs d'écran (.sr-only)

Il existe des situations où un contenu doit être invisible visuellement tout en demeurant accessible aux lecteurs d'écran. La classe .sr-only (screen-reader only) constitue le motif standard pour masquer un élément sur le plan visuel tout en le maintenant dans le DOM à l'intention des technologies d'assistance.

CSS 
/* .sr-only — visible uniquement par les lecteurs d'écran */
.sr-only {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0, 0, 0, 0);
  white-space: nowrap;
  border: 0;
}

/* Variante : remettre visible au focus (ex: Skip to content) */
.sr-only-focusable:focus {
  position: static;
  width: auto;
  height: auto;
  padding: 8px 16px;
  margin: 0;
  overflow: visible;
  clip: auto;
  white-space: normal;
  background: #3498db;
  color: white;
}

Usages courants de .sr-only

  • Libellés cachés pour les icônes seules : <button><span class="sr-only">Menu</span>☰</button>
  • Descriptions d'images pour les illustrations décoratives non décrites par alt
  • Tableaux complexes avec des en-têtes supplémentaires invisibles
  • Liens "Skip to content" : invisibles sauf au focus clavier
  • Indications de statut pour les mises à jour dynamiques via ARIA live regions
⚠️
Proscription de display: none et visibility: hidden

Les propriétés display: none et visibility: hidden masquent l'élément également aux lecteurs d'écran. Si le contenu revêt une importance pour la compréhension de la page, il convient d'utiliser .sr-only plutôt que ces propriétés. De même, l'attribut aria-hidden="true" doit être réservé aux éléments purement décoratifs.

09

Éviter le contenu généré seul (::before / ::after)

Les pseudo-éléments ::before et ::after permettent d'ajouter du contenu décoratif via CSS. Cependant, ce contenu n'est pas accessible : les lecteurs d'écran ne restituent pas le contenu de la propriété content (sauf dans certains cas particuliers pour les émojis).

CSS 
/* ✅ ACCEPTABLE — contenu purement décoratif */
.badge::before {
  content: "★ ";
  color: gold;
}

/* ❌ PROBLÉMATIQUE — information perdue pour les lecteurs d'écran */
.price::before {
  content: "Prix : ";
}
/* Le lecteur d'écran lira "25 €" et non "Prix : 25 €" */

/* ✅ SOLUTION — inclure le texte dans le HTML */
/* <span class="sr-only">Prix : </span><span class="price">25 €</span> */
💡
Règle fondamentale : le contenu textuel dans le HTML

Si le contenu est important pour la compréhension, il doit figurer dans le HTML et non dans le CSS. Les pseudo-éléments doivent être réservés au contenu décoratif (icônes, séparateurs, puces). Pour les libellés tels que « Prix : », « Nouveau ! », « Lire la suite », rédigez-les dans le HTML en recourant à .sr-only si vous souhaitez les masquer visuellement.

Exercices pratiques

EX 01 ♿ Composant accessible
  1. Créez exo-accessibilite.html avec un formulaire de contact : nom, email, message, bouton envoyer.
  2. Assurez-vous que tous les champs ont un <label> explicite.
  3. Utilisez :focus-visible pour un style de focus visible au clavier : outline de 2px bleu avec offset.
  4. Ajoutez un lien "Aller au contenu" avec la classe .sr-only-focusable.
  5. Vérifiez le ratio de contraste des couleurs choisies avec un outil comme WebAIM Contrast Checker.
  6. Ajoutez un message d'erreur visible ET lisible par les lecteurs d'écran (avec role="alert").
EX 02 🌙 Thème sombre & animations
  1. Créez exo-dark-mode.html avec une page simple : header, contenu, sidebar, footer.
  2. Définissez des variables CSS pour les couleurs (bg, text, accent, border).
  3. Utilisez prefers-color-scheme pour appliquer automatiquement un thème sombre.
  4. Ajoutez une animation CSS sur un élément (ex : un badge qui pulse).
  5. Utilisez prefers-reduced-motion pour désactiver cette animation si l'utilisateur l'a demandé.
  6. Ajoutez un bouton de basculement manuel clair/sombre avec JavaScript qui change une classe sur <html>.
🔍
Exercice bonus — Audit d'accessibilité

Prenez une page web existante et lancez un audit avec Lighthouse (dans Chrome DevTools, onglet Lighthouse). Examinez les résultats dans la catégorie "Accessibilité". Corrigez au moins 3 problèmes identifiés : contrastes, aria labels, headings hierarchy, ou focus styles.

Mini-glossaire

Accessibilité (a11y)
Pratiques visant à rendre le web utilisable par tous, y compris les personnes handicapées.
WCAG
Web Content Accessibility Guidelines — normes internationales d'accessibilité web (niveaux A, AA, AAA).
Ratio de contraste
Rapport entre la luminance du texte et du fond. Minimum 4.5:1 pour AA, 7:1 pour AAA.
Outline
Propriété CSS qui affiche un contour de focus. Ne pas supprimer sans alternative.
:focus-visible
Pseudo-classe qui applique le style de focus uniquement lors de la navigation au clavier.
:focus-within
Pseudo-classe qui cible un élément contenant un enfant focusé.
prefers-reduced-motion
Media query qui détecte la demande de réduction des animations.
prefers-color-scheme
Media query qui détecte le thème clair (light) ou sombre (dark) du système.
prefers-contrast
Media query qui détecte la préférence de contraste (more, less).
forced-colors
Media query qui détecte le mode contraste élevé du système (Windows).
sr-only
Classe CSS qui cache un élément visuellement mais le garde accessible aux lecteurs d'écran.
Lecteur d'écran
Logiciel qui lit à voix haute le contenu de l'écran (NVDA, JAWS, VoiceOver, TalkBack).
Contenu généré
Texte ou icônes ajoutés via ::before/::after et la propriété content en CSS.
