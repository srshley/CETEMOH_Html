01

Pourquoi différentes unités ?

Le CSS met à disposition plus d'une dizaine d'unités de mesure, chacune répondant à des besoins spécifiques : définir des dimensions précises, concevoir des mises en page fluides, s'adapter à la taille du viewport ou respecter les préférences de l'utilisateur. Le choix de l'unité appropriée est fondamental pour créer des designs accessibles, adaptatifs et maintenables.

Les unités CSS se répartissent en plusieurs catégories. Les unités absolues fournissent une mesure fixe et invariante, à l'image d'un mètre ruban. Les unités relatives s'adaptent au contexte d'affichage : elles sont proportionnelles à la taille de police (em, rem), aux dimensions de la fenêtre (vw, vh), ou à la taille du parent (%). Les fonctions mathématiques (calc, clamp, min, max) permettent de combiner ces unités pour créer des valeurs dynamiques.

Dans le cadre du web moderne, les unités relatives sont largement prédominantes. Elles garantissent que la mise en page s'adapte à la diversité des supports — des écrans mobiles aux écrans 4K — et respectent les préférences d'accessibilité de l'utilisateur, notamment le zoom texte. La maîtrise de ces unités distingue un développeur capable de produire des interfaces véritablement responsive.

📌
Primauté du pixel sur le web

Dans le contexte du web, une seule unité absolue est couramment employée : le pixel (px). Les autres unités (pt, cm, mm) proviennent du domaine de l'imprimerie et manquent de précision à l'écran. Pour la conception adaptative (responsive design), les unités relatives sont prédominantes.

02

Unités absolues : px, pt, cm, mm, in, pc

Les unités absolues correspondent à des mesures physiques invariantes, définies par rapport à des étalons de mesure. La plus employée dans le domaine du web est le pixel (px), qui sert de référence pour la quasi-totalité des dimensions à l'écran. Les autres unités — point (pt), centimètre (cm), millimètre (mm), pouce (in) et pica (pc) — sont héritées du domaine de l'imprimerie et trouvent leur utilité principalement dans les feuilles de style destinées à l'impression.

Un pixel CSS (px) correspond à un pixel de référence, défini comme 1/96e de pouce. Cette définition théorique ne correspond pas nécessairement à un pixel physique de l'écran : sur les écrans Retina ou à haute densité (2×, 3×), plusieurs pixels physiques sont utilisés pour restituer un seul pixel CSS. Ce mécanisme, transparent pour le développeur, garantit une taille d'affichage cohérente quelle que soit la densité de l'écran.

⚠️
Distinction entre pixel CSS et pixel physique

Un px en CSS correspond à un pixel de référence (environ 1/96 de pouce), et non nécessairement à un pixel physique de l'écran. Les écrans à haute densité (Retina 2×, 3×) utilisent plusieurs pixels physiques pour restituer un pixel CSS. Ce mécanisme est transparent pour le développeur.

03

Unités relatives à la police : em et rem

Les unités relatives à la police sont fondamentales pour la typographie et la gestion des espacements. Elles permettent aux dimensions de s'adapter à la taille de texte, garantissant ainsi une expérience accessible et cohérente. Le choix entre em et rem influence directement la prévisibilité et la maintenabilité du code.

L'unité rem (root em) est relative à la taille de police de l'élément <html> (la racine du document). Par défaut, les navigateurs fixent 1rem = 16px. En modifiant la taille de police de la racine, l'ensemble des valeurs exprimées en rem s'adapte proportionnellement. C'est l'unité recommandée pour les tailles de police car elle est prévisible : sa valeur ne dépend que d'un seul point de référence, indépendamment de l'imbrication des éléments.

L'unité em est relative à la taille de police de l'élément parent. Cette dépendance contextuelle la rend idéale pour les espacements proportionnels — padding, margin, border-radius d'un bouton — qui s'adaptent automatiquement lorsque la taille de police locale change. Toutefois, l'em s'accumule dans les éléments imbriqués, ce qui peut produire des effets de rétrécissement progressif (par exemple, font-size: .9em dans des listes imbriquées).

CSS
/* Jouez avec em et rem — voyez la différence */ html { font-size: 16px; } h1 { font-size: 2.5rem; } /* 40px — toujours pareil */ p { font-size: 1rem; } /* 16px */ .btn { font-size: 1rem; padding: .75em 1.5em; /* 12px 24px */ border-radius: .25em; /* 4px */ background: #3498db; color: white; }

CSS — em vs rem 
/* rem : valeurs prévisibles, basées sur la racine */
html { font-size: 16px; }
h1   { font-size: 2.5rem; }     /* 40px */
h2   { font-size: 1.75rem; }    /* 28px */
p    { font-size: 1rem; }       /* 16px */

/* em : valeurs contextuelles, proportionnelles au parent */
.btn { font-size: 1rem; padding: .75em 1.5em; border-radius: .25em; }
/* Agrandir le bouton = tout s'adapte */
.btn-lg { font-size: 1.25rem; } /* padding = .9375em = 15px, etc. */

/* ⚠️ Piège : imbrication d'em */
ul { font-size: .9em; }         /* 14.4px si parent 16px */
/* ul > ul > li : 0.9 × 0.9 = 0.81 × 16 = 12.96px — rétrécissement */
💡
Recommandation : rem pour le texte, em pour l'espacement

Il est recommandé d'utiliser rem pour les font-size (valeur prévisible, respectant les préférences utilisateur) et em pour les padding, margin et border-radius — les espacements s'adaptent alors automatiquement aux variations locales de taille de police.

🚀
Pour aller plus loin — Le piège de l'em cumulatif

L'utilisation de font-size en em sur des éléments imbriqués (comme les listes) provoque un effet de rétrécissement progressif bien connu. Par exemple, ul { font-size: .9em; } rendra une liste imbriquée sur 5 niveaux presque illisible (0.9⁵ ≈ 0.59 × 16px ≈ 9.4px). La solution : utiliser rem pour les tailles de police dans la liste, ou annuler l'imbrication avec ul ul { font-size: 1rem; }. Ce piège est l'une des raisons pour lesquelles rem a gagné en popularité avec l'avènement du responsive design.

04

Unités relatives au viewport (vw, vh, vmin, vmax)

Les unités viewport sont relatives aux dimensions de la fenêtre d'affichage du navigateur. Elles sont particulièrement adaptées aux sections pleine largeur, aux images hero, à la typographie adaptative et aux carrés parfaits. Le viewport correspond à la zone visible de la page, dont les dimensions varient avec le redimensionnement de la fenêtre.

L'unité la plus courante, vw (viewport width), représente 1% de la largeur du viewport. Ainsi, 50vw équivaut à la moitié de la largeur de la fenêtre. De même, vh (viewport height) représente 1% de la hauteur. Les unités vmin et vmax sont respectivement 1% de la plus petite et de la plus grande dimension du viewport, ce qui les rend utiles pour créer des éléments carrés qui s'adaptent à l'orientation de l'écran.

Référence complète des unités viewport

1vh1% de la hauteur du viewportSi viewport = 900px, 1vh = 9px
1vmin1% de la plus petite dimension (largeur ou hauteur)En portrait : vmin = vw ; en paysage : vmin = vh
1vmax1% de la plus grande dimensionInverse de vmin
1dvw / 1dvhViewport dynamique (tient compte des barres d'outils mobiles)Moderne, plus précis que vw/vh sur mobile

CSS — Viewport units 
.hero        { height: 60vh; }             /* 60% de la hauteur de la fenêtre */
.full-width  { width: 100vw; }             /* pleine largeur (attention scrollbar) */
.square      { width: 30vmin; height: 30vmin; } /* carré adaptatif */
.hero-title  { font-size: clamp(1.5rem, 4vw, 4rem); } /* typo responsive */
.safe-height { height: 100dvh; }           /* viewport dynamique (mobile safe) */
⚠️
Inclusion de la barre de défilement dans 100vw

La valeur 100vw inclut la largeur de la barre de défilement verticale, ce qui peut entraîner un débordement horizontal sur certains navigateurs. Pour éviter cet écueil, privilégiez width: 100% pour les conteneurs internes et réservez 100vw aux éléments véritablement plein écran. Sur mobile, préférez les unités de viewport dynamiques dvh / dvw pour éviter les sauts de mise en page lors de l'apparition/disparition des barres d'outils.

05

Pourcentages (%)

Les pourcentages sont relatifs à la valeur de la même propriété chez l'élément parent. Leur comportement diffère sensiblement selon la propriété à laquelle ils sont appliqués. Pour width, un pourcentage est relatif à la largeur du bloc contenant ; pour height, il nécessite une hauteur explicite sur le parent ; pour padding et margin, il est toujours relatif à la largeur du parent, quelle que soit la direction.

Cette particularité des pourcentages pour le padding vertical est contre-intuitive mais extrêmement utile : elle permet de créer des boîtes au ratio constant, notamment pour les vidéos intégrées (ratio 16:9 avec padding-top: 56.25%). Les pourcentages pour font-size sont relatifs à la taille de police du parent, à l'image de l'unité em (150% équivaut à 1.5em).

CSS — Pourcentages 
.container   { width: 80%; max-width: 1200px; margin-inline: auto; }
.sidebar     { width: 30%; }
.aspect-16-9 { padding-top: 56.25%; }     /* ratio 16:9 avec position absolute */
.child-font  { font-size: 150%; }         /* = 1.5em */
📌
Calcul des pourcentages pour padding et margin

Lorsque vous utilisez padding: 10% ou margin: 10%, le pourcentage est calculé par rapport à la largeur du parent (et non sa hauteur). Ce comportement, contre-intuitif au premier abord, est utile pour créer des boîtes au ratio constant.

06

Unités typographiques : ch, ex et autres

Deux unités moins connues mais utiles méritent une attention particulière : ch et ex. L'unité ch (character width) représente la largeur du caractère "0" (zéro) dans la police et la taille de police courantes. Dans une police à chasse fixe (monospace), 1ch correspond exactement à la largeur d'un caractère, ce qui permet de définir des largeurs de colonnes en nombre de caractères.

L'unité ex (x-height) représente la hauteur du caractère "x" minuscule dans la police courante. Elle est utile pour l'alignement vertical et la définition de tailles relatives à la hauteur des bas-de-casse (minuscules). En pratique, ex est moins utilisée que ch ou em, mais elle trouve des applications dans la typographie fine.

Depuis CSS Values Level 4, de nouvelles unités sont disponibles : cap (hauteur des capitales), ic (largeur du caractère "水" pour les textes CJK), lh (hauteur de ligne) et rlh (hauteur de ligne de la racine). Ces unités spécialisées sont principalement utiles pour la typographie avancée et les textes multilingues.

article       { max-width: 70ch; }             /* 70 caractères max par ligne */
.monospace    { width: 40ch; }                 /* 40 caractères de large */

💡
Pour aller plus loin — 70ch, la largeur de lecture idéale

Des études de lisibilité montrent qu'une ligne de texte ne devrait pas dépasser 60 à 75 caractères pour une lecture confortable. La valeur max-width: 70ch sur un article garantit cette limite de manière adaptative, car ch s'ajuste automatiquement à la police utilisée. C'est un exemple parfait d'utilisation d'une unité relative pour l'accessibilité.

07

Quand utiliser quelle unité ?

Le choix de l'unité appropriée dépend de plusieurs facteurs : le type de propriété, le comportement souhaité (fixe ou fluide), les contraintes d'accessibilité, et le contexte de l'élément. Le guide suivant résume les meilleures pratiques pour chaque situation courante.

Points de vigilance

⚠️ height: 100% sans hauteur parent

Un élément avec height: 100% ne fonctionnera que si son parent a une hauteur définie. Sans cela, le pourcentage est ignoré et l'élément s'adapte à son contenu. Solution : utilisez 100vh pour la hauteur complète.

⚠️ em imbriqué qui rétrécit

Si vous mettez font-size: .9em sur <ul>, chaque liste imbriquée rétrécit de 10% à chaque niveau. Préférez .9rem pour éviter l'effet "poupée russe".

⚠️ 100vw débordement horizontal

width: 100vw inclut la barre de défilement, ce qui peut créer un débordement horizontal. Si la scrollbar fait 17px, votre élément fera 100vw + 17px.

⚠️ Les % en padding/margin

Le padding et margin en pourcentage sont calculés par rapport à la largeur du parent, pas la hauteur. padding-top: 50% = 50% de la largeur.

💡
Stratégie recommandée pour les débutants

Il est conseillé de débuter avec rem pour les textes, % pour les largeurs, px pour les bordures et em pour les paddings de boutons. Introduisez vh/vw pour les sections hero. Progressivement, adoptez clamp() pour une typographie fluide.

08

Fonctions utiles : calc(), clamp(), min(), max()

Les fonctions mathématiques CSS permettent des calculs directement dans les déclarations de propriétés. Elles sont essentielles pour créer des valeurs fluides et adaptatives, en combinant des unités de mesure différentes. Introduites progressivement à partir de CSS3, elles sont désormais supportées par l'ensemble des navigateurs modernes.

La fonction calc() autorise le mélange d'unités de mesure différentes au sein d'une même expression arithmétique. Par exemple, calc(100% - 40px) soustrait 40 pixels fixes d'une largeur relative, permettant de créer des mise en page hybrides. La fonction clamp() constitue la méthode la plus élégante pour définir des valeurs fluides encadrées par des bornes minimale et maximale.

Référence complète de calc()

OpérationSyntaxeExemple
Additioncalc(a + b)calc(50% + 20px)
Soustractioncalc(a - b)calc(100% - 40px)
Multiplicationcalc(a * b)calc(2 * 20px)
Divisioncalc(a / b)calc(100% / 3)
Mixtecalc(a + b - c * d)calc((100% - 2rem) / 2)

Référence complète de min() et max()

FonctionDescriptionExemple

CSS — Fonctions mathématiques 
/* calc() — mélange d'unités */
.sidebar  { width: calc(33.333% - 1rem); }   /* colonne avec gouttière */
.content  { width: calc(100% - 40px); }       /* largeur moins padding */
.hero     { height: calc(100vh - 80px); }     /* plein écran moins navbar */
.fluid    { font-size: calc(1rem + .5vw); }   /* taille fluide */

/* clamp() — valeur fluide avec bornes */
h1        { font-size: clamp(1.8rem, 4vw, 3.5rem); }
.container { width: clamp(320px, 80%, 1200px); }
.card     { padding: clamp(1rem, 2vw, 2rem); }

/* min() / max() */
.wide     { width: min(100%, 1200px); }       /* max-width simplifié */
.small    { width: max(300px, 50%); }         /* min-width simplifié */
.gap      { gap: max(16px, 2vw); }           /* espacement minimum 16px */
CSS — Exemples concrets 
/* Système de conteneur responsive */
.container {
  width: clamp(320px, 80%, 1400px);
  margin-inline: auto;
  padding-inline: clamp(1rem, 2vw, 2rem);
}

/* Titre hero : taille fluide avec bornes */
.hero-title {
  font-size: calc(clamp(1.5rem, 3vw, 3rem) + .5rem);
}

/* Grille 3 colonnes avec gouttière fluide */
.grid-3 {
  display: grid;
  grid-template-columns: repeat(3, 1fr);
  gap: clamp(1rem, 2vw, 2rem);
}
💡
Syntaxe des espaces dans calc()

Les espaces autour des opérateurs + et - sont obligatoires : calc(100% - 40px) est valide, tandis que calc(100%-40px) ne l'est pas. Pour les opérateurs * et /, les espaces ne sont pas requis mais sont recommandés pour la lisibilité.

📌
clamp comme alternative à min() + max()

clamp(MIN, PRÉFÉRÉ, MAX) équivaut à max(MIN, min(PRÉFÉRÉ, MAX)). Cette fonction est particulièrement utile au quotidien : elle garantit que la valeur ne descend jamais sous le minimum ni ne dépasse le maximum, tout en restant fluide dans l'intervalle.

Exercices pratiques

EX 01 📐 Unités & espacement
  1. Créez une page avec trois blocs côte à côte de largeur 30% chacun.
  2. Utilisez vh pour donner une hauteur minimale de 50vh à la section principale.
  3. Ajoutez un bloc hero avec height: 60vh et un titre en font-size: clamp(1.5rem, 4vw, 4rem).
  4. Créez un bouton avec padding: .75em 1.5em et border-radius: .25em.
  5. Utilisez max-width: 70ch sur un article.
EX 02 🧮 Fonctions mathématiques
  1. Créez un conteneur avec width: calc(100% - 2rem) centré avec margin auto.
  2. Ajoutez une barre latérale de width: min(300px, 30%).
  3. Utilisez clamp(1rem, 2.5vw, 2rem) pour le padding d'une carte.
  4. Créez un titre dont la taille fluctue entre 1.5rem et 3.5rem selon la largeur.
  5. Utilisez max(20px, 2vw) pour un espacement qui ne descend pas sous 20px.
🔍
Exercice bonus — Système de conteneur responsive

Créez un système de conteneur avec width: clamp(320px, 80%, 1400px) centré avec margin-inline: auto. Ajoutez du padding en clamp(1rem, 2vw, 2rem). Observez l'adaptation du mobile au desktop.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • La différence entre unités absolues (px) et relatives (%, em, rem, vw, vh)
  • rem : relatif à la racine (html) — prévisible, recommandé pour font-size
  • em : relatif au parent — utile pour paddings, s'accumule dans l'imbrication
  • vw, vh : relatif au viewport — plein écran, hero sections
  • vmin, vmax : 1% de la plus petite/grande dimension du viewport
  • % : relatif au parent — attention pour height et padding
  • ch et ex : unités typographiques pour contraintes de lecture
  • Quand utiliser chaque unité selon le contexte
  • calc() : mélange d'unités, calculs arithmétiques
  • min(), max() : sélection de la plus petite/grande valeur
  • clamp() : valeur fluide avec bornes min et max
