01

Le rôle du CSS — habiller le HTML

Le langage CSS (Cascading Style Sheets, feuilles de style en cascade) a pour fonction de définir la présentation visuelle des documents HTML. Si le HTML assure la structuration sémantique du contenu — titres, paragraphes, listes, navigation —, le CSS en détermine la mise en forme, la mise en page et l'identité graphique. Cette séparation fondamentale entre structure et présentation constitue l'un des piliers de l'architecture web moderne.

Le navigateur télécharge le document HTML, construit l'arbre DOM (Document Object Model), puis applique les règles CSS pour calculer le rendu visuel final de chaque nœud. Ce processus, appelé rendering pipeline, se déroule en plusieurs étapes : le chargement des ressources, l'analyse syntaxique (parsing) du CSS, la construction de l'arbre des styles, la mise en page (layout) et enfin le dessin (painting) à l'écran. La compréhension de ce mécanisme est essentielle pour diagnostiquer les problèmes d'affichage.

Le CSS n'est pas un langage de programmation au sens impératif du terme : il ne décrit pas une séquence d'opérations, mais un ensemble de règles déclaratives qui spécifient l'apparence souhaitée. Cette nature déclarative le rend à la fois puissant pour le stylisme et parfois contre-intuitif lorsqu'il s'agit de résoudre des conflits entre règles concurrentes.

🎨
Analogie : la maison

Le HTML constitue l'ossature structurelle du document, comparable à la charpente et aux murs d'un édifice. Le CSS détermine l'apparence de cette structure : teintes, textures, éclairage, agencement spatial. Sans CSS, la page demeure fonctionnelle mais dépouillée d'habillage esthétique. Avec CSS, elle acquiert une identité visuelle cohérente et adaptée à son public.

Capacités du CSS

🎨 Couleurs & arrière-plans

Teinter le texte, les fonds, les bordures. Ajouter des dégradés, des images de fond, des motifs répétés.

📐 Mise en page

Positionner les éléments exactement où vous le souhaitez : en colonnes, en grille, centrés, collés en haut ou en bas.

🔤 Typographie

Polices, tailles, graisses, interlignage, espacement, alignement — tout ce qui rend le texte lisible et beau.

✨ Animations & transitions

Faire bouger, apparaître, disparaître, tourner, grossir des éléments — sans une ligne de JavaScript.

📱 Adaptation mobile

Un même HTML s'affiche parfaitement sur mobile, tablette et desktop grâce aux media queries.

♿ Accessibilité

Agrandir la taille du texte, augmenter les contrastes, gérer le focus clavier pour les utilisateurs malvoyants.

📌
Un langage en constante évolution

Le CSS n'est pas un langage figé. Chaque année, de nouvelles spécifications enrichissent ses capacités : les subgrid, les container queries, la pseudo-classe :has(), les @layers. Si la version de référence reste CSS3, les experts évoquent désormais un "CSS moderne" constitué de spécifications évolutives et cumulatives.

💡
Pour aller plus loin — Le rendering pipeline

Le processus de rendu d'un navigateur suit un pipeline précis : ① Parsing HTML → construction du DOM ; ② Parsing CSS → construction du CSSOM (CSS Object Model) ; ③ Render Tree → fusion du DOM et du CSSOM ; ④ Layout → calcul des dimensions et positions ; ⑤ Paint → dessin des pixels. Chaque modification d'une propriété CSS peut déclencher un ou plusieurs de ces processus. Les propriétés comme transform ou opacity sont optimisées car elles n'affectent que l'étape de compositing, évitant un recalcul complet de la mise en page.

02

Syntaxe CSS : règle, sélecteur, déclaration

Une feuille de style CSS se compose d'une série de règles. Chaque règle comprend un sélecteur — qui désigne le ou les éléments HTML ciblés — et un bloc de déclarations consistant en un ensemble de couples propriété : valeur. La syntaxe du CSS est remarquablement concise : un sélecteur, des accolades, et à l'intérieur des paires propriété-valeur séparées par des points-virgules.

Le sélecteur constitue le mécanisme de ciblage fondamental du CSS. Il peut désigner un type d'élément (p, div), une classe (.ma-classe), un identifiant (#mon-id), un attribut ([type="text"]), ou des combinaisons complexes utilisant des combinateurs (descendant , enfant direct >, adjacent +). La précision du sélecteur détermine directement la spécificité de la règle, un concept que nous explorerons en détail plus loin.

Chaque déclaration se compose d'une propriété et d'une valeur séparées par deux-points, l'ensemble étant terminé par un point-virgule. Le dernier point-virgule avant l'accolade fermante est techniquement optionnel selon la spécification, mais son omission systématique est déconseillée : l'ajout ultérieur d'une déclaration pourrait être oublié. Les commentaires en CSS s'écrivent entre /* et */ et peuvent s'étendre sur plusieurs lignes.

CSS
/* Tapez votre CSS ici — modifiez et voyez le résultat */ p { color: #2c3e50; font-size: 18px; line-height: 1.6; font-family: Georgia, serif; } h3 { color: #e74c3c; border-left: 4px solid #e74c3c; padding-left: 12px; }
APERÇU

Les trois éléments d'une règle

🎯 Sélecteur

Indique quel(s) élément(s) HTML sont ciblés. Peut être un nom de balise (p), une classe (.ma-classe), un id (#mon-id) ou une combinaison complexe.

📋 Déclaration

Couple propriété: valeur;. Chaque déclaration se termine obligatoirement par un point-virgule ;. Sans lui, la règle entière peut être ignorée.

📦 Bloc de déclarations

Ensemble de déclarations entre { }. Les accolades sont obligatoires, même pour une seule déclaration.

CSS — Sélecteurs multiples 
/* Plusieurs sélecteurs : on les sépare par une virgule */
h1, h2, h3 {
  font-family: 'Arial', sans-serif;
  color: #222;
}

/* Commentaires : entre /* et */. Ne pas hésiter à documenter */

/* Une propriété peut prendre plusieurs valeurs composées */
.bouton {
  border: 2px solid #e63946;
  padding: 8px 16px;
  border-radius: 4px;
  background: linear-gradient(to bottom, #fff, #f0f0f0);
}
⚠️
Erreur fréquente — Point-virgule oublié

Si vous oubliez le ; après une déclaration, la déclaration suivante sera ignorée (pas la courante). Le navigateur essaie de "réparer" mais le résultat est imprévisible. Le dernier point-virgule avant la fermeture est optionnel mais conseillé par convention.

💡
Pour aller plus loin — Les sélecteurs avancés

Au-delà des sélecteurs de base, CSS propose des mécanismes puissants : le sélecteur d'attribut [data-role="admin"], le sélecteur d'enfant direct ul > li, le sélecteur de frère adjacent h2 + p, et les pseudo-classes fonctionnelles comme :is() ou :where() qui permettent de regrouper des sélecteurs complexes. La spécification CSS évolue constamment : le sélecteur :has() (sélecteur "parent") est désormais supporté par tous les navigateurs modernes depuis 2024.

03

Trois méthodes pour appliquer du CSS

Trois méthodes permettent de lier du CSS à un document HTML. Chacune présente des usages, avantages et inconvénients spécifiques. Dans la pratique professionnelle, la troisième méthode — le fichier externe — est privilégiée dans la quasi-totalité des projets, car elle respecte le principe fondamental de séparation des préoccupations (separation of concerns).

Le CSS inline (attribut style sur une balise) possède la spécificité la plus élevée après !important, ce qui le rend difficile à surcharger. Il est utile pour des tests rapides mais son usage en production est généralement déconseillé. Le CSS interne (balise <style> dans le <head>) convient aux prototypes, aux pages uniques ou aux correctifs ponctuels. Le CSS externe (fichier .css lié via <link>) constitue la norme industrielle : il permet la mise en cache par le navigateur, la réutilisation entre pages, et une maintenance centralisée.

① CSS Inline

Dans l'attribut style de la balise HTML. Utile pour un test rapide. À éviter : mélange HTML/CSS, pas réutilisable, priorité maximale.

② CSS Interne

Dans une balise <style> dans le <head>. Pratique pour un fichier unique ou un prototype. Pas réutilisable entre pages.

③ CSS Externe ✅

Fichier .css séparé, lié via <link rel="stylesheet" href="style.css">. Standard professionnel : séparation totale, cache navigateur, réutilisable.

HTML — Trois méthodes 
<!-- ① INLINE — dans l'attribut style de la balise -->
<p style="color: red; font-weight: bold;">
  Ce texte est rouge et gras.
</p>

<!-- ② INTERNE — dans <head> via <style> -->
<head>
  <style>
    p { color: blue; }
  </style>
</head>

<!-- ③ EXTERNE — fichier .css séparé (recommandé) -->
<head>
  <link rel="stylesheet" href="style.css">
</head>
💡
Avantages du fichier externe

① Séparation des préoccupations : le HTML assure la structure, le CSS la présentation. ② Mise en cache : le fichier CSS est téléchargé une fois puis mis en cache par le navigateur, ce qui accélère le chargement des pages suivantes. ③ Maintenabilité : une modification unique dans le fichier CSS se répercute sur l'ensemble du site. ④ Optimisation : le fichier CSS peut être minifié pour réduire le temps de chargement.

04

La cascade : origine et ordre

Le « C » de CSS signifie Cascading (en cascade). Lorsque plusieurs règles ciblent un même élément, le navigateur applique un algorithme précis pour déterminer la règle prédominante. Cet algorithme repose sur trois critères hiérarchisés : l'origine des styles, la spécificité des sélecteurs, et l'ordre d'apparition dans le code source.

L'origine constitue le premier filtre. Le navigateur distingue trois sources : ses styles par défaut (agent utilisateur), les styles de l'auteur (le CSS que vous écrivez), et les styles de l'utilisateur (configurations personnelles du visiteur). Les styles d'agent utilisateur possèdent la priorité la plus faible, tandis que les styles de l'utilisateur avec !important surpassent toutes les autres déclarations.

À origine égale, la spécificité du sélecteur départage les règles concurrentes. Si deux déclarations ont la même spécificité, c'est la dernière déclarée dans le code source qui l'emporte — d'où l'importance de l'ordre d'importation des fichiers CSS. Ce mécanisme en cascade garantit un comportement prévisible et permet aux développeurs de structurer leurs feuilles de style de manière hiérarchique.

Les trois origines du CSS

OrigineExplicationPriorité
① Agent utilisateurStyle par défaut du navigateurFaible
② Auteur (vous)Le CSS que vous écrivezMoyenne
③ UtilisateurStyle défini par le visiteurHaute
CSS — Ordre d'apparition 
/* Ces deux règles ont la même spécificité (une balise) */
/* La dernière déclaration gagne */

p { color: red; }
p { color: blue; }
/* => Tous les <p> seront bleus car blue est déclaré après red */

/* Ordre d'importation des fichiers */
/* style.css */
p { color: green; }

/* custom.css (chargé après) */
p { color: orange; }
/* => Si custom.css est chargé après style.css, orange gagne */
📌
Cascade = algorithme en trois étapes

Le navigateur applique cet ordre pour chaque propriété : ① Origine → ② Spécificité → ③ Ordre. Si une règle gagne à l'étape 1, on ne passe pas à l'étape 2. Cet algorithme est implémenté de manière identique dans tous les navigateurs conformes aux spécifications du W3C.

05

La spécificité : calcul du score

Lorsque deux sélecteurs de même origine ciblent le même élément, celui dont la spécificité est la plus élevée l'emporte. La spécificité se calcule selon un score quadripartite noté (a, b, c, d) où chaque colonne représente un type de sélecteur : a pour les styles inline, b pour les identifiants, c pour les classes, attributs et pseudo-classes, et d pour les éléments et pseudo-éléments.

Ce système de notation implique qu'un sélecteur d'identifiant (score 0,1,0,0) l'emporte toujours sur n'importe quel nombre de sélecteurs de classe (0,0,n,0), quelle que soit la quantité. Toutefois, le sélecteur universel *, les combinateurs (>, +, ~) et la pseudo-classe :where() ne contribuent pas à la spécificité. Cette règle simple mais rigoureuse fonde la prévisibilité du CSS.

Le système de notation

Sélecteura (inline)b (id)c (classe, att, pseudo)d (balise, pseudo-élém)
style="..."1000
#contenu0100
.classe0010
[type="text"]0010
:hover0010
p0001
::before0001
* (universel)0000
CSS — Calcul de spécificité 
/* Calcul du score — chaque ligne a une spécificité différente */
p { color: red; }                    /* 0,0,0,1 = 1 */
p.important { color: orange; }       /* 0,0,1,1 = 11 */
#contenu p { color: green; }         /* 0,1,0,1 = 101 */
p.important#contenu { color: blue; } /* 0,1,1,1 = 111 */
/* style="color: purple" (inline)    /* 1,0,0,0 = 1000 */

/* Règles pratiques :
   - 1 id bat des milliers de classes
   - 1 classe bat des milliers de balises
   - L'inline bat tout (sauf !important) */
💡
Recommandation pour la gestion de la spécificité

Privilégiez l'usage des classes plutôt que des identifiants pour l'application des styles. Les classes (spécificité 0,0,1,0) offrent une plus grande flexibilité et réutilisabilité. Réservez les identifiants aux ancres de navigation et aux sélections JavaScript. Une méthodologie comme BEM (Bloc Élément Modificateur) peut vous aider à maintenir une spécificité plate et prévisible.

🚀
Pour aller plus loin — :where() et la spécificité nulle

La pseudo-classe :where(), introduite dans CSS Selectors Level 4, possède une spécificité toujours nulle, quel que soit le contenu de sa liste d'arguments. Ainsi, :where(.card, .panel) p a la même spécificité que p seul (0,0,0,1), ce qui facilite la surcharge ultérieure. À l'inverse, :is() hérite de la spécificité maximale de ses arguments. Cette distinction est cruciale pour concevoir des systèmes de composants faciles à personnaliser.

06

!important et l'héritage

Deux mécanismes peuvent modifier le comportement normal de la cascade : la déclaration !important qui impose une règle par-dessus toute autre considération, et l'héritage qui transmet automatiquement certaines propriétés des éléments parents aux éléments enfants.

L'ajout de !important après une valeur confère à cette déclaration la priorité maximale, surpassant même les styles inline. Ce mot-clé modifie le poids de la déclaration dans l'algorithme de cascade en la faisant passer dans une catégorie de priorité supérieure. Son usage intensif est généralement considéré comme une mauvaise pratique, car il brise le mécanisme naturel de cascade et rend le débogage difficile. Si vous utilisez !important fréquemment, c'est probablement le signe d'un problème de spécificité dans votre architecture CSS.

L'héritage est un mécanisme automatique par lequel certaines propriétés CSS sont transmises des éléments parents à leurs descendants. Les propriétés liées au texte — color, font-family, font-size, line-height — sont héritées, tandis que les propriétés de mise en page — margin, padding, border, width, height — ne le sont pas. Cette distinction est logique : un enfant doit naturellement hériter de la couleur du texte de son parent, mais pas de ses marges ou de ses bordures.

La déclaration !important

CSS — !important 
/* !important force la règle quoi qu'il arrive */
p { color: red !important; }

/* Même l'inline ne peut pas la surcharger */
/* <p style="color: blue"> → sera ROUGE à cause de !important */

/* ⚠️ À éviter dans la plupart des cas !
   !important rend le débogage très difficile.
   Préférez augmenter la spécificité. */

L'héritage — les enfants héritent des parents

Propriétés héritéesPropriétés NON héritées
colormargin
font-familypadding
font-sizeborder
line-heightbackground
text-alignwidth / height
visibilitydisplay
cursorposition
CSS — Héritage 
/* Héritage en action */
article {
  color: #333;
  font-family: 'Georgia', serif;
  line-height: 1.8;
}

/* Les mots-clés d'héritage */
.force-heritage {
  color: initial;    /* revient à la valeur par défaut */
  color: inherit;    /* force l'héritage */
  color: unset;      /* inherit si hérité, initial sinon */
}
⚠️
Ne pas abuser de !important

Un code avec beaucoup de !important devient rapidement ingérable. Si vous devez l'utiliser fréquemment, c'est souvent le signe que votre structure CSS a besoin d'être repensée.

🚀
Pour aller plus loin — Les valeurs globales d'héritage

En CSS, quatre mots-clés globaux permettent de contrôler l'héritage : inherit force l'héritage même pour les propriétés non héritées ; initial rétablit la valeur initiale définie par la spécification ; unset se comporte comme inherit pour les propriétés héritées et comme initial autrement ; revert rétablit la valeur qu'aurait eue la propriété sans l'intervention du CSS de l'auteur. Enfin, all: unset permet de réinitialiser l'ensemble des propriétés d'un élément en une seule déclaration, ce qui est utile dans les systèmes de conception (design systems) pour créer des composants isolés.

07

Inspecter le CSS avec les DevTools

Les outils de développement (DevTools) intégrés aux navigateurs modernes constituent des instruments essentiels pour le travail avec le CSS. Ils permettent de visualiser, tester et déboguer les styles en temps réel, sans modification du code source. La maîtrise des DevTools est une compétence fondamentale pour tout développeur web.

L'onglet « Styles » est le plus utilisé : il présente l'ensemble des règles CSS appliquées à l'élément sélectionné, classées par spécificité, avec les propriétés surchargées clairement identifiées (barrées). Chaque règle est accompagnée de sa provenance (fichier et numéro de ligne), ce qui facilite le repérage dans le code source. Les valeurs peuvent être modifiées dynamiquement par double-clic, avec un aperçu immédiat du résultat.

L'onglet « Computed » affiche la valeur finale calculée de chaque propriété après application de l'ensemble des règles. Il inclut un diagramme visuel du modèle de boîte (box model) montrant les dimensions du contenu, du padding, de la bordure et de la marge. Ces outils sont disponibles dans Chrome, Firefox, Edge et Safari avec des fonctionnalités équivalentes.

Comment ouvrir les DevTools

⌨️ Raccourcis clavier

F12 sur Chrome, Firefox, Edge
Ctrl+Maj+I (Windows/Linux)
Cmd+Opt+I (Mac)

🖱️ Clic droit

Faites un clic droit sur n'importe quel élément → "Inspecter".

Fonctionnalités clés

  • Inspecter un élément : icône 🖱️ en haut à gauche, puis cliquez sur l'élément
  • Modifier une valeur : double-clic sur une propriété dans l'onglet Styles
  • Ajouter une règle : cliquez sur le + dans l'onglet Styles
  • État de survol : cliquez sur :hov pour activer :hover, :focus
  • Box model visuel : l'onglet "Computed" montre le diagramme du box model
  • Onglet "Computé" : valeur finale calculée de chaque propriété
💡
Fonctionnalités avancées des DevTools

Les DevTools permettent également de : simuler des états CSS (:hover, :focus) via l'onglet Styles, éditer les media queries responsive, capturer des screenshots, analyser les performances de rendu, et visualiser les animations CSS frame par frame. Firefox propose un outil unique : l'éditeur de couleurs avec palettes et contrastes accessibles.

08

Normalize.css & Reset CSS

Chaque navigateur applique ses propres styles par défaut, ce qui peut entraîner des variations d'affichage significatives. Les marges du <body>, la taille des titres, le style des listes ou l'apparence des champs de formulaire diffèrent entre Chrome, Firefox, Safari et Edge. Pour garantir une base cohérente, on recourt à une feuille de style de normalisation ou de réinitialisation.

Le Reset CSS (popularisé par Eric Meyer) supprime tous les styles par défaut des navigateurs. Marges, paddings, styles de liste — tout revient à zéro. Cette approche offre une toile vierge complète, mais nécessite de redéfinir manuellement tous les styles de base, y compris ceux qui étaient utiles par défaut. Normalize.css (créé par Nicolas Gallagher et Jonathan Neal) adopte une approche plus subtile : il conserve les styles utiles par défaut, corrige les incohérences entre navigateurs et normalise les comportements pour les éléments HTML5.

Dans la pratique professionnelle, Normalize.css est souvent préféré au Reset CSS, car il préserve une base de styles fonctionnels tout en assurant la cohérence entre navigateurs. De nombreux frameworks CSS modernes (Tailwind CSS, Bootstrap 5) intègrent leur propre système de normalisation.

CSS — Reset vs Normalize 
/* ═══ RESET CSS (extrait — Eric Meyer) ═══ */
html, body, div, span, h1, h2, h3, h4, h5, h6,
p, a, ul, ol, li {
  margin: 0;
  padding: 0;
  border: 0;
  font-size: 100%;
  font: inherit;
  vertical-align: baseline;
}

/* ═══ NORMALIZE.CSS (extraits) ═══ */
html { line-height: 1.15; -webkit-text-size-adjust: 100%; }
body { margin: 0; }
h1 { font-size: 2em; margin: 0.67em 0; }
ul { list-style: disc; padding-left: 2em; }
💡
Recommandation pour vos projets

Il est recommandé d'utiliser Normalize.css comme base pour l'ensemble des projets. Placez-le en premier dans le <head>, avant votre fichier CSS personnel, de manière à établir une référence commune avant d'appliquer vos propres styles. Une alternative moderne consiste à utiliser @layer pour gérer l'ordre de cascade des différentes couches de styles.

🚀
Pour aller plus loin — @layer et la gestion de cascade

La règle @layer, introduite en CSS, permet de définir des couches de cascade explicites. Vous pouvez créer des couches nommées (par exemple @layer reset, base, components, utilities) et contrôler leur ordre de priorité. Les styles d'une couche déclarée en premier ont moins de priorité que ceux des couches suivantes, indépendamment de la spécificité. Cette fonctionnalité remplace avantageusement les conventions de nommage comme BEM pour gérer la spécificité à grande échelle.

09

Les propriétés CSS fondamentales

Les propriétés CSS qui suivent sont les plus couramment utilisées dans la conception web. Leur maîtrise est indispensable avant d'aborder des concepts plus avancés comme le box model, le positionnement ou les animations. Chaque propriété est présentée avec l'ensemble de ses valeurs possibles, ses particularités et des exemples d'utilisation.

Propriété color — couleur du texte

La propriété color définit la couleur de premier plan du contenu textuel d'un élément, ainsi que celle des bordures dont la couleur n'est pas explicitement spécifiée. C'est l'une des propriétés les plus fréquemment utilisées en CSS. La couleur peut être exprimée selon plusieurs formats.

clamp(min, préf, max)Taille fluide avec bornesclamp(1rem, 2.5vw, 2rem)
CSS — Font-size 
html  { font-size: 16px; }         /* base */
h1    { font-size: 2.5rem; }       /* 40px */
p     { font-size: 1rem; }         /* 16px */
small { font-size: .875rem; }      /* 14px */
.hero-title { font-size: clamp(1.8rem, 4vw, 4rem); } /* taille fluide */

Propriété line-height — interlignage

La propriété line-height détermine la hauteur de chaque ligne de texte. Elle influence directement la lisibilité : un interlignage trop serré fatigue la lecture, tandis qu'un interlignage excessif fragmente la cohésion textuelle. Les valeurs sans unité (facteur multiplicateur) sont recommandées car elles sont proportionnelles à la taille de police.

ValeurDescriptionComportement
normalValeur par défaut (~1.2 selon la police)Varie selon la police utilisée
<nombre>Facteur multiplicateur sans unité (recommandé)1.6 → hauteur = 1.6 × font-size
<length>Valeur absolue24px, 1.5rem
<percentage>Pourcentage de font-size160% équivaut à 1.6
CSS — Line-height 
body  { line-height: 1.6; }        /* valeur recommandée pour le texte */
h1    { line-height: 1.2; }        /* plus serré pour les titres */
.tight { line-height: 1; }         /* utile pour centrer verticalement */
p     { line-height: 1.8; }        /* interlignage aéré */

Propriété border — bordure

La propriété border est une propriété raccourcie qui définit l'épaisseur, le style et la couleur de la bordure d'un élément. Elle peut être déclinée par côté (border-top, border-right, border-bottom, border-left) ou par propriété composante (border-width, border-style, border-color).

ValeurSous-propriétéDescriptionExemple

<length>border-widthÉpaisseur expliciteborder-width: 2px

hiddenborder-styleIdentique à none, mais prioritaire dans les conflits de tableau

<color>border-colorCouleur de la bordureborder-color: #e74c3c

Propriété padding — espacement intérieur

La propriété padding définit l'espace entre le contenu d'un élément et sa bordure. Elle s'exprime avec 1 à 4 valeurs (haut, droite, bas, gauche). Le padding s'exprime en unités de longueur ou en pourcentage (relatif à la largeur du parent). Elle s'ajoute à la largeur du contenu sauf avec box-sizing: border-box.

SyntaxeDescriptionExemple
1 valeurAppliquée aux 4 côtéspadding: 16px
2 valeurs1re : haut/bas, 2e : droite/gauchepadding: 8px 16px
3 valeurshaut, droite/gauche, baspadding: 8px 16px 12px
4 valeurshaut, droite, bas, gauche (sens horaire)padding: 8px 12px 6px 12px
<percentage>% de la largeur du parent (tous côtés)padding: 5%
<length>px, em, rem, etc.padding: .75em 1.5em
CSS — Padding 
.btn     { padding: .75em 1.5em; }          /* haut/bas 12px, côtés 24px si font-size 16px */
.card    { padding: 24px; }                  /* 24px partout */
.compact { padding: 4px 8px; }               /* petit bouton */
.wide    { padding: 1rem 2rem; }             /* espacement généreux */

Propriété border-radius — coins arrondis

La propriété border-radius arrondit les coins de la bordure extérieure d'un élément. Elle peut recevoir une valeur unique pour les quatre coins, ou des valeurs différenciées par coin (haut-gauche, haut-droit, bas-droit, bas-gauche). Une barre oblique (/) permet de définir des rayons elliptiques.

ValeurDescriptionExemple
<length>Rayon d'arrondi uniformeborder-radius: 8px
<percentage>% de la dimension de la boîteborder-radius: 50% (cercle parfait)
2 valeurs1re : HG/BD, 2e : HD/BGborder-radius: 8px 16px
4 valeursHG, HD, BD, BG (sens horaire)border-radius: 4px 8px 12px 16px
v1 / v2Rayon horizontal / vertical (ellipse)border-radius: 50% / 20%
CSS — Border-radius 
.btn     { border-radius: 6px; }            /* coins légèrement arrondis */
.avatar  { border-radius: 50%; }            /* cercle parfait */
.pill    { border-radius: 9999px; }         /* forme de pilule */
.card    { border-radius: 12px 12px 0 0; }  /* seulement les coins supérieurs */

Propriété font-family — police de caractères

La propriété font-family définit la police de caractères utilisée pour le texte. Elle accepte une liste de priorités (fallback) : le navigateur utilise la première police disponible dans le système, puis la suivante, et ainsi de suite. La liste doit toujours se terminer par un nom de famille générique.

ValeurDescriptionExemple
"Police Name"Nom spécifique d'une police, entre guillemets si elle contient des espaces"Helvetica Neue", Georgia

sans-serifFamille générique sans empattementsArial, Helvetica
monospaceFamille générique à chasse fixeCourier New, Consolas
cursiveFamille générique simulant l'écriture manuscriteComic Sans MS, Brush Script
fantasyFamille générique décorativeImpact, Papyrus
system-uiPolice par défaut du système d'exploitationModerne, s'adapte à la plateforme

body { font-family: 'Inter', system-ui, -apple-system, sans-serif; }
code { font-family: 'JetBrains Mono', 'Fira Code', monospace; }
h1   { font-family: Georgia, 'Times New Roman', serif; }

Combinaison des propriétés fondamentales

Ces propriétés sont rarement utilisées de manière isolée. Leur combinaison produit des effets typographiques et de mise en forme complets. Par exemple, un bouton standard combine padding, border, border-radius, font-size et color pour produire un élément d'interface cohérent.

Mini-glossaire

CSS
Cascading Style Sheets — langage de style qui contrôle l'apparence des éléments HTML.
Règle CSS
Ensemble sélecteur + { déclarations }. Ex : p { color: red; }.
Sélecteur
Partie de la règle qui cible les éléments HTML.
Déclaration
Couple propriété : valeur terminé par un point-virgule.
Cascade
Algorithme qui détermine quelle règle s'applique en cas de conflit.
Spécificité
Score selon la précision du sélecteur (inline > id > classe > balise).
!important
Mot-clé qui force une déclaration au-dessus de toutes les autres.
Héritage
Transmission automatique de certaines propriétés des parents aux enfants.
Reset CSS
Feuille de style qui supprime tous les styles par défaut des navigateurs.
Normalize.css
Feuille de style qui harmonise les styles entre navigateurs.
