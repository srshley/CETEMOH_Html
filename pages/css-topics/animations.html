01

Introduction aux animations CSS

Les animations CSS permettent d'animer une page web sans recourir à JavaScript. Transitions, transformations, keyframes et filtres offrent un contrôle précis sur le mouvement, l'apparition et les effets visuels. Utilisées à bon escient, elles améliorent l'expérience utilisateur ; employées excessivement, elles deviennent perturbantes.

🎬
Analogie : le cinéma d'animation

Les transitions sont comme un zoom fluide entre deux plans. Les @keyframes sont un storyboard où chaque image clé définit une pose, l'ordinateur calculant les images intermédiaires (interpolation). Les filtres correspondent aux effets de post-production (flou, noir et blanc, saturation).

Quatre piliers des animations CSS

  • Transitions : fluidifient les changements d'état (hover, focus)
  • Transform : modifient la géométrie (rotation, échelle, déplacement) — accéléré GPU
  • @keyframes : animations complexes avec plusieurs étapes, en boucle ou en alternance
  • Filters : effets visuels (flou, luminosité, saturation) — animables
📌
CSS vs JavaScript pour les animations

Les animations CSS sont plus performantes (GPU), plus simples et ne nécessitent pas de bibliothèque. JavaScript est préférable pour les animations complexes (physique, timeline, scroll). Règle : si possible en CSS, faites-le en CSS.

02

Transitions CSS

Les transitions fluidifient les changements d'état (hover, focus, ajout de classe). Plutôt qu'une modification instantanée, la valeur évolue progressivement sur une durée définie selon une fonction de timing.

💡
Propriétés transitionnables

Toutes les propriétés ne sont pas animables. Les plus courantes : opacity, transform, background-color, color, box-shadow. display n'est pas animable — utilisez opacity et visibility. Évitez transition: all en production, listez les propriétés explicitement.

03

Transform : rotate, scale, translate, skew, matrix

transform modifie la forme, la taille et la position d'un élément sans affecter le flux du document. C'est la propriété la plus performante pour les animations car elle utilise l'accélération GPU (pas de reflow).

Référence complète — transform

FonctionSyntaxeDescriptionContexte d'utilisation
translate()translate(tx, ty)Déplace l'élément de tx px horizontalement et ty px verticalementAnimation de glissement (slide), centrage parfait avec translate(-50%, -50%)
translateX()translateX(tx)Déplacement horizontal uniquementDéfilement latéral, menu qui glisse de la gauche
translateY()translateY(ty)Déplacement vertical uniquementAnimation de fondu vers le haut, flottement
rotate()rotate(angle)Rotation autour du centre (ou du point défini par transform-origin)Spinner, icônes qui tournent, badges
scale()scale(sx, sy?)Mise à l'échelle. 1 = taille normale, 2 = double, 0.5 = moitiéEffet de zoom au survol, pulse, agrandissement
scaleX()scaleX(sx)Échelle horizontale uniquementBarre de progression, effets de distorsion
scaleY()scaleY(sy)Échelle verticale uniquementRévélation verticale, accordéon
skew()skew(ax, ay?)Inclinaison (déformation) de l'élément sur les axes X et YEffets 3D apparents, étiquettes inclinées
skewX()skewX(ax)Inclinaison horizontale uniquementBadges en forme de parallélogramme
skewY()skewY(ay)Inclinaison verticale uniquementEffets de perspective
matrix()matrix(a, b, c, d, tx, ty)Transformation 2D combinée via une matrice de 6 valeursUtilisé par les outils générés (pas besoin de l'écrire manuellement)

Référence complète — transform-origin

ValeurDescriptionExemple
centerPoint d'origine au centre de l'élément (défaut)Rotation normale
top leftOrigine dans le coin supérieur gaucheRotation qui part du coin
X% Y%Position personnalisée en pourcentagetransform-origin: 30% 70%
CSS 
.translate   { transform: translate(20px, 10px); }
.rotate      { transform: rotate(45deg); }
.scale       { transform: scale(1.2); }
.skew        { transform: skew(10deg, 5deg); }

/* Combinaison (ordre important !) */
.combo { transform: translateX(20px) rotate(45deg) scale(1.1); }

/* Point d'origine */
.origin-custom { transform-origin: 30% 70%; }
📌
Transform = GPU, pas CPU

Contrairement à margin, left, width ou height qui déclenchent un reflow, transform utilise exclusivement le GPU et ne déclenche que la composition. Pour des animations fluides à 60 fps, utilisez exclusivement transform et opacity.

04

@keyframes — Définir une animation

Les keyframes (images clés) définissent les étapes d'une animation avec des pourcentages (de 0 % à 100 %) ou les mots-clés from (0 %) et to (100 %). Le navigateur interpole automatiquement entre chaque étape.

Référence complète — @keyframes

SyntaxeDescriptionContexte d'utilisation
from { } to { }Animation à 2 étapes : état initial et état finalAnimations simples : fadeIn, slideIn, spin
0% { } … 100% { }Étapes multiples avec pourcentages intermédiairesAnimations complexes : pulse, shake, colorCycle avec 3+ étapes
0%, 100% { }Valeurs communes à plusieurs étapes (groupées par virgule)Éviter la répétition : état initial = état final dans une animation en boucle
CSS 
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25%      { transform: translateX(-10px); }
  75%      { transform: translateX(10px); }
}
05

Propriétés d'animation complètes

La propriété animation applique les keyframes à un élément. Elle regroupe plusieurs sous-propriétés qui contrôlent la durée, la répétition, la direction et le comportement avant/après l'animation.

Référence complète — animation-* (sous-propriétés)

PropriétéValeurs possiblesDescription
animation-nameNom des @keyframes (ex: fadeIn, pulse)Spécifie quelle règle @keyframes utiliser
animation-duration0s, .3s, 2s, 500ms, etc.Durée d'un cycle complet de l'animation
animation-timing-functionease, linear, ease-in, ease-out, ease-in-out, cubic-bezier(), steps(), step-start, step-endFonction d'accélération de l'animation (mêmes valeurs que transition-timing-function)
animation-delay0s, .5s, -1s (négatif accepté)Délai avant le début de l'animation. Négatif = commence à un état intermédiaire
animation-iteration-count1, 2, 2.5, infiniteNombre de répétitions de l'animation. Décimal possible pour une répétition partielle
animation-directionnormal, reverse, alternate, alternate-reverseSens de lecture : normal (0→100), reverse (100→0), alternate (va-et-vient), alternate-reverse (inverse)
animation-fill-modenone, forwards, backwards, bothÉtat de l'élément avant (backwards) et après (forwards) l'animation
animation-play-staterunning, pausedPermet de suspendre et reprendre une animation en cours

Référence complète — animation (shorthand)

SyntaxeDescriptionExemple
animation: name duration timing delay iteration direction fill-modeShorthand complet (ordre spécifique)animation: fadeIn .6s ease-out .2s 1 normal both
Ordrename → duration → timing → delay → iteration → direction → fill-modeLes seules valeurs obligatoires sont name et duration
Astuce délaiDeux valeurs temporelles : la 1ère = durée, la 2e = délaianimation: fadeIn .6s .2s (durée .6s, délai .2s)
CSS 
/* Sous-propriétés détaillées */
.element {
  animation-name: pulse;
  animation-duration: 2s;
  animation-timing-function: ease-in-out;
  animation-delay: 0s;
  animation-iteration-count: infinite;
  animation-direction: alternate;
  animation-fill-mode: both;
  animation-play-state: running;
}

/* Shorthand : nom durée timing délai répétition direction fill-mode */
.fade-in    { animation: fadeIn .6s ease-out .2s 1 normal both; }
.pulse-inf  { animation: pulse 2s ease-in-out infinite; }
.bounce-alt { animation: bounce 1s ease alternate infinite; }
.spinner    { animation: spin 1s linear infinite; }
06

animation-fill-mode et animation-play-state

animation-fill-mode contrôle l'état de l'élément avant et après l'animation. animation-play-state permet de suspendre et reprendre une animation. Ces deux propriétés sont essentielles pour des animations qui semblent naturelles et qui respectent l'interaction utilisateur.

Référence complète — animation-fill-mode

ValeurComportement avant l'animationComportement après l'animationContexte d'utilisation
noneAucun style particulier (défaut)Retour à l'état initialAnimation décorative qui n'altère pas l'apparence permanente
forwardsAucun style particulierConserve l'état final (100%/to)Animation d'entrée : l'élément reste visible après l'apparition
backwardsApplique l'état initial (0%/from) pendant le délaiRetour à l'état initialÉviter le flash : l'élément est déjà dans son état de départ avant le début
bothApplique l'état initial (comme backwards)Conserve l'état final (comme forwards)Recommandé pour la plupart des animations : pas de flash avant ni après

Référence complète — animation-play-state

ValeurDescriptionContexte d'utilisation
runningL'animation est en cours d'exécution (défaut)État normal de l'animation
pausedL'animation est suspendue à l'état actuelAu survol : figer une animation pour permettre la lecture. En JS : toggle play/pause
CSS 
.forwards  { animation: fadeIn 1s forwards; }
.both      { animation: fadeIn 1s .5s both; }

.spinner { animation: spin 2s linear infinite; }
.spinner:hover { animation-play-state: paused; }
⚠️
Sans fill-mode, l'élément disparaît après l'animation

Sans animation-fill-mode: both (ou forwards), un élément avec opacity: 0 dans @keyframes from sera visible avant le délai, puis disparaîtra subitement après la fin. both résout les deux : invisible avant, visible après.

07

Filters CSS : effets visuels complets

La propriété filter applique des effets visuels à un élément. backdrop-filter applique un filtre à l'arrière-plan (effet verre dépoli). Ces effets sont animables via transitions ou keyframes.

Référence complète — filter

FonctionPlage typiqueDescriptionContexte d'utilisation
blur(radius)0px à 10px+Applique un flou gaussien du rayon spécifiéEffet de verre dépoli, arrière-plan flouté, mise au point
brightness(amount)0 (noir) à 1 (normal) à 2+ (lumineux)Ajuste la luminosité : < 1 assombrit, > 1 éclaircitMode nuit, hover plus lumineux, correction d'exposition
contrast(amount)0 (gris) à 1 (normal) à 2+ (contrasté)Ajuste le contraste des couleursImages plus vives, mode contraste élevé, effets dramatiques
drop-shadow(h-offset v-offset blur spread color)Variables selon l'effetOmbre portée qui suit les contours réels de l'élément (contrairement à box-shadow)Ombre sur des PNG transparents, icônes, textes découpés
grayscale(amount)0 (couleur) à 1 (noir et blanc)Convertit l'image en nuances de grisImages en noir et blanc au repos, révélation couleur au hover
hue-rotate(angle)0deg à 360degFait tourner la teinte (cercle chromatique)Changement de couleur dynamique, effets arc-en-ciel animés
invert(amount)0 (normal) à 1 (inversé)Inverse les couleurs (négatif photo)Mode sombre automatique, effets artistiques
opacity(amount)0 (transparent) à 1 (opaque)Ajuste la transparence (similaire à la propriété opacity)Fondu, apparition/disparition progressive
saturate(amount)0 (désaturé) à 1 (normal) à 2+ (saturé)Ajuste la saturation des couleursImages plus vives, effet rétro désaturé
sepia(amount)0 (couleur) à 1 (sépia complet)Applique un filtre sépia (ton vieux papier photo)Effet vintage, photo ancienne, ambiance rétro
url(svg#id)Chemin vers un filtre SVGApplique un filtre SVG personnaliséEffets complexes impossibles en CSS pur
CSS 
.blur       { filter: blur(4px); }
.brightness { filter: brightness(1.2); }
.contrast   { filter: contrast(1.5); }
.grayscale  { filter: grayscale(1); }
.sepia      { filter: sepia(.8); }
.hue-rotate { filter: hue-rotate(90deg); }
.saturate   { filter: saturate(2); }
.invert     { filter: invert(1); }
.drop-shadow { filter: drop-shadow(4px 4px 8px rgba(0,0,0,.3)); }

/* Combinaisons */
.combo { filter: grayscale(.5) brightness(1.1) contrast(1.2); }

/* Au survol */
.card { filter: grayscale(1) blur(1px); transition: filter .3s; }
.card:hover { filter: grayscale(0) blur(0); }

/* backdrop-filter — effet verre dépoli */
.glass {
  background: rgba(255,255,255,.15);
  backdrop-filter: blur(10px);
  -webkit-backdrop-filter: blur(10px);
}
📌
backdrop-filter : l'effet verre dépoli

backdrop-filter applique un filtre à l'arrière-plan (ce qui se trouve derrière l'élément). Associé à un fond semi-transparent, il produit l'effet "glassmorphism". Supporté par tous les navigateurs depuis 2020.

08

Performance et accessibilité

Pour des animations fluides à 60 images par seconde, utilisez transform et opacity (GPU, pas de reflow). Évitez les propriétés qui déclenchent un reflow (width, height, margin, padding, top, left).

Référence complète — Performances des propriétés

PropriétéImpactOpération de renduRecommandation
transform🟢 Très performantComposite uniquement (GPU)À utiliser pour toute animation de position/taille/rotation
opacity🟢 Très performantComposite uniquement (GPU)À utiliser pour toute animation de transparence
filter🟡 MoyenPaint + CompositeUtiliser avec modération, surtout sur mobile
background-color🟡 MoyenPaint + CompositeAcceptable pour des transitions simples
box-shadow🟡 MoyenPaint + CompositeÉviter les animations d'ombres complexes
width, height🔴 CoûteuxLayout + Paint + CompositeÉviter ; utiliser transform: scale() à la place
top, left🔴 CoûteuxLayout + Paint + CompositeÉviter ; utiliser transform: translate() à la place
margin, padding🔴 CoûteuxLayout + Paint + CompositeNe pas animer
font-size🔴 CoûteuxLayout + Paint + CompositeNe pas animer

Référence complète — prefers-reduced-motion

Media queryDescriptionContexte d'utilisation (exigence WCAG)
prefers-reduced-motion: reduceL'utilisateur a demandé une réduction des mouvements dans son système d'exploitationDésactiver toutes les animations, limiter les transitions, supprimer les défilements automatiques
prefers-reduced-motion: no-preferenceL'utilisateur n'a pas de préférence particulièreComportement normal : toutes les animations sont actives
CSS 
/* ⚡ Bonnes pratiques : transform + opacity */
.animate { transform: translateX(100px); opacity: .5; }

/* 🔴 À éviter : propriétés qui déclenchent un reflow */
/* width, height, margin, padding, top, left, font-size */

/* ♿ Accessibilité : désactiver les animations */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
    scroll-behavior: auto !important;
  }
}

/* will-change : optimisation explicite */
.card { will-change: transform, opacity; }
⚠️
WCAG 2.1 — Animation from Interactions (AAA)

Le critère WCAG 2.3.3 exige que les mouvements déclenchés par une interaction puissent être désactivés via prefers-reduced-motion. Cette exigence est inscrite dans la législation de plusieurs pays pour les sites publics.

Exercices pratiques

EX 01 ✨ Animations & transitions
  1. Créez exo-animations.html avec un bouton qui change de couleur, d'échelle et d'ombre au hover.
  2. Ajoutez un spinner avec @keyframes spin.
  3. Une carte apparaît avec fadeIn et fill-mode: both.
  4. Un badge "Nouveau" pulse en continu.

  6. Animation flottante avec transform: translateY alterné.

Bilan & mini-glossaire

Compétences acquises

  • Transitions CSS : property, duration, timing-function, delay, shorthand
  • Transform : translate, rotate, scale, skew, matrix, transform-origin
  • @keyframes : from/to, 0% à 100%, étapes multiples
  • Animation : name, duration, timing, delay, iteration, direction, fill-mode, play-state, shorthand
  • Filters : blur, brightness, contrast, drop-shadow, grayscale, hue-rotate, invert, opacity, saturate, sepia, url
  • backdrop-filter pour l'effet verre dépoli
  • Performance : transform + opacity = GPU ; éviter les propriétés à reflow
  • will-change pour l'optimisation explicite
  • Accessibilité : prefers-reduced-motion (exigence WCAG)

Mini-glossaire

Transition
Animation fluide entre deux états (hover, focus, changement de classe).
Transform
Modification géométrique (déplacement, rotation, échelle). Accéléré GPU.
@keyframes
Règle CSS définissant les étapes d'une animation (pourcentages ou from/to).
animation-fill-mode
État avant/après l'animation : none, forwards, backwards, both.
animation-play-state
running (lecture) ou paused (pause).
Timing function
Fonction d'accélération : ease, linear, ease-in, ease-out, cubic-bezier, steps().
Filter
Effet visuel CSS : flou, N&B, luminosité, ombre portée, etc.
backdrop-filter
Filtre appliqué à l'arrière-plan d'un élément (effet verre dépoli).
GPU acceleration
Utilisation du processeur graphique. Transform + Opacity = 60 fps.
will-change
Propriété prévenant le navigateur qu'une propriété va être animée.
prefers-reduced-motion
Media query pour désactiver les animations (accessibilité).
