01

Les différents formats de couleurs

Le langage CSS prend en charge plusieurs formats pour définir une couleur. Chacun présente des avantages spécifiques selon le contexte d'utilisation : précision, gestion de la transparence, lisibilité du code source. Le choix du format dépend des exigences du projet : une couleur fixe sera exprimée efficacement en hexadécimal, tandis qu'une couleur nécessitant une transparence variable utilisera les notations rgba ou hsla. La spécification CSS Color Module Level 4 a modernisé l'ensemble de ces formats en autorisant l'écriture sans virgules et en unifiant la syntaxe du canal alpha.

Interchangeabilité des formats de couleurs

L'ensemble de ces formats est interchangeable : tout format valide peut être utilisé avec n'importe quelle propriété CSS acceptant une valeur de couleur (color, background, border-color, etc.). Le navigateur interprète chacun d'eux de manière équivalente. Il convient de sélectionner le format qui optimise la lisibilité et la maintenabilité du code.

02

Noms de couleurs prédéfinis

La spécification CSS définit 148 noms de couleurs reconnus par l'ensemble des navigateurs. Ceux-ci comprennent des valeurs élémentaires (red, blue) aussi bien que des nuances plus spécifiques (papayawhip, rebeccapurple). Les 16 couleurs HTML d'origine (aqua, black, blue, fuchsia, gray, green, lime, maroon, navy, olive, purple, red, silver, teal, white, yellow) constituent le jeu minimal garanti depuis les débuts du web. Les 132 couleurs supplémentaires ont été ajoutées par les spécifications CSS Color Module Level 3 et 4, incluant des noms comme crimson, coral, mediumseagreen ou slategray.

CSS — Noms de couleurs
/* Couleurs de base — les 16 couleurs HTML d'origine */ .color-red { color: red; } .color-blue { color: blue; } .color-green { color: green; } .color-yellow { color: yellow; } /* Couleurs nommées avancées */ .color-tomato { background: tomato; } .color-cornflowerblue { background: cornflowerblue; } .color-rebeccapurple { background: rebeccapurple; } .color-papayawhip { background: papayawhip; } /* Noms utiles pour le design */ .color-whitesmoke { background: whitesmoke; } .color-crimson { background: crimson; } .color-coral { background: coral; } .color-turquoise { background: turquoise; }
💡
Limites des noms de couleurs

Les noms de couleurs sont pratiques pour le prototypage rapide, mais leur gamme reste limitée. Seules 148 nuances sont disponibles, contre plus de 16 millions avec les notations hexadécimale ou rgb. Pour un design précis, on privilégiera les formats hex ou hsl. Les noms conservent une utilité dans un contexte pédagogique ou pour des styles très simples.

03

Hexadécimal (#RRGGBB, #RGB, #RRGGBBAA)

Le format hexadécimal est le plus répandu en production. Chaque paire de caractères représente une composante (Rouge, Vert, Bleu) dont la valeur s'étend de 00 à FF (soit 0 à 255 en notation décimale). Le système hexadécimal utilise une base 16 avec les symboles 0-9 puis A-F. Ainsi #ff6600 se décompose comme suit : Rouge = ff (255), Vert = 66 (102), Bleu = 00 (0). Le format court #f60 est accepté lorsque chaque paire présente deux chiffres identiques. Le canal alpha s'étend de 00 (transparent) à ff (opaque), offrant 256 niveaux de transparence.

CSS — Hexadécimal
/* #RRGGBB — format standard (6 chiffres) */ .hex-1 { color: #ff6600; } /* orange */ .hex-2 { color: #3498db; } /* bleu */ .hex-3 { color: #e74c3c; } /* rouge */ .hex-4 { color: #2ecc71; } /* vert */ /* #RGB — format court (3 chiffres), chaque chiffre est doublé */ .hex-short-1 { color: #f60; } /* = #ff6600 */ .hex-short-2 { color: #0af; } /* = #00aaff */ /* #RRGGBBAA — hex avec transparence (8 chiffres) */ .hex-alpha-1 { background: #e74c3c80; } /* rouge à 50% */ .hex-alpha-2 { background: #3498db40; } /* bleu à 25% */ .hex-alpha-3 { background: #2ecc71cc; } /* vert à 80% */
📌
Comprendre le codage hexadécimal

Le système hexadécimal utilise une base 16 avec les symboles 0-9 puis A-F. Ainsi #ff6600 se décompose comme suit : Rouge = ff (255), Vert = 66 (102), Bleu = 00 (0). Le format court #f60 est accepté lorsque chaque paire présente deux chiffres identiques : #f60 équivaut à #ff6600 car f = ff, 6 = 66, 0 = 00. Le canal alpha s'étend de 00 (transparent) à ff (opaque).

04

RGB et RGBA

Le format RGB (Red Green Blue) utilise trois nombres compris entre 0 et 255 pour définir une couleur. RGBA étend ce format en ajoutant un quatrième paramètre déterminant le niveau de transparence (canal alpha). La syntaxe moderne (CSS Color Level 4) permet d'écrire rgb(52 152 219 / .5) sans virgules, avec une barre oblique pour séparer le canal alpha. Cette évolution harmonise la notation avec les autres fonctions couleur et améliore la lisibilité.

CSS — RGB / RGBA
/* RGB — chaque canal de 0 à 255 */ .rgb-1 { background: rgb(52, 152, 219); } /* bleu */ .rgb-2 { background: rgb(46, 204, 113); } /* vert */ .rgb-3 { background: rgb(231, 76, 60); } /* rouge */ /* RGBA — avec transparence (alpha: 0 à 1) */ .rgba-1 { background: rgba(52, 152, 219, .3); } /* bleu transparent */ .rgba-2 { background: rgba(231, 76, 60, .6); } /* rouge semi-transparent */ .rgba-3 { background: rgba(0, 0, 0, .1); } /* noir très léger */ /* Syntaxe moderne : virgules optionnelles */ .rgba-new { background: rgba(52 152 219 / .5); } /* CSS Color Level 4 */
💡
Quand utiliser RGB plutôt que l'hexadécimal

RGB se révèle plus pertinent lorsque les couleurs sont manipulées par programmation ou lorsqu'on souhaite exprimer une valeur en termes de canaux (ex : « un rouge vif » = rgb(255, 0, 0) est plus immédiat que #ff0000). Le format RGBA est particulièrement adapté à la superposition de calques avec transparence, à la création d'overlays, ou à l'atténuation d'une couleur sans en modifier la teinte.

05

HSL et HSLA

HSL (Hue, Saturation, Lightness) est généralement considéré comme plus intuitif que RGB d'un point de vue perceptif. La teinte (Hue) est exprimée par un angle sur le cercle chromatique (0–360 degrés) : 0° = rouge, 60° = jaune, 120° = vert, 180° = cyan, 240° = bleu, 300° = magenta, 360° = rouge (bouclage). La saturation (S) détermine l'intensité de la couleur (0% = gris neutre, 100% = couleur vive), tandis que la luminosité (L) contrôle la clarté (0% = noir, 50% = couleur pure, 100% = blanc). Cette organisation tri-dimensionnelle permet de naviguer dans l'espace colorimétrique de manière prévisible : en faisant varier la luminosité, on génère instantanément des ombres et des reflets d'une même teinte.

Référence complète — hsl() et hsla()

L (Lightness)0%–100%Luminosité. 0% = noir, 50% = normal, 100% = blanc.Création de variantes claires/foncées

CSS — HSL / HSLA
/* HSL : teinte (0-360), saturation (0-100%), luminosité (0-100%) */ .hsl-1 { background: hsl(0, 100%, 50%); } /* rouge pur */ .hsl-2 { background: hsl(120, 100%, 50%); } /* vert pur */ .hsl-3 { background: hsl(240, 100%, 50%); } /* bleu pur */ .hsl-4 { background: hsl(210, 70%, 53%); } /* bleu comme #3498db */ /* Faire varier la saturation et la luminosité */ .hsl-sat { background: hsl(210, 100%, 53%); } /* saturation max */ .hsl-desat { background: hsl(210, 30%, 53%); } /* moins saturé, plus gris */ .hsl-light { background: hsl(210, 70%, 80%); } /* clair (pastel) */ .hsl-dark { background: hsl(210, 70%, 30%); } /* foncé */ /* HSLA — avec transparence */ .hsla-1 { background: hsla(210, 70%, 53%, .5); }
APERÇU
💡
Avantages de HSL pour la création de variations chromatiques

Avec HSL, il est possible de générer une palette complète à partir d'une teinte unique en faisant varier la saturation (vif → grisé) et la luminosité (clair → foncé). Exemple : hsl(210, 70%, 95%) pour un fond très clair, hsl(210, 70%, 53%) pour la couleur principale, hsl(210, 70%, 30%) pour un texte foncé. Cette opération s'avère considérablement plus complexe à réaliser avec les formats HEX ou RGB.

06

Opacité et transparence

Le langage CSS offre plusieurs mécanismes pour gérer la transparence : la propriété opacity (qui affecte l'intégralité de l'élément) et le canal alpha intégré aux valeurs de couleur (qui n'affecte que la propriété concernée). Cette distinction est fondamentale. opacity rend transparent tout l'élément et ses descendants, y compris le texte, les images et les éléments enfants. Le canal alpha, quant à lui, s'applique propriété par propriété : un background: rgba(0,0,0,.5) laisse le texte parfaitement opaque. Pour créer des effets de superposition (overlays), on combine généralement un fond semi-transparent avec du texte opaque, soit via une pseudo-élément, soit via un dégradé.

Référence complète — opacity et transparence

opacity: 0Élément invisible mais présent dans le flux (contrairement à display: none et visibility: hidden)Animations d'apparition, overlays invisibles

CSS — Opacité vs Alpha
/* opacity — affecte TOUT l'élément (fond + contenu) */ .opacity-full { opacity: 1; } /* opaque (défaut) */ .opacity-half { opacity: .5; } /* 50% — le texte aussi devient transparent */ .opacity-zero { opacity: 0; } /* invisible mais présent dans le flux */ /* Alpha — affecte UNE propriété seulement */ .alpha-bg { background: rgba(52, 152, 219, .3); /* seul le fond devient transparent */ color: #2c3e50; /* le texte reste opaque */ } /* Bordure transparente */ .alpha-border { border: 2px solid rgba(231, 76, 60, .5); } /* Ombre avec transparence */ .alpha-shadow { box-shadow: 0 4px 12px rgba(0, 0, 0, .15); }
⚠️
Opacity rend tout transparent, y compris le texte

L'application de opacity: .5 à un élément rend l'intégralité de son contenu (texte, images, éléments descendants) semi-transparent. Pour obtenir un fond semi-transparent avec un texte opaque, il convient d'utiliser background: rgba(...) ou background: hsla(...). Le canal alpha appliqué à la couleur de fond n'affecte que celle-ci, pas le contenu.

CSS — Overlay semi-transparent
/* Overlay semi-transparent sur une image */ .overlay-container { position: relative; } .overlay-container::after { content: ''; position: absolute; inset: 0; background: rgba(0, 0, 0, .4); /* fond semi-transparent */ } /* Solution avec dégradé */ .hero { background: linear-gradient(rgba(0,0,0,.5), rgba(0,0,0,.5)), url('hero.jpg') center/cover; }
07

currentColor — la couleur qui suit le texte

currentColor est un mot-clé représentant la valeur courante de la propriété color de l'élément. Il permet de concevoir des composants dont les bordures, les ombres, les fonds ou les icônes s'adaptent automatiquement lors d'un changement de couleur du texte. La valeur est résolue dynamiquement : si la couleur du texte change (via un sélecteur hover, une media query, ou une variable CSS), toutes les propriétés utilisant currentColor se mettent à jour simultanément. C'est particulièrement utile pour les icônes SVG (propriété fill: currentColor), les boutons outline, et les systèmes de design où un composant doit s'adapter à son contexte sans duplication de code.

CSS — currentColor
/* currentColor = la valeur de color au moment de l'utilisation */ .btn { color: #3498db; background: transparent; border: 2px solid currentColor; /* bordure = même couleur que le texte */ padding: .75em 1.5em; border-radius: 6px; transition: all .2s; } .btn:hover { color: white; background: #3498db; } /* SVG et icônes */ .icon { fill: currentColor; /* l'icône prend la couleur du texte */ width: 24px; height: 24px; }
💡
currentColor comme outil de thématisation

Avec currentColor, un composant s'adapte automatiquement à son contexte. La modification de la couleur du parent entraîne la mise à jour des bordures, icônes et ombres des éléments descendants. Cette approche est particulièrement utile dans le cadre de systèmes de design et de composants réutilisables. La combinaison avec les variables CSS permet d'obtenir des résultats encore plus puissants.

08

Bonnes pratiques et accessibilité des couleurs

Le choix des couleurs ne relève pas uniquement de considérations esthétiques — il engage également la question de l'accessibilité. Un contraste insuffisant rend le texte illisible pour des millions d'utilisateurs (malvoyance, daltonisme, conditions d'éclairage défavorables). Le WCAG (Web Content Accessibility Guidelines) définit des ratios de contraste minimaux entre le texte et son arrière-plan. Il ne faut jamais utiliser la couleur comme unique moyen de transmission d'une information importante : un champ de formulaire qui devient rouge en cas d'erreur doit également comporter un message texte ou une icône. Environ 8% des hommes présentent une forme de daltonisme (principalement la deutéranopie, difficulté à distinguer le rouge et le vert).

Ratios de contraste WCAG

NiveauTexte normal (< 18px)Texte large (≥ 18px ou bold ≥ 14px)
AA (minimum)4.5:13:1
AAA (amélioré)7:14.5:1
CSS — Accessibilité des couleurs
/* ❌ MAUVAIS — contraste insuffisant */ .bad-contrast { color: #999999; /* gris clair */ background: #f5f5f5; /* fond presque blanc — ratio ~1.7:1 */ } /* ✅ BON — contraste suffisant (AA) */ .good-contrast { color: #2c3e50; /* texte foncé */ background: #f8f9fa; /* fond clair — ratio ~10.5:1 */ } /* ✅ Ne pas se fier uniquement à la couleur */ a { color: #3498db; text-decoration: underline; /* indicateur visuel supplémentaire */ } /* ✅ Mode sombre avec variables CSS */ :root { --text: #2c3e50; --bg: #ffffff; } @media (prefers-color-scheme: dark) { :root { --text: #e4e6eb; --bg: #18191a; } }
⚠️
La couleur n'est jamais le seul vecteur d'information

Il ne faut jamais utiliser la couleur comme unique moyen de transmission d'une information importante. Exemples : un champ de formulaire qui devient rouge en cas d'erreur doit également comporter un message texte ou une icône. Un graphique utilisant des couleurs doit aussi inclure des motifs ou des étiquettes. Environ 8% des hommes présentent une forme de daltonisme (principalement la deutéranopie, difficulté à distinguer le rouge et le vert).

💡
Palette accessible : configuration minimale

Pour un site accessible, il convient de définir au minimum : une couleur de texte foncé (#2c3e50 ou #1a1a2e), un fond clair (#f8f9fa ou #ffffff), une couleur d'accent présentant un bon contraste (ex : #3498db sur fond blanc = 4.2:1), et une couleur d'erreur qui ne repose pas exclusivement sur le rouge. Il est impératif de tester avec un vérificateur de contraste avant de valider une palette.

Exercices pratiques

EX 01 🎨 Palette de couleurs complète
  1. Créez exo-colors-palette.html avec une page présentant 12 couleurs.
  2. Utilisez au moins 4 formats différents : noms, hex, rgb/hsla, hsl.
  3. Pour chaque couleur, affichez son nom, son code, et un bloc de couleur.
  4. Créez une section "Teintes dérivées" : prenez une couleur de base en hsl et faites varier la luminosité de 10% à 90%.
  5. Ajoutez une carte avec background: rgba(...) où le texte reste opaque.
  6. Utilisez currentColor sur des bordures de boutons.
EX 02 ♿ Accessibilité et contraste
  1. Créez exo-access-colors.html avec trois sections : une avec bon contraste, une avec mauvais contraste, une avec les deux.
  2. Utilisez le vérificateur WebAIM pour mesurer les ratios.
  3. Ajoutez une section "Ne pas utiliser la couleur seule" : créez des alertes (succès, erreur, info) qui utilisent des icônes ET des couleurs.
  4. Simulez le daltonisme dans les DevTools (Rendering → Emulate vision deficiencies) et ajustez les couleurs si nécessaire.
  5. Ajoutez une media query prefers-color-scheme: dark pour un mode sombre accessible.
🔍
Exercice bonus — Dégradés et transparence

Créez une page avec des dégradés utilisant rgba et hsla. Exemple : background: linear-gradient(to right, rgba(52,152,219,.8), rgba(46,204,113,.8)). Ajoutez un effet de verre (glassmorphism) avec backdrop-filter: blur(10px) et un fond rgba. Superposez du texte opaque sur des fonds transparents.
