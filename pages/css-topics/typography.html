01

font-family — Choix d'une police de caractères

La propriété font-family spécifie la police de caractères employée pour le rendu du texte. Il est impératif de toujours fournir une liste de polices de secours (fallback) séparées par des virgules, dans la mesure où il n'est pas possible de garantir qu'une police particulière soit installée sur le système de l'utilisateur. Cette pratique constitue un fondement de la robustesse typographique sur le Web.

Les valeurs sont évaluées dans l'ordre de déclaration : le navigateur utilise la première police disponible dans la liste. La liste doit toujours se terminer par une famille générique (serif, sans-serif, monospace, etc.) pour garantir qu'une police sera toujours appliquée.

Référence complète — font-family

ValeurDescriptionContexte d'utilisation
<family-name>Nom d'une police spécifique (ex: Arial, Georgia, 'Times New Roman'). Les noms contenant des espaces doivent être entre guillemets.Police choisie pour le design
serifPolice avec empattements (petits pieds aux extrémités des lettres).Texte imprimé, articles, blogs — lecture longue
sans-serifPolice sans empattements, style moderne et épuré.Écrans, interfaces, sites modernes — lecture rapide
monospaceChasse fixe : toutes les lettres occupent la même largeur.Code source, données tabulaires, terminaux
cursiveStyle manuscrit ou calligraphique.Citations, signatures, décorations
fantasyPolice décorative, non standard, ludique.Titres, effets spéciaux, sites thématiques
system-uiPolice par défaut du système d'exploitation de l'utilisateur.Performance maximale, aspect natif
ui-serifVariante serif de la police système.Interfaces natives avec empattements
ui-sans-serifVariante sans-serif de la police système.Interfaces natives modernes
ui-monospaceVariante monospace de la police système.Éditeurs de code natifs
ui-roundedPolice système arrondie.Interfaces ludiques, applications enfants
mathPolice conçue pour les notations mathématiques.Formules, équations, documents scientifiques
emojiPolice dédiée aux caractères emoji.Affichage des emojis, pictogrammes
fangsongStyle de caractères semi-cursif chinois.Typographie chinoise, documents officiels
CSS 
/* Toujours terminer par la famille générique */
body {
  font-family: 'Inter', 'Helvetica Neue', Arial, sans-serif;
}

/* Titres en serif, texte en sans-serif */
h1, h2, h3 {
  font-family: Georgia, 'Times New Roman', serif;
}

/* Code en monospace */
code, pre {
  font-family: 'Fira Code', 'JetBrains Mono', Consolas, monospace;
}

/* Police avec espace dans le nom → guillemets */
.font-custom {
  font-family: 'Playfair Display', serif;
}
💡
Polices système : une alternative performante

Les polices système sont préinstallées sur le système d'exploitation de l'utilisateur. Le recours à system-ui, -apple-system, Segoe UI ou sans-serif permet d'obtenir un rendu natif sans aucun téléchargement externe. Cette approche constitue la solution la plus performante en termes de temps de chargement : font-family: system-ui, -apple-system, 'Segoe UI', Roboto, sans-serif;.

02

font-size — Dimensionnement du texte

La propriété font-size détermine la taille du texte. Il est recommandé d'utiliser des unités relatives (rem, em, %) plutôt que des unités absolues (px) afin de respecter les préférences d'accessibilité de l'utilisateur et d'assurer une adaptation aux différents contextes de visualisation.

La valeur par défaut est medium (généralement 16px sur les navigateurs de bureau). L'unité rem est préférée car elle réfère à la taille de police de l'élément racine (html), ce qui la rend prévisible et ne dépend pas de l'imbrication des éléments.

Référence complète — font-size

ValeurDescriptionContexte d'utilisation
xx-smallTrès très petite taille (≈ 9px).Mentions légales, notes de bas de page
x-smallTrès petite taille (≈ 10px).Informations secondaires
smallPetite taille (≈ 13px).Texte secondaire, légendes
mediumTaille moyenne — valeur par défaut (≈ 16px).Texte courant des paragraphes
largeGrande taille (≈ 18px).Texte mis en évidence
x-largeTrès grande taille (≈ 24px).Sous-titres, intertitres
xx-largeTrès très grande taille (≈ 32px).Titres de section
xxx-largeExtrêmement grande taille (≈ 48px).Titres principaux, hero
smallerPlus petit que la taille de police de l'élément parent.Texte hiérarchique imbriqué
largerPlus grand que la taille de police de l'élément parent.Mise en évidence relative
<length>Valeur numérique avec unité : px, em, rem, pt, cm, mm, in, pc, q.Contrôle précis de la taille
<percentage>Pourcentage de la taille de police de l'élément parent. 150% = 1.5× la taille du parent.Dimensionnement relatif contextuel
inheritHérite de la valeur calculée de l'élément parent.Forcer l'héritage
initialRétablit la valeur initiale (medium).Réinitialisation
CSS 
/* Échelle typographique en rem (recommandé) */
html { font-size: 16px; } /* valeur par défaut */
h1   { font-size: 2.5rem; } /* 40px */
h2   { font-size: 2rem; }   /* 32px */
h3   { font-size: 1.5rem; } /* 24px */
h4   { font-size: 1.25rem; }/* 20px */
p    { font-size: 1rem; }   /* 16px */
small { font-size: .875rem; }/* 14px */
.tiny { font-size: .75rem; } /* 12px */

/* Tailles absolues (px) — à éviter pour le texte */
.px-example { font-size: 18px; }

/* Tailles relatives en % */
.percent { font-size: 150%; } /* 1.5× la taille du parent */
📌
Justification du rejet des unités px pour font-size

Lorsqu'un utilisateur a configuré son navigateur avec une taille de police par défaut de 20px (pour des raisons liées à une déficience visuelle), un texte défini en 16px demeure à 16px et ignore ce réglage utilisateur. En revanche, l'unité 1rem (équivalant à la taille définie à la racine) s'adapte automatiquement à cette préférence. Il s'agit d'un enjeu fondamental d'accessibilité.

03

font-weight & font-style

La propriété font-weight régit la graisse (épaisseur) du texte, tandis que font-style gère l'inclinaison italique. L'application de ces propriétés est conditionnée par les variantes disponibles au sein de la police utilisée. Toutes les polices ne disposent pas de l'ensemble des graisses : l'utilisation d'une graisse non prise en charge entraîne la sélection de la valeur la plus proche par le navigateur.

800Extra Bold.Titres percutants, affiches

950Extra Black — disponible dans les polices variables.Effets extrêmes, typographie display
normalÉquivalent à 400.Valeur par défaut
boldÉquivalent à 700.Gras standard
lighterPlus léger que le parent (sélectionne la graisse inférieure disponible).Hiérarchie relative

/* font-weight : valeurs numériques 100–900 */
.thin   { font-weight: 100; }
.light  { font-weight: 300; }
.normal { font-weight: 400; }
.medium { font-weight: 500; }
.bold   { font-weight: 700; }
.black  { font-weight: 900; }

/* Mots-clés équivalents */
.normal { font-weight: normal; } /* = 400 */
.bold   { font-weight: bold; }   /* = 700 */

/* font-style */
.italic  { font-style: italic; }
.normal  { font-style: normal; }
.oblique { font-style: oblique; }
⚠️
Disponibilité limitée des graisses

L'utilisation d'une valeur font-weight non prise en charge par la police conduit le navigateur à sélectionner la valeur la plus proche. Lors du recours à Google Fonts, il convient de sélectionner explicitement les graisses souhaitées dans l'URL de chargement. Les polices variables (font-variation-settings) offrent une flexibilité accrue pour le réglage fin de la graisse.

}
05

line-height — Hauteur de ligne

La propriété line-height définit l'interlignage, c'est-à-dire la hauteur allouée à chaque ligne de texte. Il s'agit d'un facteur déterminant pour la lisibilité du contenu. La valeur préconisée pour les paragraphes se situe entre 1.5 et 1.8.

La règle fondamentale est d'utiliser un nombre sans unité pour line-height, car cette valeur reste alors proportionnelle à la taille de police courante, quel que soit l'héritage.

Référence complète — line-height

ValeurDescriptionContexte d'utilisation
normalValeur par défaut, déterminée par le navigateur (généralement 1.2).Texte sans réglage spécifique
<number>Nombre sans unité multiplié par la taille de police courante. Ex: 1.6 = 1.6 × font-size. Valeur recommandée.Meilleure pratique — toutes les situations
<length>Valeur absolue avec unité. Ex: 24px, 1.5em. Déconseillé car non proportionnel.Cas spécifiques où une hauteur fixe est requise
<percentage>Pourcentage de la taille de police courante. Ex: 160%.Similaire à <number> mais hérité différemment
inheritHérite de la valeur calculée du parent.Forcer l'héritage
initialRétablit la valeur initiale (normal).Réinitialisation
CSS 
/* line-height : TOUJOURS sans unité */
.tight   { line-height: 1.2; } /* titres, dense */
.normal  { line-height: 1.5; } /* confortable */
.roomy   { line-height: 1.8; } /* aéré, citations */
.spacious { line-height: 2; }  /* très aéré */

/* Avec unité (à éviter) */
.bad { line-height: 24px; }
/* Si font-size change, 24px devient inadapté */
💡
Règle fondamentale : line-height sans unité

Il convient de toujours utiliser un nombre sans unité pour la propriété line-height. En effet, cette valeur est alors proportionnelle à la taille de police courante. Si l'on spécifie line-height: 1.6, l'interlignage représentera 1,6 fois la taille du texte, quel que soit l'héritage ou les modifications ultérieures de font-size. Cette règle est l'une des plus universellement acceptées en CSS.

⚠️
text-transform et accessibilité

La propriété text-transform: uppercase opère une transformation purement visuelle. Un lecteur d'écran restituera le texte exactement tel qu'il est rédigé dans le HTML (en minuscules si tel est le cas dans le code source). Si un contenu doit impérativement être lu en majuscules (acronyme, sigle), il convient de l'écrire en majuscules directement dans le HTML.

07

text-indent — Indentation du texte

La propriété text-indent définit le retrait de la première ligne d'un bloc de texte. Elle est traditionnellement utilisée pour signaler le début d'un paragraphe dans la typographie imprimée.

Référence complète — text-indent

ValeurDescriptionContexte d'utilisation
<length>Valeur absolue d'indentation. Ex: 2em, 40px.Indentation standard des paragraphes
<percentage>Pourcentage de la largeur du bloc contenant. Ex: 10%.Indentation proportionnelle à la largeur
0Pas d'indentation — valeur par défaut.Texte sans retrait
inheritHérite de la valeur du parent.Forcer l'héritage
initialRétablit la valeur initiale (0).Réinitialisation
CSS 
.indent { text-indent: 2em; }
/* La première ligne de chaque paragraphe est décalée de 2em */

.no-indent { text-indent: 0; }

/* Indentation négative (effet "suspendu") */
.hanging { text-indent: -2em; padding-left: 2em; }
08

letter-spacing et word-spacing

Ces propriétés permettent d'ajuster l'espacement entre les caractères (letter-spacing) et entre les mots (word-spacing). Elles s'expriment généralement en em afin de demeurer proportionnelles à la taille de police. Un léger letter-spacing positif améliore la lisibilité des textes en majuscules.

Référence complète — letter-spacing

ValeurDescriptionContexte d'utilisation
normalEspacement normal de la police — valeur par défaut.Texte standard
<length>Espacement supplémentaire entre les caractères. Valeur positive ou négative. Ex: 0.05em, -0.02em, 2px.Tracking typographique, titres espacés
inheritHérite de la valeur du parent.Forcer l'héritage

Référence complète — word-spacing

ValeurDescriptionContexte d'utilisation
normalEspacement normal entre les mots — valeur par défaut.Texte standard
<length>Espacement supplémentaire entre les mots. Ex: 0.25em, 5px.Texte aéré, justification, effets éditoriaux
inheritHérite de la valeur du parent.Forcer l'héritage
CSS 
/* letter-spacing : espacement entre les caractères */
.normal     { letter-spacing: normal; }
.tight      { letter-spacing: -0.02em; }  /* resserré — titres */
.loose      { letter-spacing: 0.05em; }   /* étalé — sous-titres */
.spread     { letter-spacing: 0.15em; }   /* très étalé — style luxe */

/* word-spacing : espacement entre les mots */
.normal-words { word-spacing: normal; }
.spaced-words { word-spacing: 0.25em; }

/* Combinaison pour un style éditorial */
.editorial {
  letter-spacing: 0.02em;
  word-spacing: 0.05em;
}
💡
Cas d'usage du letter-spacing

Un léger letter-spacing positif (0.05em à 0.1em) améliore la lisibilité des textes en majuscules. Les marques de luxe et la presse écrite recourent fréquemment à un espacement accru pour les titres. À l'inverse, un espacement négatif peut produire un effet esthétique intéressant pour les gros titres modernes, mais son application à des textes longs compromet la lisibilité.

white-space et word-break

La propriété white-space régit la gestion des espaces blancs dans le texte, tandis que word-break détermine les règles de coupure des mots en fin de ligne. Ces propriétés revêtent une importance particulière dans les interfaces utilisateur où le texte doit s'inscrire dans des conteneurs aux dimensions contraintes.

/* ═══ white-space ═══ */
.normal   { white-space: normal; }     /* défaut */
.nowrap   { white-space: nowrap; }     /* pas de retour à la ligne */

.pre-wrap { white-space: pre-wrap; }   /* pre avec retour auto */

.break-all    { word-break: break-all; } /* coupe n'importe où */

/* ═══ overflow-wrap ═══ */
.break-word   { overflow-wrap: break-word; } /* coupe si nécessaire */

/* Cas pratique : troncature avec ellipsis */
.url-container {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}

@font-face — Polices auto-hébergées

La règle CSS @font-face permet d'utiliser des polices hébergées sur votre propre serveur, sans dépendre de Google Fonts ou d'un service tiers. Elle est particulièrement adaptée aux polices commerciales, aux polices maison, ou lorsqu'un contrôle total sur la performance est requis.

/* Déclaration de la police */ @font-face { font-family: 'MaPolice'; src: url('fonts/mapolice.woff2') format('woff2'), url('fonts/mapolice.woff') format('woff'); font-weight: 400; font-style: normal; font-display: swap; } /* Utilisation */ body { font-family: 'MaPolice', 'Helvetica Neue', sans-serif; } /* Formats recommandés : WOFF2 (prioritaire), WOFF (fallback) */
RENDU

Police chargée via @font-face (simulation — Georgia fait office de police personnalisée). Modifiez le code à gauche pour expérimenter.

Bonnes pratiques relatives à @font-face

  • WOFF2 est le format le plus moderne et le plus compressé. Il convient de l'utiliser en priorité.
  • WOFF constitue la solution de secours pour les navigateurs plus anciens.
  • font-display: swap : affiche le texte avec une police de secours immédiatement, puis remplace par la police web une fois chargée. Évite le Flash of Invisible Text (FOIT).
  • Chaque variante (graisse, style) doit être déclarée dans un bloc @font-face distinct, avec le même font-family mais des valeurs font-weight/font-style différentes.

Licences et droits d'usage

Toutes les polices ne sont pas libres de droits pour une utilisation sur le Web. Il est impératif de vérifier la licence avant d'héberger une police sur son serveur. Les polices Google Fonts sont distribuées sous licence open-source (SIL Open Font License, Apache, etc.). Les polices commerciales nécessitent l'acquisition d'une licence web distincte.

11

Google Fonts et polices web

Google Fonts est un service gratuit de polices web hébergées. Il permet d'utiliser des centaines de familles de polices sans nécessité d'installation sur le serveur. L'intégration s'effectue via une balise <link> placée dans l'élément <head> du document.

HTML + CSS 
<!-- Dans le <head> du HTML -->
<link href="https://fonts.googleapis.com/css2?
family=Inter:wght@400;700;900&
family=Playfair+Display:ital@0;1
&display=swap" rel="stylesheet">

<!-- Puis dans le CSS : -->
<style>
  h1 { font-family: 'Playfair Display', serif; }
  body { font-family: 'Inter', sans-serif; }
</style>

Procédure d'utilisation de Google Fonts

  1. Accédez au site fonts.google.com
  2. Sélectionnez une police (exemples : Inter, Roboto, Open Sans, Lora, Playfair Display)
  3. Choisissez les graisses nécessaires (400 = normal, 700 = gras, 900 = black)
  4. Copiez la balise <link> fournie et insérez-la dans l'élément <head> de votre document
  5. Utilisez le nom de la police dans votre CSS via la propriété font-family
⚠️
Impact des Google Fonts sur la performance

Chaque police issue de Google Fonts engendre un chargement externe supplémentaire susceptible de ralentir l'affichage de la page. Il est recommandé de se limiter à 1 ou 2 familles. L'utilisation du paramètre display=swap dans l'URL permet d'afficher immédiatement une police de secours pendant le chargement.

Exercices pratiques

EX 01 🔤 Palette typographique complète
  1. Créez exo-typo-complete.html avec un article de blog : titre h1, sous-titre h2, plusieurs paragraphes, une citation blockquote, et une liste.
  2. Définissez une police Google Font pour les titres (ex : Playfair Display, Oswald, ou Montserrat) et une police système pour le texte.
  3. Utilisez font-size en rem pour tous les textes : 2.5rem pour h1, 1.8rem pour h2, 1.1rem pour p.
  4. Appliquez line-height: 1.7 sur les paragraphes et line-height: 1.2 sur les titres.
  5. Utilisez text-transform: uppercase et letter-spacing: .15em sur les sous-titres.
  6. Ajoutez une citation avec font-style: italic et une bordure gauche colorée.
EX 02 📐 Espacement & habillage
  1. Créez une page exo-espacement.html avec un bloc de texte large et un bloc étroit (300px).
  2. Dans le bloc étroit, utilisez overflow-wrap: break-word pour éviter qu'un long mot ne déborde.
  3. Ajoutez une ligne de texte en white-space: nowrap avec text-overflow: ellipsis.
  4. Utilisez letter-spacing: -0.02em sur un gros titre pour un effet moderne.
  5. Utilisez text-decoration: underline wavy sur un mot important.
  6. Ajoutez un lien avec text-decoration: none qui souligne au survol.
EX 03 📦 @font-face personnalisé
  1. Trouvez une police libre de droits au format WOFF2 (ex : sur Google Fonts ou Font Library) et téléchargez-la.
  2. Placez les fichiers WOFF2 dans un dossier fonts/ à côté de votre HTML.
  3. Déclarez la police avec @font-face.
  4. Déclarez une variante gras (700) supplémentaire.
  5. Utilisez la police dans votre page pour les titres, et une police système pour le texte.
  6. Vérifiez dans les DevTools (onglet Network) que les fichiers WOFF2 se chargent bien.

Exercice bonus — Échelle typographique modulaire

Bilan & mini-glossaire

Ce que vous maîtrisez

  • font-family : choix de police avec fallback et familles génériques
  • font-size en rem (recommandé) vs px, importance pour l'accessibilité
  • font-weight (100–950) et font-style (normal, italic, oblique)
  • font-variant : small-caps, ligatures, variantes numériques
  • line-height sans unité : proportionnel à la taille de police
  • text-align, text-transform, text-decoration (underline, overline, line-through, wavy)
  • text-indent pour l'indentation des paragraphes
  • letter-spacing et word-spacing en em pour l'espacement
  • white-space (nowrap, pre, pre-wrap), word-break, overflow-wrap
  • @font-face : déclaration de polices auto-hébergées au format WOFF2
  • Google Fonts : intégration via <link>, display=swap

Mini-glossaire

font-family
Propriété qui définit la police de caractères avec une liste de fallback.
Fallback
Police de secours utilisée si la police demandée n'est pas disponible.
rem
Unité relative à la taille de police de la racine (html). 1rem = 16px par défaut.
em
Unité relative à la taille de police du parent. S'accumule dans les éléments imbriqués.
line-height
Hauteur de ligne. Toujours sans unité (ex: 1.6). Essentiel pour la lisibilité.
font-weight
Graisse de la police : 100 (thin) à 950 (extra black). 400 = normal, 700 = bold.
font-variant
Variantes typographiques : petites capitales, ligatures, chiffres tabulaires.
text-transform
Modifie la casse : uppercase, lowercase, capitalize. Visuel seulement.
letter-spacing
Espacement entre les caractères. En em. Utile pour les textes en majuscules.
white-space
Gère les espaces blancs et les retours à la ligne. nowrap empêche la coupure.
word-break
Définit les règles de coupure des mots : break-all, keep-all.
@font-face
Règle CSS déclarant une police hébergée sur votre serveur. Formats : WOFF2, WOFF.
font-display
Stratégie d'affichage : swap (FOUT), block (FOIT), optional (performance).
text-overflow: ellipsis
Ajoute des points de suspension quand le texte dépasse son conteneur.
