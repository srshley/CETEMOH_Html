01

background-color — Colorer le fond

La propriété background-color définit la couleur d'arrière-plan d'un élément. Elle couvre la zone comprenant le contenu et le padding, mais s'arrête à la bordure — la marge (margin) reste toujours transparente. Cette propriété constitue la base de tout design CSS : elle établit le fond sur lequel viennent se superposer les images, dégradés et autres effets visuels.

La couleur de fond peut être exprimée dans tous les formats de couleur CSS : noms, hexadécimal, RGB/RGBA, HSL/HSLA. La valeur transparent est la valeur initiale, ce qui signifie que par défaut, les éléments n'ont pas de fond visible. L'usage de l'alpha (transparence) dans rgba() ou hsla() permet de créer des effets de superposition subtils, particulièrement utiles pour les overlays et les cartes.

Il est important de noter que background-color fait partie de la propriété raccourcie background. Lorsqu'on utilise le shorthand, si aucune couleur n'est spécifiée, la valeur par défaut transparent est appliquée. En pratique, il est recommandé de déclarer explicitement background-color avant toute image de fond, afin de fournir une couleur de secours (fallback) en cas de non-chargement de l'image.

📌
Zone couverte par background-color

La couleur de fond s'applique jusqu'à la bordure, excluant la zone de margin qui demeure toujours transparente. Pour colorer l'intégralité de la zone visible jusqu'à la bordure, assurez-vous d'inclure le padding dans les dimensions de l'élément.

🚀
Pour aller plus loin — background-color et accessibilité

Le contraste entre la couleur de fond et la couleur du texte est un critère fondamental d'accessibilité (WCAG). Le ratio de contraste doit être d'au moins 4.5:1 pour le texte normal et 3:1 pour le texte large. Les outils comme le vérificateur de contraste de Chrome DevTools permettent de valider ces ratios. Les couleurs avec alpha réduisent le contraste perçu : utilisez-les avec précaution pour les fonds texturés.

02

background-image — Images de fond

La propriété background-image insère une ou plusieurs images en arrière-plan via la fonction url(). Contrairement à la balise <img>, les images de fond sont purement décoratives : elles sont ignorées par les lecteurs d'écran et les moteurs de recherche, ce qui en fait le choix approprié pour les éléments visuels non porteurs de sens.

Par défaut, l'image se répète en mosaïque (motif) pour remplir intégralement la boîte de l'élément. Ce comportement peut être modifié avec la propriété background-repeat. Une image de fond peut également être un dégradé CSS (voir sections suivantes), car les dégradés sont traités comme des images par le moteur de rendu.

Il est possible de déclarer plusieurs images de fond en les séparant par des virgules. La première image déclarée apparaît au premier plan (au-dessus des suivantes). Cette technique permet des compositions complexes : superposition d'un motif par-dessus un dégradé, ou d'un dégradé de transparence par-dessus une image photographique pour améliorer la lisibilité du texte.

CSS — background-image

  background-image: url('../../images/hero.jpg');
  background-size: cover;
  background-position: center;
}

.card {
  background-image: linear-gradient(rgba(0,0,0,.4), rgba(0,0,0,.4)), url('photo.jpg');
}
⚠️
Distinction entre background-image et <img>

Une image de fond possède un caractère décoratif. Elle est ignorée par les lecteurs d'écran (accessibilité) et par les moteurs de recherche (SEO). Utilisez background-image pour les éléments décoratifs et <img> pour les images porteuses de sens.

03

background-repeat & background-size

Par défaut, une image de fond se répète en mosaïque pour remplir intégralement la boîte de l'élément. Ce comportement historique, hérité des premiers navigateurs, est rarement souhaité dans les designs modernes. Les propriétés background-repeat et background-size offrent un contrôle précis sur la répétition et le dimensionnement de l'image de fond.

La propriété background-size est particulièrement importante dans le responsive design. Les valeurs cover et contain permettent d'adapter automatiquement l'image à la taille du conteneur : cover remplit intégralement le conteneur en rognant l'image si nécessaire, tandis que contain affiche l'image en entier en laissant éventuellement des bandes vides. Le choix entre ces deux valeurs dépend de l'effet recherché et du format de l'image.

CSS — Répétition et dimensionnement 
.hero {
  background-image: url('../../images/hero.jpg');
  background-size: cover;
  background-position: center;
  background-repeat: no-repeat;
}

.pattern {
  background-image: url('motif.png');
  background-repeat: repeat;
}

.logo { background-size: contain; background-repeat: no-repeat; }
💡
Comparaison cover / contain

cover : l'image remplit l'intégralité du conteneur, avec un rognage éventuel. Convient aux sections hero. contain : l'image entière est visible, ce qui peut laisser des bandes vides. Utile pour les logos ou icônes en arrière-plan.

04

background-position

La propriété background-position détermine la position de départ de l'image de fond à l'intérieur de la boîte de l'élément. Elle est particulièrement utile lorsque l'image est plus grande que le conteneur (notamment avec background-size: cover) ou lorsqu'elle n'est pas répétée. La position par défaut est le coin supérieur gauche (0% 0%).

Les valeurs peuvent être exprimées par des mots-clés (top, right, bottom, left, center), en pourcentages (0% 0% à 100% 100%), ou en unités de mesure absolues. Depuis CSS3, il est possible de spécifier une position relative aux bords avec la syntaxe right 20px bottom 10px, ce qui permet un positionnement précis sans calcul mental.

Référence complète de background-position

ValeurDescriptionExemple

top / bottom / left / rightMot-clé simple pour un côtébackground-position: top
<x> <y>Deux valeurs (mots-clés, %, px)top right, 25% 75%
<x%> <y%>Pourcentage : 0%=début, 100%=fin50% 50% (équivaut à center)
<x> <y> (px)Décalage en pixels depuis le coin supérieur gauche20px 40px
right <x> bottom <y>Position depuis les bords (CSS3)right 20px bottom 10px
CSS — background-position 
.hero     { background-position: center; }
.portrait { background-position: top center; }  /* évite de couper le visage */
.decale   { background-position: right 20px bottom 15px; }
📌
background-position avec cover

Avec background-size: cover, la position détermine quelle partie de l'image est visible. center est le plus sûr. Pour un portrait, top center évite de couper le visage.

05

Dégradés linéaires (linear-gradient)

Les dégradés CSS produisent des transitions progressives entre plusieurs couleurs, sans recourir à des fichiers image. Ils sont assimilés à des images par le moteur de rendu et s'utilisent donc via la propriété background-image (ou la propriété raccourcie background). Les dégradés linéaires suivent une direction rectiligne spécifiée par un angle ou un mot-clé.

La fonction linear-gradient() accepte un ou plusieurs points d'arrêt (color-stops) qui définissent les positions où les couleurs doivent atteindre leur pleine intensité. Entre deux points d'arrêt, le navigateur calcule automatiquement la transition. La direction par défaut est to bottom (du haut vers le bas), mais elle peut être spécifiée avec les mots-clés to top, to right, to left, to bottom right, etc., ou avec un angle en degrés (45deg).

Les dégradés linéaires sont extrêmement versatiles : ils permettent de créer des fonds de page, des boutons, des overlays, des motifs rayés (avec repeating-linear-gradient), des séparateurs et bien plus encore. La combinaison d'un dégradé avec une image réelle (en arrière-plan) est une technique standard pour améliorer la lisibilité du texte sur les photos.

CSS
/* Tapez votre CSS pour le dégradé — voyez le rendu */ .ma-boite { background: linear-gradient(135deg, #667eea 0%, #764ba2 100%); height: 100px; width: 100%; border-radius: 8px; }

💡
Superposition d'un dégradé et d'une image

Un dégradé étant une image CSS, il peut être superposé à une image réelle : background: linear-gradient(rgba(0,0,0,.5), rgba(0,0,0,.5)), url('photo.jpg') center / cover;. Cette technique standard permet d'assombrir un fond pour améliorer la lisibilité du texte.

🚀
Pour aller plus loin — Dégradés coniques et motifs complexes

Au-delà de linear-gradient et radial-gradient, la fonction conic-gradient() crée des dégradés qui tournent autour d'un point central (effet de roue chromatique). Combinée avec repeating-, elle produit des motifs géométriques complexes sans aucune image. Les dégradés CSS sont également animables via la propriété transition : vous pouvez créer des transitions fluides entre deux dégradés, bien que cette technique nécessite des tailles de background identiques.

06

Dégradés radiaux (radial-gradient)

Les dégradés radiaux rayonnent à partir d'un point d'origine et se déploient en cercles ou ellipses concentriques. Cette forme de dégradé est idéale pour simuler un éclairage ponctuel (effet spotlight), créer des halos lumineux, des reflets ou des ombres internes. Le point d'origine peut être centré ou décentré.

La fonction radial-gradient() accepte des paramètres de forme (circle ou ellipse), de taille (closest-side, farthest-side, closest-corner, farthest-corner), et de position (mêmes valeurs que background-position). Par défaut, le dégradé est une ellipse centrée qui s'étend jusqu'au coin le plus éloigné (farthest-corner).

📌
Cas d'usage des dégradés radiaux

Les dégradés radiaux sont adaptés pour simuler un éclairage ponctuel (spotlight), créer des effets de profondeur ou mettre en valeur le centre d'un élément. Il convient de les utiliser avec modération — un dégradé radial trop prononcé peut nuire à la lisibilité.

07

box-shadow — Ombres portées

La propriété box-shadow ajoute une ou plusieurs ombres autour de la boîte d'un élément. Cet effet, appliqué avec mesure, confère une sensation de profondeur et de relief à l'interface. La syntaxe de box-shadow est l'une des plus riches du CSS : elle permet de contrôler le décalage horizontal et vertical, le flou, l'étalement, la couleur, et même de placer l'ombre à l'intérieur de la boîte (inset).

Le principe de conception connu sous le nom de Material Design a popularisé l'utilisation d'ombres multiples pour suggérer différentes hauteurs (élévation). Une carte située "plus haut" dans la hiérarchie visuelle reçoit une ombre plus grande et plus floue. Cette métaphore de l'élévation aide les utilisateurs à comprendre la structure hiérarchique de l'interface.

Il est important de noter que box-shadow suit la forme de la boîte, y compris les coins arrondis définis par border-radius. Les ombres multiples sont séparées par des virgules et s'empilent de l'avant vers l'arrière (la première ombre déclarée est la plus haute).

💡
Configuration standard pour les cartes

La combinaison d'ombres la plus répandue : box-shadow: 0 1px 3px rgba(0,0,0,.08), 0 4px 12px rgba(0,0,0,.04);. Au survol : 0 4px 12px rgba(0,0,0,.12), 0 12px 32px rgba(0,0,0,.08);.

🚀
Pour aller plus loin — box-shadow et performance

La propriété box-shadow est relativement coûteuse en termes de performance de rendu. Pour les animations, privilégiez l'animation de transform et opacity plutôt que box-shadow. Une alternative performante pour les ombres consiste à utiliser un pseudo-élément ::after avec un dégradé radial. Depuis l'avènement du GPU compositing, les ombres simples restent néanmoins acceptables dans la plupart des contextes.

08

Multiple backgrounds

CSS autorise la superposition de plusieurs images de fond sur un même élément. La première image dans la déclaration occupe le plan le plus élevé (premier plan, au-dessus des autres), tandis que la dernière constitue le plan le plus bas (fond lointain). Cette fonctionnalité est extrêmement puissante pour créer des effets visuels complexes sans multiplier les éléments HTML.

Chaque couche de fond peut posséder ses propres valeurs pour background-repeat, background-size, background-position et les autres propriétés du groupe background. Les valeurs de ces propriétés sont également séparées par des virgules, dans le même ordre que les images. La propriété raccourcie background permet de déclarer l'ensemble en une seule ligne, bien que la lisibilité en pâtisse souvent.

CSS — Backgrounds multiples 
.hero {
  background:
    linear-gradient(rgba(0,0,0,.4), rgba(0,0,0,.4)),
    url('../../images/hero.jpg') center / cover no-repeat;
}

.card-accent {
  background:
    linear-gradient(#e74c3c, #e74c3c) no-repeat left / 4px 100%,
    #f8f9fa;
  /* Barre latérale rouge + fond gris */
}

.complex {
  background:
    url('stars.png') repeat,
    radial-gradient(ellipse at bottom, #1b2735, #090a0f);
}
CSS — Shorthand complet 
/* Syntaxe : image repeat attachment position / size */
.element {
  background:
    url('pattern.png') repeat-x center top / 50px auto,
    linear-gradient(180deg, #fff 0%, #f0f0f0 100%) no-repeat center / 100% 100%;
}

/* Exemple : dégradé + image + couleur de secours */
.exemple {
  background:
    linear-gradient(rgba(0,0,0,.3), rgba(0,0,0,.3)),
    url('photo.jpg') center / cover no-repeat,
    #333; /* fallback si l'image ne charge pas */
}
📌
Principe d'empilement

Les arrière-plans sont empilés de gauche à droite (la première couche est la plus haute, la dernière la plus basse). Chaque couche peut posséder ses propres valeurs pour les propriétés background-repeat, background-size et background-position.

Exercices pratiques

EX 01 🌈 Dégradés créatifs
  1. Créez exo-gradients.html avec 6 blocs carrés de 150×150px.
  2. Chaque bloc a un dégradé différent : linéaire horizontal, linéaire diagonal, linéaire 4 couleurs, radial circulaire, radial décentré, radial 3 couleurs.
  3. Ajoutez un dégradé de transparence sur le dernier (rgba → transparent).
  4. Ajoutez une ombre portée à chaque bloc.
  5. Créez une hero section avec un dégradé + image par-dessus.
EX 02 🖼️ Mise en page avec backgrounds
  1. Créez une galerie de 4 cartes avec images de fond en background-size: cover.
  2. Ajoutez un overlay semi-transparent (dégradé) sur chaque carte.
  3. Chaque carte a un titre et sous-titre en blanc.
  4. Utilisez background-size: contain pour un bloc avec un logo.
  5. Ajoutez un motif répété pour un bloc décoratif.
🔍
Exercice bonus — Système d'ombres cohérent

Créez 4 niveaux d'ombre : shadow-sm (0 1px 3px), shadow-md (0 4px 12px), shadow-lg (0 8px 24px), shadow-xl (0 16px 48px). Appliquez-les à des cartes et observez la profondeur perçue.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • background-color : couleur unie, transparent, alpha
  • background-image : url(), dégradés, superposition, image-set()
  • background-repeat : repeat, no-repeat, repeat-x/y, space, round
  • background-size : cover, contain, dimensions explicites
  • background-position : mots-clés, pourcentages, pixels, bords
  • Linear-gradient : directions, angles, color-stops, transparence
  • Radial-gradient : cercle, ellipse, position, formes, taille
  • box-shadow : offset, blur, spread, couleur, multiples, inset
  • Multiple backgrounds : superposition, ordre, shorthand complet
