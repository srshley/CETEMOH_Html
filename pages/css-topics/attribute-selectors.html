01

Introduction aux sélecteurs d'attributs

Les sélecteurs d'attribut permettent de cibler les éléments HTML en fonction de la présence ou de la valeur de leurs attributs. Ils offrent une puissance de stylisation considérable sans nécessiter l'ajout de classes supplémentaires dans le code HTML. Leur spécificité est identique à celle des classes (0,0,1,0).

SyntaxeDescriptionExempleCible
[attr]Présence de l'attribut[disabled]Tous les éléments désactivés
[attr="val"]Égalité exacte[type="submit"]Boutons de soumission
[attr^="val"]Commence par (préfixe)[href^="https"]Liens sécurisés
[attr$="val"]Finit par (suffixe)[src$=".png"]Images PNG
[attr*="val"]Contient la sous-chaîne[class*="btn"]Éléments avec "btn" dans la classe
[attr~="val"]Mot dans une liste d'espaces[rel~="noopener"]Liens avec rel="noopener"
[attr|="val"]Valeur exacte ou suivie d'un tiret[lang|="fr"]Pages en français
📌
Spécificité des sélecteurs d'attribut

Les sélecteurs d'attribut possèdent la même spécificité que les sélecteurs de classe : 0,0,1,0. Ainsi, un sélecteur tel que [type="text"] est aussi spécifique que .text-field. Cette propriété permet de substituer des classes par des sélecteurs d'attribut dans de nombreux cas.

02

[attr] — Présence simple de l'attribut

Le sélecteur [attr] cible l'ensemble des éléments qui possèdent l'attribut attr, quelle que soit sa valeur. Il s'agit du sélecteur d'attribut le plus élémentaire. Idéal pour les attributs booléens (disabled, required, readonly, hidden) dont la seule présence suffit.

Référence complète — [attr]

SyntaxeDescriptionContexte d'utilisation
[attr]Sélectionne tout élément possédant l'attribut attr, sans vérifier sa valeur.Attributs booléens : [disabled], [required], [hidden], [readonly]
[disabled]Cible tous les éléments désactivés (boutons, inputs, champs).Réduire l'opacité, changer le curseur
[required]Cible les champs de formulaire obligatoires.Ajouter un indicateur visuel (bordure, astérisque)
[hidden]Cible les éléments avec l'attribut hidden.Forcer display: none
[data-toggle]Cible les éléments avec un attribut data-* spécifique.Interaction JavaScript
CSS 
/* Tous les éléments désactivés */
[disabled] {
  opacity: 0.5;
  cursor: not-allowed;
}

/* Tous les champs obligatoires */
[required] {
  border-left: 3px solid #e63946;
}

/* Tous les éléments avec un attribut hidden */
[hidden] {
  display: none !important;
}

/* Tous les liens avec target */
[target] {
  color: #8e44ad;
}
💡
Attributs booléens

Les attributs booléens (disabled, required, readonly, hidden) n'ont pas besoin de valeur explicite — leur seule présence dans le code HTML suffit. Le sélecteur [disabled] fonctionne indifféremment que l'attribut soit écrit disabled ou disabled="disabled".

03

[attr="valeur"] — Égalité exacte

Le sélecteur [attr="valeur"] cible les éléments dont l'attribut attr possède exactement la valeur spécifiée. La correspondance doit être stricte, caractère par caractère (sensible à la casse par défaut).

Référence complète — [attr="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr="valeur"]Correspondance exacte, sensible à la casse par défaut. La valeur entière de l'attribut doit correspondre exactement à la chaîne.Ciblage précis de types, états, ou valeurs spécifiques
[type="submit"]Tous les boutons de soumission.Styles spécifiques selon le type de champ
[target="_blank"]Liens qui s'ouvrent dans un nouvel onglet.Ajouter une icône de nouvelle fenêtre
[type="text"]Champs de texte.Styles personnalisés pour les inputs texte
[type="email"]Champs d'email.Styles personnalisés pour les inputs email
CSS 
/* Types de boutons */
[type="submit"] { background: #2c3e50; color: white; }
[type="reset"]  { background: #e74c3c; color: white; }
[type="button"] { background: #95a5a6; color: white; }

/* Liens externes */
[target="_blank"] { color: #2980b9; }
[target="_blank"]::after {
  content: " \2197";
  font-size: .8em;
}

/* Inputs texte et email */
[type="text"] {
  border: 2px solid #bdc3c7;
  padding: 8px 12px;
  border-radius: 4px;
}
[type="email"] {
  border: 2px solid #3498db;
  padding: 8px 12px;
  border-radius: 4px;
}
04

[attr^="valeur"] — Commence par (préfixe)

Le sélecteur [attr^="valeur"] cible les éléments dont la valeur de l'attribut commence par la chaîne spécifiée. Le symbole ^ évoque l'opérateur « commence par » (analogue à son usage dans les expressions régulières). Particulièrement utile pour catégoriser les liens selon le protocole.

Référence complète — [attr^="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr^="chaîne"]Sélectionne les éléments dont l'attribut commence par la chaîne spécifiée.Catégorisation par préfixe
[href^="https"]Liens sécurisés HTTPS.Ajouter une icône de cadenas
[href^="http"]:not([href^="https"])Liens HTTP non sécurisés (combinaison avec :not).Alerte de sécurité
[href^="mailto"]Liens d'adresse email.Icône email
[href^="tel"]Liens téléphoniques.Icône téléphone
[id^="section-"]Éléments dont l'ID commence par "section-".Styles groupés par préfixe d'ID
CSS
/* Liens HTTPS */ a[href^="https"] { color: #27ae60; } a[href^="https"]::before { content: "\1F512 "; font-size: .85em; } /* Liens mailto */ a[href^="mailto"] { color: #8e44ad; } a[href^="mailto"]::before { content: "\2709 "; } /* Liens téléphoniques */ a[href^="tel"] { color: #2980b9; font-weight: bold; } /* ID commençant par "section-" */ [id^="section-"] { border-left: 3px solid #3498db; padding-left: 1rem; }
RENDU
🔒 Lien sécurisé HTTPS
✉ contact@exemple.com
📞 +33 1 23 45 67 89
Élément avec ID "section-1"
💡
Cas d'usage : stylisation des liens

Le sélecteur [href^="https"] constitue le moyen le plus élégant pour distinguer les liens internes des liens externes. En combinant l'opérateur ^ avec la pseudo-classe :not(), il est possible de traiter différemment les liens HTTP non sécurisés, les adresses électroniques, les numéros de téléphone et les ancres internes.

05

[attr$="valeur"] — Termine par (suffixe)

Le sélecteur [attr$="valeur"] cible les éléments dont la valeur de l'attribut termine par la chaîne spécifiée. Le symbole $ évoque l'opérateur « termine par » (analogue à son usage dans les expressions régulières). Ce sélecteur est particulièrement adapté au ciblage des fichiers selon leur extension.

Référence complète — [attr$="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr$="chaîne"]Sélectionne les éléments dont l'attribut se termine par la chaîne spécifiée.Catégorisation par extension ou suffixe
[href$=".pdf"]Liens vers des fichiers PDF.Badge [PDF] après le lien
[href$=".zip"], [href$=".rar"]Liens vers des archives.Badge [Archive]
[href$=".doc"], [href$=".docx"]Liens vers des documents Word.Badge [Word]
[src$=".png"]Images au format PNG.Style spécifique selon le format d'image
[src$=".svg"]Images au format SVG.Arrière-plan ou padding pour les SVG
CSS 
/* Fichiers selon leur extension */
a[href$=".pdf"]::after {
  content: " [PDF]";
  color: #e74c3c;
  font-size: .8em;
  font-weight: bold;
}

a[href$=".zip"]::after,
a[href$=".rar"]::after {
  content: " [Archive]";
  color: #e67e22;
  font-size: .8em;
}

a[href$=".doc"]::after,
a[href$=".docx"]::after {
  content: " [Word]";
  color: #2980b9;
  font-size: .8em;
}

/* Images selon le format */
img[src$=".png"] { border: 2px solid #3498db; }
img[src$=".svg"] { background: #f8f9fa; padding: 4px; }
⚠️
Risque de faux positifs

Le sélecteur [href$=".pdf"] cible également les URLs telles que document.pdf.php ou page.pdf?version=2. Pour éviter ces correspondances intempestives, combinez-le avec [href*="."] ou adoptez une approche plus stricte. Dans les cas complexes, l'ajout d'une classe directement sur le lien peut s'avérer préférable.

06

[attr*="valeur"] — Contient la sous-chaîne

Le sélecteur [attr*="valeur"] cible les éléments dont la valeur de l'attribut contient la chaîne spécifiée, quelle que soit sa position dans la valeur. L'astérisque * évoque l'opérateur « contient ». C'est le sélecteur le plus flexible, mais aussi celui qui produit le plus de correspondances potentielles.

Référence complète — [attr*="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr*="chaîne"]Sélectionne les éléments dont l'attribut contient la sous-chaîne à n'importe quelle position.Recherche de motif flexible
[class*="btn"]Éléments dont la classe contient "btn". Cible btn, btn-primary, submit-btn.Styles génériques pour composants
[href*="wikipedia"]Liens contenant "wikipedia" dans l'URL.Marquage de sources spécifiques
[href*="amazon"]Liens contenant "amazon" dans l'URL.Marquage de liens commerciaux
[name*="email"]Champs dont l'attribut name contient "email".Styles par motif dans le nom
[id*="section"]Éléments dont l'ID contient "section".Marges de défilement pour sections
CSS 
/* Classes contenant "btn" */
[class*="btn"] {
  font-weight: bold;
  cursor: pointer;
}

/* Sources spécifiques */
[href*="wikipedia"] { color: #636363; }
[href*="amazon"]    { color: #ff9900; }

/* Champs dont le name contient "email" */
[name*="email"] {
  border-color: #3498db;
}

/* ID contenant "section" */
[id*="section"] {
  scroll-margin-top: 80px;
}
📌
Puissance et risques de collision

Le sélecteur [class*="btn"] cible indifféremment class="btn", class="btn-primary" et class="submit-btn". Cette souplesse est utile mais peut engendrer des collisions si les noms se chevauchent. Exemple : [class*="icon"] ciblerait également class="reconnaître". Pour un ciblage plus précis, préférez [class~="icon"] (mot entier) ou [class^="icon-"] (préfixe).

07

[attr~="valeur"] — Mot dans une liste

Le sélecteur [attr~="valeur"] cible les éléments dont la valeur de l'attribut contient le mot spécifié en tant que mot entier au sein d'une liste séparée par des espaces. Le tilde ~ évoque la notion de « mot ». Utile pour les attributs comme rel, class (liste de classes), ou tout attribut contenant des mots séparés par des espaces.

Référence complète — [attr~="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr~="mot"]L'attribut contient le mot exact spécifié dans une liste séparée par des espaces. Ne correspond pas à des sous-chaînes partielles.Recherche de mot entier dans une liste
[rel~="noopener"]Liens avec rel="noopener" (ou "noopener noreferrer").Sécurité des liens externes
[rel~="nofollow"]Liens avec rel="nofollow".Marquage SEO
[class~="icon"]Éléments dont la classe contient le mot exact "icon". Ne cible pas "icon-small".Styles précis par mot de classe
[data-priority~="urgent"]Attribut data-priority contenant le mot "urgent".Priorité dans les listes
CSS 
/* Attribut rel avec mot "nofollow" */
[rel~="nofollow"]::after {
  content: " [nofollow]";
  color: #e74c3c;
  font-size: .8em;
}

/* Mot exact dans l'attribut class */
[class~="icon"] {
  width: 16px;
  height: 16px;
  display: inline-block;
}

/* Attribut data contient le mot "urgent" */
[data-priority~="urgent"] {
  border-left: 4px solid #e74c3c;
  background: #fdf0ef;
}

/* Différence avec [attr*=] :
   [class~="btn"] cible class="btn primary" MAIS PAS class="btn-primary"
   [class*="btn"] cible class="btn-primary" AINSI QUE class="submit-btn" */
💡
Distinction entre [class~=] et [class*=]

La différence est subtile mais fondamentale : [class~="btn"] cible les éléments dont la liste de classes contient le mot entier « btn » (class="btn primary" correspond, mais pas class="btn-primary"). [class*="btn"] cible toute sous-chaîne contenant « btn » (class="btn-primary" et class="submit-btn" correspondent). L'opérateur ~= est à privilégier lorsqu'on recherche un mot exact dans une liste séparée par des espaces.

08

[attr|="valeur"] — Commence par un mot (suivi d'un tiret)

Le sélecteur [attr|="valeur"] cible les éléments dont la valeur de l'attribut est exactement « valeur » ou commence par « valeur- » (suivi d'un tiret). Ce sélecteur a été conçu spécifiquement pour l'attribut lang (langue), mais s'applique à tous les attributs.

Référence complète — [attr|="valeur"]

SyntaxeDescriptionContexte d'utilisation
[attr|="valeur"]Correspond si la valeur est exactement "valeur" OU commence par "valeur-" (suivi d'un tiret).Codes de langue, préfixes de version, namespaces
[lang|="fr"]Pages en français : fr, fr-FR, fr-CA, fr-BE, fr-CH.Styles typographiques par langue
[lang|="en"]Pages en anglais : en, en-US, en-GB.Guillemets, césure, typographie locale
[data-version|="2"]Version 2.x (2.0, 2.1, 2.5, 2.0.1).Styles par version de composant
[class|="icon"]Classes icon, icon-small, icon-large (mais pas iconButton).Préfixes de classes structurés
CSS 
/* Langue française (fr, fr-FR, fr-CA, fr-BE...) */
[lang|="fr"] {
  quotes: "\AB" "\BB";
}

/* Langue anglaise (en, en-US, en-GB...) */
[lang|="en"] {
  quotes: "\201C" "\201D";
}

/* Application aux guillemets */
[lang|="fr"] blockquote::before {
  content: "\AB";
}
[lang|="fr"] blockquote::after {
  content: "\BB";
}

/* Version de données */
[data-version|="2"] {
  border-color: #3498db;
}

/* Préfixes de namespace */
[class|="icon"] {
  /* icon, icon-small, icon-large */
  vertical-align: middle;
}
📌
Usage principal : l'attribut lang

Le sélecteur [lang|="fr"] est l'outil idéal pour cibler l'ensemble des variantes du français : fr, fr-FR, fr-CA, fr-BE, fr-CH. Il s'agissait de l'usage initialement prévu par la spécification CSS. On peut également l'appliquer à ses propres attributs dont les valeurs présentent une structure hiérarchique, comme data-version="2.0" ou data-version="2.1".

09

Modificateur i — Insensibilité à la casse

Par défaut, les sélecteurs d'attribut sont sensibles à la casse (case-sensitive). L'ajout du modificateur i (ou I) avant le crochet fermant rend la comparaison insensible à la casse. Le modificateur s'utilise avec =, ^=, $=, *=, mais pas avec ~= ni |=.

Référence complète — Modificateur i

SyntaxeDescriptionContexte d'utilisation
[attr="val" i]Comparaison insensible à la casse pour l'égalité exacte.Formulaires avec valeurs normalisées
[attr^="val" i]Préfixe insensible à la casse.Protocoles (HTTP, http, Http)
[attr$="val" i]Suffixe insensible à la casse.Extensions (.PDF, .pdf, .Pdf)
[attr*="val" i]Contient insensible à la casse.Recherche de motif flexible
[type="submit" i]Cible submit, SUBMIT, Submit, etc.Normalisation des types de champ
[href$=".PDF" i]Cible .pdf, .PDF, .Pdf.Extensions de fichier en toutes variantes
CSS 
/* Sensible à la casse (par défaut) */
[type="submit"] { }        /* cible "submit" mais PAS "SUBMIT" */

/* Insensible à la casse avec i */
[type="submit" i] { }      /* cible submit, SUBMIT, Submit... */

/* Exemples concrets */
[class*="btn" i] { }        /* cible btn, BTN, Btn... */

[href$=".PDF" i]::after {
  content: " [PDF]";
  color: #e74c3c;
}
/* Cible .pdf, .PDF, .Pdf, .pdf?version=2... */

/* Support : tous les navigateurs modernes depuis 2019 */
/* Ne fonctionne PAS avec [attr~=] et [attr|=] */
⚠️
Compatibilité et limitations

Le modificateur i est pris en charge par l'ensemble des navigateurs modernes depuis 2019. Toutefois, il ne fonctionne pas avec les sélecteurs [attr~=] (mot dans une liste) ni [attr|=] (préfixe avec tiret). Il est recommandé de tester sur les navigateurs cibles avant de l'utiliser en production.

10

Combinaisons avec classes et balises

Les sélecteurs d'attribut peuvent être combinés avec d'autres sélecteurs (balises, classes, pseudo-classes) pour obtenir un ciblage plus précis. On peut également enchaîner plusieurs sélecteurs d'attribut sur le même élément.

CSS 
/* Combinaison avec une balise */
a[href^="https"] { color: #27ae60; }
input[type="text"] { border: 2px solid #ccc; }

/* Combinaison avec une classe */
.btn[disabled] { opacity: 0.4; cursor: not-allowed; }
.card[data-featured="true"] { border-color: gold; }

/* Combinaison avec plusieurs attributs */
input[type="text"][required] {
  border-left: 3px solid #e74c3c;
}
a[href^="https"][target="_blank"]::after {
  content: " \2197";
}

/* Combinaison avec pseudo-classe */
input[type="text"]:focus {
  border-color: #3498db;
  box-shadow: 0 0 0 3px rgba(52,152,219,.2);
}

/* Négation d'attribut */
input:not([type="submit"]) {
  border: 1px solid #ccc;
}
a:not([href^="https"]) {
  color: #e67e22;
}
💡
Puissance des sélecteurs multiples

En combinant plusieurs sélecteurs d'attribut sur un même élément (input[type="text"][required]), on obtient un ciblage extrêmement précis sans classe dans le HTML. Cette technique est particulièrement utile pour les formulaires. La spécificité reste faible (0,0,2,0 pour deux attributs), ce qui la rend facile à surcharger si nécessaire.

CSS — Sélecteurs en action 
/* Livres disponibles */
.book[data-status="available"] {
  border-left: 4px solid #27ae60;
}
.book[data-status="available"]::before {
  content: "📗 ";
}

/* Livres empruntés */
.book[data-status="borrowed"] {
  border-left: 4px solid #e67e22;
  opacity: 0.7;
}
.book[data-status="borrowed"]::before {
  content: "📕 ";
}

/* Livres perdus */
.book[data-status="lost"] {
  border-left: 4px solid #e74c3c;
  text-decoration: line-through;
}

/* Par genre */
.book[data-genre="fantasy"] {
  background: linear-gradient(to right, #f3e5f5, transparent);
}
.book[data-genre="scifi"] {
  background: linear-gradient(to right, #e3f2fd, transparent);
}

/* Livres publiés avant 1900 */
.book[data-year$="18"] {
  font-style: italic;
}

/* Livres de plus de 400 pages */
.book[data-pages^="4"],
.book[data-pages^="5"],
.book[data-pages^="6"] {
  font-weight: bold;
}

Étapes du projet

  1. Créez la structure HTML avec des attributs data-* pour chaque livre (genre, statut, année, pages).
  2. Utilisez [data-status="available"] pour styliser les livres disponibles en vert.
  3. Utilisez [data-status="borrowed"] pour les livres empruntés en orange avec opacité réduite.
  4. Utilisez [data-status="lost"] pour les livres perdus en rouge avec texte barré.
  5. Ajoutez des icônes avec ::before selon le statut.
  6. Utilisez [data-genre="fantasy"] pour un arrière-plan coloré par genre.
  7. Utilisez [data-year$="18"] pour mettre en italique les livres du 19e siècle.
  8. Ajoutez un filtre [data-pages^="4"] pour les livres de plus de 400 pages en gras.
💡
Pourquoi utiliser data-* pour ce projet ?

Les attributs data-* sont entièrement personnalisables, accessibles en CSS (sélecteurs d'attribut) et en JavaScript (dataset). Ce motif est utilisé par de nombreux frameworks CSS pour créer des composants dynamiques sans écrire de JavaScript.

Exercices pratiques

EX 01 🎯 Sélecteurs d'attribut avancés
  1. Créez exo-attributs.html avec 10 liens différents : internes, externes HTTPS, PDF, images, emails, etc.
  2. Utilisez [href^="https"] pour ajouter un cadenas 🔒 devant les liens sécurisés.
  3. Utilisez [href$=".pdf"] pour ajouter [PDF] en rouge après les fichiers PDF.
  4. Utilisez [href^="mailto"] pour ajouter une icône email devant les adresses.
  5. Utilisez [target="_blank"]::after pour ajouter ↗.
  6. Ajoutez un style pour [disabled] sur un bouton désactivé.
EX 02 📋 Inventaire de livres
  1. Reproduisez le projet Inventaire de livres avec au moins 8 livres de genres et statuts variés.
  2. Ajoutez un attribut data-rating (1-5) et stylisez les étoiles avec ::after.
  3. Utilisez [data-rating^="5"] pour mettre en avant les livres 5 étoiles.
  4. Utilisez [data-year] pour afficher l'année entre parenthèses.
  5. Ajoutez [data-pages] et utilisez [data-pages*="0"] pour identifier les livres de plus de 100 pages.
  6. Utilisez le modificateur i pour un attribut insensible à la casse.
EX 03 🧩 Formulaires intelligents
  1. Créez un formulaire avec au moins 6 champs de types différents (text, email, password, number, checkbox, submit).
  2. Utilisez input[type="text"] et input[type="email"] pour des styles différents.
  3. Utilisez [required] pour ajouter un indicateur visuel (bordure rouge, astérisque).
  4. Combinez input[type="text"][required] pour un style encore plus spécifique.
  5. Utilisez input:not([type="submit"]) pour styliser tous les champs sauf le bouton.
  6. Ajoutez :focus sur input[type] pour un halo de focus coloré par type.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • [attr] : présence de l'attribut (ex: [disabled])
  • [attr="valeur"] : égalité exacte (ex: [type="submit"])
  • [attr^="valeur"] : commence par, préfixe (ex: [href^="https"])
  • [attr$="valeur"] : termine par, suffixe (ex: [src$=".png"])
  • [attr*="valeur"] : contient la sous-chaîne (ex: [class*="btn"])
  • [attr~="valeur"] : mot entier dans une liste (ex: [rel~="noopener"])
  • [attr|="valeur"] : valeur exacte ou suivie d'un tiret (ex: [lang|="fr"])

Mini-glossaire

[attr]
Sélecteur de présence : cible les éléments qui ont l'attribut, quelle que soit sa valeur.
[attr="val"]
Sélecteur d'égalité exacte : la valeur doit correspondre exactement.
^= (préfixe)
Sélecteur « commence par » : la valeur commence par la chaîne indiquée.
$= (suffixe)
Sélecteur « termine par » : la valeur finit par la chaîne indiquée.
*= (contient)
Sélecteur « contient » : la valeur contient la sous-chaîne à n'importe quelle position.
~= (mot)
Sélecteur « mot dans la liste » : le mot exact apparaît dans une liste séparée par des espaces.
|= (tiret)
Sélecteur « préfixe avec tiret » : valeur exacte ou valeur suivie d'un tiret.
i (insensible)
Modificateur optionnel pour rendre la comparaison insensible à la casse.
data-*
Attributs HTML personnalisés pour stocker des données. Accessibles via sélecteurs d'attribut CSS.
Spécificité
Les sélecteurs d'attribut ont une spécificité de 0,0,1,0 (comme les classes).
