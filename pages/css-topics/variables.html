01

Pourquoi des variables CSS ?

Les variables CSS (également appelées Custom Properties) sont des conteneurs de valeurs réutilisables dans l'ensemble de la feuille de style. Introduites en CSS3, elles permettent de centraliser les valeurs (couleurs, espacements, polices) et de les modifier dynamiquement, contrairement aux variables SASS ou LESS qui sont compilées une fois pour toutes et demeurent immuables au runtime.

🎨
Analogie : la peinture du salon

Sans variables, vous devez reproduire la même teinte pour chaque pot — un écart de dosage produirait des murs de couleurs différentes. Avec une variable, vous préparez un unique pot de peinture et l'utilisez sur l'ensemble des surfaces. Pour modifier toute la pièce, un seul pot est à changer.

Avantages des variables CSS

  • Réutilisabilité : une variable déclarée une fois peut être utilisée des centaines de fois
  • Lisibilité : var(--primary) est plus parlant que #2c3e50
  • Thèmes dynamiques : changer toutes les couleurs en redéfinissant quelques variables
  • Runtime : contrairement aux variables SASS, elles respectent la cascade et sont modifiables en JavaScript
📌
Variables CSS vs préprocesseurs

Les variables CSS sont natives, dynamiques, et respectent la cascade et l'héritage. Les variables SASS ($var) sont compilées et immuables au runtime. Les variables CSS remplacent avantageusement les variables de préprocesseur pour la majorité des usages.

02

Déclarer une variable avec --nom

Une variable CSS se déclare à l'aide de deux tirets -- suivis du nom choisi. La déclaration s'effectue dans une règle CSS, et la variable est accessible à l'ensemble des éléments descendants de ce sélecteur. La fonction var() permet de lire la valeur.

Référence complète — --custom-property (conventions de nommage)

AspectRecommandationExemple
PréfixeToujours commencer par ----primary
Cassekebab-case (tirets) — le camelCase est déconseillé--color-primary ✓ / --colorPrimary ✗
Nom sémantiqueCe que la variable représente, pas son apparence--color-primary ✓ / --blue ✗ (si la couleur change)
RegroupementGrouper par catégorie : couleurs, typographie, espacements, ombres--color-*, --font-*, --spacing-*
PortéeDéclarer dans :root pour une portée globale:root { --primary: #3498db; }
CSS — Live editor
:root { --primary: #3498db; --primary-dark: #2980b9; --secondary: #2ecc71; --danger: #e74c3c; --text: #2c3e50; --text-light: #7f8c8d; --bg: #f8f9fa; --radius: 8px; --shadow: 0 2px 8px rgba(0,0,0,.1); --font-main: 'Inter', sans-serif; --spacing: 1rem; --max-width: 1100px; --transition: .3s ease; } body { font-family: var(--font-main); color: var(--text); background: var(--bg); } .btn { background: var(--primary); border-radius: var(--radius); }
RENDU
Variable CSS : --primary
💡
Conventions de nommage sémantique

Utilisez des noms sémantiques plutôt que descriptifs. Préférez --color-primary à --blue. Si la couleur primaire passe du bleu au vert, --color-primary reste cohérent tandis que --blue deviendrait trompeur.

03

var() avec valeur de fallback

La fonction var() admet un deuxième argument : une valeur de fallback employée lorsque la variable n'est pas définie ou qu'elle est invalide. Cette fonctionnalité est essentielle pour créer des composants réutilisables et robustes.

Référence complète — var()

SyntaxeDescriptionContexte d'utilisation
var(--nom)Lit la valeur de la variable --nomUtilisation standard quand la variable est toujours définie
var(--nom, fallback)Lit la valeur de --nom ; si absente, utilise fallbackComposants réutilisables où l'appelant peut ou non définir la variable
var(--nom, var(--autre, fallback))Fallback chaîné : essaie --nom, puis --autre, puis la valeur littéraleSystème de thème avec plusieurs niveaux de surcharge
var(--nom, valeur)Si la variable existe mais est invalide pour la propriété, le fallback est utiliséProtection contre les valeurs mal typées (ex: un string pour une largeur)
CSS 
.element {
  color: var(--text-color, #333);
  /* Si --text-color n'existe pas → #333 */
}

.element {
  background: var(--surface, var(--bg, #fff));
  /* essaie --surface, puis --bg, puis #fff */
}

.card {
  background: var(--card-bg, white);
  color: var(--card-text, var(--text, #333));
  border-radius: var(--card-radius, 8px);
}
⚠️
Attention aux propriétés raccourcies

var() dans une propriété raccourcie comme border ou animation peut faire échouer la propriété entière si la variable est invalide. Exemple : border: var(--border-width, 2px) solid black — si la variable est invalide, toute la bordure disparaît. Privilégiez l'écriture longue.

04

Portée des variables (scope)

Les variables CSS respectent la cascade. Une variable dans :root est globale. Une variable dans un sélecteur spécifique n'est accessible qu'à ce sélecteur et à ses descendants. Cette portée locale permet de créer des composants autonomes sans risque de fuite.

CSS 
:root { --color: #3498db; --padding: 1rem; }

.card {
  --color: #2ecc71;           /* surcharge locale */
  --card-radius: 12px;         /* accessible seulement dans .card */
  color: var(--color);         /* → #2ecc71 */
}

.card__title  { color: var(--color); }  /* → #2ecc71 (hérité) */
.card__footer { --color: #e74c3c; }      /* surcharge dans le footer */

.other { color: var(--color); }          /* → #3498db (globale) */
05

Héritage et mots-clés de cascade avec les variables

Les variables CSS obéissent aux mêmes règles de cascade que les propriétés standard : héritage du parent vers l'enfant, et la déclaration la plus spécifique prévaut. Les mots-clés initial, inherit, unset et revert peuvent être utilisés pour contrôler cet héritage.

Référence complète — initial, inherit, unset, revert

Mot-cléDescriptionUtilisation avec les variables
initialRéinitialise la variable à sa valeur par défaut (inexistante pour les custom properties)--var: initial ← la variable n'a plus de valeur, le fallback s'applique
inheritForce l'héritage explicite de la valeur du parent--var: inherit ← rarement utile car l'héritage est déjà le comportement par défaut
unsetSe comporte comme inherit pour les propriétés héritées, initial sinon--var: unset ← efface la valeur ; le fallback de var() sera utilisé
revertRéinitialise à la valeur de la feuille de style d'agent utilisateur--var: revert ← supprime la valeur définie par l'auteur ; le fallback s'applique
CSS 
:root { --text-color: #333; }
article { --text-color: #1a5276; }

p { color: var(--text-color); }
/* Dans article → #1a5276, hors article → #333 */

.reset-var {
  --text-color: initial;            /* aucune valeur */
  color: var(--text-color, black);  /* → black (fallback) */
}
📌
Les variables ne sont pas des constantes

Les variables CSS peuvent être redéfinies à n'importe quel niveau et leur valeur finale est déterminée par la cascade au moment de l'utilisation. Cette propriété les rend particulièrement adaptées au theming : modifiez une variable dans un contexte et tous les enfants adoptent la nouvelle valeur.

06

Variables et thèmes (mode sombre / clair)

Les variables CSS se révèlent particulièrement utiles pour la création de thèmes. En redéfinissant les mêmes noms de variables dans des contextes différents (attribut data-theme, media query prefers-color-scheme), on modifie l'intégralité de l'apparence d'un site.

CSS 
:root {
  --bg: #ffffff;
  --text: #2c3e50;
  --primary: #3498db;
  --border: #e2e8f0;
}

[data-theme="dark"] {
  --bg: #0D1117;
  --text: #e6edf3;
  --primary: #58a6ff;
  --border: #30363d;
}

@media (prefers-color-scheme: dark) {
  :root:not([data-theme="light"]) {
    --bg: #0D1117;
    --text: #e6edf3;
  }
}

body { background: var(--bg); color: var(--text); }
💡
Stratégie de theming complète

① Déclarez les variables claires dans :root. ② Utilisez @media (prefers-color-scheme: dark) pour détecter le thème système. ③ Permettez à l'utilisateur de forcer un thème via data-theme. ④ Stockez la préférence dans localStorage. ⑤ Ajoutez transition: background .3s, color .3s sur body.

07

Manipuler les variables en JavaScript

L'avantage distinctif des variables CSS : les lire et modifier en JavaScript en temps réel via setProperty() et getPropertyValue().

JavaScript 
const root = document.documentElement;

// Lire une variable
const primary = getComputedStyle(root)
  .getPropertyValue('--primary').trim();

// Modifier une variable
root.style.setProperty('--primary', '#e74c3c');

// Thème toggle
function toggleTheme() {
  const current = root.getAttribute('data-theme');
  root.setAttribute('data-theme',
    current === 'dark' ? 'light' : 'dark');
  localStorage.setItem('theme',
    current === 'dark' ? 'light' : 'dark');
}

// Animation de variable
let hue = 0;
setInterval(() => {
  root.style.setProperty('--hue', hue++);
  root.style.setProperty('--dynamic-color',
    `hsl(${hue}, 70%, 50%)`);
}, 50);

Exercices pratiques

EX 01 🎨 Système de design avec variables
  1. Créez exo-variables.html avec un fichier CSS séparé.
  2. Définissez dans :root au moins 15 variables : couleurs, polices, espacements, ombres.
  3. Créez 3 composants : carte, bouton, badge.
  4. Chaque composant utilise exclusivement les variables définies.
  5. Ajoutez un bouton qui bascule le thème sombre/clair en JS.
EX 02 🌓 Mode sombre avec localStorage
  1. Créez exo-darkmode.html avec deux jeux de variables.
  2. Utilisez @media (prefers-color-scheme: dark).
  3. Ajoutez un bouton "🌙 / ☀️" qui bascule le thème.
  4. Stockez la préférence dans localStorage.
  5. Utilisez getComputedStyle() et setProperty().
🔍
Exercice bonus — Générateur de thème

Créez un sélecteur de couleur primaire (input type="color") qui modifie --primary en JS. Ajoutez un slider pour --spacing (4 px à 32 px). Tous les composants de la page doivent réagir instantanément.

Mini-glossaire

Custom Property
Nom officiel des variables CSS. Propriété personnalisée commençant par --.
var()
Fonction CSS qui lit une variable. Accepte un fallback en second argument.
Fallback
Valeur de secours utilisée quand la variable n'est pas définie.
:root
Sélecteur racine (<html>). Portée globale par défaut pour les variables.
Scope
Zone de validité d'une variable. Globale dans :root, locale dans un sélecteur.
initial
Mot-clé qui réinitialise une variable (perd sa valeur, le fallback s'applique).
setProperty
Méthode JS : element.style.setProperty('--nom', valeur).
getPropertyValue
Méthode JS pour lire une variable (via getComputedStyle).
prefers-color-scheme
Media query détectant le thème système (clair/sombre).
