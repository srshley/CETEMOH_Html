Pseudo-classes et Pseudo-éléments

01

Qu'est-ce qu'une pseudo-classe ?

Une pseudo-classe est un mot-clé qui s'adjoint à un sélecteur pour cibler un élément en fonction de son état ou de sa position dans l'arborescence du document. Elle se caractérise par l'usage d'un unique deux-points : en préfixe. Les pseudo-classes permettent d'appliquer des styles conditionnellement, sans avoir à modifier le HTML ou à recourir à JavaScript.

Les pseudo-éléments (notés ::) ciblent une partie spécifique d'un élément (première lettre, première ligne) ou permettent de générer du contenu virtuel (avant/après). La double notation :: a été introduite par CSS3 pour distinguer clairement les pseudo-éléments des pseudo-classes, bien que la simple notation : reste acceptée par les navigateurs pour la rétrocompatibilité.

Ces deux concepts sont fondamentaux pour la création d'interfaces riches et accessibles. Les pseudo-classes de lien (:hover, :focus) fournissent le retour visuel indispensable à l'interaction utilisateur. Les pseudo-classes structurelles (:nth-child, :first-child) permettent des mises en page sophistiquées sans classes additionnelles dans le HTML. Les pseudo-éléments (::before, ::after) ouvrent la voie au contenu généré et aux décorations purement CSS.

📌
Distinction : simple vs double notation

Historiquement, pseudo-classes et pseudo-éléments utilisaient tous la simple notation :. CSS3 a introduit la double notation :: pour les pseudo-éléments afin de les distinguer clairement. Les navigateurs modernes acceptent les deux formes, mais la convention préconise : pour les pseudo-classes et :: pour les pseudo-éléments.

02

Pseudo-classes de lien : :link, :visited, :hover, :active

Ces pseudo-classes s'appliquent aux éléments <a> et autres éléments interactifs, en fonction de l'état de la navigation. Leur ordre de déclaration est impératif pour un fonctionnement correct : l'ordre recommandé est :link → :visited → :hover → :active, dont le mnémonique est « LoVe HAte » (Love-Hate).

La pseudo-classe :link cible les liens non visités, tandis que :visited cible ceux que l'utilisateur a déjà consultés. Pour des raisons de protection de la vie privée, les styles applicables à :visited sont limités : seules les propriétés color, background-color, border-color et leurs équivalents par côté ou par direction sont autorisés. :hover s'active au passage de la souris, et :active au moment du clic.

Référence complète des pseudo-classes de lien

:visitedLien déjà visité2ePropriétés limitées (color, background-color, border-color)
:hoverSurvol par la souris3eNon disponible sur les écrans tactiles

CSS — Pseudo-classes de lien 
/* Ordre LVHA impératif : LoVe HAte */
a:link    { color: #3498db; text-decoration: none; }
a:visited { color: #8e44ad; }
a:hover   { color: #e74c3c; text-decoration: underline; }
a:active  { color: #c0392b; }

/* Accessibilité : focus clavier */
a:focus-visible { outline: 2px solid #3498db; outline-offset: 2px; }
⚠️
Ordre LVHA impératif

L'ordre :link → :visited → :hover → :active (mnémonique LoVe HAte) doit être impérativement respecté, faute de quoi certaines pseudo-classes risquent de ne pas s'appliquer. Si :hover précède :link, le survol ne fonctionnera pas car :link possède la même spécificité et survient après.

🚀
Pour aller plus loin — :focus-visible et accessibilité

La pseudo-classe :focus-visible affiche le contour de focus uniquement lorsque l'utilisateur navigue au clavier (pas lors d'un clic). Elle offre un équilibre optimal entre esthétique et accessibilité. Utilisez :focus-visible pour les contours de focus et :focus uniquement si vous souhaitez un style identique pour les deux modes. Depuis 2022, tous les navigateurs modernes supportent :focus-visible.

03

Pseudo-classes de formulaire : :focus, :disabled, :checked

Les formulaires web disposent de pseudo-classes dédiées qui reflètent l'état des champs : focus, désactivé, coché, requis, valide ou invalide. Ces sélecteurs permettent de fournir un retour visuel immédiat à l'utilisateur, améliorant ainsi l'expérience de remplissage des formulaires.

La pseudo-classe :focus s'applique lorsque l'élément reçoit le focus (clic, tabulation ou touche). Elle est essentielle pour l'accessibilité, car elle indique clairement quel champ est actif. Les pseudo-classes :valid et :invalid exploitent les attributs de validation HTML5 (required, pattern, type) pour styliser automatiquement les champs valides ou invalides, sans JavaScript.

💡
:focus-within pour les conteneurs

La pseudo-classe :focus-within cible un élément qui contient un champ recevant le focus. Elle est idéale pour styliser un groupe de champs (<fieldset>) ou un formulaire lorsque l'un de ses enfants est actif. Par exemple, form:focus-within { box-shadow: ... } met en valeur l'ensemble du formulaire.

04

Pseudo-classes structurelles : :first-child, :last-child, :nth-child

Ces pseudo-classes ciblent un élément en fonction de sa position relative parmi les éléments frères au sein du même parent. Elles sont particulièrement utiles pour la mise en forme de listes, de tableaux, de grilles et de galeries, où elles évitent de recourir à des classes CSS superflues.

La formule an+b utilisée par :nth-child() mérite une attention particulière. Le paramètre n commence à 0 et s'incrémente à chaque élément. Ainsi, 2n (pair), 2n+1 (impair), 3n+1 (1er, 4e, 7e...) ou -n+3 (trois premiers) permettent un ciblage extrêmement précis. Les mots-clés odd (impair) et even (pair) sont équivalents à 2n+1 et 2n.

CSS — Pseudo-classes structurelles 
li:first-child { font-weight: bold; }
li:last-child  { border-bottom: none; }
li:nth-child(odd)  { background: #f8f9fa; }
li:nth-child(even) { background: #e9ecef; }
li:nth-child(3n+1) { border-left: 3px solid #3498db; }  /* 1er, 4e, 7e... */

/* Différence nth-child vs nth-of-type */
section p:nth-child(2) { }    /* 2e enfant TOUT TYPE — si c'est un p */
section p:nth-of-type(2) { }  /* 2e ÉLÉMENT P — même si d'autres types précèdent */
CSS — Formule an+b 
/* n commence à 0 et s'incrémente */
li:nth-child(3n+1) { }  /* 1, 4, 7, 10... */
li:nth-child(3n+2) { }  /* 2, 5, 8, 11... */
li:nth-child(3n+3) { }  /* 3, 6, 9, 12... (= 3n) */

li:nth-child(n+4) { }   /* à partir du 4e */
li:nth-child(-n+3) { }  /* les 3 premiers */
li:nth-child(n) { }     /* tous (sélectionne tout) */
📌
Différence entre :nth-child et :nth-of-type

:nth-child() dénombre tous les enfants du parent, indépendamment de leur type. :nth-of-type() ne compte que les enfants du même type (même balise). Exemple : p:nth-of-type(2) cible le deuxième paragraphe, même s'il est précédé d'éléments d'autres types (div, img…).

05

:not() — la pseudo-classe de négation

La pseudo-classe :not() cible les éléments qui ne correspondent pas au sélecteur fourni en argument. C'est un outil puissant pour gérer les exceptions sans avoir à surcharger des styles ou à ajouter des classes spécifiques. Depuis CSS Selectors Level 4, :not() accepte une liste de sélecteurs séparés par des virgules, ce qui étend considérablement ses possibilités.

L'usage le plus fréquent de :not() est l'ajout d'un séparateur entre des éléments : li:not(:last-child) { border-bottom: 1px solid #eee; }. Cette technique évite d'avoir une bordure superflue sur le dernier élément, sans nécessiter de classe .last ou de sélecteur supplémentaire. :not() est également utile pour exclure certains types d'éléments d'une règle générale.

Référence complète de :not()

SyntaxeDescriptionExemple
:not(sélecteur)Cible les éléments ne correspondant pas au sélecteurp:not(.intro)
:not(s1, s2, ...)Liste de sélecteurs (CSS4)p:not(.intro, .excerpt)
:not(:pseudo)Négation d'une pseudo-classeli:not(:last-child)
:not([attr])Négation d'un attributinput:not([type="submit"])
:not(:disabled)Éléments activés uniquementbutton:not(:disabled)
CSS — :not() 
/* Séparateur entre éléments (sauf le dernier) */
li:not(:last-child) { border-bottom: 1px solid #eee; }

/* Exclure une classe */
p:not(.intro) { color: #555; }

/* Exclure un type d'input */
input:not([type="submit"]) { border: 1px solid #ccc; }

/* Chaînage (CSS3) */
input:not(:disabled):not([type="hidden"]) { background: white; }

/* Liste de sélecteurs (CSS4) */
p:not(.intro, .excerpt) { color: #333; }
💡
Usages courants de :not()

L'usage le plus fréquent : li:not(:last-child) { border-bottom: ... } pour ajouter un séparateur entre tous les éléments à l'exception du dernier. Cette approche évite d'avoir à recourir à une classe .last ou à un sélecteur :last-child distinct. Également utile pour exclure certains types de champs dans les formulaires.

06

Pseudo-éléments ::before et ::after

::before et ::after génèrent un enfant virtuel respectivement au début ou à la fin de l'élément ciblé. Ils sont essentiels pour insérer du contenu décoratif sans modification du code HTML. Leur présence est conditionnée par la propriété content, qui doit être déclarée même si sa valeur est vide (content: "";).

Ces pseudo-éléments ouvrent un champ d'applications extrêmement vaste : icônes décoratives, badges, guillemets de citation, séparateurs visuels, tooltips purement CSS, compteurs automatiques, overlays, et bien plus encore. Par défaut, ::before et ::after sont de type inline. Pour leur attribuer des dimensions, il convient de définir display: block ou display: inline-block.

CSS
/* Tapez votre CSS pour ::before et ::after */ .blockquote-demo::before { content: "«"; font-size: 2em; color: #ec4899; margin-right: .3em; vertical-align: middle; } .blockquote-demo::after { content: "»"; font-size: 2em; color: #ec4899; margin-left: .3em; vertical-align: bottom; } .badge-demo::after { content: "Nouveau"; font-size: .7rem; background: #e74c3c; color: white; padding: 2px 6px; border-radius: 4px; margin-left: .5rem; }
APERÇU

/* Icône décorative avant un lien */
a[href^="https"]::before { content: "🔗 "; }

/* Badge après un élément */
.nouveau::after {
  content: "Nouveau";
  font-size: .7rem;
  background: #e74c3c;
  color: white;
  padding: 2px 6px;
  border-radius: 4px;
  margin-left: .5rem;

/* Effet visuel (content vide) */
.separator::after {
  content: "";
  display: block;
  width: 50px;
  height: 3px;
  background: #3498db;
  margin-top: .5rem;
}

/* Guillemets */
blockquote::before { content: "«"; }
blockquote::after  { content: "»"; }
⚠️
Obligation de la propriété content

Les pseudo-éléments ::before et ::after ne sont visibles que si la propriété content est déclarée, fût-elle vide (content: "";). En l'absence de la propriété content, le pseudo-élément n'est pas généré.

🚀
Pour aller plus loin — Tooltip CSS avec ::after et attr()

La fonction attr() dans la propriété content permet de récupérer la valeur d'un attribut HTML. Combinée avec ::after et :hover, elle crée des tooltips purement CSS : [data-tooltip]::after { content: attr(data-tooltip); }. Cette technique évite tout JavaScript pour les infobulles simples, améliorant la performance et la maintenabilité.

07

Pseudo-éléments textuels : ::first-line, ::first-letter

Ces pseudo-éléments ciblent des parties spécifiques du texte d'un élément : la première ligne ou la première lettre. Ils sont particulièrement adaptés à la réalisation d'effets typographiques dans la tradition éditoriale — lettrines, première ligne en petites capitales, ou première lettre agrandie et colorée.

::first-letter cible la première lettre d'un bloc de texte à condition qu'elle ne soit pas précédée d'un autre contenu (comme une image). L'effet lettrine, classique dans l'imprimerie, s'obtient en combinant float: left, une font-size plus élevée et line-height: 1. ::first-line cible la première ligne d'un bloc, dont la longueur varie avec la largeur du conteneur — ce qui rend son rendu dynamique et imprévisible.

::first-linePremière ligne de textefont*, color, background, word-spacing, letter-spacing, text-decoration, line-height
::selectionTexte sélectionné par l'utilisateurbackground, color, text-shadow, cursor
::placeholderTexte indicatif d'un champfont*, color, background, text-decoration, opacity

CSS — Pseudo-éléments textuels 
/* ::first-letter — lettrine */
p.lettrine::first-letter {
  font-size: 3em;
  font-weight: bold;
  color: #e74c3c;
  float: left;
  margin-right: .2em;
  line-height: 1;
}

/* ::first-line — première ligne en gras */
p.lead::first-line {
  font-weight: bold;
  font-size: 1.1em;
  color: #2c3e50;
}

/* ::selection — personnalisation de la sélection */
::selection { background: #3498db; color: white; }

/* ::placeholder — texte indicatif */
input::placeholder { color: #999; font-style: italic; }

/* ::marker — puces colorées */
li::marker { color: #3498db; font-weight: bold; }
💡
Réalisation d'une lettrine avec ::first-letter

L'effet lettrine (première lettre agrandie) est un procédé classique de la typographie éditoriale. Combinez float: left, une font-size plus élevée et line-height: 1 pour un alignement optimal. N'omettez pas margin-right pour espacer la lettre du reste du texte.

08

Combinaisons avancées

Les pseudo-classes et pseudo-éléments peuvent être combinés entre eux ainsi qu'avec des sélecteurs classiques pour obtenir un ciblage d'une grande précision. Le chaînage de pseudo-classes (input:focus:not(:disabled)), la combinaison attribut + pseudo-classe + pseudo-élément (a[href^="https"]:hover::after), et les sélecteurs de liste :is() et :where() illustrent la puissance expressive du CSS moderne.

Les compteurs CSS (counter-reset, counter-increment, counter()) permettent une numérotation automatique des sections, titres ou éléments. Combinés avec ::before, ils produisent une table des matières ou une numérotation de chapitres entièrement gérée en CSS. Les sélecteurs :is() et :where() simplifient les sélecteurs longs tout en offrant un contrôle précis sur la spécificité.

CSS — Combinaisons avancées 
/* Pseudo-classe + pseudo-élément */
p:first-child::first-letter { font-size: 2em; }

/* Pseudo-classes chaînées */
input:focus:not(:disabled) { border-color: #3498db; }

/* Combinatoire + pseudo-classe */
ul > li:first-child { font-weight: bold; }
section:not(.intro) p:first-of-type { margin-top: 0; }

/* Attribut + pseudo-classe + pseudo-élément */
a[href^="https"]:hover::after { content: " ↗"; }

/* nth-child + :not() */
li:nth-child(3n+1):not(:last-child) { border-bottom: 2px solid #3498db; }

/* Compteurs automatiques */
body { counter-reset: section; }
h2::before {
  counter-increment: section;
  content: "Section " counter(section) " : ";
  color: #ec4899;
  font-size: .8rem;
}

/* Tooltip CSS */
[data-tooltip] { position: relative; cursor: pointer; }
[data-tooltip]::after {
  content: attr(data-tooltip);
  position: absolute;
  bottom: 100%; left: 50%; transform: translateX(-50%);
  background: #333; color: white; padding: 4px 8px;
  border-radius: 4px; font-size: .75rem; white-space: nowrap;
  opacity: 0; pointer-events: none;
  transition: opacity .2s;
}
[data-tooltip]:hover::after { opacity: 1; }

/* :is() et :where() */
:is(section, article) :is(h2, h3) { margin-top: 1.5em; }
:where(.card, .panel) p { font-size: .9rem; }
/* :where() a 0 spécificité contrairement à :is() */
💡
Différence de spécificité entre :is() et :where()

:is() hérite de la spécificité du sélecteur le plus spécifique présent dans sa liste. :where() possède toujours une spécificité nulle, quel que soit son contenu. Il est conseillé d'utiliser :where() lorsque l'on souhaite pouvoir surcharger facilement les règles.

🚀
Pour aller plus loin — :has() le sélecteur parent

La pseudo-classe :has(), parfois appelée « sélecteur parent », permet de cibler un élément en fonction de ses descendants ou de sa position relative. Par exemple, section:has(h2) cible les sections contenant un titre h2, et figure:has(figcaption) cible les figures avec légende. Supporté par tous les navigateurs modernes depuis 2024, :has() ouvre des possibilités de sélection contextualisée qui relevaient auparavant du JavaScript.
