01

Display : types de boîtes

La propriété display définit le type de boîte d'un élément. Il s'agit de la propriété la plus fondamentale pour la mise en page. Chaque élément HTML possède une valeur display par défaut déterminée par sa nature (block pour les conteneurs, inline pour le texte, etc.).

Référence complète — display

ValeurDescriptionContexte d'utilisation
blockL'élément occupe toute la largeur disponible, commence sur une nouvelle ligne. width/height respectées.Sections, titres, paragraphes, conteneurs
inlineL'élément s'écoule dans le texte, n'occupe que l'espace de son contenu. width/height ignorés.Liens, texte, balises de style (span, strong, em)
inline-blockComme inline (pas de nouvelle ligne) mais accepte width/height comme block.Boutons, cartes, éléments d'interface alignés
noneL'élément est retiré du flux et n'est pas affiché. Inaccessible aux lecteurs d'écran.Masquage complet d'éléments
contentsLa boîte de l'élément est supprimée : seuls ses enfants sont affichés. Utile pour les grilles flexbox/grid.Contournement des conteneurs superflus
flow-rootCrée un nouveau contexte de formatage de bloc (BFC). Contient les floats et les marges.Alternative moderne au clearfix
flexÉtablit un conteneur flexbox en bloc (occupe toute la largeur).Mise en page flexible unidimensionnelle
inline-flexConteneur flexbox en ligne (taille selon son contenu).Barres d'outils flexibles inline
gridÉtablit un conteneur grid en bloc.Mise en page bidimensionnelle
inline-gridConteneur grid en ligne.Grilles dans le flux du texte
table, inline-table, table-row, table-cell, table-caption, table-column, table-column-group, table-header-group, table-footer-group, table-row-groupValeurs dérivées de CSS table. Reproduisent le comportement des balises de tableau HTML.Compatibilité, mise en page tabulaire sans HTML table
list-itemL'élément se comporte comme un élément de liste (avec puce).Éléments de liste personnalisés
run-inL'élément s'intègre dans le bloc suivant s'il est inline (pris en charge limitée).Typographie avancée
CSS 
/* DISPLAY: BLOCK */
.block { display: block; }
/* Par défaut : div, p, h1-h6, ul, ol, li, section, article, nav */

/* DISPLAY: INLINE */
.inline { display: inline; }
/* Par défaut : span, a, strong, em, code, label */

/* DISPLAY: INLINE-BLOCK */
.inline-block {
  display: inline-block;
  width: 150px;
  height: 50px;
}

/* DISPLAY: NONE */
.hidden { display: none; }     /* retiré du flux */
.invisible { visibility: hidden; } /* conservé mais masqué */
📌
display: none vs visibility: hidden

display: none retire complètement l'élément du flux — il n'occupe aucun espace et est inaccessible aux lecteurs d'écran. visibility: hidden le masque visuellement tout en conservant son emplacement dans le flux, mais reste invisible pour les lecteurs d'écran. Pour un contenu masqué visuellement mais accessible, utilisez la classe .sr-only.

02

position et top/right/bottom/left

La propriété position définit le mode de positionnement d'un élément dans le document. Associée aux propriétés top, right, bottom et left, elle permet de contrôler précisément l'emplacement des éléments.

Référence complète — position

ValeurDescriptionContexte d'utilisation
staticFlux normal du document. Les propriétés top, right, bottom, left, z-index sont ignorées. Valeur par défaut.Éléments sans besoin de positionnement spécifique
relativeL'élément reste dans le flux normal mais peut être décalé visuellement via top/right/bottom/left. Sert de point d'ancrage pour les enfants en absolute.Décalage visuel fin, ancrage d'éléments absolus
absoluteL'élément est retiré du flux. Positionné par rapport à l'ancêtre positionné le plus proche (relative, absolute, fixed, sticky).Badges, superpositions, menus déroulants, centrage
fixedL'élément est retiré du flux. Positionné par rapport au viewport. Reste fixe pendant le défilement.Barres de navigation, bannières de cookies, boutons flottants
stickyAlterne entre relative et fixed : relatif jusqu'à un seuil défini par top, puis se fixe. Reste dans le flux.En-têtes de sections collants, barres latérales

Référence complète — top, right, bottom, left

ValeurDescriptionContexte d'utilisation
autoPositionnement automatique — valeur par défaut. Laisse le navigateur calculer la position.Positionnement par défaut
<length>Distance absolue depuis le bord de référence. Ex: 0, 20px, 1.5rem. Peut être négative.Positionnement précis
<percentage>Pourcentage de la hauteur (top/bottom) ou largeur (left/right) du bloc contenant. Ex: 50%.Centrage proportionnel
inheritHérite de la valeur calculée du parent.Forcer l'héritage
CSS 
/* position: static — défaut, pas d'effet de top/left */
.static-element { position: static; }

/* position: relative — décalé visuellement, espace conservé */
.relative-box {
  position: relative;
  top: 20px;
  left: 30px;
}

/* position: absolute — retiré du flux */
.badge {
  position: absolute;
  top: 10px;
  right: 10px;
}

/* Centrage parfait avec absolute */
.center-absolute {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* Overlay occupant tout le parent */
.overlay {
  position: absolute;
  top: 0; right: 0; bottom: 0; left: 0;
  background: rgba(0,0,0,.5);
}
03

Position: relative

L'élément demeure dans le flux normal (son emplacement d'origine est conservé), mais il peut être décalé visuellement à l'aide des propriétés top, right, bottom, left. Plus important encore, position: relative sert de point d'ancrage pour les éléments enfants en position: absolute.

CSS 
.relative-box {
  position: relative;
  top: 20px;
  left: 30px;
}

.relative-up {
  position: relative;
  top: -10px; /* valeurs négatives autorisées */
}

/* Point d'ancrage pour enfant absolute */
.parent-anchor {
  position: relative;
}
.child-absolute {
  position: absolute;
  top: 0;
  right: 0;
}
💡
Fonction essentielle de position: relative

Au-delà du décalage visuel, position: relative est principalement utilisé pour créer un contexte de positionnement. Un élément en position: absolute se positionne par rapport à l'ancêtre positionné le plus proche. En l'absence d'un ancêtre positionné, l'élément absolu remonte jusqu'au viewport.

04

Position: absolute

L'élément est retiré du flux normal : il n'occupe plus d'espace dans son parent et peut se superposer aux autres éléments. Il se positionne par rapport à l'ancêtre positionné le plus proche (relative, absolute, fixed ou sticky).

CSS
.parent { position: relative; width: 100%; height: 100px; background: #e8f4f8; } .badge { position: absolute; top: 10px; right: 10px; background: #e63946; color: white; padding: 4px 8px; border-radius: 4px; } .center-absolute { position: absolute; top: 50%; left: 50%; transform: translate(-50%, -50%); background: #2c3e50; color: white; padding: 6px 14px; border-radius: 4px; }
RENDU
parent relative
BADGE
Centre
⚠️
Conséquence du retrait du flux

Un élément en position: absolute n'occupe plus d'espace dans son parent. Si le parent ne contient que des enfants absolus, sa hauteur sera de 0 (phénomène d'effondrement). Il convient de définir une hauteur explicite pour le parent ou d'utiliser du padding.

05

Position: fixed

À l'instar de absolute, l'élément est retiré du flux. Toutefois, fixed se positionne toujours par rapport au viewport, indépendamment du défilement. Cette propriété est particulièrement adaptée aux barres de navigation, aux bannières de cookies et aux boutons flottants.

CSS 
.navbar {
  position: fixed;
  top: 0; left: 0; right: 0;
  height: 64px;
  background: #1a1a2e;
  color: white;
  z-index: 1000;
}

.fab {
  position: fixed;
  bottom: 24px;
  right: 24px;
  width: 56px; height: 56px;
  border-radius: 50%;
  background: #3498db;
  color: white;
  border: none;
  box-shadow: 0 4px 12px rgba(0,0,0,.3);
  cursor: pointer;
  z-index: 999;
}

.cookie-banner {
  position: fixed;
  bottom: 0; left: 0; right: 0;
  background: #2c3e50;
  color: #ecf0f1;
  padding: 1rem 1.5rem;
  z-index: 1001;
}
📌
Compensation de la hauteur d'un élément fixed

Si vous fixez une barre de navigation en haut de la page, celle-ci sort du flux : le contenu remonte en dessous. Il convient d'ajouter une valeur de padding-top équivalente à la hauteur de la barre de navigation sur l'élément <body>.

06

Position: sticky

Un élément sticky alterne entre les comportements relative et fixed en fonction du défilement. Il se comporte comme un élément relatif jusqu'à ce qu'il atteigne un seuil défini (via top), puis se fixe comme un élément fixed. Il reste confiné aux limites de son parent.

CSS 
.section-header {
  position: sticky;
  top: 80px;          /* seuil de déclenchement */
  background: white;
  z-index: 10;
  padding: .75rem 0;
  border-bottom: 2px solid #3498db;
}

.sidebar-sticky {
  position: sticky;
  top: 24px;
}

/* En-tête de tableau collant */
thead th {
  position: sticky;
  top: 0;
  background: #2c3e50;
  color: white;
  z-index: 5;
}
⚠️
Conditions requises pour le fonctionnement de sticky

Quatre conditions doivent être réunies : ① Le parent doit avoir overflow: visible (pas hidden, scroll ou auto). ② Un seuil doit être défini via top (ou equivalent). ③ Le parent doit être plus grand que l'élément sticky. ④ L'élément sticky reste confiné aux limites de son parent.

07

z-index et contexte d'empilement

Lorsque des éléments positionnés se superposent, la propriété z-index contrôle l'ordre d'empilement sur l'axe Z. Plus la valeur est élevée, plus l'élément apparaît au-dessus des autres. Le contexte d'empilement est un concept fondamental : il crée un groupe d'éléments dont les z-index sont comparés entre eux, isolés des autres groupes.

Référence complète — z-index

ValeurDescriptionContexte d'utilisation
autoL'élément n'a pas de niveau d'empilement propre. Valeur par défaut. Ne crée pas de contexte d'empilement.Éléments sans superposition
<integer>Nombre entier (positif ou négatif). Plus la valeur est élevée, plus l'élément apparaît au-dessus. Ex: 1, 100, 9999, -1.Contrôle de la superposition
initialRétablit la valeur initiale (auto).Réinitialisation

Contexte d'empilement

Un contexte d'empilement est créé par les propriétés suivantes :

  • position: relative/absolute avec z-index différent de auto
  • position: fixed ou sticky (créent toujours un contexte)
  • opacity inférieure à 1
  • transform, filter, perspective, clip-path, mask différents de none
  • isolation: isolate
  • mix-blend-mode différent de normal
  • will-change avec une propriété créant un contexte
  • contain: layout ou paint
CSS 
.box1 { position: absolute; z-index: 1; background: #e74c3c; }
.box2 { position: absolute; z-index: 2; background: #3498db; }
.box3 { position: absolute; z-index: 3; background: #2ecc71; }

/* Contexte d'empilement : l'enfant est limité au parent */
.parent-layer {
  position: relative;
  z-index: 10;  /* crée un contexte d'empilement */
}
.child-layer {
  position: absolute;
  z-index: 999; /* 999 mais limité au contexte du parent */
}
/* Un enfant avec z-index: 999 ne peut pas dépasser
   un élément avec z-index: 20 situé dans un autre contexte */

/* isolement forcé */
.isolated {
  isolation: isolate;
}
💡
Ordre d'empilement par défaut

En l'absence de z-index, l'ordre d'empilement est : fond/bordures → éléments flottés → éléments inline → éléments positionnés (dans l'ordre d'apparition). Un élément positionné apparaissant plus tard se superpose à ses prédécesseurs.

08

Float et clear

La propriété float faisait office de méthode principale de mise en page avant Flexbox et Grid. De nos jours, son usage est réservé à l'habillage de texte autour d'une image (son rôle originel). La propriété clear permet de stopper le flottement.

Référence complète — float

ValeurDescriptionContexte d'utilisation
nonePas de flottement — valeur par défaut. L'élément reste dans le flux normal.Élément standard
leftL'élément flotte à gauche de son conteneur. Le texte s'écoule à sa droite.Images dans les articles, habillage de texte
rightL'élément flotte à droite de son conteneur. Le texte s'écoule à sa gauche.Citations en encadré, images à droite
inline-startFlotte du côté de début de ligne (gauche en LTR, droite en RTL).Internationalisation
inline-endFlotte du côté de fin de ligne (droite en LTR, gauche en RTL).Internationalisation

Référence complète — clear

ValeurDescriptionContexte d'utilisation
nonePas d'effet sur le flottement — valeur par défaut.Élément standard
leftL'élément passe en dessous de tout élément flottant à gauche.Reprendre le flux après un float: left
rightL'élément passe en dessous de tout élément flottant à droite.Reprendre le flux après un float: right
bothL'élément passe en dessous de tout élément flottant (gauche et droite).Reprendre le flux après n'importe quel float

Technique du clearfix

CSS 
.float-left  { float: left;  margin-right: 1rem; }
.float-right { float: right; margin-left: 1rem; }
.clear-left  { clear: left; }
.clear-right { clear: right; }
.clear-both  { clear: both; }

/* La technique du clearfix */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* Usage moderne : habillage de texte */
.article img {
  float: left;
  margin: 0 1.5rem 1rem 0;
  max-width: 40%;
  border-radius: 8px;
}
📌
Usage moderne de float

La propriété float doit être réservée à son rôle originel : permettre au texte de s'écouler autour d'un élément (images, citations). Pour les grilles, barres latérales et mise en page générale, utilisez Flexbox ou Grid.

09

Exercices pratiques

EX 01 📐 Positionnement mis en pratique
  1. Créez exo-position.html avec un conteneur parent position: relative (400x300px, fond gris clair).
  2. Placez 4 carrés de 80x80px de couleurs différentes aux 4 coins du parent en utilisant position: absolute.
  3. Ajoutez un 5e carré centré parfaitement avec top: 50%; left: 50%; transform: translate(-50%, -50%).
  4. Créez une navbar simulée avec position: sticky qui se colle en haut au scroll.
  5. Ajoutez un bouton "Retour en haut" position: fixed en bas à droite.
EX 02 📰 Article avec image flottée
  1. Créez exo-float.html avec un article : titre, 3 paragraphes et une image.
  2. Faites flotter l'image à gauche avec float: left.
  3. Ajoutez une citation blockquote flottée à droite.
  4. Utilisez clear: both pour que le dernier paragraphe ne soit pas à côté.
  5. Ajoutez un badge "Nouveau" avec position: absolute dans le coin supérieur droit.
  6. Utilisez display: inline-block pour aligner 3 icônes horizontalement.
EX 03 📦 Contexte d'empilement
  1. Créez 3 conteneurs position: relative imbriqués.
  2. Chacun contient un carré absolu avec un z-index différent (1, 2, 3).
  3. Donnez au parent du milieu z-index: 10 pour créer un contexte d'empilement.
  4. Observez comment un enfant avec z-index: 999 reste sous un enfant avec z-index: 1 dans un parent avec z-index: 20.
🔍
Exercice bonus

Créez un bouton qui affiche une infobulle (tooltip) au survol en utilisant position: absolute par rapport au bouton en position: relative. Utilisez ::after pour la flèche.

10

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Display : block, inline, inline-block, none, contents, flow-root
  • Position relative : décalé visuellement, sert d'ancrage
  • Position absolute : retiré du flux, positionné par rapport à l'ancêtre
  • Position fixed : retiré du flux, positionné par rapport au viewport
  • Position sticky : alterne entre relative et fixed au défilement
  • top/right/bottom/left : décalages avec length ou pourcentage
  • z-index et contexte d'empilement
  • Float : usage moderne pour l'habillage de texte
  • Clear et la technique du clearfix

Mini-glossaire

Display
Type de boîte : block (toute la largeur), inline (dans le texte), inline-block (dimensionnable).
Position static
Valeur par défaut. top/left/z-index ignorés.
Position relative
L'élément peut être décalé. Sert d'ancrage pour absolute.
Position absolute
Retiré du flux. Positionné par rapport à l'ancêtre positionné.
Position fixed
Retiré du flux. Positionné par rapport au viewport, indépendant du scroll.
Position sticky
Relative jusqu'au seuil (top), puis fixed.
z-index
Ordre d'empilement sur l'axe Z. Nécessite un positionnement différent de static.
Contexte d'empilement
Groupe d'éléments isolé où les z-index sont comparés localement.
Float
Fait flotter un élément pour que le texte s'écoule autour. Usage moderne : habillage.
Clear
Propriété qui stoppe le flottement (left, right, both).
Clearfix
Technique CSS pour contenir des floats dans un parent (display: table + clear: both).
