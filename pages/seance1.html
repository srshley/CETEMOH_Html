01

Modèle Client / Serveur & le protocole HTTP

Avant d'écrire une seule ligne de HTML, il faut comprendre ce qui se passe sous le capot quand vous tapez une adresse dans votre navigateur. Ce mécanisme s'appelle le modèle client / serveur.

🍕
Analogie : la pizzeria

Votre navigateur (Chrome, Firefox) est le client qui passe commande. Le serveur est la pizzeria qui prépare et envoie. Le fichier HTML est la pizza livrée que le navigateur assemble et affiche.

Le voyage d'une requête HTTP

🌍 Navigateur → 📡 Requête HTTP
📡 Requête HTTP → 🖥️ Serveur Web
🖥️ Serveur Web → 🗄️ Base de données
🗄️ Base de données ← 🖥️ Réponse
🖥️ Réponse ← 🌍 Navigateur
💡
HTTP vs HTTPS

HTTP = données en clair. HTTPS = données chiffrées (cadenas vert). Aujourd'hui tous les sites sérieux utilisent HTTPS. En local avec vos fichiers .html, vous utilisez HTTP — c'est normal.

Le trio fondamental du web

🦴
HTML
Le squelette — structure, contenu, sens
🎨
CSS
Les vêtements — couleurs, mise en page
⚡
JavaScript
Les muscles — interactions, logique
02

Navigateur & Éditeur de code

Pour développeur web, deux outils sont indispensables : un navigateur moderne (Chrome, Firefox, Edge) et un éditeur de code (VS Code).

🌐 Navigateur

Chrome est le plus utilisé par les développeurs grâce à ses DevTools (F12). Firefox est excellent aussi. Évitez Internet Explorer.

✏️ VS Code

Éditeur gratuit et puissant. Installez l'extension Live Server pour voir vos modifications en temps réel sans recharger la page.

💡
Extension Live Server

Une fois installée, faites un clic droit sur votre fichier .html dans VS Code, choisissez "Open with Live Server". Le navigateur s'ouvre et se rafraîchit automatiquement à chaque sauvegarde.

03

Votre premier fichier HTML

Créez un dossier mon-site/ et un fichier index.html à l'intérieur. Ouvrez-le dans VS Code, puis collez ce contenu :

HTML — Premier fichier 
<h1>Bonjour le monde !</h1>
<p>Je viens de créer ma première page web.</p>
<p>Ceci est un paragraphe avec un <strong>mot important</strong>.</p>
RENDU

Bonjour le monde !

Je viens de créer ma première page web.

Ceci est un paragraphe avec un mot important.

Sauvegardez (Ctrl+S) puis ouvrez le fichier dans Chrome. Félicitations — vous venez de créer votre première page web !

📌
Live reload

Avec Live Server, pas besoin de recharger manuellement. VS Code le fait pour vous. Pour le désactiver, stoppez le serveur depuis la barre d'état.

04

Doctype & Structure obligatoire

Chaque fichier HTML valide respecte une structure fixe. Ces éléments ne sont pas optionnels :

HTML — Structure obligatoire 
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Titre de la page</title>
  </head>
  <body>
    <h1>Titre principal</h1>
    <p>Mon premier paragraphe.</p>
  </body>
</html>
RENDU

Titre principal

Mon premier paragraphe.

ÉlémentRôle
<!DOCTYPE html>Déclare HTML5 — évite le mode "quirks" qui cause des bugs d'affichage.
<html lang="fr">Racine du document. lang="fr" est crucial pour l'accessibilité et le SEO.
<head>En-tête invisible : métadonnées, CSS, scripts. Rien affiché ici.
<meta charset="UTF-8">Encodage universel. Sans ça : les accents deviennent des caractères bizarres (é → Ã©).
<meta name="viewport">Responsive mobile. Sans elle, le smartphone zoome très en arrière.
<title>Texte de l'onglet + titre Google (50-60 caractères max).
<body>Tout le contenu visible par l'utilisateur.
05

Langue & Encodage

Deux attributs sont essentiels dans tout document HTML : l'encodage et la langue. Sans eux, votre page peut s'afficher de travers.

Pourquoi UTF-8 est obligatoire

UTF-8 est l'encodage universel d'Internet. Il supporte les accents français (é, è, à, ô), les emojis 🎉, le cyrillique (Привет), l'arabe (مرحبا) et le chinois (你好). Sans la balise <meta charset="UTF-8">, le navigateur peut interpréter vos caractères avec un autre encodage (ISO-8859-1 par défaut sur certains serveurs), transformant é en Ã©.

HTML — Encodage et langue 
<!DOCTYPE html>
<html lang="fr">
<head>
  <meta charset="UTF-8">
  <title>Démonstration des accents</title>
</head>
<body>
  <p>Les accents français : é è ê ë à â ä ù û ü ô ö î ï ç.</p>
  <p>Les emojis : 🎉 🚀 💻 🌍 ✅</p>
  <p>Un peu d'anglais : <span lang="en">Hello World!</span></p>
</body>
</html>
RENDU

Les accents français : é è ê ë à â ä ù û ü ô ö î ï ç.

Les emojis : 🎉 🚀 💻 🌍 ✅

Un peu d'anglais : Hello World!

06

Meta tags essentiels

Les balises <meta> fournissent des informations aux moteurs de recherche, navigateurs et réseaux sociaux. Voici les plus utiles :

HTML — Meta tags 
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Cours complet HTML5 pour débutants.">
  <meta name="keywords" content="HTML, cours, débutant, balises">
  <meta name="author" content="Votre Nom">
  <meta name="robots" content="index, follow">
  <title>Mon site - Cours HTML</title>
</head>
RENDU

Les meta sont invisibles, mais cruciales pour le SEO.

Google affiche name="description" sous le titre dans les résultats de recherche.

AttributRôle
name="description"Description affichée sous le titre dans Google (150-160 caractères max).
name="keywords"Mots-clés (moins important aujourd'hui, mais encore utilisé par certains moteurs).
name="robots"index = autoriser l'indexation, nofollow = ne pas suivre les liens.
name="author"Nom de l'auteur de la page.
07

Outils de développement

Les DevTools sont le meilleur ami du développeur web. Accessibles avec la touche F12 (Chrome/Edge/Firefox).

Les onglets essentiels

📝 Éléments

Affiche le code HTML de la page. Vous pouvez double-cliquer pour modifier en direct — pratique pour tester des changements sans toucher au fichier source.

🎨 Styles

Montre le CSS appliqué à chaque élément. Vous pouvez désactiver ou modifier des règles en temps réel pour voir le résultat immédiatement.

📡 Console

Affiche les erreurs, les logs JavaScript. Permet d'exécuter du code JS directement. Indispensable pour le débogage.

🌐 Réseau

Montre toutes les requêtes HTTP : fichiers chargés, temps de réponse, statut (200/404/500). Essentiel pour la performance.

💡
Inspecter un élément

Faites un clic droit sur n'importe quelle partie d'une page → "Inspecter". Les DevTools s'ouvrent directement sur l'élément sélectionné. Essayez-le sur ce cours !

08

Bonnes pratiques de fichiers

Adopter les bonnes habitudes dès le début vous évitera bien des problèmes par la suite.

✅ Noms de fichiers

Uniquement des lettres minuscules, chiffres, tirets - et underscores _. Pas d'espaces, pas d'accents, pas de majuscules.
mon-fichier.html ✓
Mon Fichier.html ✗

✅ Indentation

Indentez chaque niveau d'imbrication avec 2 espaces. VS Code le fait automatiquement avec Shift+Alt+F. Un code bien indenté = un code maintenable.

✅ Toujours fermer les balises

Ne comptez pas sur le navigateur pour "réparer". Chaque balise ouvrante doit avoir sa fermante (sauf les orphelines : <br>, <hr>, <img>, <input>).

✅ Organisation des dossiers

Un dossier css/ pour les styles, js/ pour les scripts, images/ pour les images. Gardez vos fichiers HTML à la racine ou dans pages/.

Exercices pratiques

EX 01 ✏️ Votre première page personnelle
  1. Créez un dossier Seance1/ et un fichier index.html.
  2. Copiez la structure complète HTML5 (DOCTYPE, html, head, body).
  3. Personnalisez le <title> avec votre prénom.
  4. Ajoutez un <h1> avec votre nom complet.
  5. Ajoutez un paragraphe vous décrivant avec <strong> et <em>.
  6. Ajoutez <meta name="description"> et <meta name="author">.
  7. Ouvrez le fichier dans Chrome. Inspectez avec F12.
EX 02 🔧 Live Server
  1. Installez l'extension Live Server dans VS Code.
  2. Faites un clic droit sur votre fichier → "Open with Live Server".
  3. Modifiez le <h1> et sauvegardez — le navigateur se rafraîchit automatiquement.
  4. Essayez de modifier le <title> et vérifiez l'onglet.
EX 03 🔍 Exploration DevTools
  1. Ouvrez Wikipedia dans Chrome. Appuyez sur F12.
  2. Dans l'onglet Éléments, repérez <h1>, <p>, <a>, <img>.
  3. Double-cliquez sur le titre principal et modifiez-le.
  4. Dans l'onglet Console, tapez console.log("Hello !");

Mini-glossaire

Client/Serveur
Modèle où le navigateur (client) demande des ressources et le serveur les fournit. Le web fonctionne uniquement sur requête.
DNS
Domain Name System — annuaire qui traduit un nom de domaine (google.fr) en adresse IP (216.58.213.227).
HTTP / HTTPS
HyperText Transfer Protocol — protocole d'échange. HTTPS = version sécurisée par chiffrement SSL/TLS.
DOCTYPE
Déclaration HTML5. Obligatoire en première ligne. Évite le mode "quirks" des navigateurs.
UTF-8
Encodage universel des caractères. Supporte toutes les langues et les emojis.
Meta tag
Balise orpheline dans <head> qui fournit des métadonnées sur la page (description, mots-clés, etc.).
DevTools
Outils de développement intégrés au navigateur. Accessibles via F12. Permettent d'inspecter, modifier et déboguer une page.
Live Server
Extension VS Code qui lance un serveur local et recharge automatiquement la page à chaque sauvegarde.
