01

Le web : comment ça marche ?

Avant d'écrire une seule ligne de HTML, il faut comprendre ce qui se passe sous le capot quand vous tapez une adresse dans votre navigateur. Ce mécanisme s'appelle le modèle client / serveur.

🍕
Analogie : la pizzeria

Votre navigateur (Chrome, Firefox) est le client qui passe commande. Le serveur est la pizzeria qui prépare et envoie. Le fichier HTML est la pizza livrée que le navigateur assemble et affiche. Sans commande, pas de livraison : le serveur ne vous envoie rien spontanément.

💡
HTTP vs HTTPS — quelle différence ?

HTTP = données échangées en clair sur le réseau (n'importe qui peut intercepter et lire). HTTPS = les données sont chiffrées (cadenas dans la barre d'adresse). Aujourd'hui, tous les sites sérieux utilisent HTTPS. Chrome affiche "Non sécurisé" pour les sites HTTP. Pour votre apprentissage en local (fichier.html), vous utiliserez HTTP — c'est normal.

Le trio fondamental du web

Trois technologies distinctes collaborent sur chaque page web. Elles ont chacune un rôle précis et ne se remplacent pas :

🦴
HTML
Le squelette — structure, contenu, sens sémantique
→ Ce cours (Séances 1–6)
🎨
CSS
Les vêtements — couleurs, polices, mise en page, animations
→ Séances 7–12
⚡
JavaScript
Les muscles — interactions, logique, données dynamiques
→ Séances 13+
📌
Un fichier HTML seul = une page web valide

CSS et JavaScript enrichissent l'expérience, mais ils sont facultatifs. Un fichier .html ouvert dans un navigateur, sans CSS ni JS, est déjà une page web fonctionnelle. C'est pour ça que HTML est la fondation à maîtriser en premier.

02

Syntaxe HTML : balises, éléments, attributs

Le HTML se compose d'instructions entre chevrons < > appelées balises. La syntaxe est stricte mais simple une fois les trois concepts clés assimilés.

Anatomie complète d'un élément HTML

<a  href="https://exemple.com"  target="_blank" >  Visitez ce site  </a>
Balise ouvrante
Débute l'élément. Contient le nom et les attributs.
Attribut + Valeur
Info supplémentaire. Toujours dans la balise ouvrante.
Contenu
Texte, images, ou d'autres éléments HTML imbriqués.
Balise fermante
Le slash / est obligatoire. Même nom que l'ouvrante.

🏷️ Balise

Instruction entre < et >. Elle définit le rôle sémantique du contenu. Le navigateur les reconnaît et sait comment les afficher.

<p>  <h1>  <a>  <ul>  <img>

📦 Élément

Balise ouvrante + contenu + balise fermante. L'ensemble forme un élément. Les éléments peuvent être imbriqués les uns dans les autres.

<p>Mon texte</p>

⚙️ Attribut

Information placée dans la balise ouvrante. Format strict : nom="valeur". Toujours des guillemets droits, pas de guillemets typographiques.

href="url"  alt="texte"  id="nom"

🔴 Balises orphelines

Pas de contenu = pas de balise fermante. Aussi appelées void elements en anglais. Elles s'arrêtent dès la fermeture du chevron.

<br>  <hr>  <img>  <meta>  <input>

HTML — Balises orphelines en situation 
<!-- Saut de ligne forcé — <br> n'a pas de contenu -->
<p>Adresse : 12 rue des Fleurs<br>75001 Paris</p>

<!-- Séparateur horizontal thématique -->
<hr>

<!-- Image : src et alt sont des attributs, pas du contenu -->
<img src="photo.jpg" alt="Un paysage de montagne enneigé">

<!-- Métadonnée dans <head> : aucun contenu à fermer -->
<meta charset="UTF-8">
<meta name="description" content="Mon site personnel">
⚠️
Erreur fréquente — Oublier la balise fermante

Si vous oubliez </p> après un paragraphe, le navigateur essaie de "réparer" seul — avec des résultats imprévisibles selon les navigateurs. Fermez toujours vos balises. VS Code surligne en rouge les balises non fermées.

⚠️
Erreur fréquente — Entrée ≠ retour à la ligne

Le navigateur ignore les espaces multiples et les sauts de ligne dans votre éditeur. Pour forcer un saut : <br>. Pour un nouveau paragraphe : nouvelle balise <p>. Testez-le : appuyez 10 fois sur Entrée dans votre éditeur — le navigateur n'affichera aucun espace supplémentaire.

03

Structure obligatoire d'un document HTML5

Chaque fichier HTML valide respecte une structure fixe. Ces éléments ne sont pas optionnels — leur absence provoque des comportements incohérents selon les navigateurs et nuit au référencement.

HTML — Modèle à copier-coller pour tout nouveau fichier 
<!DOCTYPE html>
<html lang="fr">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <meta name="description" content="Description courte de votre page pour Google">
    <title>Titre de la page — Mon Site</title>
    <!-- Lien vers feuille de style CSS (séance 7+) -->
    <!-- <link rel="stylesheet" href="style.css"> -->
  </head>
  <body>
    <!-- Tout le contenu visible commence ici -->
    <h1>Bonjour le monde !</h1>
    <p>Ma première page web.</p>
  </body>
</html>

Hiérarchie du document — vue arborescente

<!DOCTYPE html>Déclaration de version — toujours en premier, avant tout
<html lang="fr">Racine — contient tout le document
<head>En-tête invisible — instructions pour le navigateur et Google
<meta charset="UTF-8">Encodage universel — accents, emojis, arabes, chinois…
<meta name="viewport">Responsivité mobile — indispensable pour smartphones
<meta name="description">Description affichée sous le titre dans les résultats Google
<title>Texte de l'onglet + titre cliquable sur Google (50–60 car.)
<body>Tout ce qui est VISIBLE par l'utilisateur est ici
<h1>Titre principal (un seul par page)
<p>Paragraphes de texte
<img> <a> <ul> …Images, liens, listes, formulaires, etc.
ÉlémentRôle détaillé
<!DOCTYPE html>Indique au navigateur que ce fichier est en HTML5. Sans cette ligne, le navigateur peut passer en "quirks mode" et interpréter le HTML différemment selon les versions — source de bugs visuels inexpliqués. À mettre avant tout, même avant <html>.
<html lang="fr">Balise racine qui contient l'intégralité du document. L'attribut lang est crucial : les lecteurs d'écran s'en servent pour prononcer correctement le texte dans la bonne langue, et Google l'utilise pour le référencement géographique.
<head>En-tête invisible — rien de ce qu'il contient n'est affiché directement. C'est la "coulisse" : métadonnées, CSS, scripts, informations pour les moteurs de recherche et les réseaux sociaux.
<meta charset="UTF-8">UTF-8 est l'encodage universel : il supporte les accents français (é à ü ï), les emojis 🎉, le cyrillique, l'arabe, le chinois… Sans cette ligne, vos accents s'affichent en caractères corrompus (é → Ã©).
<meta name="viewport">Rend la page responsive sur mobile. Sans elle, le navigateur mobile zoom très en arrière pour tenter de faire rentrer une mise en page "desktop" sur un petit écran — résultat : texte invisible à l'œil nu.
<title>Le texte de l'onglet du navigateur. Aussi le titre cliquable affiché dans les résultats Google. Pour le SEO : 50–60 caractères max, mot-clé principal en premier, sous la forme "Sujet principal — Nom du site".
<body>Tout le contenu visible par l'utilisateur se trouve ici. Titres, textes, images, liens, vidéos, formulaires — 99% de votre travail HTML se passe dans <body>.
⚠️
Erreur classique — Contenu visible dans <head>

Mettre des textes ou images dans <head> au lieu de <body>. Le navigateur ne les affichera pas du tout. Règle simple : si ça doit être vu → <body>. Si c'est une instruction pour le navigateur → <head>.

04

Structurer le texte : titres, paragraphes, mise en valeur

HTML offre 6 niveaux de titres et de nombreuses balises de mise en valeur sémantique. Ces balises ne choisissent pas juste un style visuel — elles donnent du sens au contenu, utilisé par les moteurs de recherche et les technologies d'accessibilité.

💡
Règle d'or — Un seul <h1> par page

Pensez à vos titres comme à un livre : <h1> = titre du livre (1 seul), <h2> = chapitres, <h3> = sous-chapitres… Ne sautez jamais de niveau (h1 → h3 sans h2). Google accorde beaucoup d'importance à cette hiérarchie pour comprendre la structure de votre contenu.

HTML — Titres, paragraphes, mise en valeur, citations 
<!-- Hiérarchie des titres -->
<h1>Titre principal de la page (un seul)</h1>
<h2>Grand chapitre</h2>
<h3>Sous-section</h3>
<h4>Sous-sous-section (rare)</h4>

<!-- Paragraphes avec mise en valeur -->
<p>Le <strong>HTML</strong> est le <em>fondement</em> du web.</p>
<p><mark>À retenir</mark> : toute balise ouvrante doit être fermée.</p>
<p><del>Ancien prix : 100€</del> <ins>Nouveau prix : 80€</ins></p>
<p>Appuyez sur <kbd>Ctrl</kbd> + <kbd>S</kbd> pour sauvegarder.</p>

<!-- Saut de ligne forcé à l'intérieur d'un paragraphe -->
<p>12 rue des Fleurs<br>75001 Paris<br>France</p>

<!-- Abréviation avec infobulle (survol souris) -->
<p>J'apprends le <abbr title="HyperText Markup Language">HTML</abbr>.</p>

<!-- Citation longue depuis une source externe -->
<blockquote cite="https://fr.wikiquote.org">
  <p>"La créativité, c'est l'intelligence qui s'amuse."</p>
  <footer>— Albert Einstein</footer>
</blockquote>

<!-- Séparateur thématique -->
<hr>

<!-- Code informatique inline -->
<p>Utilisez la balise <code>&lt;p&gt;</code> pour un paragraphe.</p>

<!-- Note secondaire -->
<p><small>&copy; 2024 Mon Site. Tous droits réservés.</small></p>

📌
Sémantique ≠ style visuel

N'utilisez pas <strong> juste pour mettre du gras. Utilisez-le parce que le contenu est réellement important. Si vous voulez du gras purement décoratif → c'est le rôle du CSS (font-weight: bold). Cette distinction est fondamentale pour l'accessibilité (WCAG) et le SEO.

05

Les listes : ul, ol et imbrication

HTML propose deux types de listes de base. Le choix entre elles est sémantique : choisissez celle qui correspond au sens de votre contenu, indépendamment de l'apparence visuelle (modifiable plus tard en CSS).

HTML — ul, ol et imbrication 
<!-- Liste NON ordonnée : l'ordre n'a pas d'importance -->
<ul>
  <li>Pain</li>
  <li>Lait</li>
  <li>Œufs</li>
</ul>

<!-- Liste ORDONNÉE : la séquence est importante -->
<ol>
  <li>Préchauffer le four à 180°C</li>
  <li>Mélanger farine, sucre et beurre</li>
  <li>Enfourner pendant 25 minutes</li>
</ol>

<!-- Imbrication : <li> peut contenir une nouvelle liste -->
<ul>
  <li>Fruits tropicaux
    <ul>
      <li>Mangue</li>
      <li>Ananas</li>
      <li>Papaye</li>
    </ul>
  </li>
  <li>Légumes racines
    <ul>
      <li>Carotte</li>
      <li>Betterave</li>
    </ul>
  </li>
</ul>

<!-- Mélange ol + ul -->
<ol>
  <li>Choisir les ingrédients
    <ul>
      <li>Option A : chocolat noir 70%</li>
      <li>Option B : chocolat au lait</li>
    </ul>
  </li>
  <li>Faire fondre au bain-marie</li>
  <li>Laisser refroidir 30 min</li>
</ol>
Rendu — Liste non ordonnée <ul>
  • Pain
  • Lait
  • Œufs
Ordre non garanti → puces • par défaut
Rendu — Liste ordonnée <ol>
  1. Préchauffer à 180°C
  2. Mélanger les ingrédients
  3. Enfourner 25 min
Numérotation automatique → 1, 2, 3…
Rendu — Liste imbriquée (ul dans ul)
  • Fruits tropicaux
    • Mangue
    • Ananas
    • Papaye
  • Légumes racines
    • Carotte
    • Betterave
💡
Indentation = lisibilité du code

Indentez chaque niveau d'imbrication avec 2 espaces (ou Tab dans VS Code). Un code bien indenté est 10× plus facile à lire et à déboguer. VS Code formate automatiquement avec Shift+Alt+F.

📌
Accessibilité — Ne pas simuler une liste avec des tirets

Évitez d'écrire <p>- Pain</p><p>- Lait</p>. Les lecteurs d'écran annoncent "liste de 3 éléments" pour une vraie <ul> — permettant à l'utilisateur de naviguer directement entre les éléments. Une fausse liste avec des tirets est juste du texte.

06

Les liens hypertextes <a> — approfondissement

Le lien est l'invention fondatrice du web. Tim Berners-Lee a conçu le web en 1989 précisément comme un système de documents reliés — d'où le nom HyperText. La balise <a> (pour anchor, ancre) concrétise cette connexion.

Les six formes de liens

🌐
Externe
href="https://..."
Vers un autre site. URL complète avec https:// obligatoire.
📄
Interne relatif
href="page.html"
Vers une autre page du même site. Chemin relatif calculé depuis le fichier courant.
⚓
Ancre interne
href="#section"
Vers une section de la même page via son id. Le scroll est animé.
📧
Email
href="mailto:..."
Ouvre le client email. On peut pré-remplir sujet et corps.
📞
Téléphone
href="tel:+33..."
Compose sur mobile. Format international obligatoire.
🪟
Nouvel onglet
target="_blank"
Ouvre dans un nouvel onglet. Toujours ajouter rel="noopener".

Chemins relatifs — comment ça marche

Un chemin relatif est calculé depuis l'emplacement du fichier HTML courant. Comprendre l'arborescence de dossiers est indispensable pour que vos liens internes fonctionnent.

Structure d'un site typique :

📁mon-site/
📄index.html← fichier courant
📄contact.htmlhref="contact.html"✓ même dossier
📁blog/
📄article.htmlhref="blog/article.html"✓ sous-dossier
📄liste.htmlhref="blog/liste.html"
📁images/
🖼️logo.pngsrc="images/logo.png"
📁css/
🎨style.csshref="css/style.css"
Si le fichier courant est blog/article.html :
href="../index.html" → remonte d'un niveau (../) pour atteindre index.html
href="../contact.html" → remonte puis pointe vers contact.html
href="liste.html" → reste dans le même dossier blog/
HTML — Tous les types de liens avec détails 
<!-- ① Lien externe -->
<a href="https://www.wikipedia.org">Aller sur Wikipedia</a>

<!-- ② Liens internes — différents chemins relatifs -->
<a href="contact.html">Page Contact (même dossier)</a>
<a href="blog/article.html">Sous-dossier blog</a>
<a href="../index.html">Dossier parent</a>
<a href="/index.html">Racine du site (chemin absolu relatif)</a>

<!-- ③ Nouvel onglet — TOUJOURS avec rel="noopener noreferrer" -->
<a href="https://exemple.com" target="_blank" rel="noopener noreferrer">
  Ouvrir dans un nouvel onglet
</a>

<!-- ④ Ancre interne — cible doit avoir un id -->
<h2 id="section-contact">Nous contacter</h2>
<a href="#section-contact">Aller à la section Contact</a>
<a href="#">Retour en haut</a>
<a href="autre-page.html#bas-de-page">Ancre sur une autre page</a>

<!-- ⑤ Email avec paramètres pré-remplis -->
<a href="mailto:contact@exemple.com">Email simple</a>
<a href="mailto:contact@exemple.com?subject=Demande de devis">Avec sujet</a>
<a href="mailto:contact@exemple.com?subject=Devis&body=Bonjour%2C%0A%0AJe souhaite...">
  Avec sujet et corps pré-rempli
</a>
<!-- Encodage URL : espace = %20, virgule = %2C, retour ligne = %0A -->

<!-- ⑥ Téléphone — format international obligatoire -->
<a href="tel:+33600000000">+33 6 00 00 00 00</a>

<!-- ⑦ Téléchargement (fichier du même domaine) -->

  Télécharger le guide PDF

⚠️
Sécurité — rel="noopener noreferrer" obligatoire avec target="_blank"

Sans rel="noopener", la page ouverte peut accéder à votre page via window.opener et la rediriger vers un site malveillant — une attaque appelée tabnapping. noreferrer empêche en plus l'envoi de l'URL de votre page en tant que référent HTTP. Ces deux valeurs ensemble protègent à la fois vos visiteurs et vous.

📌
Accessibilité — Le texte du lien doit avoir du sens seul

❌ <a href="article.html">Cliquez ici</a> — un lecteur d'écran annonce juste "cliquez ici" sans contexte.
✅ <a href="article.html">Lire notre article sur le HTML5</a> — clair et descriptif.
Si le texte visible est court (icône, "Lire la suite"), ajoutez aria-label="Lire la suite sur le HTML5" pour les lecteurs d'écran.

💡
Encodage URL dans les mailto:

Les caractères spéciaux dans les paramètres mailto doivent être encodés : espace → %20, retour à la ligne → %0A, virgule → %2C, @ → %40. Utilisez un outil comme encodeURIComponent() en JavaScript pour automatiser cela.

07

Les images <img> — approfondissement

La balise <img> est une balise orpheline qui insère une image dans le flux du document. Maîtriser ses attributs est essentiel pour la performance, l'accessibilité et le référencement.

Attributs de <img> — rôle de chacun

🖼️photo.jpg600×400 px
alt="Chat roux endormi sur canapé"
src ★
Source — chemin local ou URL. Sans src, l'image est brisée.
alt ★
Texte alternatif — lu par les lecteurs d'écran, affiché si image cassée, indexé par Google. alt="" pour les images décoratives.
width/height
Dimensions en pixels. Toujours définir les deux — évite le CLS (saut de mise en page au chargement).
loading="lazy"
Charge l'image seulement quand visible à l'écran. Améliore le temps de chargement initial.
decoding="async"
Décode l'image en parallèle du reste de la page. Améliore la réactivité.
HTML — Images : tous les cas d'usage 
<!-- ① Image basique avec attributs recommandés -->
<img src="images/chat.jpg"
     alt="Un chat roux endormi sur un canapé bleu"
     width="600" height="400"
     loading="lazy"
     decoding="async">

<!-- ② Image depuis une URL externe -->
<img src="https://picsum.photos/seed/nature/500/300"
     alt="Paysage naturel"
     width="500" height="300"
     loading="lazy">

<!-- ③ Image décorative — alt vide = ignorée par les lecteurs d'écran -->
<img src="separateur.png" alt="" width="800" height="4">

<!-- ④ Image avec légende sémantique -->
<figure>
  <img src="tokyo.jpg"
       alt="Vue aérienne de Tokyo au coucher du soleil depuis la Skytree"
       width="800" height="500"
       loading="lazy">
  <figcaption>Tokyo vu depuis la Tokyo Skytree, juillet 2024.
    Photo : <a href="https://example.com">Jean Dupont</a>
  </figcaption>
</figure>

<!-- ⑤ Image responsive avec srcset -->
<img src="image-800.jpg"
     srcset="image-400.jpg 400w,
             image-800.jpg 800w,
             image-1600.jpg 1600w"
     sizes="(max-width: 480px) 400px,
            (max-width: 1024px) 800px,
            1600px"
     alt="Paysage de montagne"
     width="1600" height="900"
     loading="lazy">

<!-- ⑥ <picture> : contrôle total (format ET art direction) -->
<picture>
  <!-- Format AVIF pour navigateurs compatibles (meilleure compression) -->
  <source srcset="photo.avif" type="image/avif">
  <!-- Format WebP pour navigateurs modernes -->
  <source srcset="photo.webp" type="image/webp">
  <!-- Fallback JPG universel -->
  <img src="photo.jpg"
       alt="Coucher de soleil sur la mer"

Guide complet de l'attribut alt

✅ Bons alt
alt="Portrait de Marie Curie en 1903, prix Nobel de physique"

alt="Graphique montrant une croissance de 40% des ventes entre 2022 et 2024"

alt="Bouton rouge Acheter maintenant"

❌ Mauvais alt
alt="image" ou alt="photo"

alt="DSC_4892_final_v2.jpg"

alt="image de l'article"

alt="image image image" (bourrage de mots-clés)
💡
Règle pratique pour rédiger un bon alt

Fermez les yeux et imaginez qu'on vous lit la page à voix haute. Que voudriez-vous entendre à la place de l'image ? C'est votre alt. Pour une image dans un article, décrivez ce qui est visible ET ce qui est pertinent pour le contexte. Pour un graphique, donnez la conclusion ou la tendance principale, pas juste "graphique".

📌
Performance — Pourquoi width et height sont essentiels

Sans ces attributs, le navigateur ne sait pas quelle place réserver avant que l'image charge. Il affiche d'abord le texte, puis quand l'image arrive, tout le contenu "saute" vers le bas. Ce phénomène s'appelle CLS (Cumulative Layout Shift). Google le pénalise depuis 2021 dans son algorithme Core Web Vitals. Un score CLS mauvais peut faire reculer votre site dans les résultats de recherche.

08

div, span, commentaires & entités HTML

Deux balises "neutres" permettent de regrouper du contenu sans lui donner de sens sémantique propre. Elles sont omniprésentes dans le développement web car elles servent de base au CSS et au JavaScript.

La différence cruciale : bloc vs en ligne

📦 <div> — Élément de bloc
<div> 1 — occupe toute la largeur
<div> 2 — force un retour à la ligne
<div> 3 — chaque div sur sa propre ligne
🖊️ <span> — Élément en ligne
Texte avant <span> A texte du milieu <span> B texte après
Les spans restent dans le flux du texte
HTML — div, span et commentaires

  <a href="acheter.html">Ajouter au panier</a>

Entités HTML — caractères réservés et spéciaux

Certains caractères sont "réservés" par HTML : < et > démarrent/finissent des balises, & démarre une entité. Pour les afficher littéralement dans le texte, il faut utiliser leur code entité.

HTML — Entités courantes 
<!-- Afficher des chevrons sans créer une balise -->
<p>La balise paragraphe s'écrit : &lt;p&gt;</p>
<!-- Résultat affiché : La balise paragraphe s'écrit : <p> -->

<!-- Afficher une esperluette -->
<p>Rock &amp; Roll</p>
<!-- Résultat : Rock & Roll -->

<!-- Espace insécable : empêche la coupure entre deux mots -->
<p>M.&nbsp;Dupont a payé 29&nbsp;€.</p>
<!-- Sans &nbsp; : "M." pourrait finir une ligne et "Dupont" commencer la suivante -->

<!-- Symboles courants -->
<p>&copy; 2024 Mon Site. Tous droits réservés.</p>
<p>Température extérieure : 37&deg;C</p>
<p>Contact&nbsp;&mdash;&nbsp;nous écrire ou appeler</p>

Exercices pratiques

EX 01 ✏️ Votre première page personnelle
  1. Créez un dossier Seance1/ et un fichier index.html à l'intérieur. Copiez la structure HTML5 complète de la section 3.
  2. Personnalisez le <title> (visible dans l'onglet) et la <meta name="description">.
  3. Dans <body> : un <h1> avec votre prénom et nom, un <h2> "Qui suis-je ?", un paragraphe vous décrivant avec au moins un <strong> et un <em>.
  4. Un <h3> "Mes centres d'intérêt" suivi d'une <ul> de 4 items. L'un doit contenir une sous-liste <ol> de 3 éléments ordonnés.
  5. Un <blockquote> avec une citation qui vous inspire (avec <footer> pour l'attribution).
  6. Un <hr> puis une <address> avec votre ville (ligne 1) et votre pays (ligne 2), séparés par <br>.
  7. Ouvrez le fichier dans Chrome. Inspectez avec F12 → onglet Éléments pour voir votre arbre HTML.
EX 02 🔗 Liens, images et entités
  1. Ajoutez un lien externe vers un site que vous aimez, avec target="_blank" rel="noopener noreferrer".
  2. Ajoutez un lien mailto: vers une adresse fictive et un tel: vers un numéro fictif.
  3. Ajoutez id="haut" à votre <h1>, puis un lien "↑ Retour en haut" tout en bas pointant vers #haut.
  4. Insérez une image depuis https://picsum.photos/seed/monprenom/500/300 (remplacez "monprenom") avec un alt descriptif, des dimensions et loading="lazy".
  5. Encadrez cette image dans <figure></figure> avec un <figcaption> de votre choix.
  6. Utilisez au moins 4 entités HTML différentes dans votre page (&copy;, &mdash;, &nbsp;, &deg;…).
  7. Ajoutez un commentaire HTML expliquant à quoi sert chaque grande section de votre code.
🔍
Exercice bonus — Inspecteur de code sur un vrai site

Ouvrez Wikipedia dans Chrome. Appuyez sur F12 → onglet Éléments. Retrouvez les balises apprises : <h1>, <p>, <a>, <img>, <ul>. Double-cliquez sur le titre de l'article pour le modifier en direct. Mini-glossaire

Balise
Instruction entre chevrons < >. Définit le rôle sémantique du contenu. Ex : <p>, <h1>, <a>.
Élément
Balise ouvrante + contenu + balise fermante = un bloc de sens. Les éléments s'imbriquent les uns dans les autres.
Attribut
Information dans la balise ouvrante. Format : nom="valeur". Ex : href="url", alt="texte", id="nom".
DOCTYPE
Déclaration HTML5 en toute première ligne. Évite le mode "quirks" et assure un comportement cohérent entre navigateurs.
Balise orpheline
Balise sans contenu ni fermeture. Ex : <br>, <img>, <hr>, <meta>, <input>.
Sémantique HTML
Principe de choisir les balises selon leur signification (pas leur style). Ex : <strong> pour importance réelle, pas juste pour le gras visuel.
Entité HTML
Code commençant par & pour afficher un caractère spécial. Ex : &copy; → ©, &lt; → <, &nbsp; → espace insécable.
id
Identifiant unique d'un élément dans toute la page. Utilisé pour les ancres (<a href="#id">) et le ciblage CSS/JS.
alt
Texte alternatif d'une image. Obligatoire : lu par lecteurs d'écran, affiché si image cassée, indexé par Google.
chemin relatif
Chemin calculé depuis le fichier courant. Ex : "images/chat.jpg", "../index.html". Recommandé pour les liens internes.
