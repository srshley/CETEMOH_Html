01

Transitions : property, duration, timing, delay

Les transitions CSS permettent de fluidifier les changements d'état (hover, focus, ajout de classe). Au lieu d'un changement instantané, la valeur évolue progressivement.

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé
CSS 
/* Shorthand : property duration timing delay */
.btn {
  background: #3498db;
  color: white;
  padding: .75em 1.5em;
  border: none;
  border-radius: 6px;
  transition: background .3s ease,
              transform .2s ease,
              box-shadow .3s ease;
  cursor: pointer;
}

.btn:hover {
  background: #2980b9;
  transform: translateY(-2px);
  box-shadow: 0 4px 12px rgba(0,0,0,.2);
}

/* Timing functions */
.ease     { transition: all .3s ease; }         /* démarre vite, ralentit à la fin (défaut) */
.linear   { transition: all .3s linear; }       /* vitesse constante */
.ease-in  { transition: all .3s ease-in; }      /* démarre lentement, accélère */
.ease-out { transition: all .3s ease-out; }     /* démarre vite, ralentit */
.ease-in-out { transition: all .3s ease-in-out; } /* lent au début et à la fin */

/* Cubic-bezier personnalisé */
.custom   { transition: all .3s cubic-bezier(.68, -.55, .265, 1.55); }

/* transition: all — à éviter (toutes les propriétés transitionnent) */
/* Préférez lister explicitement les propriétés à transitionner */
RENDU
Survolez-moi
💡
Les propriétés transitionnables

Toutes les propriétés CSS ne peuvent pas être animées en transition. Les plus courantes : opacity, transform, background-color, color, box-shadow, border. Certaines comme display ne sont pas animables. Pour un changement de display, utilisez plutôt opacity + visibility combinés. Les transitions sur width et height fonctionnent mais sont coûteuses en performance — préférez transform.

02

Transform : rotate, scale, translate

transform modifie la forme, la taille et la position d'un élément sans affecter le flux du document. C'est la propriété la plus performante pour les animations car elle utilise l'accélération matérielle GPU.

CSS 
/* TRANSLATE — déplacement */
.translate   { transform: translate(20px, 10px); }    /* X et Y */
.translate-x { transform: translateX(30px); }         /* horizontal uniquement */
.translate-y { transform: translateY(-20px); }        /* vertical (négatif = monte) */
.center      { transform: translate(-50%, -50%); }    /* centrage parfait d'un absolu */

/* ROTATE — rotation */
.rotate      { transform: rotate(45deg); }   /* 45° dans le sens horaire */
.rotate-neg  { transform: rotate(-90deg); }  /* 90° antihoraire */
.rotate-half { transform: rotate(.5turn); }  /* 180° en tours */

/* SCALE — mise à l'échelle */
.scale       { transform: scale(1.2); }      /* 120% de la taille */
.scale-x     { transform: scaleX(1.5); }     /* étirement horizontal */
.scale-y     { transform: scaleY(0.8); }     /* rétrécissement vertical */

/* SKEW — déformation (inclinaison) */
.skew        { transform: skew(10deg, 5deg); }

/* Combinaisons (ordre important) */
.combo       { transform: translateX(20px) rotate(45deg) scale(1.1); }

/* Transform-origin (point d'origine) */
.origin-center { transform-origin: center; }       /* défaut */
.origin-top    { transform-origin: top left; }
.origin-own    { transform-origin: 30% 70%; }
RENDU
rotate(15deg)
scale(1.1)
translateY
📌
Transform = GPU, pas CPU

Contrairement à margin, left, width ou height qui sollicitent le processeur (CPU) et déclenchent un reflow (recalcul complet de la mise en page), transform utilise uniquement le processeur graphique (GPU) et ne déclenche qu'un composite — beaucoup plus rapide. Pour des animations fluides à 60 images par seconde, utilisez transform et opacity exclusivement.

03

@keyframes — définir des animations

Les keyframes (images clés) définissent les étapes d'une animation. On spécifie des points de pourcentage (0% à 100%) avec les propriétés CSS à chaque étape.

CSS 
/* Définition de l'animation */
@keyframes fadeIn {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

@keyframes pulse {
  0%   { transform: scale(1); }
  50%  { transform: scale(1.05); }
  100% { transform: scale(1); }
}

@keyframes shake {
  0%, 100% { transform: translateX(0); }
  25%      { transform: translateX(-10px); }
  75%      { transform: translateX(10px); }
}

@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

@keyframes slideInLeft {
  from {
    opacity: 0;
    transform: translateX(-100%);
  }
  to {
    opacity: 1;
    transform: translateX(0);
  }
}
RENDU
Animation !
💡
from…to vs 0%…100%

Les mots-clés from et to sont équivalents à 0% et 100%. Pour les animations simples, from { } to { } est plus lisible. Pour les animations complexes avec plusieurs étapes intermédiaires, utilisez les pourcentages (0%, 25%, 50%, 75%, 100%). Les navigateurs interpolent automatiquement entre chaque étape.

04

Animation : name, duration, iteration, direction

La propriété animation applique les keyframes à un élément. Le shorthand animation regroupe toutes les sous-propriétés.

CSS 
/* Shorthand complet : name duration timing delay iteration direction fill-mode */
.fade-in {
  animation: fadeIn .6s ease-out .2s both;
  /* se déclenche après 0.2s, dure 0.6s, garde l'état final (both) */
}

/* Sous-propriétés séparées */
.spinner {
  animation-name: spin;
  animation-duration: 1s;
  animation-timing-function: linear;
  animation-delay: 0s;
  animation-iteration-count: infinite;   /* infini */
  animation-direction: normal;           /* normal, reverse, alternate, alternate-reverse */
  animation-fill-mode: both;             /* none, forwards, backwards, both */
  animation-play-state: running;         /* running, paused */
}

/* Valeurs pratiques */
.pulse-once  { animation: pulse .3s ease 2; }                 /* 2 fois */
.pulse-inf   { animation: pulse 2s ease-in-out infinite; }    /* infini */
.bounce-alt  { animation: pulse 1s ease alternate infinite; } /* aller-retour */

/* Animation au scroll avec JS (déclenchement par classe) */
.animate-on-scroll {
  opacity: 0;
  animation: fadeIn .6s ease forwards;
}
.animate-on-scroll.visible {
  /* la classe .visible est ajoutée par JS au scroll */
}
RENDU
Chargement…
⚠️
Accessibilité — prefers-reduced-motion

De nombreuses personnes souffrent de nausées ou de vertiges avec les animations. Utilisez toujours la media query @media (prefers-reduced-motion: reduce) pour désactiver ou réduire les animations : animation-duration: 0.01ms !important; et transition-duration: 0.01ms !important;. C'est une exigence WCAG (Accessibilité web). Les systèmes d'exploitation proposent cette option dans les paramètres d'accessibilité.

05

Variables CSS (Custom Properties)

Les variables CSS (Custom Properties) sont des conteneurs de valeurs réutilisables dans toute la feuille de style. Elles commencent par -- et s'utilisent avec var(). Elles respectent la cascade et l'héritage.

CSS 
/* Déclaration — souvent dans :root (portée globale) */
:root {
  --primary: #3498db;
  --primary-dark: #2980b9;
  --secondary: #2ecc71;
  --danger: #e74c3c;
  --text: #2c3e50;
  --text-light: #7f8c8d;
  --bg: #f8f9fa;
  --radius: 8px;
  --shadow: 0 2px 8px rgba(0,0,0,.1);
  --font-main: 'Inter', sans-serif;
  --spacing: 1rem;
  --max-width: 1100px;
}

/* Utilisation */
body { font-family: var(--font-main); color: var(--text); background: var(--bg); }
.btn { background: var(--primary); border-radius: var(--radius); box-shadow: var(--shadow); }
.btn:hover { background: var(--primary-dark); }

/* Valeur de fallback */
.special { color: var(--undefined-var, #333); }  /* utilise #333 si la variable n'existe pas */

/* Portée locale — redéfinition dans un contexte spécifique */
.card {
  --card-padding: 1.5rem;
  padding: var(--card-padding);
}
.card-compact {
  --card-padding: .75rem;  /* redéfinit la variable locale */
  padding: var(--card-padding);
}

/* Variables dans les media queries */
@media (max-width: 768px) {
  :root {
    --spacing: .75rem;
    --max-width: 100%;
  }
}
RENDU
Variable CSS : --primary
📌
Variables CSS vs préprocesseurs (SASS, LESS)

Les variables CSS sont natives et vivantes : elles respectent la cascade, l'héritage, et peuvent être modifiées en JavaScript ou dans les media queries. Les variables SASS ($var) sont compilées une fois pour toutes et ne changent pas au runtime. Les variables CSS remplacent avantageusement les variables de préprocesseur pour la plupart des usages. Les préprocesseurs restent utiles pour les mixins, les boucles et l'imbrication.

06

calc(), min(), max(), clamp()

Les fonctions mathématiques CSS permettent des calculs directement dans les déclarations. Elles sont essentielles pour créer des valeurs fluides et adaptatives.

CSS 
/* calc() — calcul arithmétique */
.calc {
  width: calc(100% - 40px);        /* largeur moins 40px */
  width: calc(50% - 2rem);         /* moitié moins 2rem */
  font-size: calc(1rem + .5vw);    /* taille fluide */
  height: calc(100vh - 80px);      /* plein écran moins navbar */
}

/* min() — prend la plus petite valeur */
.min-el {
  width: min(100%, 1200px);       /* jamais plus de 1200px */
  font-size: min(3rem, 6vw);      /* 3rem max, sinon 6vw */
}

/* max() — prend la plus grande valeur */
.max-el {
  width: max(300px, 50%);         /* au moins 300px */
  padding: max(1rem, 2vw);        /* au moins 1rem */
}

/* clamp() — entre min et max (déjà vu séance 9) */
.clamp {
  font-size: clamp(1rem, 2.5vw, 2.5rem);
  /* ne descend pas sous 1rem, ne dépasse pas 2.5rem */
}

/* Combinaisons */
.combo-func {
  width: clamp(300px, 60%, 800px);
  padding: calc(clamp(1rem, 2vw, 2rem) + .5rem);
}
RENDU
calc(100% - 40px)
💡
calc() avec des unités mélangées

calc() peut mélanger des unités : calc(100vh - 80px), calc(50% - 2rem). Important : les espaces autour des opérateurs + et - sont obligatoires (pour éviter la confusion avec les nombres négatifs). Pour * et /, les espaces ne sont pas nécessaires mais recommandés. min() et max() fonctionnent comme clamp() mais avec seulement deux valeurs.

07

Filters : blur, grayscale, brightness

La propriété filter applique des effets visuels (comme les filtres Instagram) à un élément. On peut les combiner et les animer.

CSS 
/* Filtres individuels */
.blur       { filter: blur(4px); }           /* flou gaussien */
.brightness { filter: brightness(1.2); }     /* luminosité (1 = normal) */
.contrast   { filter: contrast(1.5); }       /* contraste */
.grayscale  { filter: grayscale(1); }        /* noir et blanc (1 = 100%) */
.sepia      { filter: sepia(.8); }           /* effet sépia */
.hue-rotate { filter: hue-rotate(90deg); }   /* rotation des teintes */
.saturate   { filter: saturate(2); }         /* saturation */
.invert     { filter: invert(1); }           /* inversion des couleurs */
.opacity    { filter: opacity(.5); }         /* transparence (alternative à opacity) */

/* Combinaisons */
.filter-combo {
  filter: grayscale(.5) brightness(1.1) contrast(1.2);
}

/* Au survol — effet de révélation */
.card { filter: grayscale(1); transition: filter .3s; }
.card:hover { filter: grayscale(0); }

/* backdrop-filter — filtre sur l'arrière-plan (effet verre) */
.glass {
  background: rgba(255,255,255,.15);
  backdrop-filter: blur(10px);
  -webkit-backdrop-filter: blur(10px);
  /* crée un effet de verre dépoli sur ce qui se trouve derrière */
}
RENDU
📌
backdrop-filter — l'effet verre

backdrop-filter applique un filtre à l'arrière-plan d'un élément (ce qui se trouve derrière lui), pas à l'élément lui-même. Associé à une couleur de fond semi-transparente, il crée l'effet "verre dépoli" (glassmorphism) très populaire. Supporté partout depuis 2020. Exemple typique : navbar floue qui laisse deviner le contenu en dessous.

08

Méthodologie BEM — nommer ses classes

BEM (Block Element Modifier) est une convention de nommage pour les classes CSS. Elle rend le code plus lisible, maintenable et évite les conflits de cascade. Ce n'est pas une technologie mais une méthodologie.

CSS — BEM 
/* ═══ BLOC — composant autonome (nom au singulier) ═══ */
.card { }

/* ═══ ÉLÉMENT — partie du bloc (__ double underscore) ═══ */
.card__title { }     /* titre de la carte */
.card__image { }     /* image de la carte */
.card__body { }      /* corps de la carte */
.card__footer { }    /* pied de la carte */

/* ═══ MODIFICATEUR — variation (-- double tiret) ═══ */
.card--featured { }  /* carte mise en avant */
.card--compact { }   /* carte compacte */
.card__title--large { } /* titre en grand */

/* Exemple complet en HTML */
/*
<div class="card card--featured">
  <img class="card__image" src="photo.jpg" alt="">
  <div class="card__body">
    <h3 class="card__title card__title--large">Titre</h3>
    <p class="card__text">Description</p>
  </div>
  <div class="card__footer">
    <button class="card__btn">Lire la suite</button>
  </div>
</div>
*/

/* ⚠️ BEM n'interdit pas l'imbrication ou les classes utilitaires */
/* Un bloc peut contenir des blocs enfants */
RENDU
.card__title
.card__body
.card__btn
💡
Pourquoi BEM est utile

BEM résout trois problèmes du CSS à grande échelle : ① Spécificité plate : toutes les classes ont la même spécificité (0,0,1,0) → pas de guerre de sélecteurs. ② Auto-documentation : le HTML montre immédiatement la structure du composant. ③ Pas de fuite : une classe dans un bloc n'affecte jamais un autre bloc. Alternative moderne : Tailwind CSS (classes utilitaires) ou CSS Modules (isolation automatique).

Bilan & mini-glossaire — Félicitations ! 🎉

Ce que vous maîtrisez

  • Transitions CSS : property, duration, timing (ease, linear, cubic-bezier), delay
  • Transform : translate, rotate, scale, skew, transform-origin
  • @keyframes : création d'animations de 0% à 100%
  • Animation : name, duration, iteration, direction, fill-mode, play-state
  • Variables CSS : --custom, var() avec fallback, portée, héritage
  • Fonctions mathématiques : calc(), min(), max(), clamp()
  • Filters : blur, grayscale, brightness, contrast, sepia, backdrop-filter
  • Méthodologie BEM : Bloc__Élément--Modificateur

Mini-glossaire

Transition
Animation fluide entre deux états d'une propriété. Déclenchée par hover, focus, ou changement de classe.
Transform
Modification géométrique (déplacement, rotation, échelle) sans affecter le flux. Accéléré GPU.
@keyframes
Règle CSS qui définit les étapes d'une animation avec des pourcentages ou from/to.
Animation
Application de keyframes à un élément. Peut être infinie, alternée, différée.
Variable CSS
Propriété personnalisée --nom, réutilisable avec var(). Respecte la cascade et l'héritage.
calc()
Fonction mathématique CSS. Mélange d'unités possible. Ex : calc(100% - 40px).
clamp()
Fonction avec min, préféré et max. clamp(1rem, 2.5vw, 2rem). Remplit le rôle de min/max simultanément.
Filter
Effet visuel CSS (flou, noir et blanc, luminosité). backdrop-filter pour l'effet verre.
BEM
Méthodologie de nommage : Bloc__Élément--Modificateur. Structure plate et auto-documentée.
GPU acceleration
Utilisation du processeur graphique pour les animations (transform, opacity). 60 fps.
