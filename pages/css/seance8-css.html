01

display: grid — la grille bidimensionnelle

CSS Grid est le premier véritable système de mise en page bidimensionnel du CSS. Alors que Flexbox gère une seule dimension (ligne OU colonne), Grid gère les deux axes simultanément (lignes ET colonnes).

📐 Propriété → 🎯 Valeur

/* Grille simple avec 3 colonnes égales */
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  gap: 16px;
}

📌
Flexbox vs Grid — comment choisir ?

Flexbox est unidimensionnel : idéal pour des barres de navigation, des rangées de boutons, des centrages, des répartitions linéaires. Grid est bidimensionnel : idéal pour des galeries, des pages entières, des mises en page avec en-tête et pied de page, des disposition où les éléments doivent s'aligner sur deux axes. La règle : si vous gérez une seule dimension → Flexbox. Si vous gérez deux dimensions → Grid.

02

grid-template-columns & grid-template-rows

Ces deux propriétés définissent la structure de la grille : le nombre et la taille des colonnes et des lignes.

CSS 
/* TROIS COLONNES — différentes valeurs */
.grid-1 {
  grid-template-columns: 200px 1fr 1fr;
  /* colonne fixe 200px + 2 colonnes flexibles */
}
.grid-2 {
  grid-template-columns: 1fr 2fr 1fr;
  /* proportions : 25% + 50% + 25% */
}
.grid-3 {
  grid-template-columns: repeat(4, 1fr);
  /* 4 colonnes égales — repeat() évite la répétition */
}
.grid-4 {
  grid-template-columns: 100px auto 100px;
  /* auto = prend l'espace restant */
}

/* LIGNES explicites */
.grid-rows {
  grid-template-rows: auto 1fr auto;
  /* ligne haut automatique, milieu flexible, bas automatique */
  height: 100vh; /* nécessaire pour que 1fr ait un sens */
}

/* Shorthand */
.grid-layout {
  grid-template:
    "header header" 80px
    "sidebar main" 1fr
    "footer footer" 60px / 250px 1fr;
}

💡
L'unité fr — fraction de l'espace disponible

L'unité fr (fraction) est exclusive à Grid. Elle distribue l'espace restant (après les colonnes fixes) entre les colonnes déclarées en fr. 1fr 1fr 1fr = 3 colonnes égales. 1fr 2fr = la deuxième colonne est 2× plus large que la première. Les fr ne se mélangent pas avec des pourcentages de la même manière — préférez les mélanger avec des px.

03

gap, row-gap, column-gap

Comme pour Flexbox, gap définit l'espacement entre les cellules de la grille. Il s'applique à la fois aux lignes et aux colonnes.

CSS 
/* gap uniforme */
.grid { gap: 16px; }

/* gap différent par axe */
.grid { row-gap: 24px; column-gap: 16px; }

/* Ancienne syntaxe (encore supportée) */
.grid { grid-gap: 16px; }       /* déprécié, préférez gap */
.grid { grid-row-gap: 24px; }   /* déprécié */

04

grid-column & grid-row — fusionner des cellules

Ces propriétés permettent à un élément de s'étendre sur plusieurs colonnes ou plusieurs lignes, comme le colspan et rowspan des tableaux HTML.

CSS 
/* Les lignes de grille sont numérotées à partir de 1 */

/* Étendre sur 2 colonnes */
.item-wide {
  grid-column: span 2;    /* occupe 2 colonnes */
}

/* Syntaxe avec lignes de début et fin */
.item-exact {
  grid-column: 1 / 3;     /* de la ligne 1 à la ligne 3 = 2 colonnes */
  grid-row: 1 / 3;        /* de la ligne 1 à la ligne 3 = 2 lignes */
}

/* Valeurs spéciales */
.item-full {
  grid-column: 1 / -1;    /* de la première à la dernière colonne = toute la largeur */
}

/* Shorthand */
.item-header {
  grid-column: 1 / -1;    /* occupe toute la largeur */
  grid-row: 1;
}

📌
Numérotation des lignes de grille

Les lignes de grille (pas les cellules) sont numérotées de 1 à N+1 (où N = nombre de colonnes). Pour 3 colonnes, les lignes verticales sont 1, 2, 3, 4. grid-column: 1 / 3 occupe les colonnes 1 et 2 (de la ligne 1 à la ligne 3). grid-column: 1 / -1 occupe toutes les colonnes (de la première à la dernière ligne).

05

grid-area & grid-template-areas

La méthode la plus intuitive pour définir une mise en page complète : on donne des noms aux zones, et on dessine la grille avec ces noms.

CSS 
/* Définir les zones nommées dans le parent */
.layout {
  display: grid;
  grid-template-areas:
    "header header header"
    "sidebar main   main"
    "sidebar main   main"
    "footer footer footer";
  grid-template-columns: 250px 1fr 1fr;
  grid-template-rows: 80px 1fr 1fr 60px;
  gap: 0;
  height: 100vh;
}

/* Assigner chaque enfant à sa zone */
header { grid-area: header; background: #2c3e50; }
aside  { grid-area: sidebar; background: #34495e; }
main   { grid-area: main; background: #ecf0f1; }
footer { grid-area: footer; background: #2c3e50; }

/* Le '.' représente une cellule vide */
.grid-areas {
  grid-template-areas:
    "header header header"
    ".      main   ."       /* colonnes vides à gauche et droite */
    "footer footer footer";
}

header
sidebar
main

💡
grid-template-areas = mise en page déclarative

Cette approche est la plus lisible pour définir la structure d'une page. On voit en un coup d'œil la disposition. Pour changer la mise en page en responsive, il suffit de redéfinir grid-template-areas dans une media query (ex : passer de 3 colonnes à 1 colonne sur mobile). Chaque zone doit former un rectangle — Grid ne supporte pas les formes en L.

06

auto-fit & auto-fill — grilles responsives automatiques

Ces mots-clés, combinés avec repeat(), créent des grilles qui s'adaptent automatiquement au nombre de colonnes selon l'espace disponible. C'est la technique reine pour les galeries responsives sans media queries.

CSS 
/* AUTO-FIT : crée autant de colonnes que possible */
/* Au moins 200px chacune, maximum 1fr (s'étirent si espace libre) */
.auto-fit-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));
  gap: 16px;
}

/* AUTO-FILL : similaire, mais GARDE les colonnes vides */
.auto-fill-grid {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));
  gap: 16px;
}

/* Différence : auto-fit réduit les colonnes vides à 0 */
/* auto-fill conserve l'espace réservé pour des colonnes vides */
RENDU
auto-fit
auto-fit
auto-fit
📌
La recette magique : repeat(auto-fit, minmax(250px, 1fr))

C'est sans doute la ligne de CSS la plus utile pour le responsive design. Une seule déclaration crée une grille qui : ① sur grand écran, affiche 4-5 colonnes de 250px+ qui s'étirent pour remplir l'espace ② sur tablette, passe à 2-3 colonnes ③ sur mobile, passe à 1 colonne. Le tout sans une seule media query. Changez simplement la valeur minmax pour ajuster le point de rupture.

07

minmax() & l'unité fr

minmax(min, max) définit une fourchette de taille pour une colonne ou une ligne. L'unité fr distribue l'espace disponible en fractions.

CSS 
/* minmax() — taille minimale et maximale */
.grid-minmax {
  grid-template-columns: minmax(200px, 400px) 1fr;
  /* colonne 1 : entre 200px et 400px, colonne 2 : le reste */
}

/* Combinaison avec fr */
.grid-combo {
  grid-template-columns: 2fr minmax(300px, 1fr) 2fr;
  /* colonne centrale : au moins 300px, les autres : 2 parts chacune */
}

/* Avec auto-fit pour responsive automatique */
.grid-auto {
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
}

/* Pour les hauteurs aussi */
.grid-rows {
  grid-template-rows: minmax(100px, auto) 1fr minmax(60px, auto);
}
RENDU
minmax(100px,200px)
1fr (reste)
💡
fr vs % — différence fondamentale

1fr distribue l'espace restant après les colonnes en px, em, ou auto. % calcule par rapport à la largeur totale du conteneur. Si vous mélangez 200px et 50%, le pourcentage est calculé sur la largeur totale, pas sur l'espace restant — ce qui peut provoquer un overflow. Avec 200px et 1fr, le fr prend tout l'espace après les 200px. C'est plus sûr et plus prévisible.

08

Placement explicite vs implicite

Quand vous placez plus d'éléments que de cellules définies, Grid crée automatiquement de nouvelles lignes (ou colonnes). C'est le placement implicite. Vous pouvez contrôler la taille de ces pistes implicites.

CSS 
/* Grille explicite : 3 colonnes, 2 lignes définies */
.grid-explicit {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;
  grid-template-rows: 100px 100px;
  /* Si on a 10 éléments, les 4 derniers créent des lignes implicites */
}

/* Contrôle des pistes implicites */
.grid-implicit {
  grid-auto-rows: 80px;           /* toute nouvelle ligne fait 80px */
  grid-auto-columns: 200px;       /* toute nouvelle colonne fait 200px */
  grid-auto-flow: row;            /* placement par ligne (défaut) */
  grid-auto-flow: column;         /* placement par colonne */
  grid-auto-flow: dense;          /* remplit les trous automatiquement */
}

/* dense — comble les espaces vides */
.grid-dense {
  grid-auto-flow: row dense;
  /* utile pour une galerie d'images de tailles différentes */
}

📌
grid-auto-flow: dense — le mode "tetris"

Avec dense, Grid remplit automatiquement les trous laissés par des éléments qui s'étendent sur plusieurs colonnes. Par exemple, si un élément fait span 2 et qu'il ne reste qu'une colonne sur la ligne, il passe à la ligne suivante — mais dense remplit le trou avec l'élément suivant qui tient. Attention : cela change l'ordre visuel (pas l'ordre DOM), ce qui peut dérouter si le contenu a un ordre logique.

09

Exercices pratiques

EX 01 📰 Mise en page magazine
  1. Créez exo-grid-magazine.html avec une grille de 3 colonnes 1fr.
  2. Placez un article principal (span 2 colonnes, 2 lignes) avec image, titre et texte.
  3. Placez 4 articles secondaires (1 colonne, 1 ligne chacun).
  4. Ajoutez un encart "À la une" (span 1 colonne, 2 lignes) à droite de l'article principal.
  5. Utilisez gap: 20px et border-radius sur chaque article.
  6. Ajoutez un footer qui traverse toute la largeur (grid-column: 1 / -1).
EX 02 🖼️ Galerie responsive automatique
  1. Créez exo-grid-galerie.html avec une grille utilisant repeat(auto-fit, minmax(200px, 1fr)).
  2. Ajoutez 10 images (utilisez les images du dossier images/).
  3. Faites en sorte que la 1ère image fasse span 2 sur les colonnes.
  4. Utilisez grid-auto-flow: dense pour remplir automatiquement les trous.
  5. Chaque image a object-fit: cover et height: 200px.
  6. Ajoutez un titre de galerie qui traverse toute la largeur (grid-column: 1 / -1).
  7. Testez le redimensionnement de la fenêtre pour voir les colonnes s'adapter.
🔍
Exercice bonus — Holy Grail Layout

Créez le "Saint Graal" de la mise en page : header, footer, sidebar gauche, sidebar droite, contenu principal. Utilisez grid-template-areas pour définir la disposition. En mobile (media query < 768px), repassez tout en une seule colonne. C'est un classique des entretiens techniques pour les développeurs front-end. Voir MDN Grid Layouts.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • display: grid et la différence avec Flexbox (bidimensionnel vs unidimensionnel)
  • grid-template-columns et grid-template-rows avec px, fr, %, auto, repeat()
  • gap, row-gap, column-gap pour l'espacement entre les cellules
  • grid-column et grid-row avec span pour fusionner des cellules
  • grid-template-areas pour une mise en page déclarative
  • auto-fit et auto-fill pour les grilles responsives automatiques
  • minmax() pour les tailles avec fourchette minimum/maximum
  • Placement explicite vs implicite et grid-auto-flow dense

Mini-glossaire

Grid
Système de mise en page bidimensionnel (lignes + colonnes). Permet de placer des éléments sur une grille.
fr (fraction)
Unité exclusive à Grid. Distribue l'espace restant en parts égales ou proportionnelles.
repeat()
Fonction pour répéter une ou plusieurs colonnes. Ex : repeat(3, 1fr).
grid-column / grid-row
Fusion de cellules. span 2 = occupe 2 colonnes/lignes. 1 / 3 = de ligne 1 à ligne 3.
grid-template-areas
Définit des zones nommées. Permet une mise en page déclarative et lisible.
auto-fit
Crée autant de colonnes que possible, réduit à 0 les colonnes vides.
auto-fill
Crée autant de colonnes que possible, conserve l'espace des colonnes vides.
minmax(min, max)
Fonction qui définit une taille minimale et maximale pour une piste.
Piste implicite
Ligne ou colonne créée automatiquement par Grid quand le nombre d'éléments dépasse la grille explicite.
grid-auto-flow: dense
Algorithme de placement qui remplit les trous automatiquement ("mode tetris").
