Content · Padding · Border · Margin · Box-sizing · Overflow

CSS3Fondamental ~2h10 sections2 exercices
01

Content — width & height

Le Box Model est le concept le plus important du CSS. Chaque élément HTML est une boîte rectangulaire composée de quatre couches : content → padding → border → margin. Le content est la zone intérieure qui contient le contenu réel (texte, image, etc.).

CSS 
/* La boîte de contenu — ce que contient l'élément */
.box {
  width: 300px;      /* largeur de la zone de contenu */
  height: 200px;     /* hauteur de la zone de contenu */
  background: #e8f4f8;
}

/* width et height s'appliquent au CONTENU seul */
/* Les valeurs possibles : px, em, rem, %, vw, vh, auto */
.auto   { width: auto; }   /* s'adapte au contenu */
.fill   { width: 100%; }   /* remplit le parent */
.view   { width: 50vw; }   /* moitié de la largeur de la fenêtre */
RENDU
content 300×80px
📌
Les éléments bloc occupent toute la largeur par défaut

Un élément de bloc (comme <div>, <p>, <h1>) a par défaut width: auto, ce qui signifie qu'il occupe toute la largeur disponible de son parent. La hauteur, elle, s'adapte au contenu. Pour contrôler les dimensions, on définit explicitement width et/ou height.

02

Padding — espacement interne

Le padding est l'espace entre le contenu et la bordure. Il est à l'intérieur de la boîte, donc visible (prend la couleur de fond de l'élément).

CSS 
/* Shorthand : 1 à 4 valeurs (haut, droite, bas, gauche) */
.p-1 { padding: 20px; }           /* les 4 côtés */
.p-2 { padding: 10px 20px; }      /* haut/bas = 10px, gauche/droite = 20px */
.p-3 { padding: 10px 20px 30px; } /* haut 10, droite 20, bas 30, gauche = droite = 20 */
.p-4 { padding: 10px 20px 30px 40px; } /* haut, droite, bas, gauche (sens horaire) */

/* Par côté */
.p-top    { padding-top: 10px; }
.p-right  { padding-right: 20px; }
.p-bottom { padding-bottom: 10px; }
.p-left   { padding-left: 20px; }

/* Unités relatives pour padding */
.p-em     { padding: 1em 1.5em; }  /* proportionnel à la taille de police */
RENDU
padding: 20px (4 côtés)
padding: 10px 30px
padding: 5px 10px 20px 40px
💡
Mémoriser l'ordre du shorthand

L'ordre est haut → droite → bas → gauche (sens horaire). Astuce : "Haut Droite Bas Gauche" → pensez à l'horloge (12h → 3h → 6h → 9h). Si une valeur est omise, elle prend celle du côté opposé : 2 valeurs → haut/bas + droite/gauche ; 3 valeurs → haut + droite/gauche + bas.

03

Border — la bordure

La bordure entoure le padding (ou le contenu s'il n'y a pas de padding). Elle nécessite trois sous-propriétés : épaisseur (width), style, et couleur (color).

CSS 
/* Shorthand border : width style color */
.b-1 { border: 2px solid #3498db; }
.b-2 { border: 4px dashed #e74c3c; }
.b-3 { border: 1px dotted #2ecc71; }

/* Styles disponibles */
.solid   { border-style: solid; }    /* trait continu */
.dashed  { border-style: dashed; }   /* tirets */
.dotted  { border-style: dotted; }   /* points */
.double  { border-style: double; }   /* double ligne */
.groove  { border-style: groove; }   /* effet 3D encastré */
.ridge   { border-style: ridge; }    /* effet 3D relief */
.none    { border-style: none; }     /* pas de bordure */

/* Par côté */
.b-top    { border-top: 3px solid #e63946; }
.b-right  { border-right: 3px solid #3498db; }
.b-bottom { border-bottom: 3px solid #2ecc71; }
.b-left   { border-left: 3px solid #f39c12; }

/* Couleur par côté */
.b-colors { border-top: 3px solid red; border-right: 3px solid blue; }
RENDU
solid
dashed
dotted
4 couleurs différentes
📌
border ajoute à la taille totale

Par défaut (box-sizing: content-box), la bordure s'ajoute à la largeur et hauteur définies. Un élément avec width: 200px et border: 5px solid occupera 210px de large (5px à gauche + 200px de contenu + 5px à droite). C'est une source fréquente de surprises — voir la section 6 sur box-sizing.

04

Border-radius — coins arrondis

border-radius arrondit les coins d'un élément. C'est une des propriétés CSS les plus utilisées pour adoucir l'apparence des boîtes, boutons, cartes et avatars.

CSS 
/* Shorthand : 1 à 4 valeurs */
.small  { border-radius: 4px; }    /* coins légèrement arrondis */
.medium { border-radius: 8px; }    /* confortable — cartes, boutons */
.large  { border-radius: 16px; }   /* très arrondi */
.full   { border-radius: 50%; }    /* cercle / ovale */

/* Par coin */
.tl { border-top-left-radius: 10px; }
.tr { border-top-right-radius: 10px; }
.br { border-bottom-right-radius: 10px; }
.bl { border-bottom-left-radius: 10px; }

/* Deux valeurs : haut-gauche+bas-droit  |  haut-droit+bas-gauche */
.oval   { border-radius: 50px / 25px; }  /* ellipse */
.pill   { border-radius: 9999px; }       /* pilule — bouton arrondi */

/* Valeurs en pourcentage */
.percent { border-radius: 20%; } /* relatif aux dimensions de l'élément */
RENDU
4px
8px
16px
50%
9999px
💡
border-radius: 50% vs 9999px

border-radius: 50% crée un cercle parfait uniquement si l'élément est un carré (width = height). Sinon, il devient ovale. border-radius: 9999px donne une forme de "pilule" (bords complètement arrondis) quel que soit le ratio, tant que l'élément n'est pas extrêmement large. C'est la valeur recommandée pour les boutons.

05

Margin — marge externe

La margin est l'espace en dehors de la bordure. Elle sépare l'élément des autres éléments voisins. Contrairement au padding, elle est invisible (pas de couleur de fond).

CSS 
/* Shorthand : même principe que padding */
.m-1 { margin: 20px; }           /* 4 côtés */
.m-2 { margin: 10px 20px; }      /* haut/bas + droite/gauche */
.m-3 { margin: 10px auto 20px; } /* haut + auto horizontal + bas */

/* Par côté */
.mt { margin-top: 10px; }
.mr { margin-right: 20px; }
.mb { margin-bottom: 10px; }
.ml { margin-left: 20px; }

/* Centrer un élément bloc */
.center-block { margin-left: auto; margin-right: auto; width: 300px; }

/* Marges négatives */
.negatif { margin-top: -20px; }  /* remonte l'élément */
RENDU
margin-bottom: 10px
Élément suivant
margin: auto → centré

Le "margin collapse" (effondrement des marges)

Quand deux éléments bloc se suivent verticalement, leurs marges ne s'additionnent PAS — seule la plus grande est conservée. Exemple : un <h2> avec margin-bottom: 20px suivi d'un <p> avec margin-top: 10px → l'espace entre eux sera de 20px, pas 30px.

⚠️
Margin collapse — piège classique

Le collapse des marges ne se produit que pour les marges verticales (top/bottom). Les marges horizontales (left/right) s'additionnent normalement. Le collapse ne se produit pas non plus si les éléments sont en flexbox ou grid, ni si un élément a du padding ou une bordure qui sépare les deux marges. Pour éviter les surprises, utilisez une direction unique (par exemple margin-bottom sur tous les éléments sauf le dernier).

06

box-sizing : content-box vs border-box

La propriété box-sizing change la façon dont la largeur/hauteur totale est calculée. C'est la différence la plus importante à comprendre dans le Box Model.

CSS 
/* ═══ content-box (DÉFAUT) ═══ */
/* width = contenu SEULEMENT (padding et border s'ajoutent) */
.content-box {
  box-sizing: content-box;
  width: 200px;
  padding: 20px;
  border: 5px solid #333;
  /* taille totale = 200 + 40 (padding) + 10 (border) = 250px */
}

/* ═══ border-box (recommandé) ═══ */
/* width = contenu + padding + border (tout inclus) */
.border-box {
  box-sizing: border-box;
  width: 200px;
  padding: 20px;
  border: 5px solid #333;
  /* taille totale = 200px (contenu = 200 - 40 - 10 = 150px) */
}

/* Application globale — pratique courante */
*, *::before, *::after {
  box-sizing: border-box;
}
RENDU
content-box: 200px + 40 + 10 = 250px
border-box: 200px tout compris → contenu = 150px
💡
Règle d'or : box-sizing: border-box sur tout

99% des développeurs professionnels appliquent box-sizing: border-box à tous les éléments dès le début du projet. C'est tellement courant que les frameworks (Bootstrap, Tailwind) l'incluent par défaut. Avec border-box, les dimensions que vous définissez sont prévisibles : 200px = 200px, quoi que vous fassiez avec padding et border.

07

min-width, max-width, min-height, max-height

Ces propriétés fixent des limites à la taille d'un élément. Indispensables pour le responsive design : elles permettent à un élément de s'adapter tout en restant dans des bornes raisonnables.

CSS 
/* Largeur max et min */
.responsive {
  width: 80%;            /* 80% du parent */
  max-width: 1200px;     /* mais jamais plus de 1200px */
  min-width: 320px;      /* et jamais moins de 320px */
}

/* Hauteur minimale — pour les sections */
.section {
  min-height: 60vh;      /* au moins 60% de la hauteur de la fenêtre */
  /* si contenu plus long, la hauteur s'adapte automatiquement */
}

/* Hauteur max — pour les panneaux */
.panel {
  max-height: 400px;     /* pas plus de 400px de haut */
  overflow-y: auto;      /* scroll si contenu trop long */
}

/* Utilisation typique — conteneur principal */
.container {
  width: 100%;
  max-width: 1100px;
  margin: 0 auto;
  padding: 0 20px;
}
RENDU
width:80%, max-width:400px, min-width:200px
min-height: 60px
📌
Un conteneur responsive classique

Le pattern le plus répandu pour une mise en page responsive : width: 100%; max-width: 1100px; margin: 0 auto; padding: 0 20px;. Le conteneur prend toute la largeur sur mobile, mais se bloque à 1100px sur desktop. Le padding horizontal empêche le texte de toucher les bords de l'écran. Cette technique est utilisée sur des millions de sites.

08

Overflow : visible, hidden, scroll, auto

overflow contrôle ce qui se passe quand le contenu dépasse la taille définie de la boîte. C'est essentiel pour gérer les situations où le contenu est plus long que prévu.

CSS 
/* overflow: visible (DÉFAUT) — le contenu dépasse */
.visible  { overflow: visible; height: 40px; }

/* overflow: hidden — le contenu est coupé */
.hidden   { overflow: hidden; height: 40px; }

/* overflow: scroll — toujours des barres de défilement */
.scroll   { overflow: scroll; height: 40px; }

/* overflow: auto — barres seulement si nécessaire */
.auto     { overflow: auto; height: 40px; }

/* Par axe — overflow-x et overflow-y */
.x-scroll { overflow-x: auto; white-space: nowrap; }
.y-scroll { overflow-y: scroll; height: 80px; }
RENDU
hidden — le texte trop long est coupé net à la fin de la boîte.
auto — barre de défilement si le contenu dépasse la hauteur définie.
scroll horizontal — ce contenu est très large et peut défiler horizontalement.
⚠️
overflow: hidden coupe aussi les ombres

Si vous utilisez overflow: hidden sur un conteneur, les box-shadow de ses enfants seront coupées aux bords du conteneur. Pour éviter cela sans perdre le "débordement coupé", utilisez overflow: hidden sur un parent et padding sur l'élément concerné pour laisser de la place aux ombres.

CSS — overflow avec ellipsis 
/* Troncature avec points de suspension */
.truncate {
  white-space: nowrap;        /* pas de retour à la ligne */
  overflow: hidden;           /* cache le surplus */
  text-overflow: ellipsis;    /* ajoute … à la fin */
  width: 250px;
}

/* Texte multi-lignes tronqué (line-clamp) */
.clamp {
  display: -webkit-box;
  -webkit-line-clamp: 3;      /* max 3 lignes */
  -webkit-box-orient: vertical;
  overflow: hidden;
}
RENDU
Ce texte est trop long pour tenir dans 250px et sera tronqué avec des points de suspension...

Exercices pratiques

EX 01 📦 Construction d'une carte
  1. Créez exo-boxmodel-carte.html avec une carte de visite.
  2. La carte fait width: 350px avec box-sizing: border-box.
  3. Ajoutez padding: 2rem et border: 2px solid #ddd.
  4. Utilisez border-radius: 12px pour des coins arrondis.
  5. À l'intérieur : nom, titre, description, bouton de contact.
  6. Le bouton a padding: .75em 1.5em, border-radius: 9999px, border: none.
  7. Centrez la carte horizontalement avec margin: 2rem auto.
  8. Vérifiez la largeur totale dans les DevTools (onglet Computed/Box Model).
EX 02 🔲 Mise en page avec overflow et min-height
  1. Créez exo-overflow.html avec un conteneur de width: 300px et height: 100px.
  2. Remplissez-le d'un long texte qui dépasse. Testez chaque valeur d'overflow.
  3. Ajoutez une section avec min-height: 40vh et une bordure en pointillés.
  4. Créez un bloc de texte tronqué avec text-overflow: ellipsis.
  5. Ajoutez un conteneur max-width: 600px; margin: 0 auto; padding: 0 16px; et observez le comportement responsive.
  6. Testez margin: -10px sur un élément pour le superposer partiellement au précédent.
🔍
Exercice bonus — Démontrer le margin collapse

Créez deux divs avec des marges différentes (ex : margin-bottom: 30px sur le premier, margin-top: 20px sur le second). Mesurez l'espace entre eux dans les DevTools. Puis ajoutez display: flex sur le parent et constatez que les marges ne s'effondrent plus. Ajoutez aussi border: 1px solid sur le premier div et observez le même effet.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Les quatre couches du Box Model : content, padding, border, margin
  • Padding : espacement interne (1 à 4 valeurs en shorthand)
  • Border : width, style (solid, dashed, dotted…), color, par côté
  • Border-radius : coins arrondis, cercle (50%), pilule (9999px)
  • Margin : espacement externe, auto pour centrer, marges négatives
  • Le margin collapse : les marges verticales s'effondrent entre éléments bloc
  • box-sizing : content-box (défaut) vs border-box (recommandé)
  • min/max width/height pour les limites responsive
  • Overflow : visible, hidden, scroll, auto, et la troncature ellipsis

Mini-glossaire

Box Model
Modèle de boîte : chaque élément est une boîte avec content + padding + border + margin.
Padding
Espace intérieur entre le contenu et la bordure. Prend la couleur de fond de l'élément.
Border
Bordure visible autour du padding. Propriétés : width, style, color.
Margin
Espace extérieur autour de la bordure. Invisible, sert à séparer les éléments.
Margin collapse
Fusion des marges verticales adjacentes : seule la plus grande est conservée.
box-sizing
Définit comment width/height sont calculés. border-box inclut padding et border dans les dimensions.
border-radius
Arrondit les coins. 50% → cercle, 9999px → pilule, 4px → léger arrondi.
Shorthand
Propriété CSS qui regroupe plusieurs valeurs. Ex : padding, margin, border.
Overflow
Gère le dépassement du contenu. hidden coupe, scroll force les barres, auto adapte.
text-overflow
Ajoute des points de suspension quand le texte dépasse. Nécessite overflow: hidden.
