01

Sélecteur de balise (type selector)

Le sélecteur le plus simple : il cible tous les éléments HTML d'un certain type. On écrit simplement le nom de la balise, sans crochet, sans point, sans dièse.

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé
CSS 
/* Sélecteurs de type / de balise */
h1 { color: #2c3e50; }
p  { font-size: 16px; line-height: 1.6; }
a  { color: #2980b9; text-decoration: none; }
ul { list-style: square; padding-left: 1.5em; }
RENDU

Titre h4 en #2c3e50

Paragraphe en 16px, interligne 1.6.

Lien sans soulignement
📌
Portée globale

Un sélecteur de balise s'applique à tous les éléments de ce type dans la page. C'est parfait pour les styles de base (typographie, couleurs principales) mais trop large pour des cas particuliers. Pour cibler un sous-ensemble, utilisez une classe.

02

Sélecteur de classe .class

Le sélecteur de classe est le plus utilisé en CSS moderne. Il cible tous les éléments HTML qui possèdent l'attribut class correspondant. Une classe peut être appliquée à n'importe quelle balise, et une balise peut avoir plusieurs classes.

CSS 
/* Le point . devant le nom de la classe */
.important { color: #e63946; font-weight: bold; }
.small     { font-size: 0.85rem; color: #666; }
.card      { border: 1px solid #ddd; padding: 1rem; border-radius: 8px; }

/* Une balise peut avoir plusieurs classes */
/* <p class="important small"> → applique les deux */
RENDU

important

small

important + small (deux classes)

.card
💡
Convention de nommage

Utilisez le kebab-case (mots séparés par des tirets) pour nommer vos classes : .nav-link, .btn-primary, .card-title. Évitez les underscores, les camelCase, et les abréviations obscures. Un nom de classe doit décrire ce que c'est, pas à quoi ça ressemble (préférez .error-message à .red-text).

CSS — Combinaison balise + classe 
/* Combinaison : ne cible que les <p> avec cette classe */
p.intro { font-size: 1.2rem; color: #2c3e50; }

/* Sans la balise : tous les éléments avec cette classe */
.intro  { font-size: 1.2rem; color: #2c3e50; }

/* Différence : p.intro est plus spécifique (0,0,1,1 vs 0,0,1,0) */
RENDU

p.intro

span avec classe .intro
03

Sélecteur d'id #id

Le sélecteur d'id cible un élément unique grâce à son attribut id. Un id est unique dans une page — on ne peut pas avoir deux éléments avec le même id. La spécificité est très élevée (0,1,0,0).

CSS 
/* Le dièse # devant le nom de l'id */
#header    { background: #2c3e50; color: white; padding: 1rem; }
#logo      { width: 120px; height: auto; }
#main-nav  { display: flex; gap: 1rem; }
#footer    { background: #1a252f; color: #ccc; padding: 2rem; }

/* Attention : trop de spécificité rend le code rigide */
/* Préférez les classes pour le style */
RENDU
#header
⚠️
Id vs Classe — quand utiliser quoi ?

Utilisez un id pour : les ancres de navigation (#section-contact), le ciblage JavaScript (document.getElementById()), les labels de formulaires (<label for="email">). Utilisez une classe pour : tout ce qui est style CSS. Les classes sont réutilisables, ont une spécificité modérée, et ne créent pas de conflit.

04

Sélecteur universel *

L'astérisque * cible tous les éléments de la page. Sa spécificité est de 0,0,0,0 — la plus faible possible. Il est souvent utilisé pour des réinitialisations globales.

CSS 
/* Tous les éléments */
* { box-sizing: border-box; }

/* Tous les éléments dans .sidebar uniquement */
.sidebar * { font-size: 0.9rem; }

/* Tous les enfants directs de nav */
nav > * { margin: 0; padding: 0; }

/* ⚠️ * appliqué globalement peut impacter les performances
   car le navigateur évalue la règle pour CHAQUE élément */
RENDU
* { box-sizing: border-box; } est appliqué à tous.
📌
Performance du sélecteur universel

Utilisé seul (* { ... }), le sélecteur universel est lent car le navigateur doit vérifier chaque élément de la page. Préférez l'utiliser combiné à un sélecteur plus précis (.contenu * { ... }) ou dans des contexts spécifiques. L'usage le plus répandu — box-sizing: border-box — est acceptable car cette propriété ne déclenche pas de recalcul de mise en page.

05

Sélecteur d'attribut [attr]

Les sélecteurs d'attribut ciblent les éléments selon la présence ou la valeur d'un attribut HTML. Ils sont extrêmement puissants pour styliser sans classe supplémentaire.

CSS 
/* [attr] — présence de l'attribut */
[disabled] { opacity: 0.5; cursor: not-allowed; }
[required] { border-left: 3px solid #e63946; }

/* [attr="valeur"] — exact */
[type="submit"] { background: #2c3e50; color: white; }
[target="_blank"]::after { content: " ↗"; }

/* [attr^="valeur"] — commence par (prefix) */
a[href^="https"] { color: green; }
a[href^="mailto"]::before { content: "✉ "; }

/* [attr$="valeur"] — finit par (suffix) */
a[href$=".pdf"]::after { content: " [PDF]"; color: red; }
img[src$=".png"] { border: 2px solid #3498db; }

/* [attr*="valeur"] — contient */
[class*="btn"] { font-weight: bold; }
[href*="wikipedia"] { color: #636363; }
RENDU
disabled
Lien HTTPS
Document PDF
SyntaxeDescriptionExemple
[attr]Présence de l'attribut[disabled]
[attr="val"]Égalité exacte[type="submit"]
[attr^="val"]Commence par[href^="https"]
[attr$="val"]Finit par[src$=".png"]
[attr*="val"]Contient[class*="btn"]
[attr|="val"]Valeur exacte OU suivie d'un tiret[lang|="fr"]
[attr~="val"]Contient le mot (séparé par espace)[rel~="noopener"]
06

Combinateur descendant (espace)

Le combinateur descendant cible un élément qui est à l'intérieur d'un autre, à n'importe quel niveau d'imbrication. Il s'écrit avec un simple espace entre les sélecteurs.

CSS 
/* Tous les <li> dans <ul> (même imbriqués) */
ul li { margin-bottom: 0.5rem; }

/* Tous les <a> dans <nav> */
nav a { color: #ecf0f1; text-decoration: none; }

/* Tous les <p> dans .article */
.article p { line-height: 1.8; color: #444; }

/* Combinaisons plus complexes */
#sidebar .menu > li a { font-size: 0.9rem; }
/* → un <a> descendant d'un <li> enfant direct de .menu dans #sidebar */
RENDU
Lien dans nav

Paragraphe dans .article

⚠️
Attention à la profondeur excessive

Évitez les sélecteurs trop longs comme .contenu .sidebar ul li a span. Ils sont : ① très spécifiques (difficiles à surcharger), ② lents à évaluer, ③ fragiles (un changement de structure HTML les casse). Limitez-vous à 3 ou 4 niveaux maximum. Si vous dépassez, créez une classe dédiée.

07

Combinateurs > + ~

CSS propose trois autres combinateurs plus précis que l'espace descendant : l'enfant direct >, le frère adjacent +, et le frère général ~.

CSS — Les trois combinateurs 
/* ═══ > ENFANT DIRECT ═══ */
/* Uniquement les <li> qui sont enfants DIRECTS de <ul> */
/* PAS les <li> imbriqués dans une sous-liste */
ul > li { border-bottom: 1px solid #eee; }

/* ═══ + FRÈRE ADJACENT ═══ */
/* Le <p> qui suit IMMÉDIATEMENT un <h2> */
h2 + p { font-weight: bold; margin-top: 0; }

/* ═══ ~ FRÈRE GÉNÉRAL ═══ */
/* Tous les <p> qui suivent un <h2> (pas juste le premier) */
h2 ~ p { color: #555; padding-left: 1rem; }
RENDU
  • Enfant direct → bordure
  • Enfant direct → bordure

Titre h4

Frère adjacent (h4 + p) → gras

Frère général (h4 ~ p) → gris

💡
Quand utiliser chaque combinateur

> (enfant direct) : pour styler les premiers niveaux d'une liste, d'un menu, d'une grille, sans affecter les enfants imbriqués. + (adjacent) : pour espacer ou mettre en valeur le premier élément après un titre. ~ (général) : rare, utile pour styler tous les éléments après un certain point sans les précéder.

08

Pseudo-classes :hover :focus :nth-child :not

Les pseudo-classes ciblent un élément selon son état ou sa position dans le document. Elles commencent toujours par : et se placent après le sélecteur.

Pseudo-classes d'interaction

CSS 
/* HOVER : survol de la souris */
a:hover { color: #e63946; text-decoration: underline; }
.btn:hover { transform: translateY(-2px); box-shadow: 0 4px 12px rgba(0,0,0,.15); }

/* FOCUS : élément ciblé (clic, tabulation) */
input:focus { border-color: #3498db; box-shadow: 0 0 0 3px rgba(52,152,219,.2); outline: none; }
a:focus { outline: 2px solid #3498db; outline-offset: 2px; }

/* ACTIVE : au moment du clic */
.btn:active { transform: translateY(0); box-shadow: none; }

/* VISITED : lien déjà visité */
a:visited { color: #8e44ad; }
RENDU
:hover → rouge

Pseudo-classes structurelles

CSS 
/* :first-child / :last-child — premier/dernier enfant */
li:first-child { font-weight: bold; }
li:last-child  { border-bottom: none; }

/* :nth-child() — basé sur la position */
li:nth-child(odd)  { background: #f8f9fa; } /* lignes impaires */
li:nth-child(even) { background: #e9ecef; } /* lignes paires */
li:nth-child(3)    { color: #e63946; }       /* 3e élément exactement */
li:nth-child(3n+1) { border-left: 3px solid #3498db; } /* 1, 4, 7, 10… */

/* :not() — négation */
p:not(.intro) { color: #555; }  /* tous les p SAUF .intro */
input:not([type="submit"]) { border: 1px solid #ccc; }

/* :first-of-type / :last-of-type — par type, pas par position */
p:first-of-type { font-size: 1.1em; }  /* premier <p> dans le parent */
p:last-of-type  { margin-bottom: 0; }
RENDU
  • 1er : gras + fond gris clair
  • 2e : fond gris foncé
  • 3e : rouge
  • 4e
  • 5e
📌
Les pseudo-classes les plus utiles au quotidien

En pratique : :hover pour les interactions, :focus pour l'accessibilité clavier, :nth-child() pour les tableaux/listes en bandes, :not() pour les exceptions, :first-child/:last-child pour les coins arrondis ou séparateurs. Évitez :nth-child() avec des formules trop complexes (elles deviennent illisibles).

09

Exercices pratiques

EX 01 🎯 Ciblage par sélecteurs
  1. Créez exo-selecteurs.html avec un <nav> contenant 5 liens, une <section> avec 3 articles (<article>), chacun avec titre, paragraphe et liste.
  2. Dans un fichier CSS externe style-selecteurs.css : appliquez une couleur à tous les liens dans le nav.
  3. Ciblez uniquement le premier <li> de chaque liste avec :first-child et mettez-le en gras.
  4. Utilisez [href^="https"] pour ajouter une icône devant les liens externes.
  5. Utilisez :nth-child(odd) pour colorer une ligne sur deux dans une liste.
  6. Ajoutez un :hover qui souligne les liens et change leur couleur.
EX 02 🧩 Combinateurs & pseudo-classes
  1. Créez une carte de profil avec photo, nom, bio et réseaux sociaux. Utilisez des classes.
  2. Utilisez > pour styler uniquement les enfants directs de la carte (pas ceux d'une éventuelle sous-carte).
  3. Utilisez h2 + p pour mettre en valeur le premier paragraphe après le titre.
  4. Utilisez :hover sur la carte pour agrandir légèrement l'ombre portée.
  5. Utilisez :focus sur les liens des réseaux sociaux pour un contour visible au clavier.
  6. Utilisez :not(:last-child) pour ajouter un séparateur entre les éléments d'une liste horizontale.
🔍
Exercice bonus — Menu à plusieurs niveaux

Créez un menu de navigation à 2 niveaux (ul > li > ul > li). Utilisez > pour styler le premier niveau (barre horizontale) et le descendant espace pour le sous-menu (vertical). Ajoutez :hover pour faire apparaître le sous-menu. Stylez le dernier élément de chaque liste sans bordure avec :last-child.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Sélecteur de balise (type selector) : cible tous les éléments d'un type donné
  • Sélecteur de classe .nom : réutilisable, spécificité modérée, usage quotidien
  • Sélecteur d'id #nom : unique, très spécifique, à réserver aux ancres et JS
  • Sélecteur universel * : tous les éléments, faible spécificité, usage mesuré
  • Sélecteurs d'attribut [attr] : exact, préfixe ^, suffixe $, contient *
  • Combinateur descendant (espace) : n'importe quel niveau d'imbrication
  • Enfant direct >, adjacent +, général ~
  • Pseudo-classes : :hover, :focus, :active, :visited, :nth-child(), :not()

Mini-glossaire

Sélecteur
Motif qui cible des éléments HTML. Types : balise, classe (.nom), id (#nom), attribut ([attr]), universel (*).
Classe
Attribut HTML réutilisable (class="nom") ciblé en CSS par un point. Plusieurs classes possibles par élément.
Combinateur
Symbole qui relie deux sélecteurs : espace (descendant), > (enfant direct), + (adjacent), ~ (général).
Pseudo-classe
Mot-clé après : qui cible un état (:hover, :focus) ou une position (:nth-child, :first-child).
Enfant direct
Élément directement contenu dans un parent, pas dans un sous-élément. Syntaxe : parent > enfant.
Frère adjacent
Élément qui suit immédiatement un autre, au même niveau. Syntaxe : élément + frère.
nth-child
Pseudo-classe basée sur la position. Formule : an+b (odd, even, 3, 2n+1…).
Négation :not()
Pseudo-classe qui exclut les éléments correspondant au sélecteur entre parenthèses.
Kebab-case
Convention de nommage : mots en minuscules séparés par des tirets (.ma-classe).
