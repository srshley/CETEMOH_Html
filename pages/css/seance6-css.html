01

Display : block, inline, inline-block, none

La propriété display définit le type de boîte d'un élément. C'est la propriété la plus fondamentale pour la mise en page. Chaque élément HTML a un display par défaut.

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé
CSS 
/* DISPLAY: BLOCK — toute la largeur, retour à la ligne */
.block { display: block; }
/* Éléments bloc par défaut : div, p, h1-h6, ul, ol, li, section, article, nav, header, footer */

/* DISPLAY: INLINE — sur la même ligne, pas de width/height */
.inline { display: inline; }
/* Éléments inline par défaut : span, a, strong, em, img (inline-block), button */

/* DISPLAY: INLINE-BLOCK — sur la même ligne, mais accepte width/height */
.inline-block { display: inline-block; width: 150px; height: 50px; }

/* DISPLAY: NONE — invisible et NE prend PAS de place */
.hidden { display: none; }

/* visibility: hidden — invisible mais GARDE sa place */
.invisible { visibility: hidden; }

📌
display: none vs visibility: hidden

display: none retire complètement l'élément du flux : il n'occupe aucun espace et n'est pas rendu du tout. visibility: hidden cache visuellement l'élément mais garde sa place dans la mise en page. Pour l'accessibilité, display: none cache aussi aux lecteurs d'écran, tandis que visibility: hidden peut encore être visible par certains (à éviter pour le contenu important).

02

Position: static (valeur par défaut)

position: static est la valeur par défaut de tous les éléments. L'élément suit le flux normal du document : les éléments bloc s'empilent verticalement, les éléments inline s'alignent horizontalement. Les propriétés top, right, bottom, left et z-index n'ont aucun effet.

CSS 
/* Valeur par défaut — pas besoin de l'écrire */
.static {
  position: static;  /* défaut — inutile de le préciser */
  /* top, left, right, bottom n'ont aucun effet */
}

03

Position: relative

L'élément reste dans le flux normal (sa place est conservée), mais on peut le décaler visuellement avec top, right, bottom, left. Le décalage est relatif à sa position normale.

CSS 
/* L'élément garde sa place dans le flux mais est décalé visuellement */
.relative {
  position: relative;
  top: 20px;      /* décalé vers le bas de 20px */
  left: 30px;     /* décalé vers la droite de 30px */
}

/* Valeurs négatives — remonte ou va à gauche */
.relative-up {
  position: relative;
  top: -10px;     /* remonte de 10px */
}

/* La position relative sert aussi de point d'ancrage pour absolute
   (voir section suivante) */
RENDU
Normal
relative — décalé de 10px bas, 20px droite
relative — remonté de 5px
💡
Le vrai rôle de position: relative

Au-delà du décalage visuel, position: relative est surtout utilisé pour créer un contexte de positionnement : il sert de point d'ancrage pour les éléments enfants en position: absolute. Sans relative sur le parent, un élément absolute se positionnera par rapport au viewport ou au prochain parent positionné.

04

Position: absolute

L'élément est retiré du flux normal : il ne prend plus de place et peut se superposer aux autres éléments. Il se positionne par rapport à l'ancêtre positionné le plus proche (relative, absolute, fixed ou sticky).

CSS 
/* Parent doit avoir position: relative pour servir d'ancrage */
.parent {
  position: relative;  /* crée le repère */
  width: 400px;
  height: 200px;
  background: #e8f4f8;
}

/* Enfant positionné en absolu dans le parent */
.child {
  position: absolute;
  top: 20px;
  right: 20px;
  /* placé à 20px du haut et 20px de la droite du parent */
}

/* Centrer un élément absolu (technique classique) */
.center-absolute {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
  /* centré parfaitement dans le parent */
}

/* Sans parent positionné → par rapport au viewport */
.floating {
  position: absolute;
  top: 0;
  left: 0;
  /* collé en haut à gauche de la page */
RENDU
parent (relative)
absolute
⚠️
Absolute retire du flux — attention aux superpositions

Un élément en position: absolute ne prend plus de place dans le parent. Si le parent n'a que des enfants absolus, sa hauteur sera 0 (effondrement). Pour éviter cela, donnez une hauteur explicite au parent, ou utilisez une autre méthode de positionnement. Les éléments absolus peuvent se superposer — gérez l'ordre avec z-index.

05

Position: fixed

Comme absolute, l'élément est retiré du flux. Mais fixed se positionne toujours par rapport au viewport (la fenêtre du navigateur), indépendamment du défilement. Idéal pour les barres de navigation, les cookies, les boutons flottants.

CSS 
/* Barre de navigation fixe en haut */
.navbar-fixed {
  position: fixed;
  top: 0;
  left: 0;
  right: 0;
  height: 60px;
  background: #2c3e50;
  color: white;
  z-index: 1000;
  /* reste visible en haut de l'écran même en scrollant */
}

/* Bouton flottant en bas à droite */
.fab {
  position: fixed;
  bottom: 24px;
  right: 24px;
  width: 56px;
  height: 56px;
  border-radius: 50%;
  background: #3498db;
  color: white;
  box-shadow: 0 4px 12px rgba(0,0,0,.2);
}

/* Bannière de cookies en bas */
.cookies {
  position: fixed;
  bottom: 0;
  left: 0;
  right: 0;
  background: #1a1a2e;
  color: white;
  padding: 1rem;
}
RENDU
Navbar fixed (simulation)
📌
Penser à compenser la hauteur du fixed

Si vous fixez une navbar en haut, elle sort du flux : le contenu suivant remonte sous elle. Ajoutez padding-top: 60px (hauteur de la navbar) au <body> ou au premier élément après la navbar pour compenser. Les éléments fixed peuvent aussi cacher du contenu sur mobile — prévoyez des media queries.

06

Position: sticky

Un élément sticky alterne entre relative et fixed selon le scroll. Il se comporte comme relative jusqu'à ce qu'il atteigne un seuil défini (top, left, etc.), puis il se "colle" comme fixed.

CSS 
/* En-tête de section qui reste visible en scrollant */
.section-header-sticky {
  position: sticky;
  top: 80px;               /* se colle à 80px du haut du viewport */
  background: white;
  z-index: 10;
  padding: .75rem 0;
  border-bottom: 2px solid #3498db;
}

/* Sidebar qui suit le scroll */
.sidebar-sticky {
  position: sticky;
  top: 24px;               /* reste à 24px du haut du viewport */
}

/* Table avec en-tête collant */
th {
  position: sticky;
  top: 0;
  background: #2c3e50;
  color: white;
  /* les en-têtes restent visibles quand on descend dans le tableau */
}

/* ⚠️ Nécessite un conteneur parent avec overflow visible
   et une hauteur suffisante pour que le sticky ait de l'espace pour se déplacer */
RENDU
Sticky header (se colle en scrollant)
Contenu qui défile normalement en dessous.
💡
Sticky — pièges à éviter

Pour que sticky fonctionne : ① le parent doit avoir overflow en visible (pas hidden/scroll/auto). ② l'élément sticky a besoin d'au moins un seuil défini (top, bottom, etc.). ③ le parent doit avoir une hauteur supérieure à l'élément sticky pour que le défilement ait un effet. ④ sticky fonctionne dans la limite du conteneur parent — il dépasse du parent, le sticky s'arrête.

07

z-index & empilement (stacking context)

Quand des éléments se superposent (position: relative/absolute/fixed/sticky), z-index contrôle l'ordre d'empilement sur l'axe Z (profondeur). Plus la valeur est élevée, plus l'élément est au-dessus.

CSS 
/* Par défaut, z-index: auto (0) */
/* Plus la valeur est grande, plus l'élément est au-dessus */

.box1 { position: absolute; z-index: 1; }
.box2 { position: absolute; z-index: 2; }  /* au-dessus de box1 */
.box3 { position: absolute; z-index: 3; }  /* au-dessus de tous */

/* stack context */
.layer {
  position: relative;
  z-index: 10;  /* crée un nouveau contexte d'empilement */
}
.child {
  position: absolute;
  z-index: 999; /* même avec 999, l'enfant reste dans le contexte du parent */
  /* si un autre contexte a z-index 20, l'enfant z-index 999 sera dessous */
}
RENDU
z=1
z=2
z=3
⚠️
z-index ne fonctionne que sur les éléments positionnés

z-index est ignoré sur les éléments en position: static (valeur par défaut). Pour que z-index ait un effet, l'élément doit avoir position: relative, absolute, fixed ou sticky. De plus, un élément avec z-index: 9999 peut être sous un élément avec z-index: 1 si ce dernier est dans un contexte d'empilement plus élevé (créé par un parent avec un z-index).

08

Float & clear — méthode historique

float était la méthode principale de mise en page avant Flexbox et Grid. Aujourd'hui, on l'utilise encore pour faire flotter du texte autour d'une image (comme dans les magazines). Pour les mises en page complètes, préférez Flexbox ou Grid.

CSS 
/* Float : l'élément flotte à gauche ou à droite */
.float-left  { float: left;  margin-right: 1rem; }
.float-right { float: right; margin-left: 1rem; }

/* Clear : stoppe le flottement */
.clear-left  { clear: left; }    /* interdit le float à gauche */
.clear-right { clear: right; }   /* interdit le float à droite */
.clear-both  { clear: both; }    /* interdit le float des deux côtés */

/* Clearfix — technique pour qu'un conteneur flotté ait une hauteur */
.clearfix::after {
  content: "";
  display: table;
  clear: both;
}

/* Usage moderne : faire flotter une image dans un article */
.article img {
  float: left;
  margin: 0 1.5rem 1rem 0;
  max-width: 40%;
}

/* ⚠️ Ne pas utiliser float pour la mise en page de sections
   → Utiliser Flexbox (séance 7) ou Grid (séance 8) */
RENDU
Image flottée

Ce texte s'écoule autour de l'image flottée à gauche, comme dans un article de magazine. Le float est encore utile pour cet usage.

📌
Float — à l'époque des mises en page

Avant Flexbox (2015) et Grid (2017), on construisait des mises en page entières avec float et clearfix. C'était complexe, plein de hacks, et cassait facilement. Aujourd'hui, n'utilisez plus float pour la mise en page. Réservez-le au rôle pour lequel il a été conçu : faire couler du texte autour d'un élément. Pour tout le reste, utilisez Flexbox ou Grid.

Exercices pratiques

EX 01 📐 Positionnement mis en pratique
  1. Créez exo-position.html avec un conteneur parent position: relative (400×300px, fond gris clair).
  2. Placez 4 carrés de 80×80px de couleurs différentes aux 4 coins du parent en utilisant position: absolute avec top/right/bottom/left.
  3. Ajoutez un 5e carré centré parfaitement avec la technique top: 50%; left: 50%; transform: translate(-50%, -50%).
  4. Créez une navbar simulée avec position: sticky qui se colle en haut au scroll.
  5. Ajoutez un bouton "Retour en haut" position: fixed en bas à droite.
EX 02 📰 Article avec image flottée
  1. Créez exo-float.html avec un article contenant un titre, 3 paragraphes et une image.
  2. Faites flotter l'image à gauche avec float: left et une marge appropriée.
  3. Ajoutez une citation blockquote flottée à droite.
  4. Utilisez clear: both sur un élément après l'image pour que le dernier paragraphe ne soit pas à côté.
  5. Ajoutez un badge "Nouveau" avec position: absolute dans le coin supérieur droit de l'article.
  6. Expérimentez avec display: inline-block pour aligner 3 icônes de réseaux sociaux horizontalement.
🔍
Exercice bonus — Contexte d'empilement

Créez 3 conteneurs position: relative imbriqués. Chacun contient un carré absolu avec un z-index différent (1, 2, 3). Donnez au parent du milieu z-index: 10 pour créer un contexte d'empilement. Observez comment un enfant avec z-index: 999 dans le parent du milieu reste sous un enfant avec z-index: 1 dans un parent avec z-index: 20.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • display : block, inline, inline-block, none — le type de boîte de chaque élément
  • position static : flux normal du document (valeur par défaut)
  • position relative : décalé visuellement, sert d'ancrage pour absolute
  • position absolute : retiré du flux, positionné par rapport à l'ancêtre positionné
  • position fixed : retiré du flux, positionné par rapport au viewport
  • position sticky : alterne entre relative et fixed selon le scroll
  • z-index et contexte d'empilement (stacking context)
  • float : usage moderne pour le texte autour d'une image, clearfix

Mini-glossaire

Display
Type de boîte. block = toute la largeur. inline = dans le texte. inline-block = inline mais dimensionnable.
Position static
Valeur par défaut. L'élément suit le flux normal. top/left/z-index ignorés.
Position relative
L'élément reste dans le flux mais peut être décalé visuellement. Sert d'ancrage.
Position absolute
Retiré du flux. Positionné par rapport à l'ancêtre positionné le plus proche.
Position fixed
Retiré du flux. Positionné par rapport au viewport. Reste visible au scroll.
Position sticky
Relative jusqu'au seuil, puis fixed. Idéal pour les en-têtes de section.
z-index
Ordre d'empilement sur l'axe Z dans un contexte d'empilement. Valeur entière.
Contexte d'empilement
Groupe d'éléments avec un z-index de base commun. Créé par position + z-index, opacity, transform, flex.
Float
Fait flotter un élément à gauche/droite pour que le texte s'écoule autour. Usage moderne : images.
Clearfix
Technique CSS pour qu'un conteneur d'éléments flottés conserve une hauteur.
