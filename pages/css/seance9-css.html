Viewport · Media queries · Breakpoints · Mobile-first · Unités Responsives

01

La balise meta viewport

Le responsive design commence par une balise HTML qui dit au navigateur mobile comment interpréter la largeur de la page. Sans elle, les mobiles affichent le site desktop zoomé très en arrière — rendant le texte illisible.

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé
HTML — La balise viewport 
<!-- À mettre dans <head> — obligatoire pour le responsive -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">





<meta name="viewport" content="width=device-width, initial-scale=1, minimum-scale=1, maximum-scale=2">
RENDU
meta viewport indispensable pour tout site responsive.
⚠️
Sans viewport, pas de responsive

Si vous oubliez la balise viewport, votre site ne sera jamais vraiment responsive, quels que soient vos efforts CSS. Les iPhones (et autres mobiles) afficheront la page comme si elle était sur un écran de 980px de large, zoomée en arrière. L'utilisateur devra pincer-zoomer pour lire — expérience catastrophique. Les frameworks comme Bootstrap la génèrent automatiquement, mais si vous codez à la main, ne l'oubliez pas.

02

Les media queries — syntaxe

Les media queries sont des règles CSS conditionnelles : elles s'appliquent uniquement si certaines conditions sur le media type ou les caractéristiques du dispositif sont réunies (largeur d'écran, orientation, etc.).

CSS 
/* Syntaxe générale */
@media type and (condition) {
  /* règles CSS appliquées si condition vraie */
}

/* Types de média */
@media screen { }          /* écrans (ordinateur, mobile, tablette) */
@media print { }           /* impression */
@media all { }             /* tous les types (défaut) */

/* Conditions sur la largeur */
@media (min-width: 768px) { }  /* écran ≥ 768px */
@media (max-width: 767px) { }  /* écran ≤ 767px */

/* Conditions combinées (ET) */
@media screen and (min-width: 768px) and (max-width: 1024px) {
  /* tablettes en mode paysage */
}

/* OU — virgule */
@media (max-width: 480px), (orientation: landscape) { }

/* Orientation */
@media (orientation: portrait) { }   /* mobile en vertical */
@media (orientation: landscape) { }  /* mobile en horizontal */

/* Préférence de réduction de mouvement (accessibilité) */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
RENDU
Redimensionnez la fenêtre pour voir les media queries en action.
📌
Où placer les media queries ?

Deux approches : ① Dans le même fichier CSS, après les règles de base (recommandé). ② Dans un fichier CSS séparé avec l'attribut media sur la balise <link> : <link rel="stylesheet" href="mobile.css" media="(max-width: 767px)">. L'approche tout-en-un dans un seul fichier est plus facile à maintenir et évite les requêtes HTTP supplémentaires.

03

Breakpoints — points de rupture

Un breakpoint est une largeur d'écran à partir de laquelle la mise en page change. Il n'y a pas de valeurs universelles — tout dépend de votre contenu. Voici les breakpoints les plus courants.

CSS — Breakpoints standards 
/* Mobile-first (min-width) — on ajoute des styles au fur et à mesure */
/* Styles de base = mobile (0-575px) */

/* Petit écran / grand mobile (≥576px) */
@media (min-width: 576px) { }

/* Tablette (≥768px) */
@media (min-width: 768px) { }

/* Desktop (≥992px) */
@media (min-width: 992px) { }

/* Grand desktop (≥1200px) */
@media (min-width: 1200px) { }

/* Très grand écran (≥1400px) */
@media (min-width: 1400px) { }

/* ⚠️ Ces valeurs viennent de Bootstrap.
   Adaptez les vôtres à VOTRE design ! */
RENDU
≥576px ≥768px ≥992px ≥1200px
💡
Breakpoints basés sur le contenu, pas sur les appareils

N'utilisez pas les breakpoints d'un framework sans réfléchir. La meilleure approche : redimensionnez votre navigateur et ajoutez un breakpoint exactement là où le design commence à "casser" (texte qui se chevauche, marges trop grandes, colonnes trop étroites). Chaque design a ses propres points de rupture. Les valeurs Bootstrap sont un bon point de départ mais ne sont pas magiques.

04

Approche mobile-first (min-width)

L'approche mobile-first consiste à écrire d'abord le CSS pour le mobile (base), puis à ajouter des couches pour les écrans plus larges avec min-width. C'est l'approche recommandée par la plupart des experts.

CSS — Mobile-first 
/* BASE — Styles MOBILES par défaut (0px et plus) */
.grille {
  display: grid;
  grid-template-columns: 1fr;        /* 1 colonne sur mobile */
  gap: 16px;
}

/* TABLETTE — 768px et plus */
@media (min-width: 768px) {
  .grille {
    grid-template-columns: 1fr 1fr;  /* 2 colonnes */
  }
  .nav { display: flex; }
  .hamburger { display: none; }
}

/* DESKTOP — 1024px et plus */
@media (min-width: 1024px) {
  .grille {
    grid-template-columns: 1fr 1fr 1fr;  /* 3 colonnes */
  }
  .sidebar { display: block; }
}

/* Avantage : pas de !important, cascade naturelle */
RENDU
Mobile : 1 colonne
(redimensionner)
📌
Pourquoi mobile-first ?

① Performance : les mobiles téléchargent moins de CSS (les règles min-width ne s'appliquent pas). ② Progressive enhancement : le site de base fonctionne sur tous les écrans, les grands écrans ajoutent des améliorations. ③ Plus simple : partir du plus petit pour ajouter des couches est mentalement plus facile que de partir du plus grand et de retirer. ④ SEO : Google privilégie les sites mobile-friendly.

05

Approche desktop-first (max-width)

L'approche desktop-first est l'inverse : on écrit d'abord le CSS pour le grand écran, puis on "casse" la mise en page pour les écrans plus petits avec max-width. Plus intuitive pour les débutants, mais moins recommandée professionnellement.

CSS — Desktop-first 
/* BASE — Styles DESKTOP par défaut */
.grille {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr;  /* 3 colonnes */
  gap: 24px;
}

/* TABLETTE — 1023px et moins */
@media (max-width: 1023px) {
  .grille {
    grid-template-columns: 1fr 1fr;  /* 2 colonnes */
  }
}

/* MOBILE — 767px et moins */
@media (max-width: 767px) {
  .grille {
    grid-template-columns: 1fr;      /* 1 colonne */
  }
}

/* Inconvénient : le CSS de base (3 colonnes) est chargé même sur mobile */
/* Nécessite parfois !important pour surcharger les styles de base */
RENDU
Desktop-first : les styles desktop sont chargés partout, puis remplacés sur mobile.
💡
Mobile-first ou desktop-first ?

Mobile-first (min-width) est recommandé dans la grande majorité des cas. Il est plus performant, plus élégant, et mieux adapté à la réalité du web actuel (plus de trafic mobile que desktop). Desktop-first (max-width) peut être utile quand vous devez adapter un site desktop existant. Si vous démarrez un projet de zéro, partez mobile-first.

06

Unités responsives : vw, vh, %, clamp()

Les unités relatives sont au cœur du responsive design. vw, vh, % et la fonction clamp() permettent aux éléments de s'adapter à la taille de l'écran.

CSS — Unités responsives 
/* Viewport units */
.hero    { height: 60vh; }           /* 60% de la hauteur de l'écran */
.section { padding: 5vw; }          /* padding proportionnel à la largeur */
.title   { font-size: 6vw; }        /* titre qui rétrécit avec l'écran */

/* Pourcentage */
.container { width: 100%; max-width: 1100px; margin: 0 auto; }
.sidebar   { width: 30%; }
.content   { width: 70%; }

/* ═══ clamp() — la fonction reine du responsive ═══ */
/* clamp(MIN, PRÉFÉRÉ, MAX) */
.text {
  font-size: clamp(1rem, 2.5vw, 2rem);
  /* min 1rem, préféré 2.5vw, max 2rem */
  /* Sur 320px : 2.5vw = 8px → clamp → 1rem (16px)
     Sur 768px : 2.5vw = 19.2px → 19.2px
     Sur 1440px : 2.5vw = 36px → clamp → 2rem (32px) */
}

/* Padding responsive */
.card { padding: clamp(1rem, 3vw, 2.5rem); }
RENDU
clamp() : taille qui s'adapte, avec min et max.
📌
clamp() — le remplacement moderne des media queries

clamp(MIN, PRÉFÉRÉ, MAX) permet de définir une valeur fluide entre un minimum et un maximum. Pour les tailles de police, padding, gaps, c'est une alternative plus simple qu'une media query. Un titre en clamp(1.5rem, 4vw, 3rem) grandit progressivement avec l'écran sans jamais dépasser 3rem ni descendre sous 1.5rem. Plus besoin de breakpoint pour les tailles.

07

Images responsives : max-width: 100% & srcset

Une image qui dépasse de son conteneur casse la mise en page. La règle de base est simple : max-width: 100%. Pour la performance, on utilise srcset pour servir l'image à la bonne résolution.

CSS + HTML 
/* CSS — la règle d'or : l'image ne dépasse jamais son parent */
img, video, iframe {
  max-width: 100%;
  height: auto;        /* préserve le ratio aspect */
}

/* Pour une image de couverture */
.hero-img {
  width: 100%;
  height: 300px;
  object-fit: cover;   /* recadre sans déformer */
}

/* Pour une image dans un article */
.article img {
  max-width: 100%;
  height: auto;
  border-radius: 8px;
}
RENDU

srcset — servir la bonne taille

HTML — srcset et sizes 
<!-- srcset : différentes versions de l'image selon la largeur d'écran -->
<img src="photo-800.jpg"
     srcset="photo-400.jpg 400w,
             photo-800.jpg 800w,
             photo-1200.jpg 1200w"
     sizes="(max-width: 480px) 100vw,
            (max-width: 1024px) 50vw,
            800px"
     alt="Paysage">


<picture>
  <source srcset="photo.avif" type="image/avif">
  <source srcset="photo.webp" type="image/webp">
  <img src="photo.jpg" alt="Paysage">
</picture>
RENDU
srcset = images adaptées à la résolution de l'écran.
💡
La règle d'or : max-width: 100% sur toutes les images

Placez cette règle dans votre reset CSS ou normalize : img, video, iframe { max-width: 100%; height: auto; }. Cela garantit qu'aucune image ne dépasse jamais de son conteneur, sur n'importe quel écran. Sans cette règle, une image de 1920px de large dépassera de 300px sur un mobile — l'horreur responsive.

08

Approche modulaire & container queries

Une approche modulaire consiste à concevoir des composants réutilisables qui s'adaptent à leur conteneur, pas seulement à la fenêtre. C'est l'évolution du responsive design : les container queries.

CSS — Container Queries 
/* 1. Définir un conteneur de requête */
.card-container {
  container-type: inline-size;
  container-name: card;
}

/* 2. Utiliser @container au lieu de @media */
@container card (min-width: 400px) {
  .card {
    display: flex;
    flex-direction: row;    /* horizontal si assez large */
  }
  .card-img {
    width: 150px;
    height: 150px;
  }
}

/* Approche modulaire classique sans container queries */
.card {
  /* styles de base = version mobile (1 colonne) */
  display: flex;
  flex-direction: column;
}

.card-wide {
  flex-direction: row;   /* version large — classe à ajouter manuellement */
}

/* ⚠️ Container queries : support à partir de Chrome 105+, Firefox 110+ */
RENDU
Module responsive
S'adapte à son conteneur
📌
Container queries — le futur du responsive

Les container queries (ou @container) permettent à un composant de réagir à la taille de son propre conteneur, pas à la fenêtre. C'est révolutionnaire pour les composants réutilisables : le même composant peut s'afficher en mode horizontal dans une sidebar large et en mode vertical dans une sidebar étroite — sans media query sur la page entière. Supporté dans tous les navigateurs modernes depuis 2023.

09

Exercices pratiques

EX 01 📱 Page responsive complète
  1. Créez exo-responsive.html avec header, navigation, contenu principal, sidebar, footer.
  2. En mobile (< 576px) : tout en 1 colonne, menu en hamburger (caché par défaut).
  3. En tablette (≥768px) : 2 colonnes pour la section principale.
  4. En desktop (≥1024px) : 3 colonnes avec sidebar visible.
  5. Utilisez clamp() pour les tailles de texte.
  6. Images responsives avec max-width: 100%.
EX 02 🔄 Migration mobile-first
  1. Prenez un fichier CSS existant avec une approche desktop-first.
  2. Réécrivez-le en mobile-first (min-width).
  3. Observez la différence de complexité et de quantité de code.
  4. Ajoutez clamp() pour au moins 3 propriétés (font-size, padding, gap).
  5. Utilisez max-width: 100%; height: auto; sur toutes les images.
  6. Testez sur mobile réel ou dans les DevTools (mode responsive).
🔍
Exercice bonus — Container queries

Créez un composant "carte-produit" (image, titre, prix, description, bouton). Utilisez container-type: inline-size sur un parent. Faites en sorte que la carte passe en mode horizontal (flex-direction: row) quand le conteneur dépasse 400px, et reste en mode vertical en dessous. Placez la même carte dans une sidebar étroite et dans la zone principale large — elle s'adapte automatiquement.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • La balise meta viewport — obligatoire pour tout site responsive
  • Syntaxe des media queries : @media, min-width, max-width, orientation
  • Breakpoints : 576px, 768px, 992px, 1200px (adaptables au contenu)
  • Approche mobile-first (min-width) vs desktop-first (max-width)
  • Unités responsives : %, vw, vh, clamp(min, préféré, max)
  • Images responsives : max-width: 100%, srcset, picture
  • Approche modulaire et container queries (@container)

Mini-glossaire

Viewport
Zone visible du navigateur. Balise meta obligatoire pour indiquer la largeur réelle au mobile.
Media query
Règle CSS conditionnelle basée sur les caractéristiques du dispositif (largeur, orientation, etc.).
Breakpoint
Largeur d'écran à partir de laquelle la mise en page change. Défini avec min-width ou max-width.
Mobile-first
Approche qui part du mobile (styles de base) et ajoute des couches pour les écrans plus larges avec min-width.
Desktop-first
Approche inverse : styles de base pour le desktop, surcharges avec max-width pour les écrans plus petits.
clamp()
Fonction CSS : clamp(MIN, PRÉFÉRÉ, MAX). Crée une valeur fluide entre un min et un max.
srcset
Attribut HTML qui liste plusieurs versions d'une image pour différentes résolutions d'écran.
Container query
Media query basée sur la taille du conteneur parent plutôt que sur celle du viewport.
object-fit
Propriété CSS pour contrôler comment une image remplit son cadre (cover, contain).
Progressive enhancement
Stratégie qui part d'une base fonctionnelle et ajoute des améliorations pour les navigateurs/écrans plus capables.
