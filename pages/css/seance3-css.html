01

Les couleurs en CSS

CSS propose plusieurs formats pour définir une couleur. Chacun a ses avantages selon le contexte : précision, transparence, lisibilité du code.

Les différents formats

FormatSyntaxeExemple
Noms prédéfinisnomred, blue, tomato, rebeccapurple (148 noms)
Hexadécimal#RRGGBB ou #RGB#ff6600, #f60, #3498db
Hexa avec alpha#RRGGBBAA#e6394680 (50% transparent)
RGBrgb(r, g, b)rgb(231, 57, 70)
RGBArgba(r, g, b, a)rgba(231, 57, 70, .5)
HSLhsl(h, s%, l%)hsl(4, 79%, 56%)
HSLAhsla(h, s%, l%, a)hsla(4, 79%, 56%, .5)
CSS — Tous les formats de couleurs 
/* NOM — limité, mais pratique pour les couleurs de base */
.box-red    { background: red; }
.box-tomato { background: tomato; }
.box-gold   { background: gold; }

/* HEX — le plus utilisé en production */
.box-hex    { background: #3498db; }     /* bleu vif */
.box-hex2   { background: #2ecc71; }     /* vert */
.box-hex3   { background: #e74c3c; }     /* rouge */

/* HEX avec transparence (#RRGGBBAA) */
.box-hexa   { background: #e74c3c80; }   /* rouge à 50% */

/* RGB — pratique pour les couleurs programmatiques */
.box-rgb    { background: rgb(52, 152, 219); }

/* RGBA — comme RGB mais avec transparence (a = 0 à 1) */
.box-rgba   { background: rgba(52, 152, 219, .3); }

/* HSL — plus intuitif pour créer des variations */
.box-hsl    { background: hsl(210, 70%, 53%); }  /* même bleu */
.box-hsl2   { background: hsl(210, 50%, 70%); }  /* même teinte, moins saturé, plus clair */

/* HSLA — avec transparence */
.box-hsla   { background: hsla(210, 70%, 53%, .5); }
RENDU
HEX #3498db #2ecc71 #e74c3c RGBA 50% HSL
💡
Quand utiliser quel format

HEX : standard industriel, parfait pour les couleurs fixes. RGBA : quand vous avez besoin de transparence. HSL : idéal pour créer des variations (plus clair, plus foncé, moins saturé) à partir d'une teinte de base. Utilisez des variables CSS (vues en séance 10) pour stocker vos couleurs et les réutiliser.

02

Unités absolues : px, pt, cm

Les unités absolues correspondent à des mesures physiques fixes. La plus utilisée dans le web est le pixel (px). Les autres (pt, cm, mm, in, pc) viennent de l'imprimerie et sont rarement employées pour les écrans.

CSS — Unités absolues 
/* PIXEL — unité standard pour les écrans */
.border-px   { border: 2px solid #333; }    /* bordure fine */
.font-px     { font-size: 18px; }           /* taille de texte */
.padding-px  { padding: 16px 24px; }        /* espacement interne */

/* POINT — 1pt = 1/72 inch, pour l'imprimé */
.print-font  { font-size: 12pt; }

/* CENTIMÈTRE / MILLIMÈTRE — rare sur le web */
.margin-cm   { margin: 1cm 0; }

/* ⚠️ Les unités physiques (pt, cm, mm) sont imprécises à l'écran
   car la résolution réelle varie selon les moniteurs.
   Sur le web, utilisez presque toujours px ou des unités relatives. */
RENDU
Padding 16px 24px, texte 18px, bordure 2px.
📌
Le pixel CSS n'est pas un pixel physique

Un px en CSS correspond à un "pixel de référence" (environ 1/96 de pouce), pas nécessairement à un pixel physique de l'écran. Les écrans Retina (2×, 3×) utilisent plusieurs pixels physiques pour afficher un pixel CSS. C'est normal et transparent pour le développeur.

03

Unités relatives : em, rem, %, vw, vh

Les unités relatives sont calculées par rapport à une autre valeur (parent, racine, viewport). Elles sont essentielles pour le responsive design et l'accessibilité.

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé
CSS — Unités relatives 
/* ═══ % — pourcentage du parent ═══ */
.parent  { width: 600px; }
.child   { width: 50%; }  /* = 300px */

/* ═══ em — relatif à la taille de police DU PARENT ═══ */
.parent  { font-size: 16px; }
.child   { font-size: 1.5em; } /* = 24px (1.5 × 16) */
.child   { padding: 1em; }     /* = 24px (1 × font-size courant) */

/* ═══ rem — relatif à la taille de police RACINE (html) ═══ */
html     { font-size: 16px; } /* valeur par défaut */
h1       { font-size: 2rem; } /* = 32px (2 × 16) */
p        { font-size: 1rem; } /* = 16px */
.small   { font-size: .875rem; } /* = 14px */

/* ═══ vw / vh — relatif au viewport (fenêtre) ═══ */
.full-width  { width: 100vw; }  /* 100% de la largeur de la fenêtre */
.full-height { height: 100vh; } /* 100% de la hauteur de la fenêtre */
.hero-title  { font-size: 5vw; } /* 5% de la largeur de la fenêtre */

/* ═══ vmin / vmax ═══ */
.min-dim     { width: 50vmin; }  /* 50% de la plus petite dimension */
.max-dim     { height: 50vmax; } /* 50% de la plus grande dimension */
RENDU
80% du parent
1.5rem (24px si html 16px)
0.875rem (14px)
50% du parent
💡
rem recommandé, em à utiliser avec précaution

rem est plus prévisible que em car il se base toujours sur la racine (html). Avec em, la valeur s'accumule si vous imbriquez des éléments (ex : un li avec font-size: .9em dans une liste imbriquée rétrécit à chaque niveau). Préférez rem pour les tailles de police et em pour les espacements (padding, margin) qui doivent s'adapter à la taille de police locale.

04

font-family & Google Fonts

La propriété font-family définit la police de caractères. On fournit toujours une liste de fallback (polices de secours) séparées par des virgules, car on ne peut pas garantir qu'une police est installée chez l'utilisateur.

CSS — font-family avec fallback 
/* Les 5 grandes familles génériques */
.serif      { font-family: Georgia, 'Times New Roman', serif; }
.sans-serif { font-family: 'Helvetica Neue', Arial, sans-serif; }
.monospace  { font-family: 'Courier New', Consolas, monospace; }
.cursive    { font-family: 'Brush Script MT', cursive; }
.fantasy    { font-family: 'Papyrus', fantasy; }

/* Police avec espace dans le nom → guillemets */
.custom     { font-family: 'Fira Code', 'JetBrains Mono', monospace; }

/* Utilisation de Google Fonts (dans le HTML) */
/* <link href="https://fonts.googleapis.com/css2?family=Inter:wght@400;700&display=swap" rel="stylesheet"> */
.google     { font-family: 'Inter', sans-serif; }
RENDU

Georgia — police serif (avec empattements)

Arial — sans-serif (sans empattements)

Consolas — monospace (chasse fixe)

Comment utiliser Google Fonts

  1. Allez sur fonts.google.com
  2. Choisissez une police (ex : Inter, Roboto, Open Sans, Lora)
  3. Sélectionnez les graisses nécessaires (400 = normal, 700 = gras)
  4. Copiez la balise <link> fournie et collez-la dans votre <head>
  5. Utilisez le nom de la police dans votre CSS
⚠️
Performance — ne pas abuser des Google Fonts

Chaque police Google Fonts ajoute un chargement externe qui ralentit la page. Limitez-vous à 1 ou 2 familles (ex : une pour les titres, une pour le texte). Utilisez display=swap dans l'URL (affichage immédiat avec une police de secours le temps du chargement). Privilégiez les polices système quand c'est possible.

05

font-size & line-height

font-size contrôle la taille du texte. line-height contrôle l'interlignage (hauteur de ligne). Ensemble, ils déterminent la lisibilité du contenu textuel.

CSS 
/* font-size : tailles absolues et relatives */
.tres-grand  { font-size: 2.5rem; }   /* ~40px */
.grand       { font-size: 1.5rem; }   /* ~24px */
.normal      { font-size: 1rem; }     /* ~16px (défaut) */
.petit       { font-size: .875rem; }  /* ~14px */
.tres-petit  { font-size: .75rem; }   /* ~12px */

/* line-height : sans unité est recommandé */
.serre       { line-height: 1.2; }    /* serré — titres */
.confortable { line-height: 1.6; }    /* confortable — paragraphes */
.aere        { line-height: 2; }      /* aéré — citations */

/* Valeur avec unité (déconseillé pour line-height) */
/* line-height: 24px; → rigide, ne s'adapte pas si font-size change */
/* line-height: 1.6; → 1.6 × font-size, s'adapte automatiquement */
RENDU
2.5rem (titre)
1.5rem (sous-titre)
1rem, line-height 1.6 — interlignage confortable pour la lecture de longs paragraphes.
0.875rem, line-height 1.2 — serré, pour des petites notes.
💡
Règle typographique : line-height sans unité

Toujours utiliser un nombre sans unité pour line-height (ex : 1.6). Pourquoi ? Parce que la valeur est alors proportionnelle à la taille de police. Si vous mettez line-height: 24px sur un élément dont la police passe à 20px, l'interlignage ne s'adapte pas. Avec 1.5, il sera toujours de 1.5× la taille courante. C'est l'une des rares règles CSS universellement recommandées.

06

font-weight & font-style

font-weight contrôle la graisse (épaisseur) du texte. font-style gère l'italique. Ces propriétés dépendent des variantes disponibles dans la police utilisée.

CSS 
/* font-weight : valeurs numériques et mots-clés */
.fw-thin    { font-weight: 100; }  /* Thin / Hairline */
.fw-light   { font-weight: 300; }  /* Light */
.fw-normal  { font-weight: 400; }  /* Normal (défaut) */
.fw-medium  { font-weight: 500; }  /* Medium */
.fw-bold    { font-weight: 700; }  /* Bold */
.fw-black   { font-weight: 900; }  /* Black */

/* Mots-clés équivalents */
.fw-normal  { font-weight: normal; }  /* = 400 */
.fw-bold    { font-weight: bold; }    /* = 700 */

/* font-style */
.fs-normal  { font-style: normal; }
.fs-italic  { font-style: italic; }
.fs-oblique { font-style: oblique; }  /* presque identique à italic */

/* font-variant */
.fv-small   { font-variant: small-caps; } /* petites capitales */
RENDU
Thin 100
Light 300
Normal 400
Medium 500
Bold 700
Black 900
Italic
Small Caps
📌
Toutes les graisses ne sont pas disponibles

Si vous utilisez font-weight: 500 mais que la police ne contient pas cette graisse, le navigateur choisit la plus proche. Avec les Google Fonts, sélectionnez explicitement les graisses nécessaires dans l'URL de chargement pour garantir leur disponibilité. Évitez d'utiliser font-weight avec des valeurs intermédiaires (200, 600, 800) sans vérifier que la police les supporte.

07

text-align, text-transform, text-decoration

Ces trois propriétés contrôlent l'alignement, la casse et la décoration du texte. Elles sont simples mais essentielles au quotidien.

CSS 
/* ═══ text-align ═══ */
.gauche  { text-align: left; }       /* défaut */
.centre  { text-align: center; }
.droite  { text-align: right; }
.justifie { text-align: justify; }   /* pour l'imprimé, évitez sur le web */

/* ═══ text-transform ═══ */
.majuscules { text-transform: uppercase; }
.minuscules { text-transform: lowercase; }
.capitalize { text-transform: capitalize; } /* 1ère lettre de chaque mot en maj */
.normal-case{ text-transform: none; }

/* ═══ text-decoration ═══ */
.souligne   { text-decoration: underline; }
.barre      { text-decoration: line-through; }
.none       { text-decoration: none; }  /* enlever le souligné des liens */

/* Forme longue (property, style, color) */
.deco       { text-decoration: underline wavy #e63946; }
RENDU
Aligné à gauche
Centré
Aligné à droite
uppercase
chaque mot en capitale
souligné
barré
souligné ondulé rouge
⚠️
text-transform ne remplace pas le contenu HTML

text-transform: uppercase est visuel seulement → un lecteur d'écran lira le texte comme il est écrit dans le HTML, pas en majuscules. Si le texte DOIT être lu en majuscules (sigle, acronyme), écrivez-le en majuscules directement dans le HTML. Si c'est juste un style, utilisez CSS.

08

Espacement : letter-spacing, word-spacing, white-space

Ces propriétés contrôlent l'espacement entre les caractères, entre les mots, et le comportement des espaces blancs dans le texte.

CSS 
/* letter-spacing : espacement entre les caractères */
.lettre-serre   { letter-spacing: -0.02em; }  /* resserré — titres */
.lettre-normal  { letter-spacing: normal; }
.lettre-espace  { letter-spacing: 0.1em; }    /* étalé */
.lettre-tres    { letter-spacing: 0.3em; }    /* très étalé — style luxe */

/* word-spacing : espacement entre les mots */
.mots-normaux   { word-spacing: normal; }
.mots-espace    { word-spacing: 0.25em; }

/* white-space : gestion des espaces */
.ws-normal  { white-space: normal; }     /* défaut — les espaces multiples sont compressés */
.ws-nowrap  { white-space: nowrap; }     /* pas de retour à la ligne */
.ws-pre     { white-space: pre; }        /* comme <pre> — respecte les espaces */
.ws-pre-wrap{ white-space: pre-wrap; }   /* comme pre mais avec retour auto */
.ws-pre-line{ white-space: pre-line; }   /* respecte les retours ligne, pas les espaces */
RENDU
ESPACÉ
Resserré
Texte long sans retour à la ligne automatique — tronqué avec ellipsis
Ligne 1 Ligne 2 Indenté
💡
Quand utiliser letter-spacing

Un léger letter-spacing positif (0.05em–0.1em) améliore la lisibilité des textes en majuscules. Les titres de presse et de luxe utilisent souvent un espacement accru. À l'inverse, un espacement négatif peut être beau pour des gros titres très modernes, mais ne l'appliquez pas à des textes longs (illisibles).

09

Exercices pratiques

EX 01 🎨 Palette de couleurs & typographie
  1. Créez exo-typographie.html avec un article complet : titre h1, sous-titre h2, plusieurs paragraphes, une citation blockquote.
  2. Définissez une police Google Font pour les titres et une police système (serif) pour le texte.
  3. Utilisez font-size en rem pour tous les textes (2rem pour h1, 1.5rem pour h2, 1rem pour p).
  4. Appliquez line-height: 1.7 sur les paragraphes et 1.3 sur les titres.
  5. Utilisez au moins 3 formats de couleur différents (hex, rgba, hsl) pour les titres, les liens et les citations.
  6. Ajoutez text-transform: uppercase et letter-spacing: .1em sur les sous-titres.
EX 02 📐 Unités & espacement
  1. Créez une page avec trois blocs côte à côte de largeur 30% chacun, centrés dans la page.
  2. Utilisez vh pour donner une hauteur minimale de 50vh à la section principale.
  3. Ajoutez un bloc hero avec height: 60vh et un titre en font-size: 5vw.
  4. Créez un bouton avec padding: .75em 1.5em (proportionnel à sa propre taille de police).
  5. Utilisez white-space: nowrap sur un groupe de mots qui ne doivent jamais être séparés (ex : "Paris 75001").
  6. Ajoutez text-decoration: underline wavy sur un mot important.
🔍
Exercice bonus — Système de titres complet

Créez un système typographique complet pour un blog : h1 à h6 avec des tailles en rem, line-height spécifique pour chaque niveau, couleur dégradée (plus clair pour les niveaux profonds), et une variation de font-weight (h1 en 900, h6 en 400). Appliquez une marge verticale cohérente avec margin: 1.5em 0 0.5em sur tous les titres.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Les formats de couleur : hex (#RRGGBB, #RGB, #RRGGBBAA), rgb/rgba, hsl/hsla
  • Les unités absolues (px) et relatives (%, em, rem, vw, vh, vmin, vmax)
  • La différence entre em (parent) et rem (racine), et quand utiliser chacun
  • font-family avec fallback et intégration de Google Fonts
  • font-size en rem et line-height sans unité
  • font-weight (100–900) et font-style (normal, italic)
  • text-align, text-transform, text-decoration
  • letter-spacing, word-spacing, white-space

Mini-glossaire

HEX
Format hexadécimal #RRGGBB. Ex : #ff6600. Le format le plus utilisé pour les couleurs fixes.
HSL
Teinte (Hue 0–360), Saturation (0–100%), Luminosité (Lightness 0–100%). Plus intuitif pour créer des variations.
rem
Unité relative à la taille de police de la racine (html). 1rem = 16px par défaut. Prévisible et accessible.
em
Unité relative à la taille de police du parent. S'accumule dans les éléments imbriqués. Utile pour les paddings.
vw / vh
1vw = 1% de la largeur du viewport. 1vh = 1% de la hauteur du viewport.
line-height
Hauteur de ligne. Toujours sans unité (ex: 1.6). Essentiel pour la lisibilité du texte.
font-weight
Graisse de la police : 100 (thin) à 900 (black). 400 = normal, 700 = bold.
text-transform
Modifie la casse : uppercase, lowercase, capitalize. Visuel seulement, pas sémantique.
letter-spacing
Espacement entre les caractères. En em. Utile pour les textes en majuscules.
Google Fonts
Service de polices web hébergées. Gratuit, grande bibliothèque, simple à intégrer.
