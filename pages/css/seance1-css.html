01

Le rôle du CSS — habiller le HTML

Le CSS (Cascading Style Sheets, feuilles de style en cascade) est le langage qui contrôle l'apparence visuelle des pages web. Si le HTML est le squelette, le CSS est la peau, les vêtements et le maquillage.

🎨
Analogie : la maison

Le HTML construit les murs, les pièces, la charpente. Le CSS choisit la couleur des murs, le style des fenêtres, le carrelage, l'éclairage. Sans CSS, la page est fonctionnelle mais sans style — comme un chantier brut. Avec CSS, elle devient un intérieur décoré et agréable à vivre.

Ce que CSS peut faire

🎨 Couleurs & arrière-plans

Teinter le texte, les fonds, les bordures. Ajouter des dégradés, des images de fond, des motifs répétés.

📐 Mise en page

Positionner les éléments exactement où vous le souhaitez : en colonnes, en grille, centrés, collés en haut ou en bas.

🔤 Typographie

Polices, tailles, graisses, interlignage, espacement, alignement — tout ce qui rend le texte lisible et beau.

✨ Animations & transitions

Faire bouger, apparaître, disparaître, tourner, grossir des éléments — sans une ligne de JavaScript.

📱 Adaptation mobile

Un même HTML s'affiche parfaitement sur mobile, tablette et desktop grâce aux media queries.

♿ Accessibilité

Agrandir la taille du texte, augmenter les contrastes, gérer le focus clavier pour les utilisateurs malvoyants.

CSS ne peut PAS faire

  • Ajouter ou modifier du contenu textuel (sauf quelques pseudo-éléments)
  • Faire des calculs complexes ou interagir avec une base de données
  • Réagir à un clic utilisateur (survol oui, clic non — c'est le rôle de JavaScript)
📌
CSS s'améliore chaque année

Le CSS n'est pas un langage "figé". Chaque année, de nouvelles propriétés arrivent : les subgrid, container queries, has(), layers. La version actuelle est CSS3, mais on parle désormais de "CSS moderne" avec des spécifications évolutives.

02

Syntaxe CSS : règle, sélecteur, déclaration

Une feuille de style CSS est une série de règles. Chaque règle comporte un sélecteur (qui cible les éléments HTML) et un bloc de déclarations (propriété : valeur).

🎨 Sélecteur → 📐 Propriété
📐 Propriété → 🎯 Valeur
🎯 Valeur → ✨ Résultat stylisé

Anatomie d'une règle CSS

CSS 
/* Une règle CSS complète */
p {
  color: blue;
  font-size: 18px;
  line-height: 1.6;
}

/* Explication :
   p       → sélecteur (cible tous les <p>)
   color   → propriété
   blue    → valeur
   color: blue; → déclaration */
RENDU

Ce paragraphe est bleu, taille 18px.

Les trois éléments d'une règle

🎯 Sélecteur

Indique quel(s) élément(s) HTML sont ciblés. Peut être un nom de balise (p), une classe (.ma-classe), un id (#mon-id) ou une combinaison complexe.

📋 Déclaration

Couple propriété: valeur;. Chaque déclaration se termine obligatoirement par un point-virgule ;. Sans lui, la règle entière peut être ignorée.

📦 Bloc de déclarations

Ensemble de déclarations entre { }. Les accolades sont obligatoires, même pour une seule déclaration.

⚠️
Erreur fréquente — Point-virgule oublié

Si vous oubliez le ; après une déclaration, la déclaration suivante sera ignorée (pas la courante). Le navigateur essaie de "réparer" mais le résultat est imprévisible. Le dernier point-virgule avant la fermeture est optionnel mais conseillé par convention.

CSS 
/* Plusieurs sélecteurs : on les sépare par une virgule */
h1, h2, h3 {
  font-family: 'Arial', sans-serif;
  color: #222;
}

/* Commentaires : entre /* et */. Ne pas hésiter à documenter */

/* Une propriété peut prendre plusieurs valeurs */
.bouton {
  border: 2px solid #e63946;
  padding: 8px 16px;
  border-radius: 4px;
}
RENDU

Titre h3 en Arial

Bouton stylisé
03

Trois méthodes pour appliquer du CSS

Il existe trois façons de lier du CSS à un document HTML. Chacune a ses usages, ses avantages et ses inconvénients. En pratique, on utilise la troisième (fichier externe) dans la quasi-totalité des projets professionnels.

① CSS Inline

Dans l'attribut style de la balise HTML. Utile pour un test rapide. À éviter : mélange HTML/CSS, pas réutilisable, priorité maximale.

② CSS Interne

Dans une balise <style> dans le <head>. Pratique pour un fichier unique ou un prototype. Pas réutilisable entre pages.

③ CSS Externe ✅

Fichier .css séparé, lié via <link rel="stylesheet" href="style.css">. Standard professionnel : séparation totale, cache navigateur, réutilisable.

CSS — Trois méthodes 
<!-- ① INLINE — dans l'attribut style de la balise -->
<p style="color: red; font-weight: bold;">
  Ce texte est rouge et gras.
</p>

<!-- ② INTERNE — dans <head> via <style> -->
<head>
  <style>
    p { color: blue; }
  </style>
</head>

<!-- ③ EXTERNE — fichier .css séparé (recommandé) -->
<head>
  <link rel="stylesheet" href="style.css">
</head>
RENDU

CSS inline : rouge et gras.

CSS via la balise <style> : bleu.

CSS externe : vert italique.

💡
Pourquoi le fichier externe est le meilleur choix

① Séparation des préoccupations : HTML = structure, CSS = style. ② Cache navigateur : le fichier CSS est téléchargé une fois puis mis en cache — les pages suivantes chargent plus vite. ③ Maintenabilité : changer une couleur dans un fichier modifie tout le site. ④ Minification : un fichier CSS peut être compressé pour accélérer le chargement.

04

La cascade : origine et ordre

Le "C" de CSS signifie Cascading (en cascade). Quand plusieurs règles ciblent le même élément, le navigateur suit un ordre précis pour déterminer quelle règle gagne. La cascade tient compte de trois critères : l'origine, l'ordre et la spécificité.

Les trois origines du CSS

OrigineExplicationPriorité
① Agent utilisateurStyle par défaut du navigateur (ex : Chrome donne font-size: 16px aux paragraphes)Faible
② Auteur (vous)Le CSS que vous écrivez dans vos fichiersMoyenne
③ UtilisateurStyle défini par le visiteur (ex : zoom, CSS custom dans les paramètres)Haute

Ordre d'apparition : le dernier gagne

Parmi les règles de même spécificité, c'est la dernière déclarée dans le code qui l'emporte. C'est pourquoi l'ordre de vos fichiers CSS et de vos règles est important.

CSS 
/* Ces deux règles ont la même spécificité (une balise) */
/* La dernière déclaration gagne */

p { color: red; }
p { color: blue; }
/* => Tous les <p> seront bleus car blue est déclaré après red */

/* Ordre d'importation des fichiers */
/* style.css */
p { color: green; }

/* custom.css (chargé après) */
p { color: orange; }
/* => Si custom.css est chargé après style.css, orange gagne */
RENDU

Ce paragraphe est bleu (dernière déclaration gagne).

Celui-ci aussi — même sélecteur, dernier style appliqué.

📌
Cascade = algorithme en trois étapes

Le navigateur applique cet ordre pour chaque propriété de chaque élément : ① Origine (CSS auteur > agent utilisateur) → ② Spécificité (plus précis gagne) → ③ Ordre (dernier dans le code gagne). Si une règle gagne à l'étape 1, on ne passe pas à l'étape 2.

05

La spécificité : calcul du score

Quand deux sélecteurs de même origine ciblent le même élément, celui qui a la spécificité la plus élevée gagne. La spécificité se calcule comme un score à quatre chiffres (a, b, c, d).

Le système de notation

Sélecteura (inline)b (id)c (classe, att, pseudo)d (balise, pseudo-élém)
style="..."1000
#contenu0100
.classe0010
[type="text"]0010
:hover0010
p0001
::before0001
* (universel)0000
CSS 
/* Calcul du score — chaque ligne a une spécificité différente */
p { color: red; }                    /* 0,0,0,1 = 1 */
p.important { color: orange; }       /* 0,0,1,1 = 11 */
#contenu p { color: green; }         /* 0,1,0,1 = 101 */
p.important#contenu { color: blue; } /* 0,1,1,1 = 111 */
/* style="color: purple" (inline)    /* 1,0,0,0 = 1000 */

/* Règles pratiques :
   - 1 id bat 100 classes
   - 1 classe bat 100 balises
   - L'inline bat tout (sauf !important) */
RENDU

Spécificité 0,1,1,1 → bleu gagne.

Spécificité 0,0,1,1 → orange.

Spécificité 0,0,0,1 → rouge.

💡
Astuce pour éviter les conflits de spécificité

Travaillez avec des classes plutôt que des id pour le style. Les id créent une spécificité très haute qui est difficile à surcharger. Les classes (spécificité 0,0,1,0) sont plus flexibles et réutilisables. Une bonne pratique : réservez les id pour le JavaScript et les ancres, utilisez les classes pour le CSS.

06

!important et l'héritage

Deux mécanismes peuvent court-circuiter la cascade normale : !important (force une règle) et l'héritage (transmission automatique de certaines propriétés aux éléments enfants).

!important — le marteau-pilon

Ajouter !important après une valeur donne à cette déclaration la plus haute priorité possible, surpassant même l'inline. Attention : c'est un outil puissant mais dangereux — il casse la cascade naturelle et rend le code difficile à maintenir.

CSS 
/* !important force la règle quoi qu'il arrive */
p { color: red !important; }

/* Même l'inline ne peut pas la surcharger */
/* <p style="color: blue"> → sera ROUGE à cause de !important */

/* ⚠️ À éviter dans la plupart des cas !
   !important rend le débogage très difficile.
   Préférez augmenter la spécificité. */
RENDU

!important force le rouge (même si inline dit bleu).

L'héritage — les enfants héritent des parents

Certaines propriétés CSS sont héritées automatiquement des parents vers les enfants. D'autres non. C'est logique : la couleur du texte se transmet aux enfants, mais la bordure ou la marge ne se transmettent pas.

Propriétés héritéesPropriétés NON héritées
colormargin
font-familypadding
font-sizeborder
line-heightbackground
text-alignwidth / height
visibilitydisplay
cursorposition
CSS 
/* Héritage en action */
article {
  color: #333;
  font-family: 'Georgia', serif;
  line-height: 1.8;
}

/* span et p vont hériter de color, font-family et line-height */
/* Sauf si on les surcharge explicitement */
article p {
  color: #1a5276; /* surcharge la couleur héritée */
}

/* Les mots-clés d'héritage */
.force-heritage {
  color: initial;    /* revient à la valeur par défaut du navigateur */
  color: inherit;    /* force l'héritage même si normalement non hérité */
  color: unset;      /* se comporte comme inherit si hérité, initial sinon */
}
RENDU
Héritage : ce span a la couleur #333 du parent.

Ce paragraphe surcharge en #1a5276.

⚠️
Ne pas abuser de !important

Un code avec beaucoup de !important est un code qui devient rapidement ingérable. Si vous devez utiliser !important fréquemment, c'est souvent le signe que votre sélecteur manque de spécificité ou que votre structure CSS a besoin d'être repensée. La plupart des frameworks CSS l'utilisent avec parcimonie pour les utilitaires (ex : classes .hidden, .text-center).

07

Inspecter le CSS avec les DevTools

Les outils de développement (DevTools) de votre navigateur sont indispensables pour travailler avec le CSS. Ils permettent de visualiser, tester et déboguer les styles en temps réel — sans modifier vos fichiers.

Comment ouvrir les DevTools

⌨️ Raccourcis clavier

F12 sur Chrome, Firefox, Edge
Ctrl+Maj+I (Windows/Linux)
Cmd+Opt+I (Mac)

🖱️ Clic droit

Faites un clic droit sur n'importe quel élément de la page → "Inspecter". C'est le moyen le plus rapide pour cibler un élément précis.

💡
Onglet Styles — votre meilleur ami

Dans l'onglet "Styles" (ou "Styles" en français), vous voyez : ① toutes les règles CSS appliquées à l'élément sélectionné, ② la spécificité et la provenance de chaque règle, ③ les propriétés barrées par des règles plus fortes, ④ la zone de calcul du box model en bas. Vous pouvez décocher/ cocher des propriétés pour tester, modifier les valeurs en direct, et ajouter de nouvelles règles.

Fonctionnalités clés des DevTools pour le CSS

  • Inspecter un élément : cliquez sur l'icône 🖱️ en haut à gauche, puis sur l'élément dans la page
  • Modifier une valeur : double-cliquez sur n'importe quelle propriété ou valeur dans l'onglet Styles
  • Ajouter une règle : cliquez sur le + dans l'onglet Styles pour créer une nouvelle règle
  • État de survol : cliquez sur :hov pour activer :hover, :focus, :active sur l'élément
  • Box model visuel : l'onglet "Computed" (ou "Calculé") montre le diagramme du box model
  • Onglet "Computed" : affiche la valeur finale calculée de chaque propriété après la cascade
🔍
Exercice pratique avec les DevTools

Ouvrez n'importe quel site (ex : wikipedia.org). Faites clic droit → Inspecter sur le titre principal. Dans l'onglet Styles, cochez/décochez des propriétés CSS pour voir le changement en direct. Modifiez la couleur du texte. Changez la taille de la police. Cela n'affecte que votre affichage — les modifications disparaissent au rechargement de la page.

08

Normalize.css & Reset CSS

Chaque navigateur a ses propres styles par défaut (les "user-agent styles") : marges différentes sur <body>, tailles de police différentes sur les titres, etc. Pour partir d'une base cohérente, on utilise une feuille de style de normalisation ou de réinitialisation.

Reset CSS — tout remettre à zéro

Le Reset CSS (popularisé par Eric Meyer) supprime tous les styles par défaut des navigateurs. Marges, paddings, list-style, tout revient à 0. Vous reconstruisez ensuite entièrement le style.

Normalize.css — harmoniser sans tout casser

Normalize.css (créé par Nicolas Gallagher et Jonathan Neal) est plus moderne : il conserve les styles utiles par défaut, corrige les incohérences entre navigateurs et normalise les comportements. C'est l'approche recommandée aujourd'hui.

CSS — Exemple Reset vs Normalize 
/* ═══ RESET CSS (extrait — Eric Meyer) ═══ */
html, body, div, span, h1, h2, h3, h4, h5, h6,
p, a, ul, ol, li, figure, figcaption, blockquote {
  margin: 0;
  padding: 0;
  border: 0;
  font-size: 100%;
  font: inherit;
  vertical-align: baseline;
}
ol, ul { list-style: none; }
a { text-decoration: none; color: inherit; }

/* ═══ NORMALIZE.CSS (extraits) ═══ */
/* Les marges du body sont normalisées */
body { margin: 0; }

/* Les titres conservent leur importance visuelle */
h1 { font-size: 2em; margin: 0.67em 0; }

/* Les listes gardent leur style mais normalisé */
ul { list-style: disc; padding-left: 2em; }

/* Les liens restent soulignés et bleus */
a { color: inherit; text-decoration: underline; }
RENDU

Reset : aucune marge ni padding.

Titre sans style après reset.

Normalize : styles cohérents entre navigateurs.

Titre normalisé.

💡
Recommandation pour vos projets

Utilisez Normalize.css (téléchargeable sur necolas.github.io/normalize.css) comme base pour tous vos projets. Liez-le en premier dans le <head>, avant votre fichier CSS personnel. Si vous utilisez un framework comme Bootstrap, Tailwind ou Foundation, ils incluent déjà leur propre normalisation. Évitez le Reset CSS complet qui vous oblige à redéfinir manuellement des styles de base.

Exercices pratiques

EX 01 🔤 Les trois méthodes CSS
  1. Créez un fichier exo1-css-methodes.html avec la structure HTML5 de base.
  2. Ajoutez un titre <h1> "Les trois méthodes CSS" et trois paragraphes <p>.
  3. Appliquez une couleur différente à chaque paragraphe en utilisant les trois méthodes : inline sur le premier, interne (<style>) sur le deuxième, externe (fichier style-exo1.css) sur le troisième.
  4. Dans le fichier externe, ajoutez une règle qui change la police de toute la page avec font-family et une règle pour les titres.
  5. Ajoutez dans le CSS externe un commentaire expliquant la cascade.
  6. Ouvrez le fichier dans Chrome, inspectez avec F12 et observez l'onglet Styles pour voir la provenance de chaque règle.
EX 02 🎯 Spécificité & cascade
  1. Créez exo2-specificite.html avec la structure HTML5.
  2. Dans le <body>, créez un <div id="conteneur"> contenant un <p class="texte"> avec du texte.
  3. Dans <style>, créez 4 règles qui ciblent ce paragraphe avec des spécificités croissantes : p { }, .texte { }, #conteneur p { }, #conteneur .texte { }. Donnez une couleur de texte différente à chacune.
  4. Ajoutez une règle !important sur l'une d'elles et observez le changement.
  5. Dans les DevTools (F12), sélectionnez le paragraphe et observez l'ordre des règles dans l'onglet Styles. Notez lesquelles sont barrées.
  6. Modifiez l'ordre de vos règles dans le code et constatez l'effet de la cascade.
🔍
Exercice bonus — Calcul de spécificité

Calculez manuellement la spécificité de ces sélecteurs : ul li a.nav-link → ? / #header .menu ul li a → ? / style="color:red" → ? / div p:first-child → ? Vérifiez vos réponses avec un calculateur en ligne comme specificity.keegan.st.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Le rôle du CSS : mise en forme, mise en page, adaptation mobile, animations
  • Syntaxe d'une règle CSS : sélecteur { propriété: valeur; }
  • Les trois méthodes : inline, interne (<style>), externe (<link>)
  • La cascade : origine (agent, auteur, utilisateur) et ordre d'apparition
  • Le calcul de spécificité (a,b,c,d) et pourquoi les classes sont préférables aux id
  • !important et l'héritage des propriétés parent → enfant
  • Les DevTools : inspecter, modifier en direct, comprendre les règles appliquées
  • Normalize.css vs Reset CSS : quand et pourquoi les utiliser

Mini-glossaire

CSS
Cascading Style Sheets — langage de style qui contrôle l'apparence des éléments HTML.
Règle CSS
Ensemble sélecteur + { déclarations }. Ex : p { color: red; }.
Sélecteur
Partie de la règle qui cible les éléments HTML. Peut être une balise, classe, id, attribut, etc.
Déclaration
Couple propriété : valeur terminé par un point-virgule. Ex : font-size: 16px;
Cascade
Algorithme qui détermine quelle règle s'applique en cas de conflit : origine → spécificité → ordre.
Spécificité
Score calculé selon la précision du sélecteur (inline > id > classe > balise).
!important
Mot-clé qui force une déclaration au-dessus de toutes les autres. À utiliser avec modération.
Héritage
Transmission automatique de certaines propriétés CSS des parents aux enfants (couleur, police…).
Reset CSS
Feuille de style qui supprime tous les styles par défaut des navigateurs pour partir d'une base neutre.
Normalize.css
Feuille de style qui harmonise les styles entre navigateurs sans supprimer les styles utiles par défaut.
