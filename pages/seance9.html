01

<header> — En-tête

<header> représente l'en-tête d'un document ou d'une section. Il contient généralement le logo, le titre, la navigation. Il peut y avoir plusieurs headers dans une page (un pour la page, un pour chaque article).

<header> Logo, titre principal, navigation
HTML — Header 
<header>
  <img src="logo.png" alt="Mon Site">
  <h1>Mon Site Web</h1>
  <nav>
    <a href="/">Accueil</a>
    <a href="/blog">Blog</a>
    <a href="/contact">Contact</a>
  </nav>
</header>
RENDU
Mon Site Web
AccueilBlogContact
02

<nav> — Navigation

<nav> regroupe les liens de navigation principaux. On l'utilise pour le menu principal, le plan du site, la pagination. Un lecteur d'écran peut "sauter" directement à la navigation.

HTML — Nav 
<nav aria-label="Navigation principale">
  <ul>
    <li><a href="/">Accueil</a></li>
    <li><a href="/produits">Produits</a></li>
    <li><a href="/services">Services</a></li>
    <li><a href="/contact">Contact</a></li>
  </ul>
</nav>
RENDU
  • Accueil
  • Produits
  • Services
  • Contact
💡
Plusieurs nav par page

Vous pouvez avoir plusieurs <nav> (menu principal + pied de page). Distinguez-les avec aria-label : <nav aria-label="Navigation principale">.

03

<main> — Contenu principal

<main> contient le contenu unique de la page. Il ne doit y en avoir qu'un seul par page. Exclut les éléments répétés (header, nav, footer, aside).

HTML — Main 
<main>
  <article>
    <h1>Titre de l'article</h1>
    <p>Contenu principal de la page.</p>
  </article>
</main>
RENDU

Titre de l'article

Contenu principal de la page.

⚠️
Un seul <main> par page

Contrairement à header, nav, article, section, aside et footer, la balise <main> ne doit apparaître qu'une seule fois. C'est le contenu principal unique de la page.

04

<article> — Contenu autonome

<article> représente un contenu autonome : article de blog, commentaire, produit, widget. Il doit pouvoir être compris sans le reste de la page. Chaque article a son propre header, section, footer.

HTML — Article 
<article>
  <header>
    <h2>Comment apprendre le HTML</h2>
    <time datetime="2024-03-15">15 mars 2024</time>
  </header>
  <p>Le HTML est le langage de base du web...</p>
  <footer>
    <a href="#">Lire la suite</a>
  </footer>
</article>

<article>
  <header>
    <h2>Les bases du CSS</h2>
    <time datetime="2024-03-20">20 mars 2024</time>
  </header>
  <p>Le CSS permet de styliser le HTML...</p>
  <footer>
    <a href="#">Lire la suite</a>
  </footer>
</article>
RENDU

Comment apprendre le HTML

15 mars 2024

Le HTML est le langage de base du web...

05

<section> — Section générique

<section> regroupe un contenu thématiquement cohérent. Contrairement à <article>, elle n'est pas autonome. Chaque section devrait avoir un titre (<h1> à <h6>).

HTML — Section 
<section>
  <h2>Nos services</h2>
  <p>Nous proposons des formations...</p>
</section>

<section>
  <h2>Nos tarifs</h2>
  <p>Consultez notre grille tarifaire...</p>
  <table>
    <!-- tableau des prix -->
  </table>
</section>
RENDU

Nos services

Nous proposons des formations...

📌
Article dans Section ou Section dans Article ?

Si le contenu est autonome → <article>. Sinon → <section>. Un article peut contenir plusieurs sections. Une section peut contenir plusieurs articles (ex: une section "Actualités" contenant des articles).

06

<aside> — Contenu complémentaire

<aside> contient des informations complémentaires : barre latérale, encart publicitaire, liens connexes, biographie de l'auteur. Il est séparé du contenu principal mais lié contextuellement.

HTML — Aside 
<aside>
  <h3>À propos de l'auteur</h3>
  <img src="auteur.jpg" alt="Photo de Jean">
  <p>Jean est développeur web depuis 10 ans...</p>

  <h3>Articles connexes</h3>
  <ul>
    <li><a href="#">CSS Grid</a></li>
    <li><a href="#">Flexbox</a></li>
  </ul>
</aside>
RENDU

À propos de l'auteur

Jean est développeur web depuis 10 ans...

Articles connexes

  • CSS Grid
  • Flexbox
07

<footer> — Pied de page

<footer> contient les informations de pied pour son parent : copyright, liens légaux, contact, plan du site. Comme <header>, il peut y en avoir plusieurs (un pour la page, un pour chaque article).

HTML — Footer 
<footer>
  <p>&copy; 2024 Mon Site. Tous droits réservés.</p>
  <nav>
    <a href="/mentions">Mentions légales</a>
    <a href="/confidentialite">Confidentialité</a>
    <a href="/contact">Contact</a>
  </nav>
</footer>
RENDU

© 2024 Mon Site. Tous droits réservés.

Mentions légalesConfidentialitéContact
08

<time>, <details>, <summary>

<time> représente une date/heure lisible par machine (attribut datetime). <details> crée un widget pliable/dépliable, avec <summary> comme titre visible.

HTML — Time et Details 
<!-- Time : date lisible par machine -->
<p>Publié le <time datetime="2024-03-15">
  15 mars 2024</time> à
  <time datetime="14:30">14h30</time>.</p>

<p>Mise à jour :
  <time datetime="2024-06-01">
  1er juin 2024</time>
</p>

<!-- Details / Summary : accordéon -->
<details>
  <summary>Comment installer VS Code ?</summary>
  <p>Téléchargez VS Code sur code.visualstudio.com,
  puis installez l'extension Live Server.</p>
</details>

<details>
  <summary>Qu'est-ce que Live Server ?</summary>
  <p>Un serveur local qui recharge automatiquement
  votre page à chaque sauvegarde.</p>
</details>
RENDU

Publié le 15 mars 2024.

Comment installer VS Code ?

Téléchargez VS Code sur code.visualstudio.com.

Qu'est-ce que Live Server ?

Un serveur local avec rechargement automatique.

💡
Format ISO 8601 pour datetime

Le format standard est YYYY-MM-DD pour les dates et HH:MM pour les heures. Exemples : 2024-12-25, 2024-03-15T14:30:00+01:00. Les moteurs de recherche utilisent cette donnée pour afficher la date dans les résultats.

09

Exercices pratiques

EX 01 ✏️ Page sémantique complète
  1. Créez une page avec la structure sémantique complète : header, nav, main, footer.
  2. Dans main : 2 articles avec leurs propres header et footer.
  3. Chaque article a une date avec <time>.
  4. Ajoutez une section "Commentaires" avec un aside pour le profil de l'auteur.
EX 02 ✏️ FAQ avec Details
  1. Créez une page FAQ avec 5 questions/réponses.
  2. Utilisez <details> et <summary> pour chaque question.
  3. Encadrez le tout dans un <main> avec <h1>.
EX 03 🔍 Analyse d'un site existant
  1. Ouvrez un site d'actualités (Le Monde, 20 Minutes).
  2. Inspectez avec F12 → onglet Éléments.
  3. Repérez : header, nav, main, article, section, aside, footer.
  4. Notez comment ils sont organisés et imbriqués.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Header : en-tête de la page ou d'une section
  • Nav : navigation principale (plusieurs autorisées)
  • Main : contenu unique (un seul par page)
  • Article : contenu autonome et réutilisable
  • Section : groupe thématique avec titre
  • Aside : contenu complémentaire
  • Footer : pied de page ou de section
  • Time : date/heure avec format ISO 8601
  • Details/Summary : accordéon pliable

Mini-glossaire

header
En-tête. Contient introduction, logo, navigation. Peut être multiple.
nav
Navigation principale. Liens vers les sections du site.
main
Contenu principal unique. Un seul par page.
article
Contenu autonome (article de blog, commentaire, produit).
section
Groupe thématique avec son propre titre.
aside
Contenu complémentaire (barre latérale, biographie, pubs).
footer
Pied de page. Copyright, liens légaux, contact.
time
Date/heure lisible par machine. Attribut datetime obligatoire.
details
Widget pliable/dépliable. Summary pour le titre visible.
