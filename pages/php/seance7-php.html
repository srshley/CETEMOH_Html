01

session_start()

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

Une session PHP permet de stocker des données utilisateur entre les pages. session_start() démarre ou reprend une session existante. Elle doit être appelée avant tout affichage.

PHP — Démarrer une session 
<?php
// À placer TOUT EN HAUT du fichier, avant toute sortie HTML
session_start();

// Si la session n'existe pas, PHP crée un ID unique (cookie PHPSESSID)
// Si la session existe déjà, PHP la reprend
?>
<!DOCTYPE html>
<html>...
⚠️
Headers already sent

Si vous voyez "Headers already sent", c'est que du HTML ou des espaces ont été affichés avant session_start(). Vérifiez : aucun espace avant <?php, pas d'écho, pas de BOM dans le fichier.

02

$_SESSION — lire et écrire

$_SESSION est un tableau associatif qui persiste entre les pages. Chaque visiteur a sa propre session isolée.

PHP — Utilisation de $_SESSION 
<?php
session_start();

// Écrire dans la session
$_SESSION['nom'] = "Alice";
$_SESSION['age'] = 30;
$_SESSION['roles'] = ["admin", "editeur"];

// Lire depuis la session
echo $_SESSION['nom'];   // Alice

// Vérifier l'existence
if (isset($_SESSION['nom'])) {
    echo "Bonjour " . $_SESSION['nom'];
}

// Valeur par défaut
$nom = $_SESSION['nom'] ?? 'Invité';

// Modifier une valeur
$_SESSION['age'] = 31;

// Supprimer une clé
unset($_SESSION['age']);

// Vider la session (mais pas le cookie)
$_SESSION = [];
?>

03

Système de login / logout

Les sessions sont idéales pour gérer l'authentification : l'utilisateur se connecte une fois et reste identifié sur toutes les pages.

PHP — Login / Logout 
<?php
session_start();

// --- login.php ---
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $email = $_POST['email'] ?? '';
    $mdp = $_POST['mdp'] ?? '';

    // Vérification (simulée — en réalité, vérifiez en BDD)
    if ($email === 'admin@exemple.com' && $mdp === 'secret') {
        $_SESSION['user_id'] = 1;
        $_SESSION['user_email'] = $email;
        $_SESSION['logged_in'] = true;
        header('Location: dashboard.php');
        exit;
    } else {
        $erreur = "Email ou mot de passe incorrect";
    }
}

// --- dashboard.php (page protégée) ---
if (!isset($_SESSION['logged_in']) || $_SESSION['logged_in'] !== true) {
    header('Location: login.php');
    exit;
}

// --- logout.php ---
session_start();
$_SESSION = [];

// Détruire le cookie de session
if (ini_get("session.use_cookies")) {
    $params = session_get_cookie_params();
    setcookie(session_name(), '', time() - 42000,
        $params["path"], $params["domain"],
        $params["secure"], $params["httponly"]
    );
}

session_destroy();
header('Location: login.php');
exit;
?>

04

Flash messages

Un flash message est un message qui n'est affiché qu'une seule fois (typiquement après une redirection). On le stocke en session et on le supprime après affichage.

PHP — Système de flash messages 
<?php
session_start();

// Fonction utilitaire pour définir un flash
function setFlash(string $type, string $message): void {
    $_SESSION['flash'] = ['type' => $type, 'message' => $message];
}

// Fonction pour afficher ET supprimer le flash
function getFlash(): ?array {
    if (isset($_SESSION['flash'])) {
        $flash = $_SESSION['flash'];
        unset($_SESSION['flash']);
        return $flash;
    }
    return null;
}

// --- usage ---
// page1.php : après une action
setFlash('success', 'Article créé avec succès !');
header('Location: page2.php');
exit;

// page2.php : affichage
$flash = getFlash();
if ($flash) {
    $css = $flash['type'] === 'success' ? 'green' : 'red';
    echo "<div style='color:$css'>" . htmlspecialchars($flash['message']) . "</div>";
}

// Types courants : success, error, warning, info
?>
══ 05 setcookie ══ -->
05

setcookie()

Un cookie est un petit fichier stocké côté navigateur. PHP permet d'en créer avec setcookie().

PHP — Créer un cookie 
<?php
// Syntaxe : setcookie(nom, valeur, expiration, chemin, domaine, secure, httponly)

// Cookie basique — expire à la fin de la session (fermeture navigateur)
setcookie('theme', 'dark');

// Cookie avec expiration (30 jours)
setcookie('pref_langue', 'fr', time() + 86400 * 30);

// Cookie avec chemin spécifique
setcookie('admin_mode', '1', time() + 3600, '/admin/');

// Cookie sécurisé (HTTPS uniquement + inaccessible au JS)
setcookie('token', 'abc123', [
    'expires' => time() + 3600,
    'path' => '/',
    'secure' => true,      // HTTPS uniquement
    'httponly' => true,    // inaccessible au JavaScript
    'samesite' => 'Strict' // protection CSRF
]);

// setcookie() doit être appelé avant tout HTML
?>
📌
setcookie et l'ordre d'appel

Comme session_start(), setcookie() envoie un en-tête HTTP et doit être appelé avant tout affichage HTML.

06

Lire et expirer les cookies

PHP — Lire et supprimer les cookies 
<?php
// LECTURE — via $_COOKIE
$theme = $_COOKIE['theme'] ?? 'light';
$langue = $_COOKIE['pref_langue'] ?? 'fr';

echo "Thème : " . htmlspecialchars($theme);

// Vérifier l'existence
if (isset($_COOKIE['token'])) {
    echo "Token présent : " . $_COOKIE['token'];
}

// SUPPRESSION — en mettant une date passée
setcookie('theme', '', time() - 3600);           // suppression
setcookie('pref_langue', '', time() - 3600);     // suppression

// Afficher tous les cookies
echo "<pre>";
print_r($_COOKIE);
echo "</pre>";

// ATTENTION : $_COOKIE n'est mis à jour qu'au prochain chargement !
// Après setcookie('theme', ''), $_COOKIE contient encore l'ancienne valeur
// jusqu'à ce que la page soit rechargée.
?>

Sécurité des sessions

Plusieurs pratiques essentielles pour sécuriser les sessions PHP contre le vol d'identité et les attaques.

PHP — Bonnes pratiques de sécurité 
<?php
session_start();

// 1. Régénérer l'ID de session après login (empêche fixation de session)
session_regenerate_id(true);

// 2. Vérifier l'IP et User-Agent (détection de vol de session)
if (!isset($_SESSION['ip'])) {
    $_SESSION['ip'] = $_SERVER['REMOTE_ADDR'];
    $_SESSION['ua'] = $_SERVER['HTTP_USER_AGENT'];
} else {
    if ($_SESSION['ip'] !== $_SERVER['REMOTE_ADDR'] ||
        $_SESSION['ua'] !== $_SERVER['HTTP_USER_AGENT']) {
        session_destroy();
        header('Location: login.php');
        exit;
    }
}

// 3. Timeout d'inactivité
$timeout = 1800;  // 30 minutes
if (isset($_SESSION['last_activity']) &&
    (time() - $_SESSION['last_activity']) > $timeout) {
    session_destroy();
    header('Location: login.php?timeout=1');
    exit;
}
$_SESSION['last_activity'] = time();

// 4. Configuration php.ini recommandée (ou ini_set)
ini_set('session.cookie_httponly', 1);     // Cookie inaccessible au JS
ini_set('session.cookie_secure', 1);       // HTTPS uniquement
ini_set('session.cookie_samesite', 'Strict'); // Anti-CSRF
ini_set('session.use_only_cookies', 1);    // Pas d'ID dans l'URL
ini_set('session.gc_maxlifetime', 3600);   // Durée de vie session
?>

Paramètres avancés des cookies

Depuis PHP 7.3, on peut passer un tableau d'options à setcookie() pour un contrôle plus fin.

PHP — Options avancées des cookies 
<?php
// Syntaxe moderne (PHP 7.3+) avec tableau d'options
setcookie('session_token', 'valeur', [
    'expires'  => time() + 86400,
    'path'     => '/',
    'domain'   => 'monsite.com',
    'secure'   => true,           // HTTPS seulement
    'httponly' => true,           // pas d'accès JavaScript
    'samesite' => 'Lax',          // 'Strict' | 'Lax' | 'None'
]);

// SameSite explication :
// - Strict : cookie jamais envoyé pour les requêtes intersites (max sécurité)
// - Lax : cookie envoyé pour les navigations top-level (GET) — bon équilibre
// - None : toujours envoyé (nécessite secure=true)

// Lecture des paramètres de session
$params = session_get_cookie_params();
echo "<pre>";
print_r($params);
echo "</pre>";
// Array (
//   [lifetime] => 0
//   [path] => /
//   [domain] =>
//   [secure] =>
//   [httponly] =>
//   [samesite] =>
// )
?>

Exercices pratiques

EX 01 ✏️ Système d'authentification
  1. Créez login.php avec un formulaire (email, mot de passe).
  2. Simulez un utilisateur avec un tableau : $users = ["admin@exemple.com" => password_hash("1234", PASSWORD_DEFAULT)];
  3. Vérifiez le mot de passe avec password_verify().
  4. Connectez l'utilisateur et redirigez vers dashboard.php.
  5. Sur dashboard, affichez "Bienvenue, [email]" et un bouton "Déconnexion".
  6. Créez logout.php qui détruit la session et redirige vers login.
  7. Protégez dashboard.php : si pas connecté → redirection vers login.
EX 02 🍪 Préférences utilisateur
  1. Créez une page preferences.php avec un formulaire (theme: light/dark, langue: fr/en).
  2. Sauvegardez les choix en cookies avec une expiration de 30 jours.
  3. Appliquez le thème choisi sur la page (si dark → fond sombre).
  4. Ajoutez un flash message "Préférences sauvegardées !" après l'envoi.
  5. Ajoutez un lien "Réinitialiser" qui supprime les cookies.

Bilan & glossaire

Ce que vous maîtrisez

  • session_start() — démarrer/reprendre une session
  • $_SESSION — stocker et lire des données entre les pages
  • Système login/logout avec vérification
  • Flash messages pour les notifications
  • setcookie() — créer, lire et supprimer des cookies
  • Sécurité : session_regenerate_id, timeout, SameSite
  • Options avancées : httponly, secure, samesite

Mini-glossaire

Session
Stockage côté serveur de données utilisateur persistantes entre les pages.
PHPSESSID
Cookie contenant l'identifiant unique de session.
session_start
Fonction qui démarre ou reprend une session. Doit être avant tout HTML.
$_SESSION
Tableau superglobal accessible sur toutes les pages après session_start().
session_destroy
Détruit toutes les données de la session courante.
Cookie
Petit fichier stocké côté navigateur, envoyé avec chaque requête.
setcookie
Fonction PHP qui crée/modifie/supprime un cookie.
Flash message
Message stocké en session, affiché une seule fois puis supprimé.
httponly
Attribut de cookie qui le rend inaccessible au JavaScript (anti-XSS).
SameSite
Attribut qui contrôle quand le cookie est envoyé (anti-CSRF).
