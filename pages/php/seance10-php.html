Classes : class, new, public

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

La Programmation Orientée Objet (POO) organise le code en classes (plans) et objets (instances). Une classe définit des propriétés et des méthodes.

PHP — Classe et instanciation 
<?php
// Définition d'une classe
class Utilisateur
{
    // Propriétés (avec visibility)
    public string $nom;
    public string $email;
    private string $motDePasse;
    protected int $age;

    // Méthode
    public function sePresenter(): string
    {
        return "Je m'appelle {$this->nom}.";
    }

    public function verifierMotDePasse(string $mdp): bool
    {
        return password_verify($mdp, $this->motDePasse);
    }
}

// Instanciation (création d'un objet)
$alice = new Utilisateur();
$alice->nom = "Alice";
$alice->email = "alice@exemple.com";
echo $alice->sePresenter();   // Je m'appelle Alice.

// Visibilité :
// public    → accessible partout
// protected → accessible dans la classe et les classes filles
// private   → accessible uniquement dans la classe

// Typage des propriétés (PHP 7.4+)
class Produit {
    public string $nom;
    public float $prix;
    public ?string $description = null;   // nullable avec valeur par défaut
    public int $stock = 0;
}
?>
02

Constructeur (__construct)

Le constructeur est une méthode spéciale appelée automatiquement lors de l'instanciation avec new. Il sert à initialiser l'objet.

PHP — Constructeur 
<?php
class Article
{
    public function __construct(
        public string $titre,
        public string $contenu,
        public ?Utilisateur $auteur = null,
        public DateTime $createdAt = new DateTime()
    ) {
        // Le corps peut être vide — PHP 8+ gère l'assignation automatiquement
    }

    public function getExcerpt(int $longueur = 100): string
    {
        return substr($this->contenu, 0, $longueur) . '...';
    }
}

// Utilisation — le constructeur est appelé automatiquement
$article = new Article(
    titre: "Introduction à PHP 8",
    contenu: "PHP 8 a introduit de nombreuses fonctionnalités..."
);

echo $article->titre;         // Introduction à PHP 8
echo $article->getExcerpt(); // Premiers 100 caractères...

// Promotion du constructeur (PHP 8+)
// Les propriétés sont déclarées DIRECTEMENT dans les paramètres du constructeur
// Évite la redondance : propriété + paramètre + $this->propriete = $propriete
?>
💡
Promotion du constructeur (PHP 8+)

Depuis PHP 8, vous pouvez déclarer et initialiser les propriétés directement dans les paramètres du constructeur. Le code public string $titre dans les parenthèses crée automatiquement la propriété et l'assigne. C'est la syntaxe moderne à adopter.

Namespaces et use

Les namespaces évitent les collisions de noms entre classes. Ils organisent le code comme des dossiers virtuels.

PHP — Namespaces 
<?php
// src/Models/Utilisateur.php
namespace App\Models;

class Utilisateur {
    public function __construct(
        public string $nom,
        public string $email
    ) {}
}

// src/Controllers/UtilisateurController.php
namespace App\Controllers;

use App\Models\Utilisateur;   // Importe la classe depuis son namespace
use PDO;                       // Les classes natives n'ont pas besoin de use

class UtilisateurController
{
    public function __construct(
        private PDO $pdo
    ) {}

    public function liste(): array
    {
        $stmt = $this->pdo->query('SELECT * FROM utilisateurs');
        $utilisateurs = [];

        while ($row = $stmt->fetch()) {
            $utilisateurs[] = new Utilisateur($row['nom'], $row['email']);
        }

        return $utilisateurs;
    }
}

// Alias avec "as"
use App\Models\Utilisateur as User;
$user = new User("Alice", "alice@exemple.com");

// Espace global (classe sans namespace)
$pdo = new \PDO(...);          // le \ force l'espace global
?>

Composer & autoload

Composer est le gestionnaire de dépendances PHP. Il permet d'installer des bibliothèques et d'automatiser le chargement des classes.

JSON — composer.json 
{
    "name": "mon-projet/blog",
    "description": "Blog en PHP",
    "type": "project",
    "require": {
        "php": ">=8.1",
        "vlucas/phpdotenv": "^5.5",
        "symfony/var-dumper": "^6.0"
    },
    "require-dev": {
        "phpunit/phpunit": "^10.0"
    },
    "autoload": {
        "psr-4": {
            "App\\": "src/"
        }
    }
}

// Commandes
// composer install       → installe les dépendances
// composer require XXX   → ajoute une dépendance
// composer dump-autoload → régénère l'autoload
PHP — Autoload PSR-4 
<?php
// public/index.php — Point d'entrée
require_once __DIR__ . '/../vendor/autoload.php';

// Grâce à l'autoload PSR-4, toutes les classes sont chargées automatiquement
// selon leur namespace et leur emplacement dans src/

use App\Controllers\ArticleController;
use App\Models\Article;

// Structure du projet :
// mon-projet/
// ├── public/
// │   └── index.php        ← point d'entrée
// ├── src/
// │   ├── Controllers/
// │   │   └── ArticleController.php   ← namespace App\Controllers
// │   ├── Models/
// │   │   └── Article.php             ← namespace App\Models
// │   └── Views/
// │       └── articles/
// │           └── index.php
// ├── vendor/               ← généré par Composer
// ├── composer.json
// └── .env                  ← configuration

// Chargement de l'environnement
$dotenv = Dotenv\Dotenv::createImmutable(__DIR__ . '/..');
$dotenv->load();

// Utilisation des variables d'environnement
$dbHost = $_ENV['DB_HOST'];
$dbName = $_ENV['DB_NAME'];
?>
📌
PSR-4 Autoloading

Le namespace App\ correspond au dossier src/. Une classe App\Models\Article sera chargée depuis src/Models/Article.php. Le nom du fichier doit correspondre exactement au nom de la classe.

Architecture MVC

MVC (Model-View-Controller) est un patron d'architecture qui sépare le code en trois couches.

🏗️
MVC expliqué simplement

Model = les données (BDD, requêtes). View = l'affichage (HTML, templates). Controller = le chef d'orchestre qui reçoit la requête, interroge le Model, et passe les données à la View.

PHP — Exemple MVC 
<?php
// --- Model : Article.php ---
namespace App\Models;

use PDO;

class Article
{
    public function __construct(private PDO $pdo) {}

    public function findAll(): array
    {
        $stmt = $this->pdo->query('SELECT * FROM articles ORDER BY created_at DESC');
        return $stmt->fetchAll();
    }

    public function findById(int $id): ?array
    {
        $stmt = $this->pdo->prepare('SELECT * FROM articles WHERE id = :id');
        $stmt->execute(['id' => $id]);
        return $stmt->fetch() ?: null;
    }

    public function create(array $data): int
    {
        $stmt = $this->pdo->prepare('
            INSERT INTO articles (titre, contenu, auteur_id)
            VALUES (:titre, :contenu, :auteur_id)
        ');
        $stmt->execute($data);
        return (int)$this->pdo->lastInsertId();
    }
}

// --- Controller : ArticleController.php ---
namespace App\Controllers;

use App\Models\Article;

class ArticleController
{
    public function __construct(private Article $articleModel) {}

    public function index(): void
    {
        $articles = $this->articleModel->findAll();
        require_once __DIR__ . '/../Views/articles/index.php';
    }

    public function show(int $id): void
    {
        $article = $this->articleModel->findById($id);
        if (!$article) {
            http_response_code(404);
            require_once __DIR__ . '/../Views/404.php';
            return;
        }
        require_once __DIR__ . '/../Views/articles/show.php';
    }
}

// --- View : articles/index.php ---
// $articles est disponible depuis le contrôleur
?>
<h1>Blog</h1>
<?php foreach ($articles as $article): ?>
    <article>
        <h2><?= htmlspecialchars($article['titre']) ?></h2>
        <p><?= htmlspecialchars(substr($article['contenu'], 0, 200)) ?>...</p>
    </article>
<?php endforeach; ?>

Bonnes pratiques PSR

Les PSR (PHP Standards Recommendations) sont des recommandations officielles pour standardiser le code PHP.

PSRSujetRésumé
PSR-1Codage de baseBalises <?php, encodage UTF-8, namespaces, noms de classes en PascalCase
PSR-12Style de codageIndentation 4 espaces, accolades sur nouvelle ligne, espaces autour des opérateurs
PSR-4AutoloadingCorrespondance namespace → dossier. Standard de facto.
PSR-3LoggingInterface LoggerInterface pour les logs (Monolog, etc.)
PSR-7HTTP MessagesInterfaces pour requêtes/réponses HTTP
PSR-11ContainerInterface pour conteneur de dépendances
PHP — Exemple PSR-12 
<?php

declare(strict_types=1);

namespace App\Services;

use App\Models\Article;
use RuntimeException;

class ArticleService
{
    public function __construct(
        private Article $article
    ) {}

    public function publish(int $id): bool
    {
        $article = $this->article->findById($id);

        if ($article === null) {
            throw new RuntimeException('Article not found');
        }

        return $this->article->update($id, ['status' => 'published']);
    }
}
?>

Tests avec PHPUnit

PHPUnit est le framework de test standard pour PHP. Les tests sont placés dans un dossier tests/ et suivent le namespace Tests\.

PHP — Tests unitaires PHPUnit 
<?php
// tests/Models/ArticleTest.php
namespace Tests\Models;

use PHPUnit\Framework\TestCase;

class ArticleTest extends TestCase
{
    public function testTitrePeutEtreDefini(): void
    {
        $article = new \App\Models\Article(
            titre: "Mon article",
            contenu: "Contenu de test"
        );

        $this->assertEquals("Mon article", $article->titre);
        $this->assertNotEmpty($article->contenu);
    }

    public function testGetExcerpt(): void
    {
        $article = new \App\Models\Article(
            titre: "Test",
            contenu: "Lorem ipsum dolor sit amet consectetur adipiscing elit"
        );

        $this->assertEquals(
            "Lorem ipsum dolor sit amet consectetur adipiscing elit",
            $article->getExcerpt(50)
        );
        $this->assertStringEndsWith('...', $article->getExcerpt(10));
    }

    // Data provider — test avec plusieurs jeux de données
    public function prixProvider(): array
    {
        return [
            [100, 0.20, 120],
            [50,  0.10, 55],
            [200, 0.0,  200],
        ];
    }

    /** @dataProvider prixProvider */
    public function testCalculTTC($ht, $tva, $attendu): void
    {
        $resultat = $ht * (1 + $tva);
        $this->assertEquals($attendu, $resultat);
    }
}

// Lancer les tests :
// vendor/bin/phpunit tests/
// vendor/bin/phpunit tests/Models/ArticleTest.php
// vendor/bin/phpunit --filter testGetExcerpt
?>

Déploiement (.env, hébergement)

PHP — Déploiement et configuration 
<?php
// .env (NE JAMAIS COMMITER)
DB_HOST=localhost
DB_NAME=blog_prod
DB_USER=prod_user
DB_PASS=Mot2P@sseS3curise
APP_ENV=production
APP_DEBUG=false

// .env.example (version sans secrets — à commiter)
DB_HOST=localhost
DB_NAME=blog
DB_USER=root
DB_PASS=
APP_ENV=development
APP_DEBUG=true

// Chargement avec phpdotenv (vlucas/phpdotenv)
$dotenv = Dotenv\Dotenv::createImmutable(__DIR__);
$dotenv->load();
$dotenv->required(['DB_HOST', 'DB_NAME', 'DB_USER', 'DB_PASS']);

// Mode production
if ($_ENV['APP_ENV'] === 'production') {
    error_reporting(0);
    ini_set('display_errors', '0');
    ini_set('log_errors', '1');
} else {
    error_reporting(E_ALL);
    ini_set('display_errors', '1');
}

// Vérifications de déploiement
$checks = [
    'PHP >= 8.1'    => version_compare(PHP_VERSION, '8.1.0', '>='),
    'Extension PDO'  => extension_loaded('pdo_mysql'),
    'Extension GD'   => extension_loaded('gd'),
    'Extension JSON' => extension_loaded('json'),
    'Extension cURL' => extension_loaded('curl'),
];

foreach ($checks as $name => $ok) {
    echo "$name : " . ($ok ? '✓' : '✗') . "\n";
}

// Hébergements PHP populaires :
// - Platform.sh (PaaS professionnel)
// - Alwaysdata, OVH, Infomaniak (hébergement mutualisé)
// - AWS Elastic Beanstalk, Heroku (cloud)
// - VPS (DigitalOcean, Linode) avec Forge/Ploi
?>
⚠️
Sécurité en production

Ne jamais commiter le fichier .env (ajoutez-le à .gitignore). Désactivez display_errors en production. Utilisez HTTPS partout. Mettez régulièrement à jour vos dépendances Composer.

Exercices pratiques

EX 01 ✏️ Mini-framework MVC
  1. Structurez votre blog avec l'architecture MVC : public/, src/Controllers/, src/Models/, src/Views/.
  2. Créez la classe App\Models\Article avec les méthodes findAll, findById, create, update, delete.
  3. Créez la classe App\Controllers\ArticleController avec index, show, create, edit, delete.
  4. Créez un routeur simple qui analyse $_GET['page'] ou $_SERVER['REQUEST_URI'].
  5. Ajoutez composer.json avec l'autoload PSR-4 et installez phpdotenv.
  6. Déplacez la config BDD dans .env.
EX 02 🧪 Tests et finalisation
  1. Installez PHPUnit avec Composer.
  2. Créez tests/Models/ArticleTest.php avec des tests pour votre modèle Article.
  3. Testez la création, la récupération et la mise à jour d'un article.
  4. Ajoutez un test avec dataProvider pour valider différentes valeurs.
  5. Vérifiez que tous les fichiers suivent PSR-12 (outil : vendor/bin/phpcs).
  6. Préparez un fichier .env.example et un .gitignore pour le projet.
  7. Félicitations — vous avez terminé le cours PHP ! 🎉
