01

Déclaration d'une fonction

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

Une fonction est un bloc de code réutilisable auquel on donne un nom. En PHP, on utilise le mot-clé function.

PHP — Déclaration d'une fonction 
<?php
// Déclaration
function direBonjour() {
    echo "Bonjour tout le monde !<br>";
}

// Appel
direBonjour();   // Bonjour tout le monde !
direBonjour();   // On peut l'appeler plusieurs fois

// Règles de nommage
function calculerMoyenne() {}   // camelCase — convention PHP
function calculer_moyenne() {}  // snake_case — acceptable
// Les noms sont insensibles à la casse !
// Mais appelez-les toujours comme définies.

// Une fonction peut être déclarée APRÈS son appel
direBonjour();   // fonctionne — PHP lit tout le fichier avant d'exécuter
function direBonjour() {
    echo "Bonjour !";
}
?>
📌
PHP "hoiste" les fonctions

Contrairement aux variables, les fonctions déclarées avec function peuvent être appelées avant leur déclaration dans le fichier. PHP charge toutes les définitions avant d'exécuter le code.

02

Paramètres & return

Une fonction peut recevoir des données (paramètres) et en retourner (return). Dès que return est atteint, la fonction s'arrête.

PHP — Paramètres et return 
<?php
// Fonction avec paramètres
function addition($a, $b) {
    return $a + $b;
}

echo addition(3, 4);  // 7

// return stoppe l'exécution
function test($n) {
    if ($n < 0) {
        return "Nombre négatif";   // sort immédiate
    }
    return "Nombre positif ou nul";
}

// return peut retourner n'importe quel type
function getUtilisateur() {
    return ["nom" => "Alice", "age" => 30];
}
$user = getUtilisateur();

// Valeur de retour multiple (via tableau)
function getCoordonnees() {
    return ["x" => 10, "y" => 20];
}
$coord = getCoordonnees();
echo $coord["x"];   // 10

// Fonction qui n'a pas de return → retourne null
function rien() {}
var_dump(rien());   // NULL
?>

03

Valeurs par défaut

Un paramètre peut avoir une valeur par défaut. Si l'appel ne fournit pas ce paramètre, la valeur par défaut est utilisée.

PHP — Paramètres par défaut 
<?php
function saluer($nom, $salutation = "Bonjour") {
    return "$salutation, $nom !";
}

echo saluer("Alice");           // Bonjour, Alice !
echo saluer("Bob", "Salut");    // Salut, Bob !
echo saluer("Bob", "Bonsoir");  // Bonsoir, Bob !

// Règle : les paramètres par défaut doivent être APRÈS les obligatoires
// ❌ function mauvais($a = 1, $b)  → erreur
// ✅ function bon($a, $b = 1)      → correct

// Paramètres optionnels (null par défaut)
function afficherProduit($nom, $prix = null) {
    if ($prix === null) {
        return "Produit : $nom (prix non communiqué)";
    }
    return "Produit : $nom — $prix €";
}

// Arguments nommés (PHP 8+)
function creerProfil($nom, $age, $ville) {
    return "$nom, $age ans, $ville";
}

echo creerProfil(ville: "Paris", age: 30, nom: "Alice");
// Les arguments nommés permettent l'ordre flexible
?>
💡
Arguments nommés (PHP 8+)

Les arguments nommés rendent le code plus lisible et permettent d'ignorer des paramètres optionnels au milieu.

══ 04 TYPE HINTING ══ -->
04

Type hinting

PHP permet d'indiquer le type attendu pour chaque paramètre et la valeur de retour. Cela rend le code plus fiable et auto-documenté.

PHP — Type hinting 
<?php
declare(strict_types=1);  // Active le mode strict pour CE fichier

// Types sur les paramètres
function calculerPrixTTC(float $prixHT, float $tva = 0.20): float {
    return $prixHT * (1 + $tva);
}

echo calculerPrixTTC(100.0, 0.10);  // 110

// Types sur le retour
function getNom(): string {
    return "Alice";
}

// Union types (PHP 8+)
function setAge(int|string $age): void {
    echo "Âge : $age";
}
setAge(30);
setAge("trente");

// void — pas de retour
function logger(string $message): void {
    file_put_contents("log.txt", $message);
}

// Mixed — n'importe quel type
function debug(mixed $valeur): void {
    var_dump($valeur);
}

// ?type — peut être le type ou null
function getPrix(): ?float {
    return rand(0, 1) ? 19.99 : null;
}

// Types pour tableaux
function somme(array $nombres): int|float {
    return array_sum($nombres);
}
?>
⚠️
strict_types=1

Sans declare(strict_types=1), PHP convertit automatiquement les types (ex: un string "42" passé à un paramètre int sera accepté). Avec le mode strict, une erreur TypeError sera levée.

05

Portée des variables

Les variables en PHP ont une portée (scope) qui détermine où elles sont accessibles. Les variables définies en dehors d'une fonction ne sont pas accessibles à l'intérieur.

PHP — Portée : global, local, static 
<?php
$globale = "Je suis globale";

function test() {
    // ❌ Ne peut pas accéder à $globale
    // echo $globale;   // Notice : undefined variable

    $locale = "Je suis locale";
    echo $locale;   // OK — définie dans la fonction
}

test();
// echo $locale;   // ❌ Erreur — $locale n'existe pas ici

// Mot-clé global
$compteur = 0;

function incrementer() {
    global $compteur;   // importe la variable globale
    $compteur++;
}
incrementer();
echo $compteur;     // 1

// $GLOBALS — tableau de toutes les variables globales
function incrementer2() {
    $GLOBALS['compteur']++;
}

// static — variable qui persiste entre les appels
function compteurAppels() {
    static $count = 0;
    $count++;
    echo "Appel n° $count<br>";
}

compteurAppels();   // Appel n° 1
compteurAppels();   // Appel n° 2
compteurAppels();   // Appel n° 3
?>

06

Fonctions variables et closures

En PHP, les fonctions peuvent être stockées dans des variables (fonctions anonymes) et utilisées comme des valeurs. Une closure est une fonction anonyme qui capture des variables de son environnement.

PHP — Closures et fonctions anonymes 
<?php
// Fonction anonyme dans une variable
$direBonjour = function($nom) {
    return "Bonjour, $nom !";
};
echo $direBonjour("Alice");   // Bonjour, Alice !

// Closure avec use — capture une variable externe
$prefixe = "Msg: ";
$afficher = function($message) use ($prefixe) {
    echo $prefixe . $message;
};
$afficher("Hello");   // Msg: Hello

// Les closures sont typées Closure
$fn = function() {};
var_dump($fn instanceof Closure);   // true

// Passer une closure à array_map
$nombres = [1, 2, 3, 4, 5];
$carres = array_map(function($n) {
    return $n * $n;
}, $nombres);
print_r($carres);   // [1, 4, 9, 16, 25]

// Closure avec référence (&)
$total = 0;
$ajouter = function($valeur) use (&$total) {
    $total += $valeur;
};
$ajouter(10);
$ajouter(5);
echo $total;   // 15
?>

07

Fonctions fléchées fn()

Introduites en PHP 7.4, les fonctions fléchées fn() => sont une syntaxe plus courte pour les closures simples. Elles captent automatiquement les variables par valeur.

PHP — Fonctions fléchées 
<?php
$nombres = [1, 2, 3, 4, 5];

// Version closure classique
$carres = array_map(function($n) { return $n * $n; }, $nombres);

// Version fléchée (même chose)
$carres = array_map(fn($n) => $n * $n, $nombres);

// Capture automatique — pas besoin de "use"
$facteur = 2;
$doubles = array_map(fn($n) => $n * $facteur, $nombres);
// $facteur est capturé automatiquement

// Limitation : une seule expression (pas de bloc)
// fn($a, $b) => $a + $b   ✓
// fn($a, $b) => { ... }   ✗ (pas de corps multi-lignes)

// Usage avec array_filter
$pairs = array_filter($nombres, fn($n) => $n % 2 == 0);
print_r($pairs);   // [2, 4]
?>
💡
fn() vs function()

fn() => est plus court, capture automatiquement les variables (par valeur), mais ne peut contenir qu'une seule expression. Utilisez function() pour la logique complexe.

08

Include / require

PHP permet d'inclure le contenu d'un fichier dans un autre. Essentiel pour réutiliser du code (header, footer, fonctions).

InstructionComportement si fichier manquant
include 'fichier.php';Avertissement + continue (E_WARNING)
require 'fichier.php';Erreur fatale + arrêt du script (E_COMPILE_ERROR)
include_once 'fichier.php';Inclusion unique (évite les doublons)
require_once 'fichier.php';Require unique — le plus utilisé
PHP — Organisation des fichiers 
<!-- index.php -->
<?php
require_once 'config.php';
require_once 'functions.php';
require_once 'header.php';
?>

<h1>Contenu de la page</h1>

<?php require_once 'footer.php'; ?>

<!-- config.php -->
<?php
const DB_HOST = 'localhost';
const DB_NAME = 'site';

// functions.php
function nettoyer(string $data): string {
    return htmlspecialchars(strip_tags(trim($data)));
}

// header.php — HTML pur (sorti de PHP)
?>
<!DOCTYPE html>
<html>
<head>
    <title>Mon Site</title>
</head>
<body>
    <nav>...</nav>
⚠️
include vs require — quand utiliser quoi ?

Utilisez require_once pour les fichiers essentiels (config, fonctions, classes). Utilisez include pour les fichiers optionnels (widgets, templates). _once évite les inclusions multiples qui causeraient des erreurs de redéfinition.

Bilan & glossaire

Ce que vous maîtrisez

  • Déclaration et appel de fonctions avec function
  • Paramètres, return, valeurs par défaut, arguments nommés
  • Type hinting avec types scalaires, array, void, mixed, union
  • Portée : globale, locale, static, mot-clé global
  • Fonctions anonymes, closures avec use, fonctions fléchées fn()
  • Include, require, include_once, require_once

Mini-glossaire

Fonction
Bloc de code réutilisable nommé. Déclaré avec function.
Paramètre
Variable reçue par une fonction, définie dans ses parenthèses.
return
Mot-clé qui stoppe la fonction et retourne une valeur.
Type hinting
Indication du type attendu pour un paramètre ou la valeur de retour.
Portée (scope)
Zone du code où une variable est accessible.
static
Variable qui persiste entre les appels d'une fonction.
Closure
Fonction anonyme qui capture des variables avec use.
Fonction fléchée
fn() => — syntaxe courte pour une closure à une expression.
require_once
Inclusion unique et obligatoire d'un fichier PHP.
Argument nommé
PHP 8+ — passer un argument en spécifiant son nom : nom: "Alice".
