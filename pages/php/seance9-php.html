MySQL & phpMyAdmin

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

MySQL est un système de gestion de base de données relationnelle (SGBDR). Avec XAMPP, vous avez :

  • MySQL (le serveur de base de données) — accessible en ligne de commande
  • phpMyAdmin — interface web pour administrer MySQL
💡
Accéder à phpMyAdmin

Lancez Apache et MySQL dans XAMPP, puis ouvrez http://localhost/phpmyadmin/. Vous pouvez créer des bases, des tables, et exécuter des requêtes SQL directement depuis l'interface.

Concepts clés

TermeDéfinition
Base de donnéesConteneur de tables (ex: blog, ecommerce)
TableStructure organisée en lignes et colonnes (ex: utilisateurs)
Colonne (champ)Propriété d'un enregistrement (ex: nom, email)
Ligne (enregistrement)Un ensemble de valeurs (ex: un utilisateur)
Clé primaire (PK)Identifiant unique d'une ligne (id auto-incrémenté)
Clé étrangère (FK)Lien vers une clé primaire d'une autre table

02

Créer une table avec SQL

SQL — Création de tables 
-- Créer la base
CREATE DATABASE blog CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

-- Table utilisateurs
CREATE TABLE utilisateurs (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(100) NOT NULL,
    email VARCHAR(255) NOT NULL UNIQUE,
    mot_de_passe VARCHAR(255) NOT NULL,
    age INT DEFAULT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Table articles (avec clé étrangère)
CREATE TABLE articles (
    id INT AUTO_INCREMENT PRIMARY KEY,
    titre VARCHAR(200) NOT NULL,
    contenu TEXT NOT NULL,
    auteur_id INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (auteur_id) REFERENCES utilisateurs(id)
        ON DELETE CASCADE
);

-- Table categories
CREATE TABLE categories (
    id INT AUTO_INCREMENT PRIMARY KEY,
    nom VARCHAR(100) NOT NULL,
    description TEXT
);

-- Table pivot article_categorie (relation N:N)
CREATE TABLE article_categorie (
    article_id INT NOT NULL,
    categorie_id INT NOT NULL,
    PRIMARY KEY (article_id, categorie_id),
    FOREIGN KEY (article_id) REFERENCES articles(id) ON DELETE CASCADE,
    FOREIGN KEY (categorie_id) REFERENCES categories(id) ON DELETE CASCADE
);
⚠️
utf8mb4 — pour les emojis

Utilisez utf8mb4 plutôt que utf8 qui ne supporte pas les caractères sur 4 octets (emojis, certains caractères asiatiques). utf8_general_ci est plus rapide, utf8_unicode_ci plus précis.

03

Connexion PDO

PDO (PHP Data Objects) est l'extension recommandée pour accéder aux bases de données en PHP. Elle supporte plusieurs SGBD (MySQL, PostgreSQL, SQLite…) et offre des requêtes préparées sécurisées.

PHP — Connexion PDO (config) 
<?php
// config.php
const DB_HOST = 'localhost';
const DB_NAME = 'blog';
const DB_USER = 'root';
const DB_PASS = '';    // pas de mot de passe par défaut sous XAMPP
const DB_CHARSET = 'utf8mb4';

// DSN (Data Source Name)
$dsn = "mysql:host=" . DB_HOST . ";dbname=" . DB_NAME . ";charset=" . DB_CHARSET;

// Options PDO
$options = [
    PDO::ATTR_ERRMODE            => PDO::ERRMODE_EXCEPTION,
    PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
    PDO::ATTR_EMULATE_PREPARES   => false,
];

try {
    $pdo = new PDO($dsn, DB_USER, DB_PASS, $options);
} catch (PDOException $e) {
    // En production, loggez l'erreur sans l'afficher
    error_log("Erreur PDO : " . $e->getMessage());
    die("Erreur de connexion à la base de données.");
}

// Dans chaque fichier qui a besoin de la BDD :
require_once 'config.php';
// $pdo est disponible
?>
💡
Options PDO expliquées

ERRMODE_EXCEPTION : les erreurs SQL lancent des exceptions. FETCH_ASSOC : les résultats sont des tableaux associatifs. EMULATE_PREPARES = false : utilise les vraies requêtes préparées MySQL (sécurité + performance).

04

query() et fetch / fetchAll

PHP — Lire des données avec PDO 
<?php
require 'config.php';

// query() — pour les requêtes SANS paramètres dynamiques
$stmt = $pdo->query('SELECT * FROM utilisateurs');

// fetch() — une seule ligne
$utilisateur = $stmt->fetch();
echo $utilisateur['nom'] . ' - ' . $utilisateur['email'];

// fetchAll() — toutes les lignes d'un coup
$stmt = $pdo->query('SELECT * FROM utilisateurs');
$utilisateurs = $stmt->fetchAll();

foreach ($utilisateurs as $user) {
    echo $user['nom'] . '<br>';
}

// fetch column — une seule colonne
$stmt = $pdo->query('SELECT email FROM utilisateurs');
$emails = $stmt->fetchAll(PDO::FETCH_COLUMN);

// fetch key-value (première colonne = clé, seconde = valeur)
$stmt = $pdo->query('SELECT id, nom FROM utilisateurs');
$liste = $stmt->fetchAll(PDO::FETCH_KEY_PAIR);

// Nombre de lignes
$stmt = $pdo->query('SELECT COUNT(*) FROM utilisateurs');
$count = $stmt->fetchColumn();
echo "Nombre d'utilisateurs : $count";
?>

05

Requêtes préparées (prepare/execute)

Les requêtes préparées sont obligatoires dès qu'une valeur provient de l'utilisateur. Elles empêchent les attaques par injection SQL.

PHP — Requêtes préparées PDO 
<?php
require 'config.php';

// Avec marqueurs nominatifs (:nom)
$email = $_POST['email'] ?? '';

$stmt = $pdo->prepare('SELECT * FROM utilisateurs WHERE email = :email');
$stmt->execute(['email' => $email]);
$user = $stmt->fetch();

// Avec marqueurs positionnels (?)
$stmt = $pdo->prepare('SELECT * FROM utilisateurs WHERE id = ?');
$stmt->execute([$id]);
$user = $stmt->fetch();

// Plusieurs paramètres
$stmt = $pdo->prepare('
    SELECT * FROM articles
    WHERE auteur_id = :auteur
    AND created_at > :date
    ORDER BY created_at DESC
    LIMIT 10
');
$stmt->execute([
    'auteur' => 1,
    'date'   => '2024-01-01'
]);
$articles = $stmt->fetchAll();

// LIKE avec requête préparée
$recherche = '%' . $_GET['q'] . '%';
$stmt = $pdo->prepare('SELECT * FROM articles WHERE titre LIKE :q');
$stmt->execute(['q' => $recherche]);

// IN avec tableau
$ids = [1, 2, 3];
$placeholders = implode(',', array_fill(0, count($ids), '?'));
$stmt = $pdo->prepare("SELECT * FROM articles WHERE id IN ($placeholders)");
$stmt->execute($ids);
?>
⚠️
Jamais de variables dans les requêtes SQL brutes

$pdo->query("SELECT * FROM users WHERE id = $id") = ❌ injection SQL ! Utilisez toujours prepare/execute avec des marqueurs.

CRUD — Insert

PHP — Créer un enregistrement 
<?php
require 'config.php';

// Insertion simple
$stmt = $pdo->prepare('
    INSERT INTO utilisateurs (nom, email, mot_de_passe)
    VALUES (:nom, :email, :mdp)
');
$stmt->execute([
    'nom'   => 'Alice Dupont',
    'email' => 'alice@exemple.com',
    'mdp'   => password_hash('secret123', PASSWORD_DEFAULT)
]);

// Récupérer l'ID inséré
$nouvelId = $pdo->lastInsertId();
echo "Nouvel utilisateur n°$nouvelId";

// Insertion multiple (batch)
$stmt = $pdo->prepare('INSERT INTO categories (nom) VALUES (:nom)');
$categories = ['PHP', 'MySQL', 'JavaScript', 'CSS'];
foreach ($categories as $cat) {
    $stmt->execute(['nom' => $cat]);
}

// Insertion avec gestion des doublons (ON DUPLICATE KEY)
$stmt = $pdo->prepare('
    INSERT INTO utilisateurs (email, nom)
    VALUES (:email, :nom)
    ON DUPLICATE KEY UPDATE nom = VALUES(nom)
');
?>

CRUD — Select / Update / Delete

PHP — Read, Update, Delete 
<?php
require 'config.php';

// --- SELECT (read) ---
function getUtilisateur(PDO $pdo, int $id): ?array {
    $stmt = $pdo->prepare('SELECT * FROM utilisateurs WHERE id = :id');
    $stmt->execute(['id' => $id]);
    $user = $stmt->fetch();
    return $user ?: null;   // null si pas trouvé
}

function getArticles(PDO $pdo, int $limit = 10): array {
    $stmt = $pdo->prepare('
        SELECT a.*, u.nom AS auteur_nom
        FROM articles a
        JOIN utilisateurs u ON a.auteur_id = u.id
        ORDER BY a.created_at DESC
        LIMIT :limit
    ');
    $stmt->bindValue('limit', $limit, PDO::PARAM_INT);
    $stmt->execute();
    return $stmt->fetchAll();
}

// --- UPDATE ---
function updateUtilisateur(PDO $pdo, int $id, string $nom, string $email): bool {
    $stmt = $pdo->prepare('
        UPDATE utilisateurs
        SET nom = :nom, email = :email
        WHERE id = :id
    ');
    return $stmt->execute([
        'id'    => $id,
        'nom'   => $nom,
        'email' => $email
    ]);
}

// --- DELETE ---
function deleteArticle(PDO $pdo, int $id): bool {
    $stmt = $pdo->prepare('DELETE FROM articles WHERE id = :id');
    return $stmt->execute(['id' => $id]);
}
?>

Jointures INNER JOIN

Les jointures permettent de combiner des données de plusieurs tables en une seule requête.

SQL — Types de jointures 
-- INNER JOIN : uniquement les articles qui ont un auteur
SELECT a.titre, a.contenu, u.nom AS auteur
FROM articles a
INNER JOIN utilisateurs u ON a.auteur_id = u.id;

-- LEFT JOIN : tous les articles, même sans catégorie
SELECT a.titre, c.nom AS categorie
FROM articles a
LEFT JOIN article_categorie ac ON a.id = ac.article_id
LEFT JOIN categories c ON ac.categorie_id = c.id;

-- Jointure avec plusieurs tables
SELECT a.titre, u.nom AS auteur, GROUP_CONCAT(c.nom SEPARATOR ', ') AS categories
FROM articles a
JOIN utilisateurs u ON a.auteur_id = u.id
LEFT JOIN article_categorie ac ON a.id = ac.article_id
LEFT JOIN categories c ON ac.categorie_id = c.id
GROUP BY a.id
ORDER BY a.created_at DESC;
PHP — Jointures en action 
<?php
require 'config.php';

// Articles avec leur auteur
$stmt = $pdo->query('
    SELECT a.id, a.titre, u.nom AS auteur, a.created_at
    FROM articles a
    INNER JOIN utilisateurs u ON a.auteur_id = u.id
    ORDER BY a.created_at DESC
');
$articlesAvecAuteur = $stmt->fetchAll();

// Articles avec leurs catégories (relation N:N)
$stmt = $pdo->prepare('
    SELECT a.*, GROUP_CONCAT(c.nom SEPARATOR ", ") AS categories
    FROM articles a
    LEFT JOIN article_categorie ac ON a.id = ac.article_id
    LEFT JOIN categories c ON ac.categorie_id = c.id
    WHERE a.id = :id

Exercices pratiques

EX 01 ✏️ Blog — CRUD articles
  1. Créez la base "blog" et les tables (utilisateurs, articles, categories, article_categorie).
  2. Créez config.php avec la connexion PDO (en utilisateur root, pas de mot de passe).
  3. Créez create_article.php : formulaire (titre, contenu, auteur_id) + INSERT préparé.
  4. Créez articles.php : liste tous les articles avec jointure sur l'auteur.
  5. Créez article.php?id=X : affiche un article avec ses catégories.
  6. Créez delete_article.php?id=X : supprime un article (avec confirmation).
EX 02 🔍 Recherche avec jointures
  1. Créez recherche.php avec un champ de recherche.
  2. Recherchez dans les titres d'articles avec LIKE et requête préparée.
  3. Affichez pour chaque article : titre, nom de l'auteur, catégories (GROUP_CONCAT).
  4. Ajoutez un filtre par catégorie (menu déroulant).
  5. Ajoutez un compteur : "X résultats trouvés".
  6. Gérez le cas où aucun résultat n'est trouvé.

Bilan & glossaire

Ce que vous maîtrisez

  • MySQL et phpMyAdmin : créer base, tables, exécuter SQL

Mini-glossaire

MySQL
SGBDR libre, système de gestion de base de données relationnelle.
PDO
PHP Data Objects — extension d'abstraction de base de données.
DSN
Data Source Name — chaîne de connexion contenant hôte, base, charset.
Requête préparée
Requête SQL avec paramètres séparés, protège contre l'injection SQL.
Injection SQL
Attaque consistant à insérer du code SQL malveillant via des entrées utilisateur.
CRUD
Create / Read / Update / Delete — les 4 opérations de base.
INNER JOIN
Jointure qui retourne uniquement les lignes correspondant dans les deux tables.
LEFT JOIN
Jointure qui retourne toutes les lignes de la table gauche, même sans correspondance.
Clé étrangère
Colonne qui référence la clé primaire d'une autre table.
lastInsertId
Méthode PDO qui retourne l'ID du dernier enregistrement inséré.
