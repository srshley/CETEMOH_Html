Opérateurs arithmétiques

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

PHP supporte les 5 opérations de base plus l'exponentiation. Ces opérateurs fonctionnent sur les types int et float.

OpérateurNomExempleRésultat
+Addition10 + 313
-Soustraction10 - 37
*Multiplication10 * 330
/Division10 / 33.333...
%Modulo (reste)10 % 31
**Exponentiation2 ** 38
PHP — Opérations arithmétiques 
<?php
$a = 10;
$b = 3;

echo $a + $b;   // 13
echo $a - $b;   // 7
echo $a * $b;   // 30
echo $a / $b;   // 3.3333333333333
echo $a % $b;   // 1
echo $a ** $b;  // 1000

// Opérateurs combinés (assignment operators)
$x = 5;
$x += 3;    // $x = $x + 3 → 8
$x -= 2;    // 6
$x *= 2;    // 12
$x /= 3;    // 4
$x %= 3;    // 1

// Incrémentation / décrémentation
$i = 0;
$i++;   // $i = 1 (post-incrémentation)
++$i;   // $i = 2 (pré-incrémentation)
$i--;   // $i = 1
?>
💡
Modulo — applications pratiques

Le modulo % est très utile : tester si un nombre est pair ($n % 2 == 0), alterner des couleurs dans une boucle, paginer des résultats, ou valider des formats (heures, minutes).

Opérateurs de comparaison

Les opérateurs de comparaison comparent deux valeurs et retournent un booléen (true ou false). Attention à la différence entre comparaison lâche et stricte.

OpérateurNomExempleRésultat
==Égal (lâche)"5" == 5true (types ignorés)
===Identique (strict)"5" === 5false (types vérifiés)
!=Différent (lâche)"5" != 5false
!==Différent (strict)"5" !== 5true
<Inférieur3 < 5true
>Supérieur3 > 5false
<=Inférieur ou égal5 <= 5true
>=Supérieur ou égal6 >= 5true
<=>Spaceship (PHP 7+)3 <=> 5-1 (gauche < droite)
PHP — Comparaison lâche vs stricte 
<?php
var_dump(5 == "5");     // true  (lâche : conversion automatique)
var_dump(5 === "5");    // false (strict : types différents)
var_dump(0 == false);   // true  (lâche : 0 == false)
var_dump(0 === false);  // false (strict : bool ≠ int)
var_dump(null == false);// true  (lâche)
var_dump(null === false);// false (strict)

// Règle d'or : utilisez === et !== SAUF si vous voulez explicitement
// la conversion automatique
?>

Opérateurs logiques

Les opérateurs logiques combinent des expressions booléennes. Ils sont essentiels pour les conditions.

OpérateurNomExempleRésultat
&&ET (and)true && falsefalse
||OU (or)true || falsetrue
!NON (not)!truefalse
andET (basse priorité)true and falsefalse
orOU (basse priorité)true or falsetrue
xorOU exclusiftrue xor truefalse
PHP — Opérateurs logiques 
<?php
$age = 25;
$membre = true;

// &&
if ($age >= 18 && $membre) {
    echo "Accès autorisé";
}

// ||
if ($age < 12 || $age > 80) {
    echo "Tarif réduit";
}

// !
if (!$membre) {
    echo "Veuillez vous inscrire";
}

// Attention : priorité différente entre &&/|| et and/or
$result = true and false;   // $result = true (and a très basse priorité)
$result = true && false;   // $result = false (préférez &&/||)
?>

Concaténation

En PHP, le point . est l'opérateur de concaténation. Il fusionne deux chaînes en une seule.

PHP — Concaténation 
<?php
$prenom = "Alice";
$nom = "Dupont";

$full = $prenom . " " . $nom;
echo $full;                     // Alice Dupont

// Opérateur combiné .=
$message = "Bonjour";
$message .= " " . $prenom;     // Bonjour Alice
$message .= " !";              // Bonjour Alice !

echo $message;

// Concaténation vs guillemets doubles
echo "Bonjour $prenom $nom";   // Bonjour Alice Dupont
echo "Bonjour " . $prenom . " " . $nom;  // même résultat

// Concaténer des types différents
echo "Prix : " . 19.99 . " €";  // Prix : 19.99 €
?>
💡
Quand utiliser la concaténation vs les guillemets doubles

Les guillemets doubles avec "$var" sont plus lisibles pour des variables simples. La concaténation est plus claire pour des expressions complexes ou des appels de fonctions : "Prix : " . number_format($prix, 2) . " €"

Fonctions de chaînes

FonctionUtilitéExemple
strlen()Longueur d'une chaînestrlen("Bonjour") → 7
str_replace()Remplacer une sous-chaînestr_replace("e", "a", "test") → "tast"
substr()Extraire une partiesubstr("Bonjour", 0, 3) → "Bon"

PHP — Fonctions de chaînes en action 
<?php
$texte = "Bonjour tout le monde";

echo strlen($texte);                    // 22
echo str_replace("monde", "PHP", $texte); // Bonjour tout le PHP
echo substr($texte, 0, 7);              // Bonjour
echo strpos($texte, "tout");            // 8
echo strtoupper($texte);               // BONJOUR TOUT LE MONDE
echo strtolower($texte);               // bonjour tout le monde

$csv = "pomme,banane,cerise";
$fruits = explode(",", $csv);
print_r($fruits);
// Array ( [0] => pomme [1] => banane [2] => cerise )

echo implode(" | ", $fruits);           // pomme | banane | cerise
?>

Échappement & guillemets

L'échappement permet d'insérer des caractères spéciaux (guillemets, apostrophes, retours à la ligne) dans une chaîne.

PHP — Échappement des caractères 
<?php
// Échapper les guillemets dans une chaîne
echo "Il a dit \"Bonjour\" à tout le monde";
echo 'C\'est la vie';

// Caractères d'échappement (uniquement dans les guillemets doubles)
echo "Première ligne\nDeuxième ligne";   // \n = newline (retour à la ligne)
echo "Colonne 1\tColonne 2";            // \t = tabulation
echo "\\";                              // \\ = antislash littéral
echo "\$variable";                      // \$ = dollar littéral

// Heredoc — chaîne multi-lignes sans échappement
$nom = "Alice";
$html = <<<HTML
<div class="message">
    <h2>Bonjour $nom</h2>
    <p>Ceci est un heredoc.</p>
</div>
HTML;

// Nowdoc — comme heredoc mais sans interprétation des variables
$code = <<<'CODE'
    echo "Bonjour $nom";   // $nom ne sera PAS interprété
CODE;
?>
⚠️
Alterner les guillemets pour éviter l'échappement

Utilisez les guillemets simples à l'intérieur des guillemets doubles et vice-versa : "C'est simple" ou 'Il a dit "ok"' — pas besoin d'échappement.

Fonctions sur les nombres

PHP fournit de nombreuses fonctions pour vérifier et formater les nombres.

PHP — Manipulation des nombres 
<?php
// Vérifications
var_dump(is_numeric("123"));        // true
var_dump(is_numeric("abc"));        // false
var_dump(is_int(42));               // true
var_dump(is_float(3.14));           // true
var_dump(is_nan(sqrt(-1)));         // true (Not a Number)

// Arrondis
echo round(3.14159, 2);            // 3.14
echo ceil(4.1);                    // 5 (arrondi supérieur)
echo floor(4.9);                   // 4 (arrondi inférieur)

// Formatage monétaire
$prix = 1234.5678;
echo number_format($prix, 2, ',', ' ');    // 1 234,57
echo number_format($prix, 2, '.', ',');    // 1,234.57

// Aléatoire
echo rand(1, 10);                  // nombre aléatoire entre 1 et 10
echo mt_rand(1, 10);               // meilleur aléatoire (Mersenne Twister)

// Min / max
echo min(3, 7, 2, 9);              // 2
echo max(3, 7, 2, 9);              // 9

// Valeur absolue
echo abs(-42);                     // 42
?>

Constantes (define, const)

Une constante est un identifiant qui ne change pas de valeur durant l'exécution. Par convention, les constantes s'écrivent en MAJUSCULES.

PHP — Déclaration de constantes 
<?php
// Méthode 1 : define() — au moment de l'exécution, dans n'importe quel bloc
define("NOM_SITE", "MonSite.fr");
define("TAUX_TVA", 0.20);
define("ADMIN_EMAIL", "admin@exemple.com");

// Méthode 2 : const — au moment de la compilation, doit être dans la portée racine
const VERSION = "1.0.0";
const ANNEE_CREATION = 2024;

// Utilisation
echo NOM_SITE . " v" . VERSION;

// Les constantes prédéfinies de PHP
echo PHP_VERSION;           // ex: 8.2.12
echo PHP_OS;                // ex: WINNT, Linux
echo PHP_INT_MAX;           // 9223372036854775807
echo DIRECTORY_SEPARATOR;   // \ sous Windows, / sur Linux/Mac

// Les constantes magiques (__changent selon le contexte)
echo __LINE__;              // ligne courante du fichier
echo __FILE__;              // chemin complet du fichier courant
echo __DIR__;               // dossier du fichier courant
?>
📌
const vs define : quelle différence ?

const est plus rapide et doit être déclaré dans la portée racine. define() peut être utilisé à l'intérieur d'un bloc (condition, boucle, fonction) et accepte les expressions. Pour 95% des cas, utilisez const.

Mini-glossaire

Opérateur arithmétique
Symbole qui effectue un calcul : +, -, *, /, %, **
Comparaison stricte
=== et !== — vérifient la valeur ET le type.
Comparaison lâche
== et != — convertissent les types automatiquement.
Concaténation
Fusion de chaînes avec l'opérateur . (point).
Heredoc
Syntaxe <<< pour les chaînes multi-lignes avec interprétation des variables.
Nowdoc
Comme heredoc mais sans interprétation des variables (guillemets simples).
Échappement
Antislash \ devant un caractère pour lui donner un sens spécial.
Constante
Identifiant majuscule dont la valeur ne change pas durant l'exécution.
Constante magique
Constante prédéfinie (__LINE__, __FILE__…) qui change selon le contexte.
number_format
Fonction qui formate un nombre avec décimales et séparateurs.
