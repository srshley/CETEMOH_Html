01

Méthode GET

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

GET est la méthode HTTP par défaut. Les données sont envoyées dans l'URL, visibles dans la barre d'adresse. Utilisée pour les recherches, la pagination, les filtres.

PHP — Formulaire GET 
<!-- formulaire.html -->
<form action="traitement.php" method="GET">
    <label>Nom : <input type="text" name="nom"></label>
    <label>Ville : <input type="text" name="ville"></label>
    <button type="submit">Rechercher</button>
</form>

<!-- L'URL envoyée ressemblera à :
     traitement.php?nom=Alice&ville=Paris -->
⚠️
GET n'est pas sécurisé

Les données GET sont visibles dans l'URL, stockées dans l'historique du navigateur, et peuvent être mises en favori. N'utilisez JAMAIS GET pour les mots de passe, données personnelles ou actions qui modifient des données (création, modification, suppression).

02

$_GET — les paramètres d'URL

$_GET est un tableau associatif superglobal qui contient toutes les données de la chaîne de requête de l'URL.

PHP — Accès aux paramètres GET 
<?php
// URL : recherche.php?q=PHP&page=2&categorie=livres

// Accès simple
echo $_GET['q'];           // PHP
echo $_GET['page'];        // 2

// Vérifier qu'un paramètre existe
if (isset($_GET['q'])) {
    $recherche = $_GET['q'];
} else {
    $recherche = '';
}

// Avec l'opérateur null coalescent
$recherche = $_GET['q'] ?? '';
$page = (int)($_GET['page'] ?? 1);

// Afficher tous les paramètres
echo "<pre>";
print_r($_GET);
echo "</pre>";
?>

03

Méthode POST

POST envoie les données dans le corps de la requête HTTP, invisibles dans l'URL. Utilisée pour les formulaires qui modifient des données ou contiennent des informations sensibles.

PHP — Formulaire POST 
<!-- inscription.php -->
<form action="traitement.php" method="POST">
    <label>Email : <input type="email" name="email" required></label>
    <label>Mot de passe : <input type="password" name="mdp" required></label>
    <button type="submit">S'inscrire</button>
</form>

<!-- L'URL reste : traitement.php (les données sont dans le corps) -->
══ 04 $_POST ══ -->
04

$_POST — les données du formulaire

$_POST est le tableau superglobal qui contient les données envoyées via la méthode POST.

PHP — Traitement d'un formulaire POST 
<?php
// traitement.php

// Vérifier que le formulaire a été soumis
if ($_SERVER['REQUEST_METHOD'] === 'POST') {

    // Récupérer les données
    $email = $_POST['email'] ?? '';
    $mdp = $_POST['mdp'] ?? '';

    // Validation simple
    if (empty($email)) {
        $erreur = "L'email est obligatoire";
    } elseif (!filter_var($email, FILTER_VALIDATE_EMAIL)) {
        $erreur = "Email invalide";
    } elseif (empty($mdp)) {
        $erreur = "Le mot de passe est obligatoire";
    } else {
        $succes = "Inscription réussie !";
    }
}

// Noms de champs avec []
<input type="text" name="interets[]">
<input type="text" name="interets[]">

// $_POST['interets'] = ["valeur1", "valeur2"]
?>
══ 05 $_REQUEST ══ -->
05

$_REQUEST

$_REQUEST combine $_GET, $_POST et $_COOKIE. Pratique mais moins précis car on ne sait pas d'où viennent les données.

PHP — $_REQUEST 
<?php
// $_REQUEST = $_GET + $_POST + $_COOKIE
$nom = $_REQUEST['nom'] ?? 'Anonyme';

// ⚠️ Problème : ne sait pas si c'est GET ou POST
// ⚠️ Risque de confusion si un paramètre GET et POST ont le même nom
// (l'ordre de priorité est défini dans php.ini : variables_order)

// Préférez toujours $_GET ou $_POST explicitement
?>
⚠️
Évitez $_REQUEST

Utilisez explicitement $_GET ou $_POST. $_REQUEST masque la source des données et peut créer des failles de sécurité si mal utilisé.

══ 06 VALIDATION ══ -->
06

Validation des entrées

Ne faites jamais confiance aux données utilisateur. Validez toujours avant d'utiliser une entrée.

PHP — Validation des formulaires 
<?php
// filter_input — méthode recommandée pour filtrer une entrée
$email = filter_input(INPUT_POST, 'email', FILTER_VALIDATE_EMAIL);
$age = filter_input(INPUT_POST, 'age', FILTER_VALIDATE_INT, [
    'options' => ['min_range' => 1, 'max_range' => 120]
]);
$url = filter_input(INPUT_POST, 'site', FILTER_VALIDATE_URL);

// Si le filtre échoue → retourne false
if (!$email) {
    $erreurs[] = "Email invalide";
}

// Validation manuelle
function validerInscription(array $data): array {
    $erreurs = [];

    if (empty($data['nom'])) {
        $erreurs['nom'] = "Le nom est requis";
    } elseif (strlen($data['nom']) < 2) {
        $erreurs['nom'] = "Le nom doit faire au moins 2 caractères";
    }

    if (empty($data['email'])) {
        $erreurs['email'] = "L'email est requis";
    } elseif (!filter_var($data['email'], FILTER_VALIDATE_EMAIL)) {
        $erreurs['email'] = "Format d'email invalide";
    }

    if (empty($data['age'])) {
        $erreurs['age'] = "L'âge est requis";
    } elseif (!is_numeric($data['age']) || $data['age'] < 1 || $data['age'] > 120) {
        $erreurs['age'] = "Âge invalide";
    }

    return $erreurs;
}
?>
══ 07 SÉCURITÉ XSS ══ -->
07

Sécurité — XSS (Cross-Site Scripting)

Les attaques XSS consistent à injecter du code malveillant (JavaScript) via les entrées utilisateur. La protection de base : htmlspecialchars().

PHP — Protection XSS avec htmlspecialchars 
<?php
// ❌ DANGEREUX — affichage direct d'une entrée utilisateur
$nom = $_GET['nom'] ?? '';
echo "Bonjour $nom";
// Si nom = <script>alert('XSS')</script>
// → le script s'exécute dans le navigateur !

// ✅ SÉCURISÉ — échappement HTML
echo "Bonjour " . htmlspecialchars($nom, ENT_QUOTES, 'UTF-8');
// <script> devient &lt;script&gt; → texte inoffensif

// htmlspecialchars avec les bons paramètres
function e(string $value): string {
    return htmlspecialchars($value, ENT_QUOTES | ENT_SUBSTITUTE, 'UTF-8');
}

// Utilisation dans un template
$titre = $_POST['titre'] ?? '';
$contenu = $_POST['contenu'] ?? '';
?>
<h1><?= e($titre) ?></h1>
<p><?= e($contenu) ?></p>

<!-- Avec les attributs HTML, toujours échapper aussi -->
<input type="text" value="<?= e($valeur) ?>">
⚠️
Règle d'or : filtre à l'entrée, échappe à la sortie

Validez et filtrez les données quand vous les recevez. Échappez-les avec htmlspecialchars() quand vous les affichez. Ne stockez jamais de données échappées en base de données — stockez les données brutes et échappez à l'affichage.

══ 08 STRIP_TAGS & TRIM ══ -->
08

strip_tags & trim

Deux fonctions essentielles pour nettoyer les entrées : trim() supprime les espaces superflus, strip_tags() supprime les balises HTML.

PHP — Nettoyage des entrées 
<?php
// trim — supprime les espaces au début et à la fin
$nom = "  Alice  \n";
echo trim($nom);   // "Alice"

// Utile pour nettoyer les formulaires
$data = array_map('trim', $_POST);   // applique trim à tout le tableau

// strip_tags — supprime les balises HTML
$texte = "<script>alert('XSS')</script>Bonjour";
echo strip_tags($texte);   // "alert('XSS')Bonjour"

// Permet de garder certaines balises
echo strip_tags($texte, "<p><b><i>");
// Garde <p>, <b>, <i>, supprime tout le reste

// Combinaison typique pour une entrée "sûre"
function nettoyer(string $valeur): string {
    return trim(strip_tags($valeur));
}

// Pour du texte long (autoriser certaines balises)
function nettoyerTexte(string $valeur): string {
    return trim(strip_tags($valeur, "<p><br><strong><em><ul><ol><li>"));
}
?>

09

Exercices pratiques

EX 01 ✏️ Formulaire de contact
  1. Créez un fichier contact.php avec un formulaire POST (nom, email, message).
  2. Le formulaire soumet vers la même page (action="").
  3. Validez les champs : nom non vide (2+ car.), email valide, message non vide (10+ car.).
  4. Affichez les erreurs sous chaque champ en rouge.
  5. Si tout est valide, affichez "Message envoyé !" et les données nettoyées.
  6. Utilisez htmlspecialchars() pour réafficher les valeurs dans le formulaire.
  7. Protégez contre XSS avec votre fonction e().
EX 02 🔍 Recherche avec GET
  1. Créez recherche.php avec un formulaire GET (q, categorie).
  2. Créez un tableau $articles avec 5 articles (titre, categorie).
  3. Filtrez les articles selon $_GET['q'] (recherche dans le titre) et $_GET['categorie'].
  4. Affichez les résultats dans une liste HTML.
  5. Si pas de résultat, affichez "Aucun résultat trouvé".
  6. Conservez les valeurs de recherche dans le formulaire avec htmlspecialchars().

10

Bilan & glossaire

Ce que vous maîtrisez

  • GET : données dans l'URL, pour recherches et filtres
  • POST : données dans le corps, pour formulaires sensibles
  • $_GET, $_POST, $_REQUEST — les superglobales
  • Validation avec filter_input, empty, isset, filter_var
  • Sécurité XSS : htmlspecialchars() à l'affichage
  • Nettoyage : trim(), strip_tags()
  • Principe : filtre à l'entrée, échappe à la sortie

Mini-glossaire

GET
Méthode HTTP avec données dans l'URL. Visible, bookmarkable.
POST
Méthode HTTP avec données dans le corps. Invisible dans l'URL.
$_GET
Superglobale contenant les paramètres de la chaîne de requête.
$_POST
Superglobale contenant les données du corps POST.
XSS
Cross-Site Scripting — injection de code via les entrées utilisateur.
htmlspecialchars
Convertit les caractères HTML spéciaux en entités.
trim
Supprime les espaces et sauts de ligne au début et à la fin.
strip_tags
Supprime les balises HTML et PHP d'une chaîne.
filter_input
Fonction qui filtre une entrée externe (GET/POST/COOKIE).
Superglobale
Variable prédéfinie accessible partout : $_GET, $_POST, $_SERVER…
