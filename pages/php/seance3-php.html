01

if / else

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

La structure if / else est la plus fondamentale pour prendre des décisions dans votre code. Si la condition est vraie (true), le premier bloc s'exécute ; sinon, le bloc else s'exécute.

PHP — if / else 
<?php
$age = 18;

if ($age >= 18) {
    echo "Vous êtes majeur.";
} else {
    echo "Vous êtes mineur.";
}

// Sans accolades (une seule instruction)
if ($age >= 18) echo "Majeur";
else echo "Mineur";

// Style avec if: / endif; (dans les templates HTML)
if ($age >= 18):
    echo "Majeur";
else:
    echo "Mineur";
endif;
?>
⚠️
Les valeurs "falsy" en PHP

Dans une condition, ces valeurs sont considérées comme fausses : false, 0, 0.0, "", "0", null, [] (tableau vide). Tout le reste est truthy.

02

elseif

elseif permet de tester plusieurs conditions à la suite dès que la précédente est fausse.

PHP — elseif (en un mot) 
<?php
$note = 15;

if ($note >= 16) {
    echo "Très bien";
} elseif ($note >= 14) {
    echo "Bien";
} elseif ($note >= 12) {
    echo "Assez bien";
} elseif ($note >= 10) {
    echo "Passable";
} else {
    echo "Insuffisant";
}

// Important : elseif en UN mot (pas "else if" quand on utilise les accolades)
// Avec les accolades : "else if" fonctionne aussi mais "elseif" est préférable
// Avec if:/endif; il faut écrire "elseif" (pas "else if")
?>
💡
Ordre des conditions

Toujours tester les cas les plus spécifiques en premier. Si vous testez $note >= 12 avant $note >= 16, le "Très bien" ne sera jamais atteint.

══ 03 SWITCH ══ -->
03

switch

switch compare une valeur contre plusieurs cas. Utile quand vous avez plus de 3-4 cas sur une même variable.

PHP — switch 
<?php
$jour = "mardi";

switch ($jour) {
    case "lundi":
        echo "Début de semaine";
        break;
    case "mardi":
    case "mercredi":
    case "jeudi":
        echo "Milieu de semaine";
        break;
    case "vendredi":
        echo "Bientôt le week-end";
        break;
    case "samedi":
    case "dimanche":
        echo "C'est le week-end !";
        break;
    default:
        echo "Jour invalide";
        break;
}

// Attention : sans break, PHP continue sur le cas suivant (fall-through)
// Parfois utile : cas groupés ci-dessus (mardi/mercredi/jeudi)
?>
⚠️
switch utilise la comparaison LÂCHE (==)

switch("2") correspondra au cas case 2: car il utilise == et non ===. Soyez vigilant avec les chaînes qui ressemblent à des nombres.

══ 04 MATCH ══ -->
04

match (PHP 8)

Introduit en PHP 8, match est une version améliorée de switch : il retourne une valeur, utilise la comparaison stricte (===), et ne nécessite pas de break.

PHP — match (PHP 8+) 
<?php
$jour = "mardi";

$resultat = match ($jour) {
    "lundi"    => "Début de semaine",
    "mardi",
    "mercredi",
    "jeudi"    => "Milieu de semaine",
    "vendredi"  => "Bientôt le week-end",
    "samedi",
    "dimanche"  => "C'est le week-end !",
    default     => "Jour invalide"
};

echo $resultat;

// match peut aussi tester des conditions booléennes
$age = 20;
$statut = match (true) {
    $age < 12   => "Enfant",
    $age < 18   => "Adolescent",
    $age < 65   => "Adulte",
    default      => "Senior"
};
echo $statut;   // Adulte

// match lance une UnhandledMatchError si aucun cas ne correspond (sans default)
?>
💡
match vs switch : résumé

match utilise === (strict), retourne une valeur, pas de break, pas de fall-through accidentel. Si vous êtes en PHP 8, préférez match à switch.

══ 05 TERNAIRE ══ -->
05

Opérateur ternaire ? :

Le ternaire est une version condensée de if/else qui retourne une valeur. Syntaxe : condition ? valeur_si_vrai : valeur_si_faux

PHP — Opérateur ternaire 
<?php
$age = 20;

// Version if/else classique
if ($age >= 18) {
    $status = "Majeur";
} else {
    $status = "Mineur";
}

// Version ternaire (équivalent strict)
$status = ($age >= 18) ? "Majeur" : "Mineur";

// Ternaire imbriqué (à utiliser avec modération)
$note = 15;
$mention = ($note >= 16) ? "Très bien" :
           (($note >= 14) ? "Bien" :
           (($note >= 12) ? "Assez bien" : "Passable"));

// Ternaire raccourci (PHP 7+) : $x ?: $y
// Retourne $x si truthy, sinon $y
$nom = $nom ?: "Anonyme";   // équivaut à : $nom ? $nom : "Anonyme"
?>
⚠️
Évitez les ternaires imbriqués trop complexes

Un ternaire imbriqué est difficile à lire. Pour plus de 2 niveaux, préférez if/elseif ou match qui seront beaucoup plus clairs pour votre équipe (et pour vous-même dans 6 mois).

══ 06 NULL COALESCENT ══ -->
06

Null coalescent ?? et ??=

L'opérateur null coalescent ?? retourne la première valeur si elle existe et n'est pas null ; sinon il retourne la seconde. Indispensable pour les formulaires et les tableaux.

PHP — Null coalescent et ??= 
<?php
// ?? (null coalescent) — PHP 7+
$nom = $_GET['nom'] ?? "Anonyme";
// Si $_GET['nom'] existe et n'est pas null → $nom = $_GET['nom']
// Sinon → $nom = "Anonyme"

// Équivalent long :
$nom = isset($_GET['nom']) ? $_GET['nom'] : "Anonyme";

// ??= (null coalescent assignment) — PHP 7.4+
$prenom = null;
$prenom ??= "Inconnu";      // $prenom = "Inconnu" car null
$prenom ??= "Jamais";       // $prenom reste "Inconnu" (déjà défini)

// Chaînage possible
$ville = $_GET['ville'] ?? $_POST['ville'] ?? "Non définie";

// Différence avec ?: (ternaire raccourci)
$test = 0;
echo $test ?? "fallback";   // 0 (car 0 n'est pas null)
echo $test ?: "fallback";   // "fallback" (car 0 est falsy)
?>
💡
?? vs ?: — ne les confondez pas

?? teste null/isset : 0 ?? "fallback" donne 0. ?: teste truthy : 0 ?: "fallback" donne "fallback". Le ?? est généralement ce que vous voulez pour les formulaires.

══ 07 isset ET empty ══ -->
07

isset() et empty()

Deux fonctions essentielles pour vérifier l'existence et le contenu d'une variable ou d'un index de tableau.

FonctionRetourne true si…Exemple
isset($var)la variable existe et n'est pas nullisset($x) → false si $x n'existe pas ou vaut null
empty($var)la variable n'existe pas OU est "falsy"empty($x) → true si $x vaut "", 0, null, false, []
PHP — isset et empty en pratique 
<?php
// isset — la variable existe et n'est pas null
$nom = "Alice";
var_dump(isset($nom));       // true
var_dump(isset($inconnu));   // false (variable inexistante)

$ville = null;
var_dump(isset($ville));     // false (null)

// empty — la variable n'existe pas OU est falsy
var_dump(empty($nom));       // false ("Alice" est truthy)
var_dump(empty(0));          // true
var_dump(empty(""));         // true
var_dump(empty([]));         // true
var_dump(empty($foo));       // true (variable inexistante — pas d'erreur)

// Cas typique : formulaire
if (empty($_POST['email'])) {
    echo "L'email est obligatoire";
}

// Vérifier qu'un index existe dans un tableau
if (isset($_GET['page'])) {
    $page = $_GET['page'];
}
?>
══ 08 STRICTE VS LÂCHE ══ -->
08

Comparaison stricte vs lâche : le guide

Comprendre quand utiliser === vs == est l'une des compétences les plus importantes pour éviter les bugs en PHP.

PHP — Comparaison détaillée 
<?php
// Tableau des pièges de ==
var_dump(0 == false);         // true  ← piège !
var_dump("0" == false);       // true  ← piège !
var_dump("" == false);        // true  ← piège !
var_dump("php" == 0);         // true  ← GROS piège !
var_dump(null == false);      // true
var_dump(null == 0);          // true
var_dump("123abc" == 123);    // true  ← conversion en nombre

// === élimine tous ces problèmes
var_dump(0 === false);        // false ✓
var_dump("0" === false);      // false ✓
var_dump("" === false);       // false ✓
var_dump("php" === 0);        // false ✓
var_dump(null === false);     // false ✓

// Règle d'or
// - Utilisez === par défaut (sécurité)
// - Utilisez == seulement quand vous voulez la conversion automatique
//   (ex: in_array() avec le 3e paramètre à true par défaut ? non)
?>
⚠️
Le piège classique : in_array() et array_search()

Par défaut, in_array() et array_search() utilisent la comparaison lâche (==). Passez true en 3e paramètre pour la comparaison stricte : in_array("cherche", $tableau, true)

Bilan & glossaire

Ce que vous maîtrisez

  • if/else et if/elseif/else pour toutes les conditions
  • switch avec break et attention au fall-through
  • match pour une alternative stricte et expressive (PHP 8+)
  • L'opérateur ternaire ?: et le ternaire raccourci
  • Null coalescent ?? et ??= pour les valeurs par défaut
  • isset() vs empty() et leurs différences
  • Comparaison stricte === vs lâche == et les pièges
  • Les valeurs "falsy" en PHP : false, 0, "", null, [], "0"

Mini-glossaire

Condition
Expression booléenne qui détermine quelle branche du code exécuter.
Truthy / Falsy
Valeurs considérées comme vraies ou fausses dans un contexte booléen.
Ternaire
Opérateur ? : qui retourne une valeur selon une condition. Forme condensée de if/else.
Null coalescent
?? — retourne le premier opérande non-null. Synonyme de isset() ? :
Fall-through
Comportement de switch où l'absence de break exécute le cas suivant.
match
Expression PHP 8+ qui retourne une valeur avec comparaison stricte.
isset
Fonction qui vérifie si une variable existe et n'est pas null.
empty
Fonction qui vérifie si une variable n'existe pas ou est falsy.
Comparaison lâche
== — compare les valeurs après conversion automatique des types.
Comparaison stricte
=== — compare les valeurs SANS conversion de type.
