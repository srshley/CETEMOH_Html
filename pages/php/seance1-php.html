01

C'est quoi PHP ?

PHP (Hypertext Preprocessor) est un langage de programmation côté serveur créé en 1994 par Rasmus Lerdorf. Contrairement au HTML/CSS/JS qui s'exécutent dans le navigateur, PHP s'exécute sur le serveur avant d'envoyer la page au client.

🍕
Analogie : la pizzeria

L'HTML/CSS/JS, c'est la pizza livrée (le client voit ça). PHP, c'est le cuisinier dans la cuisine du serveur : il prépare la pizza selon votre commande. Le client ne voit jamais le cuisinier — seulement le résultat de son travail.

💡
PHP dynamique vs HTML statique

Un fichier .html est statique : le serveur l'envoie tel quel. Un fichier .php est d'abord exécuté par le serveur, puis le résultat HTML est envoyé. Cela permet de générer du contenu personnalisé (heure actuelle, données d'un formulaire, contenu depuis une base de données…).

Le voyage d'une requête PHP

🌍 Navigateur → 📨 Formulaire POST
📨 Formulaire POST → 🖥️ PHP ($_POST)
🖥️ PHP ($_POST) → 📄 Réponse HTML

⚠️
PHP ne s'exécute pas dans le navigateur

Vous ne pouvez pas ouvrir un fichier .php en double-cliquant comme un fichier HTML. Il a besoin d'un serveur (Apache, Nginx) avec PHP installé pour être interprété. C'est le rôle de XAMPP.

02

XAMPP & environnement

XAMPP est un package tout-en-un qui installe Apache (serveur web), PHP, MySQL (base de données) et phpMyAdmin. Il vous permet de développer en PHP localement sur votre machine.

💡
L'alternative : WAMP, MAMP, Laragon

XAMPP fonctionne sur Windows, Mac et Linux. WAMP est spécifique à Windows. MAMP pour Mac. Laragon est plus moderne sous Windows. Tous fournissent le même trio Apache + PHP + MySQL.

Installation et structure

Fichiers — Arborescence XAMPP 
C:\xampp\
  ├── htdocs\          ← Dossier web racine (vos projets ici)
  │   ├── mon-site\
  │   │   ├── index.php
  │   │   └── contact.php
  │   └── autre-projet\
  ├── apache\          ← Serveur Apache
  ├── php\             ← Interpréteur PHP
  ├── mysql\           ← Base de données MySQL
  └── phpMyAdmin\      ← Interface web pour MySQL
⚠️
Accéder à vos fichiers PHP

Ne double-cliquez pas sur un fichier .php. Lancez XAMPP, démarrez Apache, puis ouvrez votre navigateur sur http://localhost/mon-site/index.php. Le dossier htdocs correspond à la racine http://localhost.

03

Balises <?php ?>

PHP s'insère dans un fichier HTML grâce à des balises spéciales. Tout ce qui est en dehors des balises PHP est envoyé directement au navigateur comme HTML standard. Tout ce qui est à l'intérieur est exécuté par le serveur.

PHP — Balises d'ouverture et fermeture 
<!DOCTYPE html>
<html lang="fr">
<head>
  <title>Ma première page PHP</title>
</head>
<body>
  <h1>Bonjour</h1>

  <?php
    // Tout ceci est du code PHP
    echo "<p>Ce texte a été généré par PHP !</p>";
  ?>

  <p>Ce texte est du HTML statique normal.</p>
</body>
</html>
BaliseUsage
<?php ?>Balise standard — toujours utilisable, recommandée
<?= ?>Raccourci pour <?php echo ?> — affiche une valeur rapidement
<? ?>Balise courte — dépréciée, ne l'utilisez pas (désactivée sur certains serveurs)
PHP — Syntaxe courte <?= ?> 
<!-- Équivalents : -->
<?php echo "Bonjour"; ?>
<?= "Bonjour" ?>  <!-- même résultat, plus court -->

<!-- Très utile dans les templates HTML : -->
<h1><?= $titre ?></h1>
<p>Bienvenue, <?= $prenom ?> !</p>
📌
PHP pur — pas besoin de fermeture

Si un fichier ne contient que du PHP (pas de HTML après), omettez la balise fermante ?> pour éviter d'envoyer des espaces ou retours à la ligne involontaires dans la sortie HTML.

04

Echo et print

echo et print sont deux constructions du langage qui envoient du texte dans la sortie HTML. echo est légèrement plus rapide et accepte plusieurs arguments.

PHP — echo et print 
<?php
// echo — le plus utilisé
echo "Bonjour tout le monde !";
echo "Bonjour", " ", "monde", " !";  // arguments multiples

// print — retourne 1, utile dans une expression
print "Bonjour !";

// Les parenthèses sont optionnelles
echo "Bonjour";
echo("Bonjour");

// Afficher du HTML
echo "<h2>Titre </h2>";
echo '<a href="page.php">Lien</a>';
?>
PHP — Exemple avec guillemets 
<?php
$nom = "Marie";
echo "Bonjour $nom !";      // guillemets doubles : $nom est interprété
echo 'Bonjour $nom !';      // guillemets simples : $nom est littéral
?>
💡
Guillemets doubles vs simples

En PHP, les guillemets doubles interprètent les variables et les caractères d'échappement (\n, \t…). Les guillemets simples sont littéraux : tout est pris tel quel, plus rapide à analyser.

05

Variables $nom

En PHP, toute variable commence par le symbole $. PHP est un langage faiblement typé : vous n'avez pas besoin de déclarer le type d'une variable — il est déterminé automatiquement par la valeur que vous assignez.

PHP — Déclaration et règles des variables 
<?php
// Déclaration — le $ est obligatoire
$nom = "Jean";
$age = 25;
$prix = 19.99;
$estActif = true;

// Règles de nommage
$maVariable       // camelCase — convention recommandée
$mon_produit      // snake_case — acceptable
$MaVariable       // PascalCase — réservé aux classes
$_cache           // Underscore au début = "privé" (convention)
$var1             // Peut contenir des chiffres (pas au début)

// ❌ Erreurs courantes
// $1var          → chiffre au début
// $ma-var        → tiret interdit
// $ma variable   → espace interdit

// PHP est sensible à la casse
$nom = "Alice";
$Nom = "Bob";   // variable différente !
echo $nom;      // Alice
echo $Nom;      // Bob
?>
⚠️
Variable non définie = notice PHP

Si vous utilisez une variable qui n'existe pas (echo $inconnu;), PHP génère une notice (avertissement). En développement, affichez toutes les erreurs avec error_reporting(E_ALL);. En production, masquez les notices.

06

Types : string, int, float, bool, null

PHP supporte 10 types de données. Voici les 5 types de base que vous utiliserez quotidiennement.

TypeDescriptionExemple
stringChaîne de caractères"Bonjour", 'texte'
intNombre entier42, -7, 0
floatNombre à virgule flottante3.14, -0.5
boolBooléen (vrai/faux)true, false
nullAucune valeurnull
PHP — Les types de base 
<?php
// string
$prenom = "Alice";
$message = 'Bonjour le monde';

// int
$age = 30;
$annee = 2026;

// float
$prix = 19.99;
$pi = 3.14159;

// bool
$estConnecte = true;
$estAdmin = false;

// null — variable sans valeur
$resultat = null;

// Conversion automatique
$total = 10 + "5";     // 15 (string converti en int)
$message = "Vous avez " . 30 . " ans";  // 30 converti en string
?>
💡
PHP fait du "type juggling"

PHP convertit automatiquement les types selon le contexte : "10" + 5 donne 15 (int). "10" . 5 donne "105" (string). Ce comportement est pratique mais peut causer des bugs — utilisez les opérateurs de comparaison stricte (===) pour les éviter.

07

var_dump & print_r

Ces deux fonctions sont vos meilleures amies pour le débogage. Elles affichent la structure et la valeur d'une variable, y compris son type.

PHP — var_dump et print_r 
<?php
$nom = "Alice";
$age = 30;
$prix = 19.99;
$estOK = true;
$rien = null;

// var_dump — le plus complet (type + valeur + longueur)
var_dump($nom);    // string(5) "Alice"
var_dump($age);    // int(30)
var_dump($prix);   // float(19.99)
var_dump($estOK);  // bool(true)
var_dump($rien);   // NULL

// print_r — plus lisible pour les tableaux et objets
$fruits = ["pomme", "banane", "cerise"];
print_r($fruits);
// Array ( [0] => pomme [1] => banane [2] => cerise )
?>
💡
Debug propre avec <pre>

Entourez var_dump() ou print_r() de balises <pre> pour un affichage formaté dans le navigateur : echo '<pre>'; var_dump($var); echo '</pre>';

08

Commentaires

PHP supporte trois styles de commentaires. Les commentaires sont ignorés par l'interpréteur — ils sont là pour vous et votre équipe.

PHP — Les 3 styles de commentaires 
<?php
// ① Commentaire sur une ligne — style C++ (le plus courant)
$nom = "Alice";  // Vous pouvez commenter en fin de ligne

// ② Commentaire sur une ligne — style shell
$age = 30;  # Moins fréquent, mais valide

/*
 * ③ Commentaire multi-lignes — style C
 * Utile pour désactiver un bloc de code
 * ou pour décrire une fonction complexe
 */

/*
echo "Ce code est temporairement désactivé";
$test = "Ne sera pas exécuté";
*/
?>
📌
Bonnes pratiques

Commentez le pourquoi, pas le quoi. Un code bien écrit est auto-documenté. Utilisez les commentaires pour expliquer les choix d'implémentation, les cas particuliers ou les TODO.

Exercices pratiques

EX 01 ✏️ Premier script PHP
  1. Créez un dossier php-seance1/ dans htdocs.
  2. Créez un fichier index.php avec la structure HTML5 complète.
  3. Dans la balise <body>, insérez du PHP pour afficher "Bonjour, je m'appelle [Votre Prénom]" en utilisant echo.
  4. Créez une variable $age, assignez-lui votre âge, et affichez-la avec echo.
  5. Utilisez var_dump() pour afficher les types de 3 variables différentes.
  6. Ajoutez un commentaire expliquant chaque section de votre code.
  7. Testez en ouvrant http://localhost/php-seance1/index.php.
EX 02 🔍 Exploration des types
  1. Déclarez 5 variables de types différents : string, int, float, bool, null.
  2. Affichez chaque variable avec echo et observez le rendu.
  3. Utilisez var_dump() sur chaque variable pour voir son type exact.
  4. Testez les conversions : additionnez un string et un int, concaténez un int avec un string.
  5. Essayez var_dump("10" + 5) et var_dump(10 . 5) — observez la différence.

Bilan & glossaire

Ce que vous maîtrisez

  • PHP est un langage côté serveur qui génère du HTML dynamique
  • XAMPP fournit Apache + PHP + MySQL pour le développement local
  • Les balises <?php ?> délimitent le code PHP dans un fichier
  • echo et print envoient du texte dans la sortie HTML
  • Les variables commencent par $ et sont faiblement typées
  • Les 5 types de base : string, int, float, bool, null
  • var_dump() affiche le type et la valeur, print_r() est plus lisible
  • 3 styles de commentaires : //, #, /* */
