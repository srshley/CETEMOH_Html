Types input avancés

HTML5 a introduit de nouveaux types d'<input> qui améliorent l'expérience utilisateur et la validation côté navigateur : number, date, url, tel, color, range.

number— Nombres
date— Date
url— URL
tel— Téléphone
color— Couleur
range— Curseur

Checkbox & Radio

Les checkboxes permettent de sélectionner plusieurs options. Les radio buttons imposent un choix unique dans un groupe (même attribut name).

HTML — Checkbox et Radio 
<!-- Checkbox : choix multiples -->
<fieldset>
  <legend>Centres d'intérêt</legend>
  <label>
    <input type="checkbox" name="interets" value="html">
    HTML
  </label>
  <label>
    <input type="checkbox" name="interets" value="css">
    CSS
  </label>
  <label>
    <input type="checkbox" name="interets" value="js">
    JavaScript
  </label>
</fieldset>

<!-- Radio : choix unique -->
<fieldset>
  <legend>Niveau</legend>
  <label>
    <input type="radio" name="niveau" value="debutant">
    Débutant
  </label>
  <label>
    <input type="radio" name="niveau" value="intermediaire">
    Intermédiaire
  </label>
  <label>
    <input type="radio" name="niveau" value="avance">
    Avancé
  </label>
</fieldset>

03

Input file — upload de fichier

<input type="file"> permet à l'utilisateur de sélectionner un fichier. L'attribut accept filtre les types autorisés. multiple permet de sélectionner plusieurs fichiers.

HTML — File input 
<!-- Fichier unique -->
<label for="cv">CV (PDF) :</label>
<input type="file" id="cv" name="cv"
  accept=".pdf,.doc,.docx">

<!-- Plusieurs fichiers -->
<label for="photos">Photos :</label>
<input type="file" id="photos" name="photos[]"
  accept="image/*" multiple>
RENDU

⚠️
Formulaire avec fichier : method="POST" et enctype

Pour envoyer un fichier, le formulaire doit avoir method="POST" ET enctype="multipart/form-data". Sans cet attribut, le fichier n'est pas transmis.

04

Validation native HTML5

HTML5 propose une validation côté navigateur sans JavaScript. Les attributs clés : required (champ obligatoire), minlength/maxlength (longueur du texte), type (email, url, number, etc.).

HTML — Validation native 
<form>
  <!-- Champ obligatoire -->
  <label>Nom (obligatoire) :</label>
  <input type="text" name="nom" required>

  <!-- Longueur minimale et maximale -->
  <label>Mot de passe (6-20 car.) :</label>
  <input type="password" name="pass"
    minlength="6" maxlength="20" required>

  <!-- Email avec validation du format -->
  <label>Email :</label>
  <input type="email" name="email" required>

  <button type="submit">Valider</button>
</form>
RENDU

05

Pattern — validation par regex

L'attribut pattern valide le champ avec une expression régulière (regex). Utile pour les formats spécifiques : code postal, téléphone, date personnalisée, etc.

HTML — Pattern 
<!-- Code postal français (5 chiffres) -->
<label>Code postal :</label>
<input type="text" name="cp"
  pattern="[0-9]{5}"
  title="5 chiffres, ex: 75001"
  required>

<!-- Téléphone français -->
<label>Téléphone :</label>
<input type="tel" name="tel"
  pattern="0[1-9][0-9]{8}"
  title="10 chiffres, ex: 0612345678"
  required>
RENDU

💡
L'attribut title avec pattern

Toujours ajouter un title décrivant le format attendu. Le navigateur l'affiche dans le message d'erreur quand la validation échoue. Sans title, le message est "Format non valide" — peu utile.

06

Min, max, step

Ces attributs contrôlent les valeurs autorisées pour les champs numériques (number, range, date, etc.) : min (valeur minimum), max (maximum), step (pas d'incrément).

HTML — Min, max, step 
<!-- Age : 18 à 120 ans -->
<input type="number" name="age"
  min="18" max="120" value="25">

<!-- Note sur 20, pas de 0.5 -->
<input type="number" name="note"
  min="0" max="20" step="0.5" value="10">

<!-- Curseur de 0 à 100, pas de 10 -->
<input type="range" name="volume"
  min="0" max="100" step="10" value="50">

<!-- Date de naissance : pas avant 1900 -->
<input type="date" name="naissance"
  min="1900-01-01" max="2024-12-31">
RENDU

07

Attributs avancés

Hormis la validation, plusieurs attributs modifient le comportement des champs : autofocus (focus automatique), disabled (désactivé), readonly (lecture seule), multiple (sélection multiple).

HTML — Attributs avancés 
<!-- Focus automatique au chargement -->
<input type="text" name="recherche"
  autofocus placeholder="Rechercher...">

<!-- Champ désactivé (non envoyé) -->
<input type="text" name="promo"
  value="CODE10" disabled>

<!-- Lecture seule (envoyé mais non modifiable) -->
<input type="text" name="email"
  value="user@exemple.com" readonly>

<!-- Sélection multiple (select) -->
<select name="couleurs[]" multiple size="4">
  <option value="rouge">Rouge</option>
  <option value="bleu">Bleu</option>
  <option value="vert">Vert</option>
</select>

08

Organisation avancée des formulaires

Combinez les techniques apprises : fieldset/legend pour les groupes, label pour l'accessibilité, validation native pour l'expérience utilisateur, et attributs avancés pour le comportement.

HTML — Formulaire complet 
<form action="/inscription" method="POST">
  <fieldset>
    <legend>Identité</legend>

    <label for="prenom">Prénom :</label>
    <input type="text" id="prenom" name="prenom"
      required autofocus>

    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom" required>

    <label for="naissance">Date naissance :</label>
    <input type="date" id="naissance"
      name="naissance"
      min="1900-01-01" max="2010-12-31">
  </fieldset>

  <fieldset>
    <legend>Compte</legend>

    <label for="email">Email :</label>
    <input type="email" id="email"
      name="email" required>

    <label for="pass">Mot de passe :</label>
    <input type="password" id="pass"
      name="pass" minlength="8" required>
  </fieldset>

  <button type="submit">Créer mon compte</button>
</form>

Exercices pratiques

EX 01 ✏️ Formulaire d'inscription complet
  1. Créez un formulaire d'inscription avec : prénom, nom, email, téléphone, date naissance, mot de passe, confirmation.
  2. Utilisez required, minlength, pattern pour le téléphone.
  3. Ajoutez autofocus sur le prénom.
  4. Groupez avec fieldset : "Identité" et "Sécurité".
EX 02 ✏️ Sondage
  1. Créez un sondage avec : nom, email, âge (number), couleur préférée (color).
  2. Une question checkbox : "Langages maîtrisés" (HTML, CSS, JS, PHP).
  3. Une question radio : "Niveau d'expérience".
  4. Un curseur range : "Satisfaction générale" de 0 à 10.
  5. Un file input pour uploader une image de profil.
EX 03 🔧 Validation avancée
  1. Créez un champ code postal avec pattern="[0-9]{5}".
  2. Créez un champ téléphone avec pattern="0[1-9][0-9]{8}".
  3. Ajoutez title à chaque pattern pour le message d'erreur.
  4. Testez la validation en cliquant sur submit sans remplir.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Types input avancés : number, date, url, tel, color, range
  • Checkbox (choix multiples) et Radio (choix unique)
  • File input avec accept et multiple
  • Validation native : required, minlength, maxlength
  • Pattern : validation par regex avec title informatif
  • Min, max, step pour les champs numériques
  • Attributs : autofocus, disabled, readonly, multiple
  • Organisation avancée avec fieldset/legend

Mini-glossaire

required
Rend un champ obligatoire. Le formulaire ne peut pas être soumis sans.
pattern
Expression régulière pour valider le format du champ.
min / max
Bornes minimale et maximale pour les champs numériques et dates.
step
Pas d'incrément pour number et range (ex: 0.5, 10).
disabled
Champ désactivé, non soumis, grisé.
readonly
Champ en lecture seule, soumis mais non modifiable.
autofocus
Focus automatique sur le champ au chargement de la page.
multiple
Autoriser la sélection de plusieurs fichiers ou options.
accept
Filtre les types de fichiers autorisés pour file input.
enctype
Type d'encodage du formulaire. multipart/form-data pour les fichiers.
