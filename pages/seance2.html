01

Balises ouvrantes et fermantes

Le HTML se compose d'instructions entre chevrons < > appelées balises. La grande majorité des balises fonctionnent par paires : ouvrante et fermante.

📦
Analogie : les boîtes

Imaginez que chaque balise est une boîte. La boîte s'ouvre <p>, on met du contenu dedans, puis on la referme </p>. Sans fermeture, la boîte reste ouverte et le contenu "déborde".

HTML — Balises ouvrantes et fermantes 
<!-- Balise ouvrante : <nom> -->
<p>Ceci est un paragraphe.</p>
<!-- Balise fermante : </nom> (slash avant le nom) -->

<h1>Titre principal</h1>
<h2>Sous-titre</h2>
<a href="https://exemple.com">Lien cliquable</a>
RENDU

Ceci est un paragraphe.

02

Éléments et imbrication

Un élément = balise ouvrante + contenu + balise fermante. Les éléments peuvent être imbriqués les uns dans les autres. C'est ce qui crée la structure arborescente du document HTML.

⚠️
Règle impérative : Nidification correcte

On ferme d'abord le dernier ouvert : <strong><em>Texte</em></strong> ✓
On ne croise pas : <strong><em>Texte</strong></em> ✗

HTML — Imbrication correcte 
<!-- Imbrication correcte : on ferme du plus récent au plus ancien -->
<p>Ce texte contient un <strong>mot important <em>et très important</em></strong>.</p>

<!-- Liste imbriquée dans un paragraphe n'est PAS valide -->
<div>
  <h2>Titre de section</h2>
  <ul>
    <li>Item 1</li>
    <li>Item 2</li>
  </ul>
</div>
RENDU

Ce texte contient un mot important et très important.

Titre de section

  • Item 1
  • Item 2

04

Commentaires HTML

Les commentaires sont invisibles dans le navigateur mais visibles dans le code source. Ils servent à laisser des notes, documenter son code, ou désactiver temporairement des blocs.

HTML — Commentaires 
<!-- Ceci est un commentaire -->
<p>Ce texte est visible.</p>

<!--
  Commentaire
  multi-lignes
  pour documenter une section
-->

<!-- Désactiver temporairement du code : -->
<!--
<p>Ce paragraphe est masqué.</p>
<img src="ancienne-photo.jpg" alt="Ancienne photo">
-->

<!-- TODO: ajouter le formulaire de contact ici -->

<p><!-- commentaire au milieu d'une balise -->Suite du texte</p>

Ce texte est visible.

05

Entités HTML

Certains caractères sont "réservés" par HTML. Pour les afficher littéralement, on utilise des entités commençant par & et finissant par ;.

HTML — Entités courantes 
<!-- Chevrons (pour afficher < sans créer une balise) -->
<p>La balise paragraphe : &lt;p&gt;</p>

<!-- Esperluette -->
<p>Rock &amp; Roll</p>

<!-- Espace insécable (empêche la coupure) -->
<p>M.&nbsp;Dupont a 29&nbsp;€.</p>

<!-- Symboles courants -->
<p>&copy; 2024 Tous droits réservés.</p>
<p>Il fait 37&deg;C aujourd'hui.</p>
<p>Contact &mdash; nous écrire</p>
RENDU

La balise paragraphe : <p>

Rock & Roll

M. Dupont a 29 €.

© 2024 Tous droits réservés.

Il fait 37°C aujourd'hui.

Contact — nous écrire

06

Balises orphelines

Les balises orphelines (void elements) n'ont pas de contenu et donc pas de balise fermante. Elles s'arrêtent d'un seul coup.

HTML — Balises orphelines 
<!-- Saut de ligne -->
<p>12 rue des Fleurs<br>75001 Paris</p>

<!-- Séparateur horizontal -->
<hr>

<!-- Image -->
<img src="photo.jpg" alt="Description">

<!-- Input -->
<input type="text" placeholder="Votre nom">

<!-- Meta -->
<meta charset="UTF-8">
<meta name="description" content="Mon site">
RENDU

12 rue des Fleurs
75001 Paris

Indentation & lisibilité

Le navigateur ignore les espaces et retours à la ligne dans le code HTML. Mais pour le développeur, l'indentation est essentielle. Un code bien indenté se lit comme un livre.

💡
Raccourci VS Code

Sélectionnez tout (Ctrl+A) puis Shift+Alt+F pour formater automatiquement tout le document.

08

Validation W3C

Le W3C (World Wide Web Consortium) fournit un validateur officiel pour vérifier que votre code HTML est correct. Un code valide = meilleur SEO, meilleure accessibilité, moins de bugs.

💡
Validateur en ligne

Rendez-vous sur validator.w3.org. Trois options : saisir une URL, uploader un fichier, ou coller le code directement. Corrigez les erreurs une par une — le validateur indique la ligne exacte de chaque problème.

⚠️
Erreurs fréquentes détectées par le validateur

Balise non fermée · Attribut sans guillemets · Mauvais ordre d'imbrication · Doctype manquant · Tag inconnu · Valeur d'attribut invalide. VS Code surligne aussi ces erreurs en rouge.

09

Exercices pratiques

EX 01 ✏️ Structure commentée
  1. Créez un fichier seance2-exo1.html avec la structure HTML5 complète.
  2. Ajoutez un commentaire avant chaque section : <!-- HEADER -->, <!-- CONTENU -->, etc.
  3. Ajoutez un <h1> avec un id="top".
  4. Ajoutez 3 paragraphes utilisant des entités : &copy;, &mdash;, &nbsp; et &lt;.
EX 02 🔧 Attributs globaux
  1. Créez un <div> avec class="container" et id="main".
  2. À l'intérieur, mettez un <p> avec un attribut title donnant une info bulle.
  3. Ajoutez un <span> avec data-user-id="123".
  4. Ajoutez un élément avec hidden — vérifiez qu'il n'apparaît pas.
EX 03 🔍 Validation W3C
  1. Allez sur validator.w3.org.
  2. Sélectionnez "Validate by File Upload" et choisissez votre fichier.
  3. Lisez les erreurs et corrigez-les une par une.
  4. Répétez jusqu'à obtenir "Passed validation".

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Balises ouvrantes et fermantes — fonctionnement en paires
  • Éléments et imbrication correcte (fermer du plus récent au plus ancien)
  • Attributs globaux : id, class, title, style, lang, data-*, hidden, tabindex
  • Commentaires HTML pour documenter et désactiver du code
  • Entités HTML : &lt;, &gt;, &amp;, &nbsp;, &copy;, &deg;, &mdash;
  • Balises orphelines : br, hr, img, input, meta
  • Indentation et formatage automatique VS Code
  • Validation W3C pour un code conforme aux standards

Mini-glossaire

Balise ouvrante
Démarre un élément. Format : <nom>.
Balise fermante
Termine un élément. Format : </nom> avec un slash.
Élément
Balise ouvrante + contenu + balise fermante = un bloc complet.
Imbrication
Placement d'un élément à l'intérieur d'un autre. Les éléments s'emboîtent.
Attribut
Information dans la balise ouvrante. Format : nom="valeur".
Commentaire HTML
Note invisible dans le navigateur. <!-- texte -->.
Entité HTML
Code commençant par & pour afficher un caractère réservé.
Balise orpheline
Balise sans contenu ni fermeture. Aussi appelée void element.
W3C Validator
Outil officiel pour vérifier la conformité du code HTML aux normes.
Indentation
Décalage visuel pour représenter la hiérarchie des éléments imbriqués.
