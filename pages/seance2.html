02

Les attributs globaux

Les attributs globaux sont utilisables sur presque tous les éléments HTML, quelle que soit la balise. Ils définissent des comportements transversaux : identité, accessibilité, données personnalisées, interaction clavier.

Attributs globaux les plus importants

AttributRôle détaillé
idIdentifiant unique dans la page entière. Utilisé pour les ancres (href="#mon-id"), le CSS (#mon-id { }) et le JS (document.getElementById()). Jamais deux éléments avec le même id sur une page.
classClasse CSS — peut être partagée par plusieurs éléments. On peut mettre plusieurs classes séparées par des espaces : class="bouton rouge large". C'est la base de toute feuille de style CSS.
titleInfobulle au survol de l'élément. Utile pour les abréviations, les icônes, les éléments dont le sens n'est pas évident seul.
styleCSS inline directement sur l'élément. À éviter sauf urgence — préférez les classes CSS dans un fichier séparé. Contenu = règles CSS : style="color:red; font-size:18px;"
langLangue du contenu de cet élément (remplace le lang global de <html>). Permet d'avoir une phrase en anglais dans une page française : <span lang="en">Hello world</span>.
data-*Attributs de données personnalisés. Stockent des informations utiles pour JavaScript sans affecter l'affichage. Le nom peut être tout ce que vous voulez après data-. Ex : data-user-id="42", data-color="blue".
tabindexOrdre de navigation au clavier (touche Tab). tabindex="0" = rend un élément focusable dans l'ordre naturel. tabindex="-1" = focusable uniquement par script, pas par Tab. Valeur positive = ordre forcé (déconseillé).
hiddenAttribut booléen — masque l'élément (équivaut à display:none en CSS). Ne prend pas de valeur : <div hidden>.
contenteditableRend l'élément éditable par l'utilisateur directement dans le navigateur. contenteditable="true". Utilisé par certains éditeurs rich-text.
HTML — Attributs globaux en situation 
<!-- id : unique dans la page -->
<h2 id="introduction">Introduction</h2>
<a href="#introduction">Aller à l'introduction</a>

<!-- class : partageable, plusieurs classes possibles -->
<p class="texte-important mise-en-valeur">Contenu crucial.</p>
<p class="texte-important">Aussi important.</p>

<!-- title : infobulle au survol -->
<abbr title="HyperText Markup Language" id="html-abbr">HTML</abbr>
<img src="icone.png" alt="Aide" title="Cliquez pour afficher l'aide">

<!-- lang : langue d'un passage spécifique -->
<p>En anglais on dit <span lang="en">"developer"</span>.</p>

<!-- data-* : données pour JavaScript -->
<button data-product-id="42" data-action="add-cart">Ajouter au panier</button>
<div data-theme="dark" data-user-role="admin">Panneau d'administration</div>

<!-- hidden : masque l'élément -->
<p hidden>Ce texte est masqué mais reste dans le code source.</p>

<!-- contenteditable : zone éditable -->
<div contenteditable="true" style="border: 1px solid #ccc; padding: 8px;">
  Cliquez ici pour modifier ce texte directement dans le navigateur.
</div>
💡
id vs class — règle mnémotechnique

id = comme un numéro de sécurité sociale — unique, un seul élément y répond. class = comme un groupe scolaire — plusieurs élèves peuvent être dans la même classe. En CSS : #mon-id cible un seul élément, .ma-classe cible tous les éléments de cette classe.

03

Listes avancées et liste de définitions

Les listes <ol> disposent d'attributs puissants pour contrôler la numérotation. Et HTML offre un troisième type de liste souvent méconnu : la liste de définitions <dl>.

Attributs avancés de <ol>

AttributValeursEffet
type1 A a I iType de marqueur : chiffres arabes (défaut), lettres majuscules, lettres minuscules, chiffres romains majuscules, chiffres romains minuscules.
startn'importe quel entierValeur de départ. start="5" commence à 5. start="3" avec type="A" commence à C.
reversed(attribut booléen)Inverse l'ordre de numérotation (compte à rebours). Ne prend pas de valeur.
HTML — ol avec type, start et reversed 
<!-- Chiffres romains majuscules -->
<ol type="I">
  <li>Introduction</li>
  <li>Développement</li>
  <li>Conclusion</li>
</ol>
<!-- Affiche : I. Introduction / II. Développement / III. Conclusion -->

<!-- Lettres majuscules, départ à C (start="3") -->
<ol type="A" start="3">
  <li>Option C</li>
  <li>Option D</li>
  <li>Option E</li>
</ol>

<!-- Compte à rebours (reversed) -->
<ol reversed>
  <li>Top 3</li>
  <li>Top 2</li>
  <li>Top 1 !!!</li>
</ol>
<!-- Affiche : 3, 2, 1 -->

<!-- Chaque <li> peut aussi avoir sa propre valeur avec value -->
<ol>
  <li value="10">Dixième étape</li>
  <li>Onzième étape</li>
  <li value="20">Vingtième étape (saut)</li>
</ol>
Rendu — type="I" (romain)
  1. Introduction
  2. Développement
  3. Conclusion
Rendu — reversed (compte à rebours)
  1. Top 3
  2. Top 2
  3. Top 1 !!!

04

Liens avancés : attributs supplémentaires

En plus de href, target et rel vus en séance 1, la balise <a> dispose d'attributs supplémentaires utiles pour améliorer l'expérience utilisateur.

AttributRôle détaillé
downloadForce le téléchargement du fichier au lieu de l'ouvrir. La valeur optionnelle suggère le nom du fichier téléchargé. Limite : ne fonctionne que pour les fichiers du même domaine (restrictions CORS).
hreflangIndique la langue de la page cible. Utilisé par les moteurs de recherche pour le SEO multilingue. Ex : hreflang="en", hreflang="de".
typeType MIME du contenu lié. Aide le navigateur à décider comment traiter le lien. Ex : type="application/pdf", type="image/png".
referrerpolicyContrôle les informations de référent envoyées lors du clic. no-referrer = n'envoie pas d'où vient l'utilisateur. Utile pour la confidentialité.
HTML — Liens avancés avec attributs supplémentaires 
<!-- Téléchargement forcé avec nom de fichier suggéré -->
<a href="documents/cv.pdf" download="CV_Jean_Dupont_2024.pdf">
  Télécharger mon CV (PDF)
</a>

<!-- Téléchargement sans nom suggéré (garde le nom original) -->
<a href="rapport-annuel.xlsx" download>Rapport annuel (Excel)</a>

<!-- Lien vers page en anglais (SEO multilingue) -->
<a href="https://en.wikipedia.org/wiki/HTML" hreflang="en" target="_blank" rel="noopener">
  Wikipedia HTML (version anglaise)
</a>

<!-- Lien PDF avec type MIME (aide le navigateur) -->
<a href="catalogue.pdf" type="application/pdf" target="_blank">
  Voir le catalogue (PDF, s'ouvre dans un nouvel onglet)
</a>

<!-- Email avec sujet et corps pré-remplis -->
<a href="mailto:contact@exemple.com?subject=Demande de devis&body=Bonjour,%0A%0AJe souhaite obtenir un devis.">
  Nous écrire (sujet pré-rempli)
</a>
<!-- %0A = retour à la ligne dans le corps de l'email -->

<!-- Lien avec politique de référent -->
<a href="https://partenaire.com" referrerpolicy="no-referrer" target="_blank" rel="noopener">
  Lien confidentiel (sans information de provenance)
</a>
⚠️
Limite de download — CORS

L'attribut download ne fonctionne que pour les fichiers hébergés sur votre propre domaine. Pour un fichier sur un autre domaine (ex : Google Drive, CDN tiers), le navigateur ignore download et ouvre le fichier normalement. C'est une restriction de sécurité du navigateur (politique CORS — Cross-Origin Resource Sharing).

05

Images avancées : responsive et performance

En séance 1, nous avons vu les attributs essentiels de <img>. Voici les attributs avancés pour créer des images adaptatives (responsive) et optimiser les performances de chargement.

AttributRôle détaillé
loadinglazy = charge l'image seulement quand l'utilisateur scrolle pour la voir. eager = charge immédiatement (défaut). À utiliser sur toutes les images sous le pli de la page.
srcsetFournit plusieurs versions de l'image (petite, moyenne, grande). Le navigateur choisit automatiquement la plus adaptée selon la résolution de l'écran.
sizesIndique quelle taille l'image occupera selon la largeur de l'écran (en CSS). Aide le navigateur à choisir la bonne image dans srcset avant même de charger la page.
width / heightDimensions en pixels. Toujours les définir — le navigateur réserve l'espace avant le chargement, évitant le "Cumulative Layout Shift" (CLS), critère de performance de Google.
decodingasync = décode l'image en parallèle du reste de la page (améliore les performances). sync = décode avant d'afficher le reste (bloquant).
HTML — Images optimisées et responsive 
<!-- Image avec lazy loading (recommandé pour toutes les images hors écran) -->
<img src="photo.jpg"
     alt="Description précise"
     width="800" height="600"
     loading="lazy"
     decoding="async">

<!-- Image responsive avec srcset : 3 résolutions fournies -->
<img src="image-800.jpg"
     srcset="image-400.jpg 400w,
             image-800.jpg 800w,
             image-1200.jpg 1200w"
     sizes="(max-width: 480px) 400px,
            (max-width: 900px) 800px,
            1200px"
     alt="Paysage de montagne enneigé"
     width="1200" height="800"
     loading="lazy">

<!-- <picture> : encore plus de contrôle (différents formats selon le navigateur) -->
<picture>
  <!-- WebP (format moderne, plus léger) pour les navigateurs compatibles -->
  <source srcset="photo.webp" type="image/webp">
  <!-- Fallback JPG pour les navigateurs anciens -->
  <img src="photo.jpg" alt="Paysage" width="800" height="600" loading="lazy">
</picture>
📌
Pourquoi toujours mettre width et height ?

Sans dimensions, le navigateur ne sait pas quelle place réserver avant que l'image charge. Résultat : le texte sous l'image "saute" quand l'image arrive. Ce phénomène s'appelle Cumulative Layout Shift (CLS) — Google le pénalise dans son algorithme de classement depuis 2021.

Les tableaux — approfondissement complet

Un tableau HTML sert exclusivement à afficher des données tabulaires — informations naturellement organisées en lignes et colonnes. Il ne sert pas à la mise en page.

📌
Les balises d'un tableau

<table> conteneur · <caption> titre · <thead> en-tête · <tbody> corps · <tfoot> pied · <tr> ligne · <th> cellule d'en-tête · <td> cellule de données · <colgroup> / <col> colonnes

HTML — Tableau minimal avec caption et th 
<table>

    <th>Points</th>

L'attribut scope — accessibilité des tableaux

L'attribut scope sur les <th> indique si la cellule d'en-tête s'applique à une colonne (col), une ligne (row), un groupe de colonnes (colgroup) ou un groupe de lignes (rowgroup). C'est essentiel pour les lecteurs d'écran.

Matinscope="row" HTMLCSSJavaScript
Après-midiscope="row" ProjetProjetCorrection

Attributs importants des cellules

AttributÉlémentRôle
colspan="N"<td> <th>Fusionne la cellule sur N colonnes (horizontalement). La valeur = nombre de colonnes occupées.
rowspan="N"<td> <th>Fusionne la cellule sur N lignes (verticalement). Les cellules absorbées disparaissent du code.
scope="col"<th>Indique que ce th est un en-tête de colonne. Obligatoire pour l'accessibilité.
scope="row"<th>Indique que ce th est un en-tête de ligne. Permet aux lecteurs d'écran d'associer données et en-têtes.
headers="id"<td>Pour les tableaux complexes : liste les id des th associés à cette cellule.
span="N"<col>La colonne couvre N colonnes (équivalent de colspan pour <col>).
⚠️
Ne jamais utiliser les tableaux pour la mise en page

Dans les années 1990, les développeurs utilisaient <table> pour créer des colonnes et positionner des éléments. Cette pratique est aujourd'hui strictement à éviter : illisible pour les lecteurs d'écran (annonce des données tabulaires là où il y a de la mise en page), difficile à maintenir, et pénalisé par Google. La mise en page se fait avec CSS Flexbox et CSS Grid.

07

Balises sémantiques HTML5

HTML5 (2014) a introduit des balises qui décrivent la fonction des zones de la page. Ces balises remplacent les <div> anonymes et donnent du sens à la structure — crucial pour l'accessibilité, le SEO et la lisibilité du code.

🏗️
Analogie : les pièces d'une maison

Avant HTML5, tout le monde vivait dans des <div> anonymes — comme si toute une maison n'avait que des "pièces". HTML5 a nommé les pièces : <header> = entrée, <nav> = couloir, <main> = salon, <aside> = bureau, <footer> = cave. Un plan de maison avec des noms de pièces est bien plus lisible qu'un plan avec des rectangles numérotés.

Schéma d'une page sémantique HTML5

<header> Logo, titre du site, slogan — en-tête de la page ou d'une section
<nav> Menu de navigation principal, fil d'Ariane, liens de catégories
<main> Contenu central et unique de la page — un seul par page
<article> Contenu autonome et redistribuable : billet de blog, fiche produit, commentaire
<section> Groupe thématique avec un titre — chapitre, onglet, catégorie
<aside> Contenu complémentaire : barre latérale, publicité, liens connexes, biographie de l'auteur
<footer> Pied de page : copyright, mentions légales, liens secondaires, contact
BaliseÉquivalent avant HTML5Quand l'utiliser
<header><div id="header">En-tête de page (logo + nav) OU en-tête d'un article/section. Peut apparaître plusieurs fois sur une page.
<nav><div id="navigation">Groupe de liens de navigation importants. Ne pas l'utiliser pour chaque groupe de liens — réservez aux navigations principales.
<main><div id="main">Contenu central unique de la page. Un seul par document. Les lecteurs d'écran permettent de sauter directement au main.
<article><div class="article">Contenu autonome — peut être extrait, redistribué ou syndiqué seul (RSS, email). Ex : billet de blog, commentaire, fiche produit.
<section><div class="section">Regroupement thématique. Doit avoir un titre (<h2>…). Si le contenu peut être autonome → préférer <article>.
<aside><div id="sidebar">Contenu tangentiellement lié au contenu principal. Peut être retiré sans que la page perde son sens.
<footer><div id="footer">Pied de page ou pied d'un article/section. Peut apparaître plusieurs fois.
HTML — Page complète avec balises sémantiques HTML5 
<body>
  <header>
    <h1><a href="/">MonBlog</a></h1>
    <nav>
      <ul>
        <li><a href="/">Accueil</a></li>
        <li><a href="/blog">Articles</a></li>
        <li><a href="/contact">Contact</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <header>
        <!-- <header> peut aussi aller dans un <article> -->
        <h2>Comment apprendre le HTML efficacement</h2>
        <p>Publié le <time datetime="2024-03-15">15 mars 2024</time></p>
      </header>

      <section>
        <h3>Les fondamentaux</h3>
        <p>Commencez par maîtriser la structure de base...</p>
      </section>

      <section>
        <h3>La pratique quotidienne</h3>
        <p>Codez au moins 20 minutes par jour...</p>
      </section>

      <footer>
        <!-- pied de l'article -->
        <p>Tags : <a href="/tag/html">HTML</a>, <a href="/tag/web">Web</a></p>
      </footer>
    </article>
  </main>

  <aside>
    <h2>Articles récents</h2>
    <ul>
      <li><a href="/blog/css">Introduction au CSS</a></li>
      <li><a href="/blog/js">Débuter en JavaScript</a></li>
    </ul>
  </aside>

  <footer>
    <p>&copy; 2024 MonBlog. <a href="/mentions">Mentions légales</a></p>
  </footer>
</body>
📌
Sémantique ≠ style visuel

Ces balises n'ont aucune apparence visuelle particulière par défaut — elles ressemblent à des <div>. Leur valeur est invisible mais réelle : un lecteur d'écran annonce "région de navigation principale" pour <nav>, "contenu principal" pour <main>. Google comprend la structure de votre page. Et votre collègue qui lit le code comprend instantanément le rôle de chaque zone.

08

Entités HTML supplémentaires & time

Nous avons vu les entités fondamentales en séance 1. Voici des entités typographiques courantes en français, plus la balise <time> souvent oubliée mais très utile.

HTML — Entités typographiques et balise time 
<!-- Guillemets français typographiques -->
<p>&laquo;&nbsp;Bonjour le monde&nbsp;&raquo;</p>
<!-- Résultat : « Bonjour le monde » (espaces insécables avant et après) -->

<!-- Tirets typographiques -->
<p>Paris&ndash;Lyon en TGV</p>         <!-- Plage : Paris–Lyon -->

💡
La balise <time> — invisible mais précieuse

<time datetime="2024-03-15">15 mars 2024</time> permet aux moteurs de recherche et aux applications de calendrier de comprendre exactement quelle date ou heure vous mentionnez. Google peut afficher cette date dans les résultats de recherche (rich snippets). L'attribut datetime accepte les formats ISO 8601 : AAAA-MM-JJ, HH:MM, AAAA-MM-JJTHH:MM.

EX 01📋 Listes avancées
  1. Créez une liste ordonnée de vos 5 films préférés en chiffres romains (type="I").
  2. Créez une liste de définitions <dl> avec 4 termes informatiques et leurs définitions.

  1. Créez une galerie de 4 images depuis https://picsum.photos/seed/NOM/600/400 (remplacez NOM par des mots différents).

  3. Encadrez chaque image dans <figure> + <figcaption> + <a> (image cliquable vers l'URL originale).
  4. Ajoutez un attribut data-category="nature" (ou autre catégorie) sur chaque <figure>.
EX 03📊 Tableau de planning
  1. Créez un emploi du temps sur 5 jours × 4 créneaux horaires.
  2. Utilisez <th scope="col"> pour les jours et <th scope="row"> pour les heures.
  3. Ajoutez au moins une fusion colspan="2" (cours de 2 heures) et une rowspan="2" (activité sur 2 jours).

EX 04🏗️ Page sémantique complète

  3. Dans <article> : deux <section> avec des <h2>.

Mini-glossaire

colspan
Fusionne une cellule sur N colonnes (horizontalement). La valeur = nombre de colonnes occupées.
rowspan
Fusionne une cellule sur N lignes (verticalement). Les cellules "absorbées" ne doivent plus apparaître dans le code.
thead / tbody / tfoot
Structuration sémantique d'un tableau en en-tête, corps et pied. Permet la répétition de l'en-tête à l'impression et lors du scroll horizontal.
sémantique HTML5
Principe de choisir les balises selon leur signification structurelle. header/nav/main/article/section/aside/footer remplacent les div anonymes.
data-*
Attribut personnalisé pour stocker des données dans un élément HTML. Accessible en JavaScript via dataset. Ex : data-user-id="42".
loading="lazy"
Différer le chargement d'une image jusqu'à ce qu'elle entre dans la zone visible. Améliore le temps de chargement initial.
srcset
Liste de versions de l'image à différentes résolutions. Le navigateur choisit automatiquement la plus adaptée à l'écran.
CLS
Cumulative Layout Shift — déplacement des éléments au chargement. Pénalisé par Google. Prévenu en définissant width et height sur les images.
<time>
Balise pour les dates et heures. L'attribut datetime (format ISO 8601) rend la date lisible par les machines : moteurs de recherche, agendas.
CORS
Cross-Origin Resource Sharing — politique de sécurité du navigateur restreignant les requêtes entre domaines différents. Explique pourquoi download ne fonctionne pas sur des fichiers distants.
