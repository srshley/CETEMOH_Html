01

Les formulaires — à quoi ça sert ?

Les formulaires sont le principal mécanisme d'interaction utilisateur → serveur sur le web. Connexion, inscription, recherche, commande, prise de contact — tout passe par un formulaire HTML. C'est là que le web devient vraiment bidirectionnel.

📌
Rappel séances 1 & 2

Structure HTML5 complète · Balises sémantiques header/main/footer · Tableaux avec scope · Listes de définitions · Chemins relatifs et absolus · Formats d'images — tout ceci est utilisé dans cette séance.

📝
Collecter
Recueillir des données de l'utilisateur (nom, email, mot de passe…)
✅
Valider
Vérifier que les données sont correctes avant envoi (HTML5 natif)
📨
Envoyer
Transmettre les données au serveur via GET ou POST
📋
Analogie — Le formulaire papier numérisé

Un formulaire HTML est exactement comme un formulaire papier : des champs à remplir (<input>), des cases à cocher (type="checkbox"), des listes déroulantes (<select>), des libellés qui indiquent quoi mettre où (<label>), et un bouton d'envoi (type="submit"). La différence : le navigateur peut vérifier les réponses avant l'envoi.

02

<form> — action, method et structure

La balise <form> est le conteneur de tous les champs. Ses deux attributs fondamentaux définissent où envoyer les données (action) et comment les envoyer (method).

GET vs POST — quelle différence ?

method="GET"

Les données sont ajoutées dans l'URL : ?nom=Alice&age=30

✓ URL partageable et mémorisable
✓ Résultats indexables par Google
✗ Données visibles dans l'URL
✗ Limité en taille (~2000 caractères)
✗ Jamais pour des données sensibles

Utiliser pour : formulaires de recherche, filtres, navigation

method="POST"

Les données sont envoyées dans le corps de la requête HTTP (invisibles dans l'URL)

✓ Données non visibles dans l'URL
✓ Pas de limite de taille
✓ Adapté aux données sensibles
✗ URL non partageable
✗ Non indexé par Google

Utiliser pour : connexion, inscription, commande, contact

HTML — Structure complète d'un formulaire 
<form action="/traitement.php" method="post" novalidate>
  <!--
    action : URL de traitement (page serveur qui reçoit les données)
    method : "get" ou "post"
    novalidate : désactive la validation native (pour tester)
    autocomplete : "on" (défaut) ou "off"
    enctype : "multipart/form-data" obligatoire si le form contient des fichiers
  -->

  <!-- Champs ici -->
  <label for="nom">Votre nom</label>
  <input type="text" id="nom" name="nom" required>

  <button type="submit">Envoyer</button>
</form>

<!-- Formulaire de recherche (GET : l'URL devient /recherche?q=html) -->
<form action="/recherche" method="get">
  <label for="q">Rechercher</label>
  <input type="search" id="q" name="q" placeholder="HTML, CSS, JavaScript...">
  <button type="submit">Rechercher</button>
</form>

<!-- Formulaire d'envoi de fichier (enctype obligatoire) -->
<form action="/upload" method="post" enctype="multipart/form-data">
  <label for="fichier">Choisir un fichier</label>
  <input type="file" id="fichier" name="fichier" accept=".pdf,.jpg,.png">
  <button type="submit">Envoyer le fichier</button>
</form>

Anatomie visuelle d'un formulaire

<form action="/login" method="post">

03

<input> — la balise la plus polyvalente du web

La balise <input> est une balise orpheline dont le comportement change entièrement selon la valeur de son attribut type. HTML5 a ajouté de nombreux types spécialisés qui améliorent l'expérience mobile et la validation.

📌
L'attribut name est indispensable

Sans name, les données du champ ne sont pas envoyées au serveur. C'est le nom de la variable côté serveur. Exemple : name="email" → le serveur reçoit $_POST["email"] en PHP.

<input type="text" name="prenom" id="prenom" placeholder="Votre prénom">

<!-- Email — valide le format automatiquement -->
<input type="email" name="email" id="email" placeholder="vous@exemple.com" required>

<!-- Mot de passe — masque la saisie -->
<input type="password" name="password" id="password" minlength="8">

<!-- Nombre avec limites -->
<input type="number" name="age" id="age" min="0" max="120" step="1" value="25">

<!-- Téléphone — affiche clavier téléphone sur mobile -->
<input type="tel" name="telephone" id="telephone" pattern="[0-9]{10}"
       placeholder="0600000000">

<!-- URL — valide le format https://... -->
<input type="url" name="site" id="site" placeholder="https://exemple.com">

<!-- Date — sélecteur de calendrier natif -->
<input type="date" name="naissance" id="naissance"
       min="1900-01-01" max="2024-12-31">

<!-- Heure -->
<input type="time" name="rdv" id="rdv" min="09:00" max="18:00" step="900">
<!-- step="900" = paliers de 15 minutes (900 secondes) -->

<!-- Curseur glissant -->
<input type="range" name="satisfaction" id="satisfaction"
       min="1" max="10" value="5" step="1">

<!-- Couleur -->
<input type="color" name="couleur" id="couleur" value="#3B82F6">

<!-- Case à cocher -->
<input type="checkbox" name="newsletter" id="newsletter" value="oui" checked>

<!-- Boutons radio (même name = groupe exclusif) -->
<input type="radio" name="genre" id="h" value="homme">
<input type="radio" name="genre" id="f" value="femme">
<input type="radio" name="genre" id="autre" value="autre">

<!-- Fichier avec types acceptés -->
<input type="file" name="cv" id="cv" accept=".pdf,.doc,.docx" multiple>

<!-- Champ caché (invisible, envoie une valeur fixe) -->
<input type="hidden" name="source" value="page-accueil">

<!-- Boutons de formulaire -->
<input type="submit" value="Envoyer">
<input type="reset" value="Tout effacer">
<button type="submit">Envoyer (préféré à input submit)</button>

04

<label> — lier le texte au champ

La balise <label> est obligatoire pour l'accessibilité. Elle lie textuellement un libellé à son champ. Sans label, un lecteur d'écran ne peut pas annoncer à l'utilisateur ce qu'il doit saisir dans le champ.

✅ Label explicite (recommandé)

L'attribut for du label correspond au id du champ. Le clic sur le label active le champ.

HTML
<label for="email">
  Adresse email
</label>
<input type="email"
       id="email"
       name="email">

✅ Label enveloppant

L'input est à l'intérieur du label. Pas besoin de for/id. Pratique pour checkbox et radio.

HTML
<label>
  <input type="checkbox"
         name="cgv"
         value="oui">
  J'accepte les CGU
</label>
HTML — Formulaire complet avec labels corrects 
<form action="/inscription" method="post">

  <!-- label explicite : for="id" -->
  <label for="prenom">Prénom *</label>
  <input type="text" id="prenom" name="prenom" required autocomplete="given-name">

  <label for="email">Adresse email *</label>
  <input type="email" id="email" name="email" required autocomplete="email">

  <label for="mdp">Mot de passe * (8 caractères minimum)</label>
  <input type="password" id="mdp" name="password" required minlength="8"
         autocomplete="new-password">

  <label for="naissance">Date de naissance</label>
  <input type="date" id="naissance" name="naissance"
         min="1900-01-01" max="2010-01-01">

  <!-- Radio avec label enveloppant -->
  <p>Genre :</p>
  <label><input type="radio" name="genre" value="h"> Homme</label>
  <label><input type="radio" name="genre" value="f"> Femme</label>
  <label><input type="radio" name="genre" value="a"> Autre / Non précisé</label>

  <!-- Checkbox avec label enveloppant -->
  <label>
    <input type="checkbox" name="newsletter" value="oui">
    Je souhaite recevoir la newsletter (optionnel)
  </label>

  <label>
    <input type="checkbox" name="cgv" value="accepte" required>
    J'accepte les <a href="/cgu">conditions générales d'utilisation</a> *
  </label>

  <button type="submit">Créer mon compte</button>
  <p><small>* Champs obligatoires</small></p>
</form>
Rendu navigateur — Formulaire d'inscription
Genre :
   

* Champs obligatoires

⚠️
Jamais de champ sans label

Un input sans label associé est inaccessible pour les utilisateurs de lecteurs d'écran. Le navigateur annonce juste "champ de saisie" sans préciser à quoi il sert. C'est aussi illégal dans certains pays (WCAG 2.1 niveau AA). N'utilisez pas l'attribut placeholder comme substitut au label — le placeholder disparaît dès que l'utilisateur commence à saisir.

05

<textarea> et <select> — zones de texte et listes déroulantes

<textarea> — zone de texte multilignes

Contrairement à <input type="text">, <textarea> est une balise à fermeture (pas orpheline). Le contenu entre les balises = valeur par défaut. L'utilisateur peut redimensionner le champ en tirant le coin.

HTML — textarea avec tous ses attributs 
<label for="message">Votre message</label>
<textarea
  id="message"
  name="message"
  rows="6"
  cols="50"
  placeholder="Décrivez votre demande en détail..."
  minlength="20"
  maxlength="1000"
  required
  spellcheck="true"
></textarea>

<!--
  rows : nombre de lignes visibles
  cols : largeur approximative en caractères (souvent ignoré, CSS préféré)
  resize via CSS : textarea { resize: vertical; } ou none / horizontal / both
-->
Rendu — textarea
0 / 1000 caractères

<select> — liste déroulante

Permet à l'utilisateur de choisir parmi une liste d'options prédéfinies. Plus pratique qu'une série de boutons radio quand il y a plus de 4-5 choix.

HTML — select, option et optgroup 
<!-- Select simple -->
<label for="pays">Pays</label>
<select id="pays" name="pays" required>
  <option value="">-- Choisir un pays --</option> <!-- valeur vide = placeholder -->
  <option value="fr" selected>France</option>       <!-- selected = sélectionné par défaut -->
  <option value="be">Belgique</option>
  <option value="ch">Suisse</option>
  <option value="ca">Canada</option>
</select>

<!-- Select avec groupes d'options (optgroup) -->
<label for="langage">Langage préféré</label>
<select id="langage" name="langage">
  <optgroup label="Langages web">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
    <option value="js">JavaScript</option>
  </optgroup>
  <optgroup label="Langages back-end">
    <option value="php">PHP</option>
    <option value="python">Python</option>
    <option value="java">Java</option>
  </optgroup>
</select>

<!-- Sélection multiple (Ctrl+clic ou Cmd+clic) -->
<label for="competences">Compétences (plusieurs sélections possibles)</label>
<select id="competences" name="competences[]" multiple size="5">
  <option value="html">HTML</option>
  <option value="css">CSS</option>
  <option value="js">JavaScript</option>
  <option value="react">React</option>
  <option value="sql">SQL</option>
</select>
Rendu — select simple et avec optgroup

<datalist> — autocomplétion avec suggestions

Combine la liberté d'un champ texte avec des suggestions prédéfinies. L'utilisateur peut choisir dans la liste ou saisir sa propre valeur.

HTML — datalist pour autocomplétion 
<label for="ville">Ville</label>
<input type="text" id="ville" name="ville"
       list="villes-suggestions"
       placeholder="Commencez à taper...">

<datalist id="villes-suggestions">
  <option value="Paris">
  <option value="Lyon">
  <option value="Marseille">
  <option value="Bordeaux">
  <option value="Toulouse">
  <option value="Nantes">
</datalist>
<!-- L'input et le datalist sont liés par list="id-du-datalist" -->
06

<fieldset> et <legend> — grouper les champs

Ces deux balises permettent de regrouper sémantiquement des champs connexes. Les lecteurs d'écran annoncent le nom du groupe avant chaque champ — essentiel pour les formulaires longs.

HTML — Formulaire structuré avec fieldset et legend 
<form action="/commande" method="post">

  <!-- Groupe 1 : Informations personnelles -->
  <fieldset>
    <legend>👤 Informations personnelles</legend>

    <label for="nom">Nom *</label>
    <input type="text" id="nom" name="nom" required autocomplete="family-name">

    <label for="prenom">Prénom *</label>
    <input type="text" id="prenom" name="prenom" required autocomplete="given-name">

    <label for="email">Email *</label>
    <input type="email" id="email" name="email" required autocomplete="email">

    <label for="tel">Téléphone</label>
    <input type="tel" id="tel" name="telephone" autocomplete="tel">
  </fieldset>

  <!-- Groupe 2 : Adresse de livraison -->
  <fieldset>
    <legend>📦 Adresse de livraison</legend>

    <label for="adresse">Adresse *</label>
    <input type="text" id="adresse" name="adresse" required autocomplete="street-address">

    <label for="cp">Code postal *</label>
    <input type="text" id="cp" name="code_postal" pattern="[0-9]{5}"
           required autocomplete="postal-code" placeholder="75001">

    <label for="ville">Ville *</label>
    <input type="text" id="ville" name="ville" required autocomplete="address-level2">
  </fieldset>

  <!-- Groupe 3 : Choix de livraison (radio buttons) -->
  <fieldset>
    <legend>🚚 Mode de livraison</legend>

    <label>
      <input type="radio" name="livraison" value="standard" checked>
      Standard (3–5 jours) — Gratuit
    </label>
    <label>
      <input type="radio" name="livraison" value="express">
      Express (24h) — 5,90 €
    </label>
    <label>
      <input type="radio" name="livraison" value="relais">
      Point relais (2–3 jours) — 2,90 €
    </label>
  </fieldset>

  <!-- Groupe 4 : Paiement (fieldset désactivé) -->
  <fieldset disabled>
    <legend>💳 Paiement (bientôt disponible)</legend>
    <label for="carte">Numéro de carte</label>
    <input type="text" id="carte" name="carte" placeholder="1234 5678 9012 3456">
  </fieldset>
  <!-- disabled sur fieldset = tous les champs du groupe sont désactivés -->

  <button type="submit">Valider la commande</button>
</form>
Rendu — fieldset et legend
👤 Informations personnelles
🚚 Mode de livraison
💡
disabled sur fieldset — désactiver tout un groupe

Ajouter disabled sur un <fieldset> désactive tous les champs à l'intérieur en une seule ligne — sans modifier chaque input. Pratique pour les étapes suivantes d'un formulaire multi-étapes, ou une fonctionnalité temporairement indisponible.

07

Validation native HTML5 — sans JavaScript

HTML5 offre une validation intégrée déclenchée à la soumission du formulaire. Le navigateur vérifie les champs et affiche des messages d'erreur natifs, traduits dans la langue de l'utilisateur — sans une seule ligne de JavaScript.

Démonstration visuelle — champ valide vs invalide

✅ Champ valide
✓ Format email correct
❌ Champ invalide
✗ Veuillez inclure un "@" dans l'adresse email.
HTML — Attributs de validation HTML5 
<!-- required : champ obligatoire -->
<input type="text" name="nom" required>

<!-- minlength / maxlength : longueur du texte -->
<input type="text" name="pseudo" minlength="3" maxlength="20"
       placeholder="3 à 20 caractères">

<!-- min / max : valeur numérique ou date -->
<input type="number" name="age" min="18" max="99">
<input type="date" name="rdv" min="2024-01-01" max="2024-12-31">

<!-- step : incrément autorisé -->
<input type="number" name="prix" min="0" max="1000" step="0.01">
<input type="time" name="heure" min="09:00" max="17:00" step="900">
<!-- step="900" = paliers de 15 minutes (900 secondes) -->

<!-- pattern : expression régulière personnalisée -->
<input type="text" name="code_postal"
       pattern="[0-9]{5}"
       title="5 chiffres exactement (ex: 75001)"
       placeholder="75001">

<input type="tel" name="telephone"
       pattern="0[0-9]{9}"
       title="10 chiffres commençant par 0 (ex: 0600000000)">

<input type="text" name="identifiant"
       pattern="[a-zA-Z0-9_]{4,16}"
       title="4 à 16 caractères : lettres, chiffres ou underscore">

<!-- Mot de passe fort : min 8 chars, lettre ET chiffre -->
<input type="password" name="password"
       pattern="(?=.*[A-Za-z])(?=.*\d).{8,}"
       title="Minimum 8 caractères avec au moins une lettre et un chiffre">
AttributS'applique àComportement
requiredTous les champsBloque la soumission si le champ est vide. Affiche "Ce champ est obligatoire."
minlengthtext, email, password, textareaLongueur minimale en caractères. Le champ doit avoir au moins N caractères.
maxlengthtext, email, password, textareaLongueur maximale. Bloque la saisie au-delà de N caractères (pas de message d'erreur).
minnumber, date, time, rangeValeur minimale. Pour une date : format AAAA-MM-JJ.
maxnumber, date, time, rangeValeur maximale.
stepnumber, time, rangeIncrément accepté. step="0.01" autorise les décimales. step="any" = toute valeur.
patterntext, tel, email, urlExpression régulière que la valeur doit respecter. Toujours accompagner d'un title explicatif.
titleToutMessage affiché dans l'infobulle de validation (explique le format attendu).
💡
Validation HTML5 ≠ validation serveur

La validation HTML5 est faite côté client (navigateur) : rapide et agréable pour l'utilisateur, mais contournable. Un utilisateur malveillant peut désactiver la validation via DevTools ou envoyer des requêtes directes au serveur. Toujours valider aussi côté serveur (PHP, Python, Node.js…) — la validation HTML5 est un confort UX, pas une sécurité.

📌
novalidate — pendant le développement

Ajoutez novalidate sur la balise <form> pour désactiver temporairement toute validation native. Utile quand vous testez l'envoi du formulaire sans vouloir remplir tous les champs. À retirer avant la mise en production.

08

Attributs avancés — UX et accessibilité

Ces attributs améliorent l'expérience utilisateur (autocomplétion, focus automatique, aide à la saisie) et l'accessibilité (ARIA).

AttributValeursRôle détaillé
placeholdertexteTexte indicatif grisé qui disparaît à la saisie. Ne remplace pas le label. Utile pour les exemples de format.
autocompletevaleur sémantiqueAide le navigateur à proposer les bonnes valeurs mémorisées. Utiliser les valeurs standardisées : name, email, tel, street-address, postal-code, current-password…
autofocus(booléen)Donne automatiquement le focus à ce champ au chargement de la page. Un seul par page. Pratique sur les pages de connexion.
readonly(booléen)L'utilisateur voit la valeur mais ne peut pas la modifier. La valeur est quand même envoyée au serveur (différent de disabled).
disabled(booléen)Champ grisé, non interactif. La valeur n'est pas envoyée au serveur.
valuetexteValeur par défaut pré-remplie. L'utilisateur peut la modifier (différent de readonly).
nameidentifiantIndispensable — clé utilisée côté serveur. Sans name, le champ n'est pas envoyé.
multiple(booléen)Sur type="file" : sélection de plusieurs fichiers. Sur <select> : sélection multiple (Ctrl+clic).
accepttypes MIME ou extensionsSur type="file" : filtre les fichiers dans le sélecteur. Ex : accept=".pdf,.jpg,image/*".
inputmodenumeric decimal tel…Contrôle le type de clavier affiché sur mobile, indépendamment du type. Plus flexible que le type seul.
aria-describedbyid d'un élémentPointe vers un élément qui décrit le champ (aide contextuelle, contraintes). Lu par les lecteurs d'écran.
aria-invalidtrue / falseIndique explicitement aux lecteurs d'écran si le champ est en erreur (utile avec validation JavaScript).
HTML — Attributs avancés en situation 
<!-- autofocus : focus immédiat au chargement -->
<input type="text" name="recherche" id="recherche"
       placeholder="Rechercher..."
       autofocus
       autocomplete="off">

<!-- readonly : visible mais non modifiable (envoyé au serveur) -->
<label for="ref">Référence commande</label>
<input type="text" id="ref" name="reference"
       value="CMD-2024-00842"
       readonly>

<!-- disabled : champ inactif (NON envoyé au serveur) -->
<label for="promo">Code promo</label>
<input type="text" id="promo" name="code_promo"
       placeholder="PROMO2024"
       disabled>

<!-- autocomplete sémantique -->
<input type="text" name="prenom" autocomplete="given-name">
<input type="text" name="nom" autocomplete="family-name">
<input type="email" name="email" autocomplete="email">
<input type="tel" name="tel" autocomplete="tel">
<input type="text" name="rue" autocomplete="street-address">
<input type="text" name="cp" autocomplete="postal-code">
<input type="password" name="mdp" autocomplete="current-password">
<input type="password" name="new_mdp" autocomplete="new-password">

<!-- inputmode : clavier mobile personnalisé -->
<input type="text" name="code_pin" inputmode="numeric"
       pattern="[0-9]{4}" placeholder="Code PIN (4 chiffres)">

<!-- Champ avec aide contextuelle accessible -->
<label for="mdp2">Mot de passe</label>
<input type="password" id="mdp2" name="mdp2"
       required minlength="8"
       aria-describedby="aide-mdp">
<p id="aide-mdp">
  <small>Minimum 8 caractères. Au moins une majuscule et un chiffre.</small>
</p>

<!-- Champ avec multiple fichiers -->
<label for="photos">Téléverser des photos (plusieurs possibles)</label>
<input type="file" id="photos" name="photos[]"
       accept="image/*"
       multiple>
💡
autocomplete sémantique — les vraies valeurs

Les valeurs de autocomplete suivent un standard HTML : given-name (prénom), family-name (nom), email, tel, street-address, postal-code, country, current-password, new-password, one-time-code (OTP/SMS)… Le navigateur peut ainsi pré-remplir correctement le bon champ au bon endroit, même si vos attributs name sont différents.

09

Exercices pratiques

EX 01📝 Formulaire de contact
  1. Créez une page contact.html avec la structure HTML5 complète et les balises sémantiques header/main/footer.
  2. Dans <main> : un formulaire method="post" action="/contact" avec un seul <fieldset>.
  3. Champs requis : Nom (text, required), Email (email, required), Sujet (select avec 5 options groupées en 2 optgroup), Message (textarea, 6 lignes, minlength=20, maxlength=2000).
  4. Champs optionnels : Téléphone (tel, pattern), Fichier joint (file, accept=".pdf,.jpg,.png").
  5. Tous les champs doivent avoir un <label> correctement associé via for/id.
  6. Un bouton submit et un bouton reset.
  7. En dessous du formulaire : une liste <dl> avec les coordonnées de contact (adresse, téléphone, email, horaires).
EX 02🔐 Formulaire d'inscription complet
  1. Créez inscription.html. Formulaire method="post" action="/inscription" en 3 <fieldset> : "Identité", "Compte", "Préférences".
  2. Identité : Nom, Prénom, Date de naissance (date, max = aujourd'hui - 18 ans), Genre (3 radio buttons).
  3. Compte : Email (email, required), Pseudo (text, minlength=3, maxlength=20, pattern=^[a-zA-Z0-9_]+$), Mot de passe (password, minlength=8), Confirmation MDP (password).
  4. Préférences : Centres d'intérêt (select multiple), Newsletter (checkbox), CGU (checkbox required).
  5. Ajoutez des autocomplete sémantiques sur tous les champs appropriés.
  6. Ajoutez aria-describedby sur le champ mot de passe pointant vers un paragraphe d'aide.
  7. Testez la validation en cliquant Soumettre avec des champs vides ou mal remplis.
EX 03🛒 Formulaire de commande avec tableau
  1. Créez commande.html. En haut : un tableau récapitulatif de commande (thead/tbody/tfoot, colspan, scope).
  2. En dessous : un formulaire en 2 fieldsets — "Livraison" et "Mode de paiement".
  3. Livraison : adresse, code postal (pattern 5 chiffres), ville, pays (select).
  4. Mode de paiement : 3 radio buttons (Carte bancaire, PayPal, Virement). Le troisième désactivé (disabled).
  5. Un champ type="hidden" contenant l'identifiant de commande fictif.
  6. Ajoutez tous les autocomplete appropriés pour l'adresse.
EX 04🔍 Formulaire de recherche avancée
  1. Créez recherche.html. Formulaire method="get" action="/resultats" (pas de fieldset nécessaire).
  2. Champs : Mot-clé (search, autofocus), Catégorie (select), Prix min/max (number, min=0, step=5), Date de parution (date, min="2000-01-01"), Langue (datalist avec 6 suggestions).
  3. Bouton submit "Rechercher" ET un lien "Réinitialiser les filtres" pointant vers la même page.
  4. Observez dans DevTools : après la soumission, l'URL contient les paramètres (comportement GET).
🧪
Astuce — Tester vos formulaires sans serveur

Pour voir les données envoyées sans avoir de serveur, changez temporairement l'action vers https://httpbin.org/post (pour POST) ou https://httpbin.org/get (pour GET). Ce service gratuit renvoie les données reçues au format JSON — parfait pour déboguer.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Balise <form> : action (destination), method (GET vs POST), enctype (fichiers), novalidate
  • Différence GET (URL, recherche) vs POST (corps, données sensibles)
  • Tous les types d'input : text, email, password, number, tel, url, date, time, range, color, checkbox, radio, file, hidden, submit, reset
  • Association label/input : for/id (label explicite) et label enveloppant
  • textarea : zone de texte multilignes avec rows, minlength, maxlength
  • select / option / optgroup : liste déroulante et groupes d'options
  • datalist : suggestions d'autocomplétion sur un champ texte
  • fieldset / legend : grouper sémantiquement les champs
  • Validation native : required, minlength, maxlength, min, max, step, pattern, title
  • Attributs avancés : placeholder, autocomplete, autofocus, readonly, disabled, multiple, accept, inputmode
  • Accessibilité : aria-describedby, aria-invalid, jamais de champ sans label

Mini-glossaire

<form>
Conteneur du formulaire. action = URL de traitement, method = GET ou POST.
<input>
Balise orpheline dont le comportement dépend du type. La plus polyvalente du HTML.
<label>
Libellé d'un champ. Lié via for="id-du-champ" ou en enveloppant l'input. Obligatoire pour l'accessibilité.
<textarea>
Zone de texte multilignes. Balise à fermeture. Contenu entre les balises = valeur par défaut.
<select>
Liste déroulante. Les choix sont des <option>, regroupables en <optgroup>.
<datalist>
Liste de suggestions pour un <input> via l'attribut list="id-datalist". L'utilisateur peut aussi saisir sa propre valeur.
<fieldset>
Groupe visuel et sémantique de champs. disabled sur fieldset désactive tous ses champs.
<legend>
Titre du fieldset, toujours premier enfant du fieldset.
required
Attribut booléen — bloque la soumission si le champ est vide.
pattern
Expression régulière que la valeur doit respecter. Toujours accompagner d'un title explicatif.
autocomplete
Valeur sémantique indiquant au navigateur quelles données mémorisées proposer. Ex : given-name, email, postal-code.
GET vs POST
GET : données dans l'URL (recherche, filtres). POST : données dans le corps HTTP (connexion, inscription, commande).
