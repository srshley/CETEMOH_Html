01

C'est quoi JavaScript ?

🖱️ Clic utilisateur → 📡 Événement DOM
📡 Événement DOM → ⚡ addEventListener
⚡ addEventListener → 🎯 Fonction callback
🎯 Fonction callback → ✨ DOM mis à jour

JavaScript (souvent abrégé JS) est le langage de programmation côté client du web. Il permet de rendre une page dynamique : réagir à un clic, valider un formulaire, animer un élément, charger des données sans recharger la page.

⚡
Les muscles de la page

HTML est le squelette, CSS les vêtements, JavaScript les muscles. Sans JS, la page est statique. Avec JS, elle devient interactive.

Où s'exécute JavaScript ?

Principalement dans le navigateur (Chrome, Firefox, Safari, Edge). C'est ce qu'on appelle du client-side JS. Depuis 2009 avec Node.js, JS peut aussi s'exécuter côté serveur.

ECMAScript — la norme derrière JS

JavaScript est basé sur une norme appelée ECMAScript (ES). Chaque année, une nouvelle version apporte des fonctionnalités :

VersionAnnéeApports clés
ES52009Strict mode, JSON, Array methods
ES6 / ES20152015let/const, fonctions fléchées, classes, modules, Promises, template strings
ES20172017async/await, Object.entries/values
ES20202020Optional chaining (?.), nullish coalescing (??)
ES20232023Array findLast, toSorted, toReversed
💡
JS n'est pas Java

Malgré le nom similaire, JavaScript et Java sont deux langages totalement différents. JavaScript a été nommé ainsi pour des raisons marketing à l'époque. Ne confondez pas !

02

Où écrire du JavaScript

Il y a trois endroits principaux pour placer votre code JavaScript :

1. Dans une balise <script> dans le HTML

HTML — Script inline 
<!DOCTYPE html>
<html>
<body>
  <h1>Ma page</h1>
  <script>
    alert('Hello depuis le script !');
  </script>
</body>
</html>

2. Dans la console du navigateur

Ouvrez les outils développeur (F12 ou Ctrl+Maj+I), onglet Console. Tapez directement du JS et validez avec Entrée. Idéal pour tester rapidement.

3. Dans un fichier .js externe (recommandé)

HTML — Lien vers fichier externe 
<!-- Tout en bas du body, avant </body> -->
<script src="app.js"></script>
JavaScript — app.js 
// app.js — tout le code JS va ici
console.log('app.js est chargé');
let nom = 'Alice';
console.log('Bonjour ' + nom);
⚠️
Toujours placer <script> avant </body>

Si vous mettez le script dans <head>, le navigateur bloquera le rendu HTML le temps de charger et exécuter le JS. En plaçant le script en bas du body, le HTML est déjà affiché quand JS s'exécute. Alternative : l'attribut defer.

03

let et const — les variables modernes

Depuis ES6 (2015), on déclare les variables avec let et const.

let — variable qui peut changer

Utilisez let quand la valeur va changer au fil du temps. Portée au bloc {}.

let age = 25;
age = 26; // OK
let age = 30; // ERREUR !

const — valeur constante

Utilisez const pour les valeurs qui ne doivent pas changer. Obligatoire d'assigner une valeur dès la déclaration.

const prenom = 'Alice';
prenom = 'Bob'; // ERREUR !
const PI = 3.14159;
💡
Règle d'or : const par défaut, let si nécessaire

Utilisez toujours const par défaut. Si vous avez besoin de réassigner la variable, passez à let. Cela rend le code plus prévisible et évite les erreurs.

⚠️ const ne rend pas l'objet/tableau immuable : on peut modifier ses propriétés, mais pas réassigner la variable.

JavaScript — let vs const 
let compteur = 0;      // va changer
compteur++;             // 1

const nom = 'Alice';    // ne changera jamais
// nom = 'Bob';         // TypeError

const couleurs = ['rouge', 'vert']; // objet immuable ?
couleurs.push('bleu');  // OK ! car on modifie, pas réassigne
// couleurs = [];       // ERREUR : réassignation

console.log(compteur, nom, couleurs);
// → 1 "Alice" ["rouge", "vert", "bleu"]

04

var — l'ancienne façon

Avant ES6, on utilisait var. Elle existe encore mais il est déconseillé de l'utiliser dans du code moderne.

Caractéristiquevarlet / const
Portée (scope)FonctionBloc {}
HoistingOui (undefined)Oui (mais pas initialisé — TDZ)
RedéclarationPossibleImpossible
RéassignationPossiblelet oui, const non
JavaScript — Problèmes de var 
// Problème 1 : pas de portée de bloc
if (true) {
  var x = 10;   // var ignore le bloc
}
console.log(x); // 10 — fuite !!

if (true) {
  let y = 20;   // let reste dans le bloc
}
console.log(y); // ReferenceError

// Problème 2 : redéclaration silencieuse
var nom = 'Alice';
var nom = 'Bob'; // pas d'erreur
console.log(nom); // Bob

// Problème 3 : hoisting avec undefined
console.log(z);  // undefined (pas d'erreur !)
var z = 5;
⚠️
N'utilisez plus var dans du code moderne

Les problèmes de var (portée de fonction, hoisting dangereux, redéclaration) sont source de bugs. Tout code ES6+ doit utiliser let et const. Même si vous voyez var dans du code existant, écrivez systématiquement en let/const.

Les types de données

JavaScript est un langage faiblement typé : une variable peut changer de type. Mais chaque valeur a un type bien défini.

Les types primitifs

TypeExempleDescription
string"Bonjour", 'A'Chaîne de caractères — guillemets simples, doubles ou backticks
number42, 3.14, NaN, InfinityNombres entiers ou décimaux (il n'y a qu'un seul type number)
booleantrue, falseVrai ou faux — utilisé dans les conditions
nullnull"Rien" — valeur intentionnellement vide
undefinedundefined"Pas encore défini" — valeur par défaut d'une variable déclarée sans valeur
symbolSymbol('id')Valeur unique et immuable (avancé)
bigint9007199254740991nEntiers très grands (ES2020)
JavaScript — Les types en pratique 
const nom     = 'Alice';        // string
const age     = 25;              // number
const prix    = 19.99;           // number (décimal)
const majeur  = true;            // boolean
const voiture = null;            // null (vide intentionnel)
const taille  = undefined;       // undefined (pas définie)
const id      = Symbol('id');    // symbol
const big     = 123n;            // bigint

// Différence clé : null vs undefined
let a;          // undefined (déclarée mais pas initialisée)
let b = null;   // null (explicitement vide)

console.log(a); // undefined
console.log(b); // null

// NaN est un number (même si c'est "Not a Number")
console.log(typeof NaN);        // "number"
console.log(0 / 0);             // NaN
console.log(Infinity);          // Infinity

06

typeof — connaître le type d'une valeur

L'opérateur typeof renvoie une chaîne indiquant le type de la valeur.

JavaScript — typeof en action 
console.log(typeof 'Bonjour'); // "string"
console.log(typeof 42);        // "number"
console.log(typeof 3.14);      // "number"
console.log(typeof true);      // "boolean"
console.log(typeof null);      // "object" — c'est un bug historique !
console.log(typeof undefined); // "undefined"
console.log(typeof {});        // "object"
console.log(typeof []);        // "object"
console.log(typeof function(){}); // "function"
console.log(typeof Symbol());  // "symbol"
console.log(typeof 123n);      // "bigint"

// Attention !
console.log(typeof null === 'object'); // true (bug)
// Pour tester null : valeur === null
⚠️
typeof null === "object" — un bug historique

Depuis la création de JS, typeof null renvoie "object". C'est un bug reconnu mais jamais corrigé car trop de code existant en dépendrait. Pour tester null, utilisez valeur === null.

07

Template strings (littéraux de gabarit)

Les template strings (ou littéraux de gabarit) permettent d'intégrer des expressions dans une chaîne avec ${}, et de créer des chaînes multi-lignes.

JavaScript — Template strings 
const prenom = 'Alice';
const age = 25;

// Ancienne façon (concaténation)
const msg1 = 'Je m\'appelle ' + prenom + ' et j\'ai ' + age + ' ans.';

// Nouvelle façon (template string)
const msg2 = `Je m'appelle ${prenom} et j'ai ${age} ans.`;

console.log(msg1); // Je m'appelle Alice et j'ai 25 ans.
console.log(msg2); // Je m'appelle Alice et j'ai 25 ans.

// Multi-lignes
const poeme = `
  Roses are red,
  Violets are blue,
  JavaScript is awesome,
  And so are you.
`;
console.log(poeme);

// Expressions à l'intérieur
const a = 10, b = 5;
console.log(`${a} + ${b} = ${a + b}`); // 10 + 5 = 15

// Appels de fonctions
function saluer(n) { return `Bonjour ${n} !`; }
console.log(saluer('Alice')); // Bonjour Alice !
💡
Backticks `` — toujours disponibles

Les backticks (``) fonctionnent partout où JS est exécuté (navigateurs, Node.js). Préférez-les systématiquement aux guillemets pour les chaînes contenant des variables, et pour toutes les chaînes multi-lignes.

08

Les commentaires

Les commentaires sont ignorés par JavaScript. Ils servent à documenter le code pour les humains.

JavaScript — Commentaires 
// --- Commentaire sur une seule ligne ---

// Ceci est un commentaire ligne — tout après // est ignoré
let x = 5; // On peut aussi commenter en fin de ligne

// --- Commentaire multi-lignes ---

/*
  Ceci est un commentaire
  qui s'étend sur plusieurs lignes.
  Très utile pour désactiver temporairement du code.
*/

/*
  Exemple : configuration utilisateur
  On initialise les valeurs par défaut
*/
const config = {
  theme: 'dark',
  lang: 'fr'
};

// --- JSDoc (commentaire de documentation) ---

/**
 * Calcule le carré d'un nombre
 * @param {number} n - Le nombre à élever au carré
 * @returns {number} Le résultat
 */
function carre(n) {
  return n * n;
}
📌
Un code bien écrit n'a pas besoin de commentaires

Préférez des noms de variables et fonctions explicites plutôt que des commentaires. Un commentaire qui dit "c'est compliqué" est un signe qu'il faut simplifier le code. Les commentaires doivent dire pourquoi, pas quoi.

Exercices pratiques

EX 01 🧪 Variables & types
  1. Créez un fichier exercice1.js et liez-le à une page HTML.
  2. Déclarez une variable const ville = "Paris". Affichez-la dans la console.
  3. Déclarez let population = 2161000. Puis modifiez-la pour population = 11000000 (aire urbaine).
  4. Utilisez typeof pour afficher le type de ville, population, true, null et undefined.
  5. Créez un template string : `${ville} a ${population} habitants.` et affichez-le.
  6. Ajoutez un commentaire expliquant chaque étape.
EX 02 🔄 let vs const vs var
  1. Déclarez var test = "ancien". Déclarez-la à nouveau avec var test = "nouveau". Que se passe-t-il ?
  2. Essayez de redéclarer une variable let avec le même nom. Observez l'erreur.
  3. Dans un if (true) { }, déclarez une variable let et une var. Essayez d'accéder aux deux après le bloc. Que constatez-vous ?
  4. Déclarez const MA_CONST = 42. Essayez de la modifier. Notez l'erreur.
  5. Déclarez const tableau = [1, 2, 3]. Ajoutez 4 avec tableau.push(4). Pourquoi ça marche ?
  6. Dans la console du navigateur, tapez document.title = "Mon titre JS". Observez le changement dans l'onglet.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Rôle de JavaScript : client-side, ECMAScript, évolution ES6+
  • Les 3 façons d'inclure JS : script tag, console, fichier .js externe
  • let pour les variables qui changent, const pour les constantes

  • Les 7 types primitifs : string, number, boolean, null, undefined, symbol, bigint
  • typeof pour inspecter le type d'une valeur
  • Template strings avec backticks `` et interpolation ${}
  • Commentaires //, /* */ et JSDoc /** */

Mini-glossaire

ECMAScript (ES)
Norme standardisée sur laquelle JavaScript est basé. ES6 (2015) a apporté des changements majeurs.
let
Mot-clé pour déclarer une variable réassignable avec une portée de bloc.
const
Mot-clé pour déclarer une variable non réassignable. La valeur ne peut pas être remplacée.
Template string
Chaîne délimitée par des backticks `` permettant l'interpolation ${} et le multi-lignes.
typeof
Opérateur retournant le type d'une valeur sous forme de chaîne (ex: "number", "string").
Hoisting
Comportement où les déclarations var et function sont remontées en haut de leur portée avant l'exécution.
TDZ (Temporal Dead Zone)
Zone entre le début du bloc et la déclaration let/const où la variable existe mais n'est pas accessible.
null
Valeur primitive représentant l'absence intentionnelle de valeur.
undefined
Valeur par défaut d'une variable déclarée mais non initialisée.
