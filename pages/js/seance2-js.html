Opérateurs arithmétiques

🖱️ Clic utilisateur → 📡 Événement DOM
📡 Événement DOM → ⚡ addEventListener
⚡ addEventListener → 🎯 Fonction callback
🎯 Fonction callback → ✨ DOM mis à jour

Les opérateurs arithmétiques permettent d'effectuer des calculs mathématiques.

OpérateurNomExempleRésultat
+Addition5 + 38
-Soustraction5 - 32
*Multiplication5 * 315
/Division6 / 32
%Modulo (reste)7 % 31
**Puissance (ES2016)2 ** 38
++Incémentationlet i=1; i++2
--Décrémentationlet i=1; i--0
JavaScript — Opérateurs arithmétiques 
const a = 10, b = 3;

console.log(a + b);  // 13
console.log(a - b);  // 7
console.log(a * b);  // 30
console.log(a / b);  // 3.3333... (division décimale)
console.log(a % b);  // 1  (reste de 10 ÷ 3)
console.log(a ** b); // 1000 (10³)

// Priorité des opérateurs (comme en maths)
console.log(2 + 3 * 4);  // 14 (pas 20)
console.log((2 + 3) * 4); // 20

// Incrémentation
let compteur = 0;
compteur++; // 1
compteur++; // 2
compteur--; // 1
console.log(compteur); // 1
📌
Division par zéro

JS ne génère pas d'erreur en cas de division par zéro. Il renvoie Infinity ou -Infinity. 0 / 0 donne NaN (Not a Number).

Opérateurs de comparaison

Les opérateurs de comparaison comparent deux valeurs et renvoient un booléen (true ou false).

OpérateurSignificationExempleRésultat
===Égalité stricte (valeur ET type)5 === "5"false
!==Inégalité stricte5 !== "5"true
==Égalité faible (avec conversion)5 == "5"true
!=Inégalité faible5 != "5"false
>Supérieur5 > 3true
<Inférieur5 < 3false
>=Supérieur ou égal5 >= 5true
<=Inférieur ou égal5 <= 5true
JavaScript — Comparaison stricte vs faible 
// === et !== sont à PRÉFÉRER (stricts)
console.log(5 === 5);     // true
console.log(5 === '5');   // false (number vs string)
console.log(5 !== '5');   // true

// == et != font conversion de type (à éviter)
console.log(5 == '5');    // true (convertit '5' en number)
console.log(0 == false);  // true (0 est falsy)
console.log('' == false); // true (chaîne vide est falsy)

// Pièges de ==
console.log(null == undefined); // true (mais ce sont des types différents)
console.log([] == false);       // true (conversion surprenante)

// Comparaisons de chaînes (ordre alphabétique)
console.log('abc' < 'abd');     // true
console.log('pomme' > 'poire'); // false (m > i en lexicographique)

// Règle d'or : TOUJOURS utiliser === et !==
⚠️
Ne JAMAIS utiliser == ou !=

Les opérateurs faibles (==, !=) effectuent une conversion de type implicite, ce qui mène à des bugs difficiles à détecter. Utilisez toujours === (égalité stricte) et !== (inégalité stricte).

Opérateurs logiques

Les opérateurs logiques combinent plusieurs conditions booléennes.

OpérateurNomExempleRésultat
&&ETtrue && truetrue
||OUtrue || falsetrue
!NON!truefalse
JavaScript — Opérateurs logiques 
const age = 20;
const permis = true;

// && (ET) — toutes les conditions doivent être vraies
if (age >= 18 && permis) {
  console.log('Vous pouvez conduire');
}

// || (OU) — au moins une condition vraie
const carte = 'Visa';
if (carte === 'Mastercard' || carte === 'Visa') {
  console.log('Paiement accepté');
}

// ! (NON) — inverse la valeur booléenne
const estNuit = false;
if (!estNuit) {
  console.log('Il fait jour');
}

// Double !! pour convertir en booléen
console.log(!!'Bonjour'); // true
console.log(!!0);         // false
console.log(!!'');        // false

if / else if / else

La structure conditionnelle la plus utilisée.

JavaScript — if/else if/else 
const note = 15;

if (note >= 18) {
  console.log('Excellent');
} else if (note >= 14) {
  console.log('Très bien');    // ← s'exécute
} else if (note >= 10) {
  console.log('Passable');
} else {
  console.log('Insuffisant');
}

// Conditions imbriquées
const age = 65;
const aCarteSenior = true;

if (age >= 60) {
  if (aCarteSenior) {
    console.log('Réduction 50%');
  } else {
    console.log('Réduction 20%');
  }
} else {
  console.log('Plein tarif');
}

// Attention aux blocks : sans {}, seule la première ligne est conditionnée
if (false)
  console.log('Ceci ne s\'affiche pas');
  console.log('Ceci s\'affiche TOUJOURS !');
// Toujours utiliser {} même pour une ligne
══ 05 SWITCH ══ -->
05

switch

switch est utile quand on a plusieurs cas pour une même valeur.

JavaScript — switch 
const jour = 3;
let nomJour;

switch (jour) {
  case 1:
    nomJour = 'Lundi';
    break;
  case 2:
    nomJour = 'Mardi';
    break;
  case 3:
    nomJour = 'Mercredi';     // ← ce cas
    break;                     // ← sort du switch
  case 4:
    nomJour = 'Jeudi';
    break;
  case 5:
    nomJour = 'Vendredi';
    break;
  case 6:
  case 7:
    nomJour = 'Week-end';     // cas groupés
    break;
  default:
    nomJour = 'Jour invalide';
}

console.log(nomJour); // Mercredi

// switch avec chaînes
const couleur = 'rouge';
switch (couleur) {
  case 'rouge':
  case 'vert':
  case 'bleu':
    console.log('Couleur primaire');
    break;
  default:
    console.log('Autre couleur');
}
⚠️
Ne pas oublier le break

Sans break, l'exécution continue sur le cas suivant (c'est le fall-through). Parfois voulu (cas groupés), mais souvent un bug. Utilisez break systématiquement.

06

Opérateur ternaire ? :

Le ternaire est une forme concise de if/else qui renvoie une valeur. Syntaxe : condition ? valeur_si_vrai : valeur_si_faux.

JavaScript — Opérateur ternaire 
const age = 20;

// Version if/else
let statut;
if (age >= 18) {
  statut = 'Majeur';
} else {
  statut = 'Mineur';
}

// Version ternaire (équivalente)
const statut2 = age >= 18 ? 'Majeur' : 'Mineur';

console.log(statut);  // Majeur
console.log(statut2); // Majeur

// Ternaire imbriqué (à utiliser avec parcimonie)
const note = 15;
const mention = note >= 18 ? 'Excellent'
               : note >= 14 ? 'Très bien'
               : note >= 10 ? 'Passable'
               : 'Insuffisant';

console.log(mention); // Très bien

// Ternaire pour une assignation conditionnelle
const largeur = 800;
const type = largeur > 768 ? 'Desktop' : 'Mobile';
📌
Quand utiliser le ternaire ?

Pour des conditions simples à deux branches. Si la logique est complexe ou nécessite plusieurs instructions, préférez if/else classique pour la lisibilité.

07

Truthy & Falsy

En JavaScript, les valeurs sont truthy (considérées comme vraies) ou falsy (considérées comme fausses) dans un contexte booléen.

❌ Valeurs falsy (6 seulement)

  • false
  • 0
  • '' ou "" (chaîne vide)
  • null
  • undefined
  • NaN

✅ Valeurs truthy (tout le reste)

  • true
  • 1, -1, 3.14
  • 'texte', '0'
  • [] (tableau vide)
  • {} (objet vide)
  • function(){}
JavaScript — Truthy et Falsy en pratique 
// Dans un if, toute valeur est convertie en booléen
if ('Bonjour') {
  console.log('Chaîne non-vide → truthy'); // s'exécute
}

if (0) {
  console.log('Ceci ne s\'affiche JAMAIS');
}

if ([]) {
  console.log('Tableau vide → truthy'); // s'exécute
}

// Utilisation pratique : valeur par défaut
const prenom = ''; // chaîne vide (falsy)
if (prenom) {
  console.log(`Bonjour ${prenom}`);
} else {
  console.log('Bonjour anonyme'); // s'exécute
}

// Attention : '0' est truthy !
if ('0') console.log('truthy');  // s'affiche
if (0) console.log('falsy');     // ne s'affiche pas

08

Short-circuit && et ||

Les opérateurs && et || n'évaluent pas toujours les deux opérandes. Ils s'arrêtent dès que le résultat est déterminé (court-circuit).

JavaScript — Short-circuit evaluation 
// || (OU) — renvoie la première valeur truthy
// ou la dernière si tout est falsy
console.log('' || 'default');     // "default"
console.log(0 || 42);             // 42
console.log(null || 'fallback');  // "fallback"
console.log('Alice' || 'Bob');    // "Alice" (première truthy)
console.log(0 || '' || null);     // null (tout falsy → dernier)

// Utile pour valeur par défaut
const nom = '';
const affichage = nom || 'Inconnu';
console.log(affichage); // Inconnu

// && (ET) — renvoie la première valeur falsy
// ou la dernière si tout est truthy
console.log('Alice' && 42);       // 42 (dernière truthy)
console.log(null && 'texte');     // null (première falsy)
console.log(0 && 'texte');        // 0
console.log(true && 'suite');     // "suite"

// Utile pour exécution conditionnelle
const estConnecte = true;
estConnecte && console.log('Utilisateur connecté');

// Combinaison
const age = 0;
const message = age || 'Âge non spécifié';
console.log(message); // Âge non spécifié
// Mais attention : age = 0 est falsy, alors que 0 est un âge valide
⚠️
Piège du short-circuit avec 0 et ''

const prix = 0 || 10 → 10 (car 0 est falsy). Si 0 est une valeur valide, utilisez ?? (nullish coalescing) : prix ?? 10 ne traite que null/undefined comme valeurs à remplacer.

Exercices pratiques

EX 01 🧮 Calculatrice conditionnelle
  1. Déclarez deux variables const a = 15, b = 4.
  2. Affichez le résultat de a + b, a - b, a * b, a / b, a % b, a ** b.
  3. Si a est pair (utilisez %), affichez "pair", sinon "impair".
  4. Comparez a et b : affichez le plus grand des deux.
  5. Utilisez un ternaire pour afficher si a est positif, négatif ou nul.
  6. Testez avec == : 5 == "5" vs 5 === "5". Que constatez-vous ?
EX 02 🌤️ Convertisseur de notes
  1. Créez une variable const note = 13.
  2. Utilisez if/else if/else pour afficher : "Excellent" (≥ 18), "Très bien" (≥ 16), "Bien" (≥ 14), "Passable" (≥ 10), "Insuffisant" (< 10).
  3. Refaites la même logique avec un ternaire chaîné.
  4. Testez les valeurs falsy : écrivez if (""), if (0), if ([]). Lesquels s'exécutent ?
  5. Utilisez || pour définir une valeur par défaut : const couleur = null; const affiche = couleur || "aucune";
  6. Utilisez switch pour convertir un chiffre (1-7) en nom de jour.

Mini-glossaire

Opérande
Valeur sur laquelle un opérateur agit. Ex: dans a + b, a et b sont les opérandes.
Égalité stricte (===)
Compare valeur ET type. 5 === "5" est false car les types diffèrent.
Modulo (%)
Reste de la division entière. Utile pour tester pair/impair : n % 2 === 0.
Falsy
Valeur qui se comporte comme false dans un contexte booléen. Les 6 : false, 0, '', null, undefined, NaN.
Court-circuit (short-circuit)
Évaluation partielle : && s'arrête à la première valeur falsy, || à la première truthy.
Ternaire
Opérateur à 3 parties : condition ? valeur_si_vrai : valeur_si_faux.
