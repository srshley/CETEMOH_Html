01

Synchrone vs asynchrone

🖱️ Clic utilisateur → 📡 Événement DOM
📡 Événement DOM → ⚡ addEventListener
⚡ addEventListener → 🎯 Fonction callback
🎯 Fonction callback → ✨ DOM mis à jour

En JavaScript, certaines opérations prennent du temps : requêtes réseau, lecture de fichier, timer. Le code synchrone bloque l'exécution. Le code asynchrone permet de continuer.

JavaScript — Synchrone vs asynchrone 
// ── Synchrone (bloquant) ──
console.log('Début');
for (let i = 0; i < 1000000000; i++) {} // boucle longue
console.log('Fin');
// → "Début" ... (attente) ... "Fin"

// ── Asynchrone (non-bloquant) ──
console.log('Début');
setTimeout(() => {
  console.log('Timer terminé');
}, 1000);
console.log('Fin');
// → "Début", "Fin", (1 seconde) "Timer terminé"

// JavaScript a un modèle "event loop" :
// 1. Call stack exécute le code synchrone
// 2. Les opérations async vont dans une file d'attente
// 3. Quand la stack est vide, la file est traitée

console.log('A');
setTimeout(() => console.log('B'), 0); // même avec 0ms
console.log('C');
// → A, C, B (toujours B en dernier !)

// Pourquoi ? Parce que setTimeout est async,
// même avec 0ms, il passe par la file d'attente
02

Callbacks (setTimeout)

Un callback est une fonction passée à une autre fonction, exécutée plus tard.

JavaScript — Callbacks 
// setTimeout — exécute après un délai
setTimeout(() => {
  console.log('1 seconde plus tard');
}, 1000);

// setInterval — exécute périodiquement
const id = setInterval(() => {
  console.log('Toutes les 500ms');
}, 500);

// Arrêter après 3 secondes
setTimeout(() => {
  clearInterval(id);
  console.log('Intervalle arrêté');
}, 3000);

// Callback personnalisé
function attendreEtDire(message, callback) {
  console.log('Préparation...');
  setTimeout(() => {
    console.log(message);
    if (callback) callback();
  }, 1000);
}

attendreEtDire('Hello !', () => {
  console.log('Callback exécuté !');
});

// Problème : callback hell (enfer des callbacks)
function etape1(cb) { setTimeout(() => { console.log(1); cb(); }, 1000); }
function etape2(cb) { setTimeout(() => { console.log(2); cb(); }, 1000); }
function etape3(cb) { setTimeout(() => { console.log(3); cb(); }, 1000); }

// Pyramid of doom :
etape1(() => {
  etape2(() => {
    etape3(() => {
      console.log('Fini');
    });
  });
});
// Les Promises résolvent ce problème

Promises (resolve, reject, then, catch)

Une Promise est un objet qui représente une valeur future. Elle a 3 états : pending (en attente), fulfilled (résolue), rejected (rejetée).

JavaScript — Promises 
// Créer une Promise
const maPromise = new Promise((resolve, reject) => {
  const succes = true;

  setTimeout(() => {
    if (succes) {
      resolve('✅ Opération réussie');
    } else {
      reject('❌ Opération échouée');
    }
  }, 1000);
});

// Utiliser la Promise
maPromise
  .then(resultat => {
    console.log(resultat); // "✅ Opération réussie"
  })
  .catch(erreur => {
    console.error(erreur);
  })
  .finally(() => {
    console.log('Terminé');
  });

// Exemple concret : simuler une API
function fetchUser(id) {
  return new Promise((resolve, reject) => {
    setTimeout(() => {
      if (id > 0) {
        resolve({ id, nom: 'Alice', age: 25 });
      } else {
        reject(new Error('ID invalide'));
      }
    }, 500);
  });
}

// Chaînage de .then()
fetchUser(1)
  .then(user => {
    console.log(user.nom); // Alice
    return user.age;
  })
  .then(age => {
    console.log(`${age} ans`);
  })
  .catch(err => {
    console.error(err.message);
  });

04

Promise.all — exécution parallèle

Promise.all exécute plusieurs promesses en parallèle et attend que toutes soient résolues.

JavaScript — Promise.all 
function fetchUser(id) {
  return new Promise(resolve => {
    setTimeout(() => resolve({ id, nom: `User ${id}` }), Math.random() * 1000);
  });
}

// Lancer 3 requêtes en parallèle
const promesses = [fetchUser(1), fetchUser(2), fetchUser(3)];

Promise.all(promesses)
  .then(users => {
    console.log(users);
    // → [{ id:1, nom:"User 1" }, { id:2, nom:"User 2" }, { id:3, nom:"User 3" }]
    console.log('Tous les utilisateurs chargés');
  })
  .catch(err => {
    console.error('Au moins une requête a échoué', err);
  });

// Promise.allSettled — ne rejette jamais, donne l'état de chaque
const p1 = Promise.resolve('OK');
const p2 = Promise.reject('ERREUR');

Promise.allSettled([p1, p2]).then(resultats => {
  console.log(resultats);
  // → [{ status:"fulfilled", value:"OK" }, { status:"rejected", reason:"ERREUR" }]
});

// Promise.race — la première promesse qui se résout
Promise.race([fetchUser(1), fetchUser(2)])
  .then(plusRapide => console.log('Plus rapide:', plusRapide));

// Promise.any — la première qui réussit (ignore les rejets)
Promise.any([Promise.reject('A'), Promise.resolve('B')])
  .then(r => console.log(r)); // "B"

async / await — syntaxe moderne

async/await permet d'écrire du code asynchrone qui ressemble à du synchrone. C'est du sucre syntaxique sur les Promises.

JavaScript — async/await 
// async transforme une fonction en Promise
async function saluer() {
  return 'Bonjour !';
}

saluer().then(msg => console.log(msg)); // "Bonjour !"

// await attend la résolution d'une Promise
function attendre(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function exemple() {
  console.log('Début');
  await attendre(1000); // 1 seconde
  console.log('Après 1 seconde');
  await attendre(500);
  console.log('Après 500ms de plus');
}
exemple();

// Éviter le callback hell / then chaîné
function fetchUser(id) {
  return new Promise(resolve => {
    setTimeout(() => resolve({ id, nom: `User ${id}` }), 500);
  });
}

async function afficherUtilisateur(id) {
  // Au lieu de .then().then()
  const user = await fetchUser(id);
  console.log(`Nom: ${user.nom}`);
  return user;
}

afficherUtilisateur(1);

// Parallèle avec await + Promise.all
async function chargerTout() {
  const [u1, u2] = await Promise.all([
    fetchUser(1),
    fetchUser(2)
  ]);
  console.log(u1, u2);
}

Fetch API — requête GET

fetch est l'API moderne pour faire des requêtes HTTP. Elle retourne une Promise.

JavaScript — Fetch GET 
// JSONPlaceholder — API gratuite pour tests
const API_URL = 'https://jsonplaceholder.typicode.com';

// ══ GET avec Promises ══
fetch(`${API_URL}/posts/1`)
  .then(response => {
    // response.ok = true si statut 200-299
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}`);
    }
    return response.json(); // réponse parsée en JSON
  })
  .then(data => {
    console.log(data);
    // → { userId: 1, id: 1, title: "...", body: "..." }
  })
  .catch(err => console.error('Erreur:', err));

// ══ GET avec async/await ══
async function getPost(id) {
  try {
    const response = await fetch(`${API_URL}/posts/${id}`);
    if (!response.ok) throw new Error(`Erreur ${response.status}`);
    const post = await response.json();
    console.log(post.title);
    return post;
  } catch (err) {
    console.error('Échec du chargement:', err);
  }
}

getPost(1);

// ══ Récupérer et afficher une liste ══
async function getUsers() {
  const response = await fetch(`${API_URL}/users`);
  const users = await response.json();
  users.forEach(u => console.log(`${u.name} — ${u.email}`));
}

getUsers();

// Autres formats : response.text(), response.blob(), response.formData()
📌
JSONPlaceholder est votre ami

jsonplaceholder.typicode.com est une API gratuite pour tester vos requêtes. Ressources : /posts, /comments, /users, /todos. Idéal pour apprendre fetch.

Fetch POST — envoyer des données

Pour envoyer des données au serveur, on utilise la méthode POST avec body et headers.

JavaScript — Fetch POST 
const API_URL = 'https://jsonplaceholder.typicode.com';

// ══ POST avec async/await ══
async function creerPost(titre, corps) {
  const nouveauPost = {
    userId: 1,
    title: titre,
    body: corps
  };

  try {
    const response = await fetch(`${API_URL}/posts`, {
      method: 'POST',             // Méthode HTTP
      headers: {
        'Content-Type': 'application/json', // Type de données envoyées
      },
      body: JSON.stringify(nouveauPost)     // Convertir en JSON
    });

    if (!response.ok) {
      throw new Error(`Erreur ${response.status}`);
    }

    const data = await response.json();
    console.log('Post créé:', data);
    // → { userId:1, title:"...", body:"...", id:101 }
    return data;
  } catch (err) {
    console.error('Erreur création:', err);
  }
}

creerPost('Mon titre', 'Mon contenu');

// ══ PUT (mise à jour complète) ══
async function mettreAJourPost(id, titre) {
  const response = await fetch(`${API_URL}/posts/${id}`, {
    method: 'PUT',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ title: titre })
  });
  return response.json();
}

// ══ DELETE ══
async function supprimerPost(id) {
  const response = await fetch(`${API_URL}/posts/${id}`, {
    method: 'DELETE'
  });
  if (response.ok) console.log('Post supprimé');
}

// ══ PATCH (mise à jour partielle) ══
async function patcherPost(id, champs) {
  const response = await fetch(`${API_URL}/posts/${id}`, {
    method: 'PATCH',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify(champs)
  });
  return response.json();
}

try/catch — gestion d'erreurs

La gestion d'erreurs asynchrones est essentielle. try/catch fonctionne avec async/await.

JavaScript — try/catch 
// ══ try/catch basique ══
try {
  console.log('Tentative...');
  JSON.parse('json invalide');
  console.log('Ceci ne s\'affiche pas');
} catch (erreur) {
  console.error('Erreur attrapée:', erreur.message);
} finally {
  console.log('Toujours exécuté');
}

// ══ Avec async/await et fetch ══
async function chargerDonnees(url) {
  try {
    const response = await fetch(url);

    // Vérifier le statut HTTP
    if (!response.ok) {
      throw new Error(`HTTP ${response.status}: ${response.statusText}`);
    }

    const data = await response.json();
    return data;

  } catch (erreur) {
    // Erreur réseau ou HTTP
    if (erreur instanceof TypeError) {
      console.error('Erreur réseau (pas de connexion ?)');
    } else {
      console.error('Erreur API:', erreur.message);
    }

    // Valeur par défaut
    return { fallback: true, data: [] };
  }
}

// ══ Gestion granulaire ══
async function apiCall(url) {
  try {
    const res = await fetch(url);
    if (res.status === 404) throw new Error('Ressource introuvable');
    if (res.status === 500) throw new Error('Erreur serveur');
    return await res.json();
  } catch (err) {
    if (err.name === 'AbortError') {
      console.log('Requête annulée');
    }
    throw err; // Re-lancer si besoin
  }
}

// ══ Timeout avec AbortController ══
async function fetchWithTimeout(url, timeoutMs = 5000) {
  const controller = new AbortController();
  const timeout = setTimeout(() => controller.abort(), timeoutMs);

  try {
    const response = await fetch(url, { signal: controller.signal });
    return await response.json();
  } finally {
    clearTimeout(timeout);
  }
}

Exercices pratiques

EX 01 ⏱️ Timer & Promises
  1. Créez une fonction delay(ms) qui retourne une Promise résolue après ms millisecondes.
  2. Utilisez-la avec .then() pour afficher "1", "2", "3" avec 1 seconde d'intervalle.
  3. Réécrivez avec async/await.

  1. Faites un fetch GET vers https://jsonplaceholder.typicode.com/users.
  2. Affichez la liste des noms dans une <ul> du DOM.
  3. Ajoutez un champ de recherche qui filtre les utilisateurs par nom (en temps réel).
  4. Créez un bouton "Ajouter" qui fait un POST vers /users avec un formulaire.
  5. Gérez les erreurs : si fetch échoue, affichez "Erreur de chargement".
  6. Ajoutez un bouton "Recharger" qui récupère à nouveau les données.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Synchrone (bloquant) vs asynchrone (event loop)
  • Callbacks avec setTimeout/setInterval
  • Promises : new Promise, resolve, reject, then, catch, finally
  • Promise.all, allSettled, race, any
  • async/await : syntaxe moderne pour l'asynchrone
  • Fetch GET : récupérer des données d'une API
  • Fetch POST/PUT/DELETE avec headers et body JSON
  • try/catch pour gérer les erreurs asynchrones

Mini-glossaire

Asynchrone
Opération qui ne bloque pas le thread principal. JS utilise l'event loop pour gérer les opérations différées.
Promise
Objet représentant une valeur future. États : pending, fulfilled, rejected.
async
Mot-clé qui transforme une fonction en fonction asynchrone retournant une Promise.
await
Mot-clé qui suspend l'exécution de la fonction async jusqu'à la résolution de la Promise.
fetch
API navigateur pour effectuer des requêtes HTTP. Retourne une Promise de Response.
JSONPlaceholder
API REST gratuite pour tests et prototypage avec des données factices.
AbortController
Mécanisme pour annuler une requête fetch en cours (timeout, navigation).
