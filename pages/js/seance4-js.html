01

Objets littéraux {}

🖱️ Clic utilisateur → 📡 Événement DOM
📡 Événement DOM → ⚡ addEventListener
⚡ addEventListener → 🎯 Fonction callback
🎯 Fonction callback → ✨ DOM mis à jour

Un objet est une collection de propriétés clé/valeur. C'est le type de données le plus important en JavaScript.

JavaScript — Créer des objets 
// Syntaxe littérale (recommandée)
const personne = {
  nom: 'Alice',
  age: 25,
  ville: 'Paris'
};

console.log(personne);
// → { nom: "Alice", age: 25, ville: "Paris" }

// Clés sans guillemets (sauf si caractères spéciaux)
const etudiant = {
  nom: 'Bob',
  'nom-complet': 'Bob Martin',  // guillemets nécessaires
  age: 22
};

// Objet vide puis ajout
const voiture = {};
voiture.marque = 'Tesla';
voiture.modele = 'Model 3';
voiture['annee'] = 2024;

console.log(voiture);
// → { marque: "Tesla", modele: "Model 3", annee: 2024 }

// Clés dynamiques avec crochets
const propriete = 'couleur';
voiture[propriete] = 'rouge';
console.log(voiture.couleur); // rouge

// Shorthand property (ES6)
const x = 10, y = 20;
const point = { x, y }; // équivaut à { x: x, y: y }
console.log(point); // { x: 10, y: 20 }
02

Accès aux propriétés : dot vs bracket

Deux façons d'accéder aux propriétés d'un objet.

JavaScript — Dot notation vs bracket notation 
const personne = {
  nom: 'Alice',
  age: 25,
  'adresse.ville': 'Paris'  // clé avec point
};

// Dot notation (point) — privilégiée
console.log(personne.nom);    // Alice
console.log(personne.age);    // 25

// Bracket notation (crochets) — pour cas particuliers
console.log(personne['nom']); // Alice
console.log(personne['adresse.ville']); // Paris

// Quand utiliser les crochets ? 
// 1. Clé dynamique (variable)
const cle = 'age';
console.log(personne[cle]); // 25

// 2. Clé avec caractères spéciaux
// personne.adresse.ville  // ERREUR
console.log(personne['adresse.ville']); // OK

// 3. Clé qui n'existe pas → undefined
console.log(personne.taille); // undefined

03

Méthodes d'objet

Une méthode est une fonction attachée à un objet.

JavaScript — Méthodes d'objet 
const personne = {
  nom: 'Alice',
  age: 25,

  // Méthode classique (ES5)
  presentation: function() {
    return `Je m'appelle ${this.nom}`;
  },

  // Méthode concise (ES6) — recommandée
  saluer() {
    return `Salut, je suis ${this.nom}`;
  },

  // Méthode fléchée — attention à this !
  // direAge: () => `${this.age}` // NE PAS UTILISER
};

console.log(personne.presentation()); // Je m'appelle Alice
console.log(personne.saluer());       // Salut, je suis Alice

// Ajouter une méthode après coup
personne.ageDansUnAn = function() {
  return this.age + 1;
};
console.log(personne.ageDansUnAn()); // 26
══ 04 THIS ══ -->
04

Le mot-clé this

this fait référence au contexte d'exécution. Sa valeur change selon comment la fonction est appelée.

JavaScript — Les règles de this 
// 1. Contexte global
console.log(this); // window (navigateur) ou global (Node.js)

// 2. Dans une méthode d'objet → l'objet lui-même
const voiture = {
  marque: 'Tesla',
  demarrer() {
    console.log(`${this.marque} démarre`);
  }
};
voiture.demarrer(); // Tesla démarre

// 3. Fonction classique → global (ou undefined en strict)
function test() {
  'use strict';
  console.log(this); // undefined
}
test();

// 4. Fonction fléchée → this du contexte parent
const personne = {
  nom: 'Alice',
  saluer: function() {
    const dire = () => `Bonjour ${this.nom}`;
    return dire();
  }
};
console.log(personne.saluer()); // Bonjour Alice

// 5. Problème classique : this perdu dans un callback
const utilisateur = {
  prenom: 'Bob',
  amis: ['Alice', 'Charlie'],
  listerAmis() {
    this.amis.forEach(function(ami) {
      // ICI this n'est PLUS utilisateur !
      // console.log(this.prenom); // ERREUR
    });

    // Solution 1 : fonction fléchée
    this.amis.forEach(ami => {
      console.log(`${this.prenom} est ami avec ${ami}`);
    });

    // Solution 2 : .bind(this)
    this.amis.forEach(function(ami) {
      console.log(`${this.prenom} est ami avec ${ami}`);
    }.bind(this));
  }
};
utilisateur.listerAmis();

05

Factory functions

Une factory function est une fonction qui crée et retourne un objet. Alternative simple aux classes.

JavaScript — Factory functions 
function creerPersonne(nom, age, ville) {
  return {
    nom,
    age,
    ville,
    presentation() {
      return `${this.nom}, ${this.age} ans, ${this.ville}`;
    },
    estMajeur() {
      return this.age >= 18;
    }
  };
}

const alice = creerPersonne('Alice', 25, 'Paris');
const bob = creerPersonne('Bob', 16, 'Lyon');

console.log(alice.presentation()); // Alice, 25 ans, Paris
console.log(bob.estMajeur());      // false

// Factory avec validation
function creerVoiture(marque, modele) {
  if (!marque || !modele) {
    throw new Error('Marque et modèle requis');
  }
  return {
    marque,
    modele,
    description() {
      return `${this.marque} ${this.modele}`;
    }
  };
}

const tesla = creerVoiture('Tesla', 'Model 3');
console.log(tesla.description()); // Tesla Model 3

06

Object.keys / values / entries

Ces méthodes statiques permettent de manipuler les propriétés d'un objet sous forme de tableau.

JavaScript — Object.keys/values/entries 
const etudiant = {
  nom: 'Alice',
  age: 22,
  filiere: 'Informatique',
  moyenne: 15.5
};

// Object.keys() → tableau des clés
console.log(Object.keys(etudiant));
// → ["nom", "age", "filiere", "moyenne"]

// Object.values() → tableau des valeurs
console.log(Object.values(etudiant));
// → ["Alice", 22, "Informatique", 15.5]

// Object.entries() → tableau de paires [clé, valeur]
console.log(Object.entries(etudiant));
// → [["nom","Alice"], ["age",22], ["filiere","Informatique"], ["moyenne",15.5]]

// Utile pour parcourir avec for...of
for (const [cle, valeur] of Object.entries(etudiant)) {
  console.log(`${cle}: ${valeur}`);
}

// Calculer la somme des valeurs numériques
const notes = { maths: 15, français: 12, anglais: 18 };
const somme = Object.values(notes).reduce((acc, v) => acc + v, 0);
console.log(somme); // 45

07

Boucle for...in sur les objets

for...in parcourt les clés énumérables d'un objet.

JavaScript — for...in sur objets 
const produit = {
  id: 101,
  nom: 'Casque audio',
  prix: 79.99,
  stock: true
};

for (const cle in produit) {
  console.log(`${cle}: ${produit[cle]}`);
}
// → id: 101, nom: Casque audio, prix: 79.99, stock: true

// Filtrer les propriétés héritées
for (const cle in produit) {
  if (produit.hasOwnProperty(cle)) {
    console.log(`Propre: ${cle}`);
  }
}

// Alternative moderne : Object.keys() + for...of
for (const cle of Object.keys(produit)) {
  console.log(produit[cle]);
}

// Vérifier si une propriété existe
console.log('nom' in produit);      // true
console.log('couleur' in produit);  // false
console.log(produit.hasOwnProperty('nom')); // true

08

Optional chaining ?. (ES2020)

L'opérateur ?. permet d'accéder à une propriété sans erreur si l'objet intermédiaire est null ou undefined.

JavaScript — Optional chaining 
const utilisateur = {
  nom: 'Alice',
  adresse: {
    rue: '12 rue des Fleurs',
    codePostal: '75001'
  },
  preferences: null
};

// Sans optional chaining (vieux style)
const ville1 = utilisateur.adresse ? utilisateur.adresse.ville : undefined;
console.log(ville1); // undefined

// Avec optional chaining (moderne)
const ville2 = utilisateur.adresse?.ville;
console.log(ville2); // undefined

// Chaînage multiple
const rue = utilisateur.adresse?.rue;
console.log(rue); // "12 rue des Fleurs"

// Avec propriété inexistante
const cp = utilisateur.adresse?.codePostal;
console.log(cp); // "75001"

// Avec null
const theme = utilisateur.preferences?.theme;
console.log(theme); // undefined (pas d'erreur)

// Appel de méthode optionnel
const resultat = utilisateur.calculer?.() ?? 'Méthode inexistante';
console.log(resultat); // "Méthode inexistante"

// Accès à un élément de tableau
const premierAmi = utilisateur.amis?.[0];
console.log(premierAmi); // undefined (pas d'erreur)

// Combinaison avec ?? (nullish coalescing)
const couleur = utilisateur.preferences?.couleur ?? 'bleu';
console.log(couleur); // bleu
⚠️
Ne pas abuser du ?.

Utilisez ?. quand vous attendez vraiment que la valeur puisse être null/undefined. Ne l'utilisez pas pour masquer des bugs — si une propriété doit exister, laissez-la générer une erreur pour mieux déboguer.

  1. Écrivez une factory creerCompte(nom, solde) qui retourne un objet avec nom, solde, deposer(montant), retirer(montant), afficher().

Mini-glossaire

Objet littéral
Collection de propriétés clé/valeur notée {}. Le type de données le plus fondamental en JS.
Dot notation
Accès aux propriétés avec un point : objet.propriete. Syntaxe préférée.
Bracket notation
Accès aux propriétés avec des crochets : objet['propriete']. Pour clés dynamiques ou spéciales.
this
Mot-clé qui référence le contexte d'exécution. Sa valeur change selon l'appel (méthode, globale, fléchée).
Factory function
Fonction qui crée et retourne un nouvel objet. Alternative simple aux constructeurs/classes.
Optional chaining (?.)
Opérateur qui court-circuite l'accès à une propriété si la valeur est null/undefined, évitant les erreurs.
Nullish coalescing (??)
Opérateur qui renvoie l'opérande droit seulement si l'opérande gauche est null/undefined (pas falsy).
