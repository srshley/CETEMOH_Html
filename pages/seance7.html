01

La balise <form> : action et method

<form> est le conteneur de tout formulaire. Deux attributs sont essentiels : action (URL de traitement) et method (GET ou POST).

GET

Les données sont visibles dans l'URL. Limité à ~3000 caractères. Pour les recherches, filtres.

?nom=Jean&age=28
POST

Les données sont dans le corps de la requête (invisibles). Pas de limite de taille. Pour les formulaires avec données sensibles.

Données cachées
HTML — Formulaire GET 
<form action="/recherche" method="GET">
  <input type="text" name="q">
  <button type="submit">Rechercher</button>
</form>
RENDU
02

Input : text, password, email

<input> est la balise la plus polyvalente des formulaires. Son type (type) détermine son comportement et l'apparence sur mobile. Les types de base : text, password, email.

HTML — Types input basiques 
<form action="#" method="POST">
  <p>
    <label>Nom :</label>
    <input type="text" name="nom">
  </p>
  <p>
    <label>Email :</label>
    <input type="email" name="email">
  </p>
  <p>
    <label>Mot de passe :</label>
    <input type="password" name="pass">
  </p>
  <button type="submit">Envoyer</button>
</form>
RENDU

03

Label — accessibilité

<label> associe un texte à un champ de formulaire. Deux méthodes : l'attribut for qui pointe vers id du champ, ou l'imbrication du champ dans le label. Les labels sont obligatoires pour l'accessibilité.

HTML — Label 
<!-- Méthode 1 : for + id (recommandée) -->
<label for="prenom">Prénom :</label>
<input type="text" id="prenom" name="prenom">

<!-- Méthode 2 : imbrication -->
<label>
  Nom :
  <input type="text" name="nom">
</label>
RENDU

⚠️
Chaque champ doit avoir un label

Un champ sans label est inaccessible. Les lecteurs d'écran ne peuvent pas annoncer le nom du champ. Si le design ne permet pas de label visible, utilisez aria-label.

04

Button : submit, reset, button

<button> crée un bouton cliquable. L'attribut type détermine son action : submit (envoie le formulaire), reset (réinitialise), button (action personnalisée avec JavaScript).

HTML — Types de boutons 
<form action="#" method="POST">
  <input type="text" name="nom">

  <!-- Envoie le formulaire -->
  <button type="submit">Envoyer</button>

  <!-- Réinitialise tous les champs -->
  <button type="reset">Effacer</button>

  <!-- Action personnalisée (JS) -->
  <button type="button" onclick="alert('Bonjour !')">
    Cliquez-moi
  </button>
</form>
RENDU

05

Select & Option

<select> crée une liste déroulante. Chaque option est définie par <option> avec value. On peut grouper les options avec <optgroup>.

HTML — Select 
<label for="pays">Pays :</label>
<select id="pays" name="pays">
  <option value="">— Choisir —</option>
  <option value="fr">France</option>
  <option value="be">Belgique</option>
  <option value="ch">Suisse</option>
  <option value="ca">Canada</option>
</select>

<br><br>

<!-- Avec groupes -->
<label for="sujet">Sujet :</label>
<select id="sujet" name="sujet">
  <optgroup label="Technique">
    <option value="html">HTML</option>
    <option value="css">CSS</option>
  </optgroup>
  <optgroup label="Administratif">
    <option value="facture">Facture</option>
    <option value="info">Information</option>
  </optgroup>
</select>
RENDU

06

Textarea — zone de texte

<textarea> crée une zone de texte multi-lignes. Les attributs cols et rows définissent sa taille. Contrairement à <input>, c'est une balise avec contenu (pas orpheline).

HTML — Textarea 
<label for="message">Votre message :</label>
<textarea id="message" name="message"
  rows="5" cols="40"
  placeholder="Écrivez votre message ici..."></textarea>
RENDU

07

Fieldset & Legend

<fieldset> regroupe des champs liés thématiquement. <legend> donne un titre au groupe. Essentiel pour l'accessibilité et l'organisation visuelle des formulaires longs.

HTML — Fieldset + Legend 
<form action="#" method="POST">
  <fieldset>
    <legend>Informations personnelles</legend>

    <label for="nom">Nom :</label>
    <input type="text" id="nom" name="nom">

    <label for="email">Email :</label>
    <input type="email" id="email" name="email">
  </fieldset>

  <fieldset>
    <legend>Adresse</legend>

    <label for="rue">Rue :</label>
    <input type="text" id="rue" name="rue">

    <label for="cp">Code postal :</label>
    <input type="text" id="cp" name="cp">
  </fieldset>

  <button type="submit">Envoyer</button>
</form>
RENDU

08

Placeholder & value

placeholder affiche un texte indicatif dans le champ (disparaît à la saisie). value définit la valeur par défaut. Pour les champs pré-remplis, préférez value.

HTML — Placeholder et value 
<!-- Placeholder : indicatif, disparaît à la saisie -->
<input type="text" name="recherche"
  placeholder="Rechercher un article...">

<!-- Value : valeur pré-remplie (modifiable) -->
<input type="text" name="ville"
  value="Paris">

<!-- Placeholder sur textarea -->
<textarea name="bio" rows="3"
  placeholder="Parlez-nous de vous..."></textarea>
RENDU

📌
Placeholder ≠ Label

Le placeholder ne remplace pas un vrai <label>. Il disparaît quand on tape, ce qui pose problème d'accessibilité. Toujours avoir un label visible ou un aria-label.

09

Exercices pratiques

EX 01 ✏️ Formulaire de contact
  1. Créez un formulaire de contact avec : nom, email, sujet (select), message (textarea).
  2. Utilisez method="POST" et action="/contact".
  3. Chaque champ a un <label> avec for.
  4. Ajoutez un bouton submit "Envoyer" et un bouton reset "Effacer".
  5. Encadrez dans un <fieldset> avec <legend>.
EX 02 ✏️ Inscription
  1. Créez un formulaire d'inscription : prénom, nom, email, mot de passe, confirmation.
  2. Utilisez les types appropriés (email, password).
  3. Ajoutez des placeholders pertinents.
EX 03 🔍 Enquête
  1. Créez un sondage avec 2 fieldset : "À propos de vous" et "Vos préférences".
  2. Utilisez au moins : input text, email, select, textarea.

10

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Form : action (URL de traitement) et method (GET/POST)
  • Input : text, password, email
  • Label : for+id pour l'accessibilité
  • Button : submit, reset, button
  • Select : liste déroulante avec options et optgroup
  • Textarea : zone de texte multi-lignes
  • Fieldset et legend pour grouper les champs
  • Placeholder (indicatif) vs value (valeur par défaut)

Mini-glossaire

form
Conteneur du formulaire. Attributs action et method obligatoires.
input
Champ de saisie polyvalent. Le type change le comportement.
label
Étiquette associée à un champ via for+id. Essentiel pour l'accessibilité.
button
Bouton avec trois types : submit, reset, button.
select
Liste déroulante. Les options sont définies par <option>.
textarea
Zone de texte multi-lignes. rows et cols pour la taille.
fieldset
Groupe de champs thématiques. Legend donne le titre.
placeholder
Texte indicatif qui disparaît à la saisie.
value
Valeur par défaut d'un champ (pré-remplie).
method
GET (visible dans l'URL) ou POST (dans le corps de la requête).
