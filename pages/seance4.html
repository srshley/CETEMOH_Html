01

Vidéo HTML5 — <video>

Avant HTML5 (2008), intégrer une vidéo nécessitait un plugin tiers — Flash, QuickTime, RealPlayer. Aujourd'hui la balise <video> est native dans tous les navigateurs modernes : aucun plugin, aucune dépendance externe.

📺
Analogie — La chaîne TV avec décodeur de secours

Pensez aux sources multiples comme des chaînes de secours : si le navigateur ne supporte pas MP4, il essaie WebM, puis OGG. Si aucune source ne fonctionne, le texte de repli s'affiche — comme un message "signal indisponible" sur votre télé.

Syntaxe de base

HTML
<!-- Forme minimale -->
<video src="film.mp4" controls></video>

<!-- Forme recommandée — sources multiples + accessibilité -->
<video
  width="800"
  height="450"
  controls
  poster="miniature.jpg"
  preload="metadata"
  muted
  loop
  playsinline
  aria-label="Démonstration de la balise video HTML5">

  <source src="film.webm" type="video/webm">
  <source src="film.mp4"  type="video/mp4">
  <source src="film.ogv"  type="video/ogg">

  <!-- Sous-titres / transcription -->
  <track kind="subtitles" src="sous-titres-fr.vtt" srclang="fr" label="Français" default>
  <track kind="subtitles" src="subtitles-en.vtt"   srclang="en" label="English">
  <track kind="captions"  src="audio-desc.vtt"     srclang="fr" label="Description audio">

  <!-- Texte de repli pour navigateurs très anciens -->
  <p>Votre navigateur ne supporte pas la balise vidéo HTML5.
     <a href="film.mp4">Télécharger la vidéo</a> (MP4, 42 Mo).</p>
</video>

Tous les attributs de <video>

AttributValeur(s)Rôle
srcURLSource unique (alternativement, utilisez des balises <source>)
controlsbooléenAffiche les contrôles natifs du navigateur (lecture, volume, plein écran)
width / heightpixelsDimensions de la zone de lecture — évite le layout shift
posterURL imageImage affichée avant la lecture (comme la jaquette d'un DVD)
autoplaybooléenLecture automatique — nécessite muted sur la plupart des navigateurs
mutedbooléenSoupe coupé au démarrage — obligatoire pour autoplay
loopbooléenLecture en boucle infinie
preloadnone / metadata / autoContrôle le chargement anticipé : metadata = durée + dimensions seulement
playsinlinebooléeniOS : empêche le passage en plein écran automatique — nécessaire pour mobile
crossoriginanonymous / use-credentialsAutorise CORS pour les ressources cross-domain

Les formats vidéo — lequel choisir ?

FormatExtensionCodecSupportRecommandation

.webmvideo/webmVP9 / AV1✅ Chrome, Firefox, EdgeMettre en premier — meilleure compression

💡
Ordre des sources — important !

Le navigateur essaie les <source> dans l'ordre et utilise le premier qu'il supporte. Placez toujours WebM en premier (meilleure compression = fichier plus léger), puis MP4 en filet de sécurité universel.

Sous-titres avec <track> — accessibilité et SEO

La balise <track> lie un fichier de sous-titres au format WebVTT (.vtt) à votre vidéo. C'est indispensable pour l'accessibilité (personnes sourdes ou malentendantes) et pour le référencement (Google peut indexer le contenu de vos sous-titres).

VTT
WEBVTT

Bienvenue dans ce tutoriel HTML5.

00:00:03.500 --> 00:00:07.000
Aujourd'hui nous allons apprendre la balise <video>.

00:00:07.000 --> 00:00:11.200
Elle permet d'intégrer des vidéos sans plugin.

Intégrer YouTube ou Vimeo

YouTube et Vimeo ne fournissent pas de fichier vidéo direct — ils utilisent des iframes (voir section 04). Mais si vous hébergez vos propres vidéos, <video> est la solution.

02

Audio HTML5 — <audio>

La balise <audio> fonctionne exactement comme <video> : même logique de sources multiples, mêmes attributs booléens, même texte de repli. Seul manque : pas d'attribut poster ni width/height puisqu'il n'y a pas d'image à afficher.

HTML
<!-- Forme minimale -->
<audio src="musique.mp3" controls></audio>

<!-- Forme recommandée -->
<audio
  controls
  preload="metadata"
  aria-label="Extrait sonore — Introduction au HTML5">

  <source src="extrait.ogg" type="audio/ogg">
  <source src="extrait.mp3" type="audio/mpeg">
  <source src="extrait.wav" type="audio/wav">

  <p>Votre navigateur ne supporte pas l'audio HTML5.
     <a href="extrait.mp3">Télécharger l'audio</a></p>
</audio>

<!-- Audio en boucle pour fond musical (toujours muted + autoplay) -->
<audio autoplay muted loop>
  <source src="ambiance.webm" type="audio/webm">
  <source src="ambiance.mp3"  type="audio/mpeg">
</audio>

.wavaudio/wavSans pertes✅ UniverselSons courts (effets UI) — fichier lourd

🎵
Stratégie de sources pour l'audio

Fournissez toujours OGG + MP3 minimum. OGG est plus léger et open-source ; MP3 est le filet universel. Pour les applications exigeantes (jeux), ajoutez WAV pour des sons très courts (latence nulle).

Accessibilité et audio

Un lecteur audio seul n'est pas suffisant pour l'accessibilité. Accompagnez toujours un contenu audio important d'une transcription textuelle — soit visible sur la page, soit dans un <details> expansible juste en dessous.

HTML
<figure>
  <figcaption>🎙️ Interview — Introduction au HTML5 (3 min 42)</figcaption>
  <audio controls>
    <source src="interview.ogg" type="audio/ogg">
    <source src="interview.mp3" type="audio/mpeg">
  </audio>
</figure>

<!-- Transcription accessible -->
<details>
  <summary>📝 Lire la transcription</summary>
  <div>
    <p><strong>Interviewer :</strong> Qu'est-ce que le HTML5 ?</p>
    <p><strong>Invité :</strong> Le HTML5 est la cinquième version du langage...</p>
  </div>
</details>
03

Canvas — dessin avec JavaScript

La balise <canvas> est une zone de dessin bitmap pilotée par JavaScript. Contrairement aux SVG (vectoriels et dans le DOM), le canvas est un tableau de pixels : parfait pour les jeux, les graphiques dynamiques et les traitements d'image en temps réel.

🎨
Analogie — La toile et le pinceau

<canvas> est la toile vierge. JavaScript (via le contexte 2D) est le pinceau. Vous dessinez instruction par instruction : tracez un rectangle, remplissez-le de rouge, écrivez du texte, dessinez un cercle… chaque instruction s'exécute sur la toile immédiatement.

Structure de base du canvas

HTML
<!-- Déclaration HTML : juste width et height -->
<canvas
  id="monCanvas"
  width="600"
  height="400"
  aria-label="Graphique de ventes mensuelles — Voir le tableau ci-dessous"
  role="img">
  <!-- Contenu de repli pour navigateurs sans canvas -->
  <p>Votre navigateur ne supporte pas le canvas HTML5.</p>
</canvas>
JS
// 1. Récupérer la référence de l'élément canvas
const canvas = document.getElementById('monCanvas');

// 2. Obtenir le contexte de rendu 2D
const ctx = canvas.getContext('2d');

// ── Rectangles ──────────────────────────────────
ctx.fillStyle   = '#3B82F6';          // couleur de remplissage
ctx.fillRect(50, 50, 200, 100);       // fillRect(x, y, largeur, hauteur)

ctx.strokeStyle = '#2DD4BF';          // couleur de contour
ctx.lineWidth   = 3;
ctx.strokeRect(300, 50, 200, 100);    // rectangle vide (contour)

ctx.clearRect(80, 80, 60, 40);        // efface une zone

// ── Texte ────────────────────────────────────────
ctx.fillStyle = '#E6EDF3';
ctx.font      = 'bold 24px Syne, sans-serif';
ctx.fillText('Bonjour Canvas !', 50, 230);

// ── Chemin (path) — cercle ────────────────────────
ctx.beginPath();
ctx.arc(400, 280, 60, 0, Math.PI * 2);  // (x, y, rayon, début, fin)
ctx.fillStyle = '#A78BFA';
ctx.fill();

// ── Ligne ────────────────────────────────────────
ctx.beginPath();
ctx.moveTo(50,  350);
ctx.lineTo(550, 350);
ctx.strokeStyle = '#F59E0B';
ctx.lineWidth   = 2;
ctx.stroke();

Canvas vs SVG — quand choisir quoi ?

Canvas <canvas>

✅ Jeux vidéo, animations temps réel
✅ Traitement d'image pixel par pixel
✅ Graphiques avec +1000 éléments
✅ Effets visuels complexes
✅ Manipulation de pixels (filtres photo)
❌ Non accessible (pas dans le DOM)
❌ Pas de clic par élément
❌ Qualité pixelisée en zoom

SVG <svg>

✅ Icônes, logos (infiniment scalable)
✅ Graphiques interactifs (chaque élément cliquable)
✅ Animations CSS possibles
✅ Indexable par les moteurs de recherche
✅ Accessible (du texte dans le DOM)
❌ Lent avec beaucoup d'éléments
❌ Pas adapté aux jeux
❌ Pas de manipulation de pixels

💡
Canvas et accessibilité

Le canvas est opaque aux lecteurs d'écran. Si votre canvas affiche des informations (graphique de données, carte), vous devez OBLIGATOIREMENT fournir un équivalent textuel : soit un tableau HTML juste dessous, soit des attributs ARIA (role="img" + aria-label), soit les deux.

04

iframes — intégrer d'autres pages

La balise <iframe> (inline frame) permet d'incorporer une page web complète dans une autre. C'est le mécanisme utilisé par YouTube, Google Maps, Spotify et la plupart des widgets tiers pour s'intégrer dans vos pages.

🖼️
Analogie — La fenêtre dans un mur

Un iframe est comme percer une fenêtre dans votre mur pour voir la maison du voisin. Votre page reste votre page, mais à travers la fenêtre vous voyez le contenu d'un autre site. Ce site voisin n'a aucun accès à votre maison (sécurité des iframes).

HTML
<!-- YouTube embed -->
<iframe
  width="560"
  height="315"
  src="https://www.youtube.com/embed/qz0aGYrrlhU"
  title="Introduction au HTML5 — Tutoriel complet"
  frameborder="0"
  allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture"
  allowfullscreen
  loading="lazy">
</iframe>

<!-- Google Maps embed -->
<iframe
  src="https://www.google.com/maps/embed?pb=!1m18!1m12!..."
  width="600"
  height="450"
  style="border:0;"
  allowfullscreen
  loading="lazy"
  referrerpolicy="no-referrer-when-downgrade"
  title="Localisation — Paris, France">
</iframe>

<!-- Page locale ou intranet -->
<iframe
  src="formulaire-contact.html"
  width="100%"
  height="500"
  title="Formulaire de contact intégré"
  sandbox="allow-forms allow-scripts">
</iframe>

L'attribut sandbox — sécurité des iframes

L'attribut sandbox isole le contenu de l'iframe : par défaut tout est interdit, puis on autorise explicitement ce qu'on veut. C'est la bonne pratique pour les contenus tiers non maîtrisés.

Valeur sandboxCe qui est autorisé
allow-formsSoumettre des formulaires
allow-scriptsExécuter du JavaScript
allow-same-originConserver l'origine (cookies, stockage)
allow-popupsOuvrir des popups / nouvelles fenêtres
allow-top-navigationNaviguer dans la fenêtre parente
allow-modalsOuvrir des boîtes de dialogue
⚠️
Iframe responsive — le problème du ratio

Un iframe avec width="560" height="315" fixe n'est pas responsive. La technique standard : enveloppez l'iframe dans un <div> avec position:relative; padding-bottom:56.25%; height:0; et l'iframe avec position:absolute; width:100%; height:100%;. Le ratio 56.25% correspond au format 16:9 (315/560 × 100).

05

Web Storage — stocker des données côté client

Avant HTML5, les cookies étaient le seul moyen de stocker des données dans le navigateur. Le Web Storage introduit deux mécanismes bien plus simples et puissants — sans limite des 4 Ko des cookies, sans envoyer les données au serveur à chaque requête.

localStorage

Données persistantes — survivent à la fermeture du navigateur et au redémarrage de l'ordinateur. Idéal pour les préférences utilisateur (thème sombre, langue).

Capacité : ~5 Mo par domaine

sessionStorage

Données de session — effacées dès que l'onglet est fermé. Idéal pour l'état d'un formulaire multi-étapes ou un panier temporaire.

Capacité : ~5 Mo par onglet

Visualisation — ce qu'il y a dans le localStorage

localStorage — domaine : monsite.com
theme → "dark"
langue → "fr"
user_prefs → {"fontSize":16,"notifications":true}
derniere_visite → "2024-09-15T14:32:00Z"

L'API Web Storage en JavaScript

JS
// ── localStorage ─────────────────────────────────

// Stocker une valeur (toujours des strings)
localStorage.setItem('theme', 'dark');
localStorage.setItem('prenom', 'Alice');

// Lire une valeur
const theme = localStorage.getItem('theme');   // → "dark"
const absent = localStorage.getItem('xyz');    // → null (si inexistant)

// Supprimer une clé
localStorage.removeItem('theme');

// Tout effacer (attention !)
localStorage.clear();

// Nombre d'éléments stockés
console.log(localStorage.length);  // → 1

// ── Stocker des objets (JSON) ──────────────────────
const prefs = { fontSize: 16, notifications: true, langue: 'fr' };
localStorage.setItem('user_prefs', JSON.stringify(prefs));

// Récupérer l'objet
const prefsStockees = JSON.parse(localStorage.getItem('user_prefs'));
console.log(prefsStockees.fontSize);  // → 16

// ── Pattern défensif recommandé ───────────────────
function sauvegarderTheme(theme) {
  try {
    localStorage.setItem('theme', theme);
  } catch (e) {
    // localStorage peut être indisponible (navigation privée, quota dépassé)
    console.warn('Impossible de sauvegarder le thème :', e);
  }
}

function chargerTheme() {
  return localStorage.getItem('theme') || 'light';  // 'light' par défaut
}

// ── sessionStorage — même API, périmètre différent ──
sessionStorage.setItem('etape_formulaire', '2');
sessionStorage.setItem('panier_temp', JSON.stringify([{ id: 1, qty: 2 }]));
⚠️
Ce que Web Storage ne remplace PAS

Ne stockez jamais dans localStorage : mots de passe, tokens d'authentification sensibles, données médicales ou bancaires. Le localStorage est accessible à tout JavaScript sur le même domaine — une attaque XSS peut tout lire. Pour les données sensibles, utilisez des cookies HttpOnly; Secure; SameSite=Strict gérés côté serveur.

Inspecter le Web Storage — DevTools

Dans Chrome ou Firefox : F12 → onglet Application (Chrome) ou Stockage (Firefox) → Local Storage / Session Storage. Vous pouvez ajouter, modifier et supprimer des entrées en direct depuis cet outil.

APIs HTML5 modernes

HTML5 a introduit un ensemble d'APIs JavaScript qui étendent les capacités du navigateur bien au-delà de l'affichage de texte : géolocalisation, glisser-déposer, travail hors ligne, notifications… Ces APIs sont appelées par JavaScript mais déclenchées ou configurées en HTML.

🌍 Geolocation
Récupère la position GPS de l'utilisateur (avec sa permission).
navigator.geolocation
🔔 Notifications
Affiche des notifications natives du système d'exploitation.
Notification API
🖱️ Drag & Drop
Glisser-déposer d'éléments HTML et de fichiers depuis le bureau.
draggable + events
📶 Online / Offline
Détecte si l'utilisateur a une connexion internet active.
navigator.onLine
📋 Clipboard
Lire et écrire dans le presse-papier du système (avec permission).
navigator.clipboard
🗄️ IndexedDB
Base de données côté client pour grandes quantités de données structurées.
window.indexedDB

Geolocation — exemple complet

HTML
<button id="btn-position">📍 Obtenir ma position</button>
<p id="resultat-geo">Cliquez pour activer la géolocalisation.</p>
JS
document.getElementById('btn-position').addEventListener('click', () => {
  const output = document.getElementById('resultat-geo');

  // 1. Vérifier le support
  if (!('geolocation' in navigator)) {
    output.textContent = '❌ Géolocalisation non supportée.';
    return;
  }

  output.textContent = '⏳ Récupération de la position…';

  // 2. Demander la position (déclenche une popup de permission)
  navigator.geolocation.getCurrentPosition(
    // Succès
    (position) => {
      const { latitude, longitude, accuracy } = position.coords;
      output.innerHTML = `
        ✅ Latitude  : <strong>${latitude.toFixed(6)}</strong><br>
        Longitude : <strong>${longitude.toFixed(6)}</strong><br>
        Précision : ${Math.round(accuracy)} mètres
      `;
    },
    // Erreur (permission refusée, timeout…)
    (error) => {
      const messages = {
        1: 'Permission refusée par l'utilisateur.',
        2: 'Position indisponible (GPS hors portée).',
        3: 'Délai dépassé (timeout).'
      };
      output.textContent = '❌ ' + (messages[error.code] || 'Erreur inconnue.');
    },
    // Options
    { enableHighAccuracy: true, timeout: 10000, maximumAge: 60000 }
  );
});

Drag & Drop natif HTML5

HTML
<!-- Rendre un élément "draggable" -->
<div
  draggable="true"
  id="carte-1"
  ondragstart="glisser(event)"
  class="carte">
  🃏 Carte déplaçable
</div>

<!-- Zone de dépôt -->
<div
  id="zone-depot"
  ondrop="deposer(event)"
  ondragover="event.preventDefault()"
  class="zone-depot">
  📦 Déposez ici
</div>
JS
function glisser(e) {
  // Stocker l'id de l'élément glissé
  e.dataTransfer.setData('text/plain', e.target.id);
}

function deposer(e) {
  e.preventDefault();
  const id = e.dataTransfer.getData('text/plain');
  const element = document.getElementById(id);
  e.target.appendChild(element);
}

Images responsive — picture & srcset

Une image de 2000 px de large chargée sur un téléphone de 390 px, c'est une catastrophe pour les performances. HTML5 résout ce problème avec deux mécanismes complémentaires : l'attribut srcset sur <img> et la balise <picture>.

srcset — proposer plusieurs résolutions

L'attribut srcset liste plusieurs versions de la même image. Le navigateur choisit automatiquement la plus adaptée selon la densité de pixels de l'écran et la taille d'affichage.

320px
Téléphone
photo-320.webp
768px
Tablette
photo-768.webp
1200px
Desktop
photo-1200.webp
HTML
<!-- srcset avec descripteurs de largeur (w) -->
<img
  src="photo-800.jpg"
  srcset="
    photo-320.webp  320w,
    photo-640.webp  640w,
    photo-800.jpg   800w,
    photo-1200.jpg 1200w,
    photo-1600.jpg 1600w
  "
  sizes="
    (max-width: 480px)  100vw,
    (max-width: 768px)  90vw,
    (max-width: 1200px) 66vw,
    50vw
  "
  alt="Paysage montagneux au lever du soleil"
  width="800"
  height="534"
  loading="lazy">

<!-- srcset avec descripteurs de densité (x) — pour icônes/logos -->
<img
  src="logo.png"
  srcset="logo.png 1x, logo@2x.png 2x, logo@3x.png 3x"
  alt="Logo MonSite"
  width="200"
  height="60">

<picture> — art direction et formats modernes

La balise <picture> va plus loin que srcset : elle permet de servir des formats différents (WebP, AVIF) avec un fallback, et d'adapter le cadrage de l'image selon la taille d'écran (art direction).

HTML
<picture>
  <!-- Format AVIF — meilleure compression (support partiel) -->
  <source
    srcset="hero-mobile.avif 480w, hero-desktop.avif 1200w"
    sizes="(max-width:768px) 100vw, 50vw"
    type="image/avif">

  <!-- Format WebP — bon support moderne -->
  <source
    srcset="hero-mobile.webp 480w, hero-desktop.webp 1200w"
    sizes="(max-width:768px) 100vw, 50vw"
    type="image/webp">

  <!-- Art direction — image carrée sur mobile, panoramique sur desktop -->
  <source
    media="(max-width: 768px)"
    srcset="hero-portrait.jpg">

  <!-- Fallback JPEG universel — TOUJOURS obligatoire -->
  <img
    src="hero-desktop.jpg"
    alt="Vue aérienne de la ville de Paris"
    width="1200"
    height="630"
    loading="lazy">
</picture>

Attributs data-* — données personnalisées

Les attributs data-* permettent de stocker des données personnalisées directement dans le HTML, associées à un élément. Ils servent de pont entre votre HTML et JavaScript — sans polluer les attributs standard, sans variables globales JS.

🏷️
Analogie — Les étiquettes sur les produits

Imaginez chaque élément HTML comme un produit en rayon. Les attributs data-* sont les étiquettes que vous collez dessus : data-prix="29.90", data-categorie="electronique", data-stock="12". Ces infos sont invisibles au visiteur mais lisibles instantanément par JavaScript.

Syntaxe et lecture

HTML
<!-- Carte produit avec données métier -->
<article
  class="carte-produit"
  data-id="42"
  data-prix="29.90"
  data-categorie="electronique"
  data-stock="8"
  data-promo="true">

  <h3>Casque Bluetooth XS Pro</h3>
  <p>Prix : 29,90 €</p>
  <button class="btn-ajout">Ajouter au panier</button>
</article>

<!-- Bouton avec action et cible -->
<button
  data-action="supprimer"
  data-cible-id="article-7"
  data-confirmation="Êtes-vous sûr ?">
  🗑️ Supprimer
</button>

<!-- Onglets — état courant -->
<nav class="onglets">
  <button data-onglet="accueil"  data-actif="true">Accueil</button>
  <button data-onglet="profil"   data-actif="false">Profil</button>
  <button data-onglet="reglages" data-actif="false">Réglages</button>
</nav>
JS
// ── Lire les data-* ──────────────────────────────
const article = document.querySelector('.carte-produit');

// Via .dataset (propriété camelCase)
console.log(article.dataset.id);          // "42"
console.log(article.dataset.prix);        // "29.90"
console.log(article.dataset.categorie);   // "electronique"
// data-en-stock → dataset.enStock (tiret → camelCase)

// Via getAttribute (nom exact avec "data-")
console.log(article.getAttribute('data-stock'));  // "8"

// ── Écrire/modifier un data-* ────────────────────
article.dataset.stock = '7';          // modifie data-stock en HTML aussi !
article.dataset.nouveauChamp = 'abc'; // crée data-nouveau-champ

// ── Supprimer ────────────────────────────────────
delete article.dataset.promo;         // supprime data-promo de l'élément

// ── Cas d'usage — système d'onglets ──────────────
document.querySelectorAll('[data-onglet]').forEach(btn => {
  btn.addEventListener('click', () => {
    // Désactiver tous
    document.querySelectorAll('[data-onglet]').forEach(b => {
      b.dataset.actif = 'false';
    });
    // Activer celui cliqué
    btn.dataset.actif = 'true';
    afficherOnglet(btn.dataset.onglet);
  });
});

// ── Sélecteurs CSS sur data-* ────────────────────
// [data-actif="true"] { font-weight: bold; border-bottom: 2px solid blue; }

EX 01 🎬 Galerie multimédia
  1. Créez un dossier Seance4/ avec index.html. Ajoutez la structure HTML5 complète avec les balises sémantiques vues en séance 2.

  5. Ajoutez une image responsive avec <picture> proposant au moins WebP + JPEG et un srcset avec 3 résolutions.
  6. Vérifiez que la page passe la validation W3C (validator.w3.org).
EX 02 💾 Préférences persistantes avec localStorage
  1. Créez une page avec deux boutons : "Thème clair" et "Thème sombre". Ajoutez une classe CSS .dark sur <body> qui change les couleurs.
  2. En JavaScript, lorsque l'utilisateur clique sur un bouton, basculez la classe .dark sur <body> ET sauvegardez le choix dans localStorage sous la clé 'theme'.
  3. Au chargement de la page, lisez localStorage.getItem('theme') et appliquez le thème sauvegardé automatiquement.
  4. Ajoutez un troisième bouton "Réinitialiser" qui efface l'entrée localStorage et repasse en thème clair.
  5. Testez : rechargez la page — votre choix doit persister. Ouvrez les DevTools → Application → Local Storage pour voir l'entrée.
EX 03 🏷️ Tableau filtrable avec data-*
  1. Créez un tableau HTML de 8 films (titre, année, genre, note/5) avec data-genre et data-note sur chaque <tr>.
  2. Ajoutez un <select> "Filtrer par genre" avec les options : Tous, Action, Comédie, Science-fiction, Drame.
  3. En JavaScript, écoutez le change du select. Parcourez toutes les lignes du tableau et cachez (style.display = 'none') celles dont le dataset.genre ne correspond pas au filtre. "Tous" les affiche toutes.
  4. Ajoutez un deuxième filtre "Note minimum" (1 à 5) avec un <input type="range"> qui cache les films sous la note choisie.
  5. Affichez le nombre de films visibles dans un <span> mis à jour dynamiquement.

Bilan & mini-glossaire

Ce que vous maîtrisez

  • Balise <video> : sources multiples, formats (MP4/WebM), <track> pour les sous-titres, tous les attributs
  • Balise <audio> : formats (MP3/OGG/WAV), transcription accessible avec <details>
  • Canvas : obtenir un contexte 2D, dessiner rectangles, texte, cercles, lignes — et quand préférer SVG
  • Iframes : intégrer YouTube/Maps, attribut sandbox et ses valeurs, technique du ratio responsive
  • localStorage vs sessionStorage : API complète (setItem, getItem, removeItem, clear), stockage JSON
  • APIs HTML5 : Geolocation (getCurrentPosition, gestion d'erreurs), Drag & Drop, Online/Offline
  • Images responsive : srcset avec descripteurs w et x, attribut sizes, balise <picture> pour art direction et formats
  • Attributs data-* : lire/écrire via .dataset, convention kebab-case → camelCase, cibler en CSS

Mini-glossaire

Codec
Algorithme de compression/décompression d'un flux vidéo ou audio. Ex : H.264 (vidéo), Opus (audio).
WebVTT
Format de fichier texte pour sous-titres et légendes (.vtt). Associé à une vidéo via <track>.
Canvas
Zone de dessin bitmap en HTML. Pilotée uniquement par JavaScript. Contenu non accessible au DOM.
Contexte 2D
Objet JavaScript renvoyé par canvas.getContext('2d') qui expose toutes les méthodes de dessin.
iframe
Fenêtre dans la page qui affiche un document HTML externe. Le contenu est isolé de la page hôte.
sandbox (iframe)
Attribut de sécurité qui restreint les actions du contenu de l'iframe. Liste blanche d'autorisations.
localStorage
Stockage clé/valeur côté navigateur, persistant après fermeture. Limité à ~5 Mo par domaine.
sessionStorage
Même API que localStorage mais effacé à la fermeture de l'onglet.
srcset
Attribut de <img> listant plusieurs sources. Le navigateur choisit selon la taille et la densité de l'écran.
data-*
Attributs HTML personnalisés pour stocker des métadonnées sur un élément. Accessibles en JS via element.dataset.
WebP / AVIF
Formats d'image modernes à haute compression. WebP : support universel ; AVIF : encore partiel mais meilleur ratio.
